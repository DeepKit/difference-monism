# 属集变迁存在论 (ASTO)

> **定位**：ASTO 是 `DM -> 工程 / 治理 / AI` 的应用桥接框架。  
> 它不单独承担终极本体论任务，而负责把最小结构承诺压成可操作的分析语言、审议流程与运行协议。

> **适用范围声明**：AI 只是 ASTO 的一个高压力应用场景，不是唯一场景。  
> ASTO 同样面向软件工程、组织治理、制度设计、平台规则、教育、公共决策、城市治理与个人实践等需要处理`变迁 / 边界 / 责任 / 例外`的问题域。

> **作者 / Author**：Yi Fu（付毅，ODDFounder，fuyi.it@live.cn）

## 统一发布模板

### 适用范围

- 用于把 DM 的最小结构承诺翻译成跨场景分析、审议、设计和裁决语法
- 用于 AI、软件工程、组织治理、制度设计、平台规则、教育、公共决策和个人实践等高变迁场景
- 用于公开征评、预印本占位和应用桥接讨论

### 不是什么

- 不是与 DM 平行竞争的第二终极本体论
- 不是只服务 AI 对齐的单场景理论
- 不是可以脱离人类裁决与边界约束独立运行的自动决策手册

### 边界

- 哲学地基与强本体主张应优先回引 DM
- ASTO 主要负责结构压缩、案例编码、边界判断与运行协议，不直接替代文明治理总论或工程执行细则
- 当前首轮公开主包以 `U00 + P01-P11 + P99.1-4` 为准，其他活跃支线继续按状态分层处理

### 与上游关系

- ASTO 直接建立在 DM 之上，是 `DM -> ASTO -> ECET -> TAT -> ODD` 中的第二层
- ECET 继续把 ASTO 的桥接语法推进到文明演化与治理判断
- TAT 与 ODD 不应跳过 ASTO 而直接宣称自足
