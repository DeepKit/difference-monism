# 属集变迁存在论 (ASTO)

> **定位**：ASTO 是 `DM -> 工程 / 治理 / AI` 的应用桥接框架。  
> 它不单独承担终极本体论任务，而负责把最小结构承诺压成可操作的分析语言、审议流程与运行协议。

> **适用范围声明**：AI 只是 ASTO 的一个高压力应用场景，不是唯一场景。  
> ASTO 同样面向软件工程、组织治理、制度设计、平台规则、教育、公共决策、城市治理与个人实践等需要处理`变迁 / 边界 / 责任 / 例外`的问题域。

> **作者 / Author**：Yi Fu（付毅，ODDFounder，fuyi.it@live.cn）

## 统一发布模板

### 适用范围

- 用于把 DM 的最小结构承诺翻译成跨场景分析、审议、设计和裁决语法
- 用于 AI、软件工程、组织治理、制度设计、平台规则、教育、公共决策和个人实践等高变迁场景
- 用于公开征评、预印本占位和应用桥接讨论

### 不是什么

- 不是与 DM 平行竞争的第二终极本体论
- 不是只服务 AI 对齐的单场景理论
- 不是可以脱离人类裁决与边界约束独立运行的自动决策手册

### 边界

- 哲学地基与强本体主张应优先回引 DM
- ASTO 主要负责结构压缩、案例编码、边界判断与运行协议，不直接替代文明治理总论或工程执行细则
- 当前对外主链以 `U00 + P03 + P05a + P06 + P04 + P99.1-5` 为准
- `U00` 当前已承担 `P05a-lite` 的压缩职责，暂不另拆独立 `P05a-lite`
- `P05b / P07-P11` 作为扩展层与场景层按需附加，不再默认与主链同强度外发

### 与上游关系

- ASTO 直接建立在 DM 之上，是 `DM -> ASTO -> ECET -> TAT -> ODD` 中的第二层
- ECET 继续把 ASTO 的桥接语法推进到文明演化与治理判断
- TAT 与 ODD 不应跳过 ASTO 而直接宣称自足
- `COP / LMM / WSM` 在使用状态语言、结构编码与责任流向前置字段时，也应优先回引 ASTO

## DM 风险防火墙

### 最小依赖

即使不先接受 `DM` 的强版本，`ASTO` 至少仍只依赖以下弱前提：

1. 场景中存在可区分的 `状态`
2. 状态之间存在 `变迁`
3. 变迁受 `边界 / 角色 / 条件 / 例外` 影响
4. 这些对象可以被压成一套统一语法

### `DM` 受争议时如何阅读本层

若 `DM` 的强本体论承诺受争议，
`ASTO` 当前最稳的降级阅读方式是：

- 先把它读成跨场景结构语言
- 先把它读成分析、审议、设计与裁决的桥接语法
- 暂不把它读成只能由 `DM` 唯一推出的派生层

### 本层可独立成立的部分

- 状态语言
- 角色与边界语言
- 变迁与例外语法
- 跨场景结构压缩能力

### 本层仍强依赖 `DM` 的部分

- `ASTO` 作为 `DM` 唯一正统桥接器的强宣称
- 某些把 `DM` 强本体论直接翻译成应用结论的句子

### 本层防火墙句

> **即使不接受 `DM` 的全部强形而上学承诺，`ASTO` 仍可作为跨场景结构语法使用；`DM` 在本层首先是优选解释器，而不是唯一许可证。**
