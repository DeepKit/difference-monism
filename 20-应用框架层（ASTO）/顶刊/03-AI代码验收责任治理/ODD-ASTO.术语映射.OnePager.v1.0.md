# ODD ↔ ASTO 术语映射（一页版）

| ODD术语 | ASTO术语 | 对齐说明 |
|:---|:---|:---|
| Artifact（产出物） | 物化态/存在的可验证节点 | 原子性、可封板、可替换；A→B→C 擢升与网络协同 |
| Contract（契约） | 属集（Attribute-Set） | 可执行规范，定义边界/不变量，要求可测试/可审计/可版本 |
| Sealing / Unsealing（封板/解封） | 稳态（Stable State）/小跃迁 | 封板形成可上线稳态；解封进入“对话→赛马→对抗→再封板”循环 |
| Mutation Testing（变异测试） | 信任转移的技术基座 | 以“杀死变异体”检验测试套件强度，替代人工逐行审查 |
| Horse Race（赛马机制） | 并行探索与裁决 | 低成本试错 + 证据化选择；时间盒与熔断 |
| Adversarial Testing（对抗测试） | 结构在坏天气下的有效性 | 合同/性质/Fuzz、红队、Chaos、变异测试 |
| Seal Bundle（稳态包） | 封板对象清单 | 产出物版本、契约版本、依赖版本窗口、发布路径、收敛目标 |
|| Motility（动变性） | 动变性 | 可解封/回滚/拔插的升级能力 |

### **术语锚点：属集**

> 属集，是存在在时间切片上可被指认的属性集合；
> 属集的变迁，构成了存在的全部历史。
>
> 我们不讨论存在“本来是什么”，
> 只讨论它在时间中“此刻呈现为什么”。
