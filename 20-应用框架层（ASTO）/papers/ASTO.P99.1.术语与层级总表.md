# ASTO.P99.1 术语与层级总表

> **定位**：本文件是 ASTO 首轮公开预印本的术语纪律与层级附录，用于固定 `U00 -> P03 -> P05a -> P06 -> P04 -> P99` 阅读链中的最低层级纪律与核心术语单义。  
> **公开口径**：`首轮公开预印本 / 公开征评附录 / 术语与层级支撑件`  
> **作者 / Author**：Yi Fu（付毅，ODDFounder，fuyi.it@live.cn）

---

## 一、总定位

ASTO 在当前项目中的更稳定位是：

> **DM 提供本体 - 认识 - 规范地基，ASTO 提供工程 - 治理 - AI 的应用桥接语法。**

由此引出三条纪律：

1. ASTO 不单独承担终极本体论任务。
2. 凡超出应用框架所需的强本体主张，默认回引 DM。
3. ASTO 的价值在于可编码、可审议、可熔断、可回写。

---

## 二、四类主张

| 类别 | 作用 | 代表文档 | 说明 |
|:---|:---|:---|:---|
| **结构公设** | 给出最小结构描述语言 | `P05a` | 用于回答“系统如何被拆解” |
| **经验推论** | 给出可证伪的操作推论 | `P03 / P05a / P05b` | 用于回答“如何判断与修正” |
| **价值公设** | 给出底线与裁决方向 | `P06 / P04` | 用于回答“为什么不能这么做” |
| **运行协议** | 给出停机、审议、复盘接口 | `U00 / P06` | 用于回答“现在该怎么做” |

---

## 三、核心术语单义表

| 术语 | 当前固定写法 | 最短定义 | 不再混用为 |
|:---|:---|:---|:---|
| **一元** | 最小结构起点 | 可被扰动的统一结构场域 | 最终实体、本源神学 |
| **属集** | 扰动条件下的属性配置切片 | 条件性真实的可操作切片 | 对象本体、终极描述 |
| **开放边界 / 存在剩余** | 本体论未尽性 | 模型无法穷尽存在的剩余 | 规范红线 |
| **禁行红线** | 规范义边界 | 一旦触碰必须触发 `FREEZE` 的禁止事项 | 本体论剩余 |
| **复数性** | 人的结构性守护 | 不可替代性 + 对话可能性 + 行动空间 | 单一效率指标 |
| **人类裁决权** | 价值公设 | 最终裁决权由人类保留 | 能力比较结论 |
| **FREEZE** | 保护性熔断 | 自动停机并进入人工审议 | 故障、失败、放弃判断 |

## 四、文档分工

| 文档 | 当前职责 | 不再承担 |
|:---|:---|:---|
| `U00` | 唯一最短入口、运行时压缩接口 | 独立终极本体论 |
| `P03` | 认识论与证据化认知 | 终极价值奠基 |
| `P05a` | 结构公设 + 派生推论 + 协议接口 | 把所有内容都写成同强度“公理” |
| `P06` | 规范边界、审议与熔断 | 替代结构层 |
| `P04` | 宣言、行动语言、治理框架 | 重复造本体地基 |
| `history.md` | 优化与封板归档 | 官方主文 |

---

## 五、使用纪律

1. 若正文继续把“开放边界”与“禁行红线”混写，优先修术语而不是补解释。
2. 若一个主张无法判断是结构公设、经验推论、价值公设还是运行协议，则不得直接进入正式主文。
3. 若新术语不能替代旧问题，只会增加歧义，则禁止进入正文。
4. 后续所有结构性修订，至少同步更新：
   - `P99.1`
   - `P99.2`
   - `P99.4`

## 六、当前最短判断

ASTO 的核心优化方向不是“继续发明术语”，而是：

> **把主张分层，把术语单义，把裁决程序化。**
