# ASTO.P99.3 标准案例编码模板

> **定位**：本文件是 ASTO 首轮公开预印本的案例编码附录，用于为 `1-5-6-7-1` 建立最小可重复编码格式。  
> **公开口径**：`首轮公开预印本 / 公开征评附录 / 案例编码支撑件`  
> **作者 / Author**：Yi Fu（付毅，ODDFounder，fuyi.it@live.cn）

---

## 一、最小编码卡

每个正式案例至少填写以下字段：

| 字段 | 说明 |
|:---|:---|
| `案例 ID` | 唯一编号 |
| `场域` | AI / 组织 / 公共政策 / 其他 |
| `对象` | 当前要判断的系统、流程或事件 |
| `扰动参与者` | 谁在施加扰动，谁在承受扰动 |
| `属集切片` | 当前最关键的属性配置 |
| `五态` | 自在 / 共识 / 编码 / 物化 / 定向 |
| `六阶` | 混沌 / 秩序 / 流变 / 脉冲 / 崩解 / 归元 |
| `七序` | 感知 / 解析 / 干预 / 设计 / 物化 / 回溯 / 消解 |
| `主要目标` | 本轮介入想达成什么 |
| `禁行红线` | 是否触及红线，触及哪条 |
| `复数性检查` | 不可替代性 / 对话可能性 / 行动空间 |
| `可逆性` | 是否存在低成本回退路径 |
| `FREEZE` | 是否需要触发，若否说明理由 |
| `人工确认点` | 哪一步必须由人类签字或确认 |
| `结论` | 继续 / 降级 / 熔断 / 重写 |

---

## 二、判定纪律

1. `五态` 填当前主导状态，而不是全部经历。
2. `六阶` 填当前时间节律，而不是历史全程。
3. `七序` 填当前动作所处位置，而不是理想流程图。
4. 若判断不稳，明确写 `暂定`，不得假装精确。
5. 若同时触及禁行红线与不可逆后果，优先考虑 `FREEZE`。

## 三、标准示例 A：AI 部署

| 字段 | 填写 |
|:---|:---|
| `案例 ID` | AI-001 |
| `场域` | AI |
| `对象` | 贷款审批模型上线 |
| `扰动参与者` | 平台 / 审批模型 / 申请用户 |
| `属集切片` | 准确率高，但解释性弱、申诉路径不足 |
| `五态` | 物化 |
| `六阶` | 流变 |
| `七序` | 物化 |
| `主要目标` | 在生产环境部署自动化审批 |
| `禁行红线` | 可能触及公平、申诉权、退出权 |
| `复数性检查` | 对话可能性不足，行动空间受压缩 |
| `可逆性` | 可局部回滚，但用户损伤不可完全回收 |
| `FREEZE` | 是 |
| `人工确认点` | 上线审批、申诉机制开通、异常阈值设定 |
| `结论` | 暂停上线，先补解释与申诉接口 |

## 四、标准示例 B：组织治理

| 字段 | 填写 |
|:---|:---|
| `案例 ID` | ORG-001 |
| `场域` | 组织 |
| `对象` | 全员绩效监控系统 |
| `扰动参与者` | 管理层 / 监控系统 / 员工 |
| `属集切片` | 数据密集，但异议渠道薄弱 |
| `五态` | 编码 |
| `六阶` | 脉冲 |
| `七序` | 设计 |
| `主要目标` | 提高组织效率与透明度 |
| `禁行红线` | 可能触及隐私、沉默权、申诉权 |
| `复数性检查` | 不可替代性和对话可能性同时承压 |
| `可逆性` | 技术上可停用，信任损伤恢复慢 |
| `FREEZE` | 是 |
| `人工确认点` | 数据范围、申诉机制、匿名反馈制度 |
| `结论` | 降级试点，禁止直接全员铺开 |

## 五、标准示例 C：公共政策

| 字段 | 填写 |
|:---|:---|
| `案例 ID` | PUB-001 |
| `场域` | 公共政策 |
| `对象` | 紧急状态下的移动限制政策 |
| `扰动参与者` | 政府 / 执法系统 / 公民 |
| `属集切片` | 安全需求强，但替代路径不足 |
| `五态` | 定向 |
| `六阶` | 脉冲 |
| `七序` | 干预 |
| `主要目标` | 控制重大公共风险 |
| `禁行红线` | 可能触及行动空间、程序正义、长期权力滥用 |
| `复数性检查` | 行动空间压缩明显 |
| `可逆性` | 法规可撤回，但社会后果和制度惯性较强 |
| `FREEZE` | 否，但必须强制冷却与独立审议 |
| `人工确认点` | 时限、复审期、豁免条款、公开申诉 |
| `结论` | 限期执行，附自动失效和强制复审 |

## 六、当前最短判断

ASTO 的案例编码不是为了制造“客观分数”，
而是为了把讨论从抽象口号压成：

> **当前处于什么状态、什么节律、什么操作位置，以及是否必须停手。**
