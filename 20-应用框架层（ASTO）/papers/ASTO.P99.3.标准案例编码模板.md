# ASTO.P99.3 标准案例编码模板

> **定位**：本文件是 ASTO 的跨层案例编码附录，用于为 `1-5-6-7-1` 与下游 `ECET / TAT / COP / ODD / LMM / WSM` 建立最小可重复编码格式。  
> **公开口径**：`首轮公开预印本 / 公开征评附录 / 案例编码支撑件`  
> **作者 / Author**：Yi Fu（付毅，ODDFounder，fuyi.it@live.cn）

---

## 一、最小编码卡

每个正式案例至少填写以下字段：

| 字段 | 说明 |
|:---|:---|
| `案例 ID` | 唯一编号 |
| `场域` | AI / 组织 / 公共政策 / 其他 |
| `对象` | 当前要判断的系统、流程或事件 |
| `扰动参与者` | 谁在施加扰动，谁在承受扰动 |
| `属集切片` | 当前最关键的属性配置 |
| `五态` | 自在 / 共识 / 编码 / 物化 / 定向 |
| `六阶` | 混沌 / 秩序 / 流变 / 脉冲 / 崩解 / 归元 |
| `七序` | 感知 / 解析 / 干预 / 设计 / 物化 / 回溯 / 消解 |
| `边界状态` | 清晰 / 模糊 / 可争议 / 不可绕过 |
| `例外标记` | 无 / 低阶例外 / 高阶例外 / 禁行红线 |
| `责任流向模式` | 常规 / 下滑 / 蔓生 / 引爆 / 沉积 / 割裂 |
| `主要目标` | 本轮介入想达成什么 |
| `禁行红线` | 是否触及红线，触及哪条 |
| `复数性检查` | 不可替代性 / 对话可能性 / 行动空间 |
| `可逆性` | 是否存在低成本回退路径 |
| `FREEZE` | 是否需要触发，若否说明理由 |
| `人工确认点` | 哪一步必须由人类签字或确认 |
| `结论` | 继续 / 降级 / 熔断 / 重写 |

## 二、跨层共用字段

若该案例还要继续流入下游层，
建议同步补齐以下共享字段：

| 字段 | 下游用途 |
|:---|:---|
| `COP 分流状态` | `RESOLVE / MIXED / FREEZE / UNKNOWN / REFER` |
| `TAT 门槛状态` | `R = 0 / R = 1` |
| `TAT 责任锚定度` | `r` 的当前判断 |
| `ODD 门禁状态` | `ALLOW / LIMITED / FREEZE / ROLLBACK` |
| `LMM 接引状态` | 尚未显影 / 已显影 / 可接引 / 已转介 |
| `WSM 局型标签` | `水 / 木 / 火 / 土 / 金` |

这些字段不是要求 `ASTO` 替下游层做判断，
而是要求案例在跨层流转时保留同一张病历。

---

## 三、判定纪律

1. `五态` 填当前主导状态，而不是全部经历。
2. `六阶` 填当前时间节律，而不是历史全程。
3. `七序` 填当前动作所处位置，而不是理想流程图。
4. `边界状态` 回答的是当前接口是否清晰、可申诉、可回接。
5. `责任流向模式` 允许先写 `常规`，只有出现明显滑移时再写异常模式。
6. 若判断不稳，明确写 `暂定`，不得假装精确。
7. 若同时触及禁行红线与不可逆后果，优先考虑 `FREEZE`。

## 七、当前最短判断

ASTO 的案例编码不是为了制造“客观分数”，
而是为了把讨论从抽象口号压成：

> **当前处于什么状态、什么节律、什么操作位置、责任沿何种模式流动，以及是否必须停手。**
