# ASTO P 系列公开入口

> **定位**：ASTO 是 `DM -> 多场景实践` 的应用桥接框架。  
> 它把最小结构承诺翻译成可用于分析、审议、设计与裁决的语言。

> **公开口径**：ASTO 不是与 DM 平行竞争的第二终极本体论。  
> 它更适合被表述为：`跨场景应用框架 / 工作中的理论桥 / 持续收束中的公开文档组`。

> **作者 / Author**：Yi Fu（付毅，ODDFounder，fuyi.it@live.cn）

> **当前状态**：本目录是 ASTO 的公开 `P` 系列入口，但不是把目录内所有文件自动视作同强度主包的许可。当前对外主链以 `U00 + P03 + P05a + P06 + P04 + P99.1-5 + 预印本发布说明` 为准；`P01 / P02` 固定为外围入口，`P05b / P07-P11` 继续按扩展层、场景稿或补充件处理。

## 统一发布模板

### 不是什么

- 不是把 `read_p*.txt`、缓存稿、归档稿和历史归档一并外发的材料堆
- 不是与 DM 平行竞争的第二终极本体论文库
- 不是只服务 AI 的窄场景说明集

### 边界

- 强本体主张优先回引 DM
- 当前目录只负责 ASTO 的公开 `P` 系列与附录入口，不替代 ECET / TAT / ODD 的下游层级入口
- 当前可稳定外发的主对象以 `U00 + P03 + P05a + P06 + P04 + P99.1-5` 为准
- `U00` 当前兼作 `P05a-lite` 的压缩运行稿，暂不另拆独立 `P05a-lite`
- `P05b / P07-P11` 默认按扩展层与场景层附加，不与主链同强度并列

---

## 适用范围

ASTO 不只用于 AI。

当前更稳的表述是：ASTO 同时可用于以下场景中的结构分析、边界判断与行动设计：

- AI 对齐与人机协作
- 软件工程与自动化治理
- 组织设计与流程裁决
- 制度治理与公共决策
- 平台规则与社区审议
- 教育、城市与跨领域复杂系统
- 个体在高变迁环境中的实践判断

AI 只是其中一个压力最高、最容易暴露边界问题的应用场景。

## 各文档职责

- `P01-P02`：外围入口与免疫性前言，不与主链同强度并列。
- `P03`：说明有限主体为何会系统性出错，以及为什么要转向证据化认知。
- `P05a`：提供最小结构语法与桥接层分层接口。
- `P05b`：体验映射扩展稿，不属于当前最小主链。
- `P06`：给出价值边界、熔断纪律与不可程序化的保留区。
- `P04`：把结构与价值压成行动语言、治理协议与责任框架。
- `P04A`：P04 的传播伴随件，不构成规范主文。
- `P07-P11`：扩展层与场景层，用于专题外发或补充比较。
- `P99.1-P99.5`：主链默认伴随件，其中 `P99.5` 承接规范转译与哲学借鉴说明。

## 对外纪律

- 不把 ASTO 说成“终极世界观”。
- 不把 ASTO 说成“只服务 AI”。
- 不把结构承诺、经验推论、价值公设和运行协议混写成同一种强度。
- 凡强本体主张，优先回引 DM；凡应用桥接与裁决语言，优先回引 ASTO。
