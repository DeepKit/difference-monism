# 差异一元论 / Difference Monism

> 一个以`差异`为最小起点的哲学本体论框架。
> 它讨论的核心问题不是“世界由哪些现成实体构成”，而是“存在如何可能”。
>
> **作者 / Author**：Yi Fu（付毅，ODDFounder，fuyi.it@live.cn）

---

## 当前定位

本仓库当前以`哲学版本`为主，不再把自己包装成：

- 可直接证伪的自然科学理论
- 商业方法论入口
- 产品化/变现型项目主页

当前主轴是：

1. `本体论地基`
差异、结构、边界、位形变换、时间、持存、规律、因果、涌现

2. `层级展开`
从结构到生命、主体、认识、责任与文明

3. `与主要传统对话`
同结构实在论、过程哲学、现象学、佛学缘起论等展开比较

公开发布时，当前默认口径是：

- `DM` 作为主发布对象，按“统一送审版 / 征评版 / 预印本”口径对外
- `ASTO` 作为 `DM -> 工程 / 治理 / AI / 多场景实践` 的应用桥接框架配套公开
- 整个当前公开主链按 `DM -> ASTO -> ECET -> TAT -> ODD` 理解；历史 `RM / Resonance Monism / 共振一元论 / 共扰一元论` 命名默认按早期材料处理

## 统一发布模板

### 适用范围

- 用于整个理论体系的总入口、发布导航和层级关系说明
- 用于指导外部读者理解 `DM -> ASTO -> ECET -> TAT -> ODD` 的最短阅读路径
- 用于预印本占位、Zenodo 版本化和公开征评前的总口径统一

### 不是什么

- 不是把全仓库所有历史材料、缓存稿、讨论稿一次性对外抛出的材料堆
- 不是宣称整套体系已经完成同行评审、长期验证和全球制度落地的终局版本
- 不是要求外部读者一开始就阅读所有层级全文的唯一入口

### 边界

- 根入口只负责统一定位、统一导航和统一发布口径，不替代各层主文
- 具体哲学论证以 DM 主文为准，具体应用桥接以 ASTO 为准，具体文明/责任/工程展开以下游各层为准
- 历史命名阶段材料可保留作溯源参考，但不自动进入当前公开主包

### 与上游关系

- 根入口是整套体系的导航层，不位于五层主链之内，也不与任何一层竞争
- 它负责把当前公开主链固定为 `DM -> ASTO -> ECET -> TAT -> ODD`
- 它同时负责把早期 `RM / Resonance Monism / 共振一元论 / 共扰一元论 / MFO` 材料压回历史命名阶段

## 与 BetterCiv 的边界

`一元论` 与 `BetterCiv` 不是两个彼此竞争的项目，而是同一套体系中的上下游分工。

### `一元论` 负责什么

`一元论` 是整套体系的**理论母库 / 法源母本**，主要承担：

- 底层本体与哲学地基
- 应用桥接框架与统一解释语言
- 文明、责任、工程等主链理论展开
- 预印本、征评稿、论文稿与理论发布口径
- 概念、命题、边界、失败条件与学术对话

压缩一句话：

**`一元论` 负责回答“这套理论为什么成立”。**

### `一元论` 不直接负责什么

- 不直接承担 BetterCiv 的现实事务调度
- 不直接充当产品中控台、执行母库或日常战役指挥部
- 不把短期市场动作、产品文案、行动日志、现实推进记录混入理论主链
- 不让前台变现压力反向偷改上位理论定义

### `BetterCiv` 负责什么

`BetterCiv` 是整套体系的**现实执行母库 / 部署系统**，主要承担：

- 纲领、战略、事务、产品、学术与归档的物理分层
- 理论在现实中的部署、转译、执行、验证与累积
- 冷启动传播、产品推进、现实承接与研究复核
- 以 `LMM + COP` 把高位理论压缩为现实触点、诊断泊位、港口承接与责任闭合接口
- 以产品收入为现实军费，反哺理论传播、案例积累、学术推进与下一轮部署
- 行动计划、产品资产、执行记录与现实中控

压缩一句话：

**`BetterCiv` 负责回答“这套理论如何运行”。**

补一句：

**`BetterCiv` 默认首先服务内部操作者，而不是充当整套体系的对外项目主页。**

### 论文体系放置规则

- **理论论文母本**、预印本、征评稿、正式送审稿、论文法源与对外发布总口径，主位放在 `一元论`
- **要拿去发表的论文总规划**，主位也放在 `一元论`，因为它属于整套理论如何对外成立的发布路线，而不是执行层事务表
- **案例证据、复核包、匿名化、补证清单、盲评 SOP、投稿执行表、期刊筛选与投稿推进记录**，放在 `BetterCiv/05_学术型`

压缩一句话：

**论文战略与论文法源在 `一元论`，论文作战与投稿执行在 `BetterCiv/05_学术型`。**

### 共同纪律

- 理论法源优先留在 `一元论`
- 现实执行、产品推进与部署编排优先落在 `BetterCiv`
- `BetterCiv` 不得反向篡改 `一元论` 的上位定义
- `一元论` 不替代 `BetterCiv` 的现实中控与执行编制
- 对外产品与销售入口可以降维表达，但不得反向偷改主链理论边界

## 体系总览 (System Architecture)

当前应同时区分三条链，而不是把“法源顺序”和“运行顺序”写成一条线。

### 公开法源链

`DM -> ASTO -> ECET -> TAT -> ODD`

用于公开发布、论文主位与理论论证。

### 运行执行链

`LMM -> COP -> TAT -> ODD -> Harness`

用于现实显影、诊断分流、责任裁决、工程编译与受控执行。

### 反馈回流链

`Harness -> ODD -> COP -> LMM`

用于把运行时事件、事故、回滚与复审重新送回诊断和前线承接。

### 八层统一理解

| 层级 | 名称 | 当前最短职责 |
|---|---|---|
| 1 | `DM` | 世界最小地基，定义差异、结构、边界 |
| 2 | `ASTO` | 把哲学母法压成跨场景可翻译语法与统一前置编码 |
| 3 | `ECET` | 说明治理边界何时成立、何时失真 |
| 4 | `TAT` | 规定责任门槛与裁决条件 |
| 5 | `ODD` | 把裁决与诊断编译成工程门禁、验证、封存、回滚 |
| 6 | `COP` | 对复杂现实做诊断压缩与风险分流 |
| 7 | `LMM` | 在现实前线显影问题场并完成首次责任闭合 |
| 8 | `Harness` | 在运行时执行拦截、留痕、冻结、回滚 |

最短句：

> **公开法源上看是 `DM -> ASTO -> ECET -> TAT -> ODD`，现实运行上看是 `LMM -> COP -> TAT -> ODD -> Harness`。**

补充：

> **`五行察局学（WSM）` 当前收纳在 `LMM` 之下，作为其内部二级理论，用于把责任流向与结构困局压成前线可调用的识别语言。**

再补一句当前现行口径：

> **`ASTO` 当前不只负责“应用桥接”，还负责给 `TAT / COP / ODD / LMM / WSM` 提供统一结构编码、节律判断与边界前置字段。**
