# 企业拆弹专家·60篇文章发布计划

> **定位**：企业拆弹专家——帮成功创始人在系统爆炸前找到并拆除责任黑洞  
> **发布周期**：2025年3月 — 2026年5月，一周一篇，共60篇14个月  
> **发布平台**：虎嗅（权威建立）+ 知乎（流量沉淀）+ 私域微信群（转化闭环）  
> **传播三引擎**：傍大款｜引争议｜促转发

---

## 传播设计说明

### 三引擎评估与补强

| 引擎 | 原版问题 | 补强方案 |
|---|---|---|
| **傍大款** | 案例零散，无系列感 | 新增"大款解剖"专栏标签，每月至少1篇知名企业拆弹，形成条件反射 |
| **引争议** | 几乎没有反直觉标题 | 新增10篇争议性标题，主动挑战"管理学常识"，让同行想反驳 |
| **促转发** | 转发动机设计不足 | 每篇结尾设计"转发钩子"，让董事长有理由转给高管或同行 |

### 渠道矩阵（三层）

```
第一层：公域获客
  虎嗅    → 深度长文 3000-5000字，建立权威，吸引高净值读者
  知乎    → 同主题精简版，问答形式，搜索流量沉淀

第二层：私域沉淀（新增）
  微信群  → 每篇文章发布后，在1个垂直创始人社群分享
            （领教工坊校友群、行业协会群、EMBA同学群）
            目标：每篇至少进入1个300人以上的群

第三层：转化闭环（新增）
  个人微信 → 文章结尾引导"扫码加我"
  首次接触 → 发送《企业责任黑洞自检清单》（PDF）作为见面礼
  持续跟进 → 每周发1条私域内容，不在公域发布
```

### 转发钩子设计（每篇文章结尾标配）

**A类钩子（让董事长转给高管）**：
> "如果你的高管不理解为什么这是个问题，把这篇文章发给他。"

**B类钩子（让董事长转给同行）**：
> "你认识的创始人里，有几个正在经历这个？"

**C类钩子（让读者保存备用）**：
> "收藏这篇，三个月后再看，你会发现它说的是你现在的处境。"

## 文章类型说明

| 标签 | 含义 | 传播引擎 |
|---|---|---|
| 🔴 触发恐惧 | 让他停下来，感到背脊发凉 | 促转发 |
| 🔵 大款解剖 | 用知名企业案例建立框架权威 | 傍大款 |
| 🟡 争议炸弹 | 反直觉观点，让同行想反驳 | 引争议 |
| 🟢 建立认知 | 植入新概念，让他用你的语言思考 | 促转发 |
| 🟠 提供出口 | 转化文章，让有需求的人联系你 | 转化 |
| 🟣 演绎预估 | 前瞻预测，建立你的判断权威 | 傍大款+引争议 |

## 执行原则

### 不可违背的底线
1. **第40、44、49、52篇**必须有真实案例后才写，绝不编造
2. 争议类文章（🟡）不攻击具体个人，只攻击概念和制度
3. 大款解剖类（🔵）如涉及在世创始人，只引用公开信息，不做个人品德判断

### 节奏控制
- **每周一篇**：虎嗅发深度长文（3000-5000字）
- **同步知乎**：同主题精简版（800-1500字），问答形式
- **私域分发**：每篇发布后，在1个垂直创始人社群分享片段，引导阅读全文
