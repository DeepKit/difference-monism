Theories

Three layers. One direction.

ODD — Output-Driven Development

Engineering Layer

Code is liability. Output is the asset. Stop reviewing process — start verifying results.

In the LLM era, AI generates code at near-zero cost. But the cognitive cost of humans reviewing that code is enormous. ODD inverts the traditional model: instead of trusting process to guarantee results, we define verifiable contracts upfront and let machines validate machine output.

Core Loop

  • Contract — Define what "done" means before any code is written
  • Execute — Let AI generate. The process is a black box. Don't read it.
  • Verify — Run automated gates. Pass, Fail, or Freeze.
  • Seal — Hash-bind passing output with its test report. Immutable evidence.

The Three-State Model

  • 🟢 PASS — Output meets contract. Ship it.
  • 🔴 FAIL — Output rejected. Rewrite.
  • 🟡 FREEZE — Contract has undefined boundaries. Human judgment required.

Empirical data: 172-task controlled experiment. Defect rate reduced by 59.5% (p=0.029).

ECET — Evolutionary Constraint Existential Theory

Governance Layer

Every decision consumes resources and produces consequences. Both are physical facts. Physical facts must have physical owners.

ECET derives governance principles from three unavoidable physical constraints — not from ethics, not from social contract, but from thermodynamics, evolutionary selection, and Gödelian incompleteness.

Three Constraints

  • Energy Constraint — Resources are finite. Decisions must have owners.
  • Adaptive Selection — Structures with misaligned accountability are unstable and will be eliminated.
  • Incompleteness — No cognitive model can fully describe its environment. No structure is final.

Shadow Sovereignty

When decision-making power exists but accountability is diffuse, we get Shadow Sovereignty. It doesn't collapse immediately — it runs under the cover of efficiency and convenience, until consequences can no longer be assigned.

ASTO — Attribute Set Transformation Ontology

Ontological Layer

Existence is not a thing. It is a process of transformation.

ASTO provides a new meta-language for describing any system — human, AI, organization, or civilization. Instead of "subject-object-action," it uses "perturbation-slice-transformation." Instead of linear time, it maps existence across five states, six phases, and seven sequences.

Core Concepts

  • Attribute Set — The basic unit of existence. Not a substance, but a pattern of properties.
  • Perturbation — The fundamental operation. All change is perturbation.
  • Forbidden Element — The irreducible remainder. What cannot be totalized: dignity, privacy, the right to exit.
  • FREEZE — When a forbidden element is touched, the system must stop and call for human judgment.
