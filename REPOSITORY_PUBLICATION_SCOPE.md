# 仓库公开发表范围说明

> 用途：说明本仓库哪些内容属于公开法源主包，哪些属于可公开补充，哪些属于受限执行与经营材料。
>
> 配套文件：
> - `README.md`
> - `SNAPSHOT_MANIFEST.md`
> - `理论公开分类清单.md`
>
> 说明：本文件用于仓库快照、预印本挂载、对外导出与引用边界控制，不替代正式法律意见。

---

## 一、当前仓库角色

`一元论` 当前是整套体系的理论母库 / 法源母本。

它当前承载两条不同但相关的链：

1. 公开法源链  
   `DM -> ASTO -> ECET -> TAT -> ODD`
2. 扩展执行链  
   `ASTO -> LMM -> COP -> TAT -> ODD -> Harness`

发布时，不应把两条链混成一条。

最短句：

> 本仓库首先用于公开理论法源，其次用于挂接必要的接口与导航，不等于把全部执行层和经营层全文公开。

---

## 二、正式公开主包

以下对象构成当前仓库的正式公开主包，可作为整仓快照中的优先引用对象：

这些对象负责：

它们是公开快照的根入口，但不替代各层主文。

## 五、三类发布边界

如果用最小方式理解当前仓库，可以把对象分成三类：

### A 类：先确权后发布

适用对象：

- `DM`
- `ASTO`
- `ECET`
- `TAT`
- `ODD`
- `LMM` 公共理论面
- `WSM` 核心理论面
- `ASTO.P99.6` 这类接口法补充件
- 关键论文主稿

动作原则：

- 先统一作者、版本、引用入口
- 再公开全文、预印本或送审包

### B 类：可直接推广

适用对象：

- 根目录总纲件
- 八层总图
- 各层摘要页
- 导航层
- 科学对话层
- 工具层安全入口
- `WSM.001` 中可切出的前线识局摘要段

动作原则：

- 可以直接用于演讲、介绍、短文、合作沟通
- 但不等于放出全部底稿与执行细节

### C 类：保留用于收入

适用对象：

- `COP`
- `Harness`
- `LMM` 私有层
- 根目录跨层执行协议件
- `Progee2` 落地链
- 真实案例、参数、阈值、模板、复审库、误判库

动作原则：

- 原则可讲，全文不放
- 可做脱敏展示，不做全量共享
- 优先作为服务、产品、顾问、实施与授权资产保留

## 六、对外引用纪律

### 1. 整仓引用

若引用整仓快照，优先引用：

- `CITATION.cff`
- `README.md`
- `SNAPSHOT_MANIFEST.md`
- 本文件

### 2. 分层引用

若引用具体理论内容，优先引用：

- 对应层 `README`
- 对应层主文
- 对应层发布说明
- 对应层预印本或论文主稿

不要优先引用：

- `better.md`
- `history.md`
- 临时执行件
- 归档旧稿
- 执行样例包

### 3. 运行链引用

若确需对外介绍运行执行链，应优先使用摘要性、结构性、脱敏化材料，不应默认开放：

- 完整阈值
- 完整模板
- 全案例证据链
- 收费承接路径
- 可直接复刻的交付 SOP

---

## 七、导出与发布纪律

若要导出公开快照，建议遵守以下顺序：

1. 先确认 `A 类` 对象的作者、版本、Canonical 入口已锁定。
2. 再用 `B 类` 对象生成外部介绍、总图、摘要页与导读页。
3. `C 类` 对象只保留脱敏结构说明，不直接导出全文。
4. `99-归档/`、各层 `better/history`、内部工作区默认不进入正式对外压缩包。
5. 若有需要保留的执行件，只能以“摘要接口说明”形式出现，不可与理论主文混放。

---

## 八、最短结论

> 本仓库可以公开，但应按三层处理：  
> `DM -> ASTO -> ECET -> TAT -> ODD` 作为正式法源主包；  
> 导航、摘要与总图作为公开补充；  
> `COP / Harness / LMM 私有层 / Progee2 / 执行协议件` 作为受限资产保留。
