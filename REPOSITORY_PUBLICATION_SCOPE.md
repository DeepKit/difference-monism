# 仓库公开发表范围说明

> **仓库角色**：本仓库当前作为 `DM -> ASTO -> ECET -> TAT -> ODD` 理论栈的公开快照容器。
>
> **作者 / Author**：Yi Fu（付毅，ODDFounder，fuyi.it@live.cn）
>
> **快照日期**：2026-03-20

## 1. 当前公开口径

本仓库可以作为对外公开的整仓快照发布，但它不是“所有文件同等权威”的材料堆。

对外应区分三类对象：

1. `正式主包`
2. `公开补充件`
3. `历史留档 / 工作区保留件`

当前唯一推荐的主链理解方式是：

`DM -> ASTO -> ECET -> TAT -> ODD`

历史命名如 `RM / Resonance Monism / 共振一元论 / 共扰一元论 / MFO` 默认按历史阶段材料理解，不与当前主链竞争正典地位。

## 2. 正式主包

以下目录构成当前理论栈的正式公开主链：

- `10-哲学核心层（DM）/`
- `20-应用框架层（ASTO）/`
- `30-文明演化层（ECET）/`
- `40-责任架构层（TAT）/`
- `50-工程方法层（ODD）/`

这些目录的 `README`、`PUBLISH_READINESS.md`、主文档与预印本发布说明，构成外部读者的优先引用入口。

## 3. 公开补充件

以下目录允许随仓库一起公开，但默认按补充、接口或发布辅助理解：

- `00-索引与导航/`
- `60-科学对话层（DM.S）/`
- `70-其他工具层（DM.O）/`
- `80-学术投稿（RM.papers）/` 中已明确标记为预印本发布包、支撑件、投稿流程说明的部分

这些内容可用于导航、补充、对照与发布支持，但通常不作为第一引用对象。

## 4. 历史留档与工作区保留件

以下目录允许保留在仓库中，但默认不与正式主包混读：

- `90-传播与变现/`
- `99-归档/`

其中材料主要按传播稿、转译稿、市场化工作线、历史稿、旧命名阶段材料理解。

## 5. 不纳入正式引用对象的内容

即使它们暂时存在于仓库中，也不应被当作正式发表内容引用：

- 版本控制元数据，如 `.git/`
- 本地产生的缓存、测试缓存、编译中间物、临时数据库、运行时二进制与压缩包
- 已明确标记为任务记录、讨论记录、PoC 工作区、投稿流程件、历史残稿的内容
- 未来若重新生成的 `.Story/`、`.works/`、`__pycache__/`、`.pytest_cache/` 一类本地工作目录

## 6. 发布纪律

若把本仓库作为公开快照上传到预印本或仓储平台，建议遵守以下纪律：

1. 根入口、`README`、本文件与 `SNAPSHOT_MANIFEST.md` 一起发布。
2. 以各层 `README`、`PUBLISH_READINESS.md`、预印本发布说明作为分层解释器。
3. 不把历史命名阶段材料当成当前正式主张自动引用。
4. 不把传播稿、案例草案、产品笔记、投稿流程件误写成正式主文。

## 7. 引用建议

若引用整仓快照，优先引用：

- `CITATION.cff`
- 本文件
- `SNAPSHOT_MANIFEST.md`

若引用具体理论内容，优先引用对应层的主文或发布包，而不是引用整个仓库所有目录。
