# 学术投稿区说明

> **作者 / Author**：Yi Fu（付毅，ODDFounder，fuyi.it@live.cn）
>
> **当前口径**：`历史探索稿 + 公开预印本发布工作区 + 单焦点论文送审清稿区`

本目录当前包含两类内容：

1. `预印本发布包/`
   当前对外手工上传预印本平台时，应优先使用这里已经整理好的 `DM / ASTO / ECET / TAT / ODD` 发布包。
   这一类目录优先承担“理论全本发布预印本包”职责。
2. `RM-PAPERS-*`、`RM论文体系.md`、`sci.tasks.md`
   这些文件属于早期 `RM / Resonance Monism / 共振一元论 / 共扰一元论` 命名阶段的探索稿或内部工作稿，保留作理论溯源与写作记录，不作为当前默认公开主包。

另外，当前已开始新增一类内容：

3. `DM-P04.*`、`DM-P09.*` 单焦点稿
   用于把 `DM` 从整包征评推进到单篇可送审稿；当前优先方向仍是 `P04.硬问题不是两种实体的桥接而是内部性的形成.投稿体草案` 与 `P09.危机压缩必须可复归.投稿体草案`，两稿正文现已统一到 `送审清稿` 口径，后续主要工作改为外投执行而非继续扩写正文。
   当前已补出四个直接可用的外投强化件：
   `DM.代表作对外入口.一页纸.md`
   `DM.代表作外投执行清单.md`
   `匿名送审版.硬问题不是两种实体的桥接，而是内部性的形成.md`
   `匿名送审版.危机压缩必须可复归.md`
   并已把 `投稿申请包/DM-P04-申请包` 与 `投稿申请包/DM-P09-申请包` 填到可执行状态：`paper/zh`、`paper/blind`、`submission/` 与 `materials/` 都已具有可直接使用的内容件。
   本轮进一步补出：
   `投稿申请包/`
   作为“单篇论文申请包”主目录，专门承接中英文版、申请指南、配套资料与代码说明。
   当前已一次性补出 6 个申请包，并统一补齐包内投稿说明、投稿信摘要资料页与补充材料清单。

此外，
已开始补建一个次优先级的支撑短稿：

4. `DM-P00.*` 起点竞争稿
   用于把 `P00` 中“为什么是差异 + 非任意约束性，而不是关系 / 过程 / 事件 / 信息 / 约束”这条起点竞争线抽成独立短文；当前已同时具备 `代表作骨架` 与 `投稿体草案` 两个层级，其功能仍是服务 `P00/P11`，优先级低于 `P04/P09`。

当前更稳定的公开主链应理解为：

`DM -> ASTO -> ECET -> TAT -> ODD`

如需查看仓库内当前层级名、历史命名与文件名前缀的统一说明，优先参考：

`00-索引与导航/DM.全栈术语映射与命名关系表.md`

当前整套论文体系的现行总规划，优先参考：

- [`../论文体系总规划（现行版）.md`](../论文体系总规划（现行版）.md)
- [`../学术论文总导航.md`](../学术论文总导航.md)
- [`D:/_Progs/学术论文总导航.md`](D:/_Progs/学术论文总导航.md)

因此：

- 若你现在要对外发布，占位对象应从 `预印本发布包/` 选择，而不是直接上传本目录根部的历史稿。
- 若你现在要打顶刊 / 顶会 / 严肃期刊单篇投稿，应优先使用 `投稿申请包/` 下的标准目录，而不是继续散文件提交。
- 根目录中的早期 `RM` 文稿可继续保留，但应按“历史探索稿 / 溯源材料”理解，不应与当前公开征评主包混用。
- 若你现在要推进正式投稿，应优先围绕 `DM-P04.*`、`DM-P09.*` 进入外投执行：目标期刊版式、匿名版、逐条核引与投稿材料，而不是继续扩写整包入口文案。
- 若你现在要对外介绍这套理论，优先使用 `DM.代表作对外入口.一页纸.md`。
- 若你现在要准备送审材料，优先使用 `匿名送审版.*` 与 `DM.代表作外投执行清单.md`。
