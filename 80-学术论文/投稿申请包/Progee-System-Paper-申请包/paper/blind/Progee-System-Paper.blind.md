# An Evidence-First, State-Machine-Governed System for AI-Assisted Software Engineering

**Status**: blind-submission working draft

## Abstract

AI-assisted software engineering can accelerate implementation, but it also introduces unverifiable claims, weak test oracles, and audit gaps that make many tool reports difficult to trust. This paper presents a system centered on validation-gate state-machine governance and evidence-first persistence. The system organizes execution into explicit state transitions, binds claims to durable evidence objects, and provides a reproducible E2E-based evaluation path. Rather than claiming final maturity, the paper states its current limits in artifact stability, evaluation breadth, and external validity.

## Keywords

AI-assisted software engineering; state-machine governance; evidence-first persistence; auditability; reproducibility; artifact evaluation

## Core contributions

1. A validation-gate state-machine governance model for AI-assisted software engineering.
2. An evidence-first persistence path that turns execution claims into auditable evidence objects.
3. A minimal reproducible evaluation route built on E2E tests and artifact-oriented verification.
