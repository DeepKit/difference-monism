# Progee 结果与主张映射表

## 作用

本表把“论文说了什么”与“审稿人能核什么”压成一张最短映射表，避免 supplement 只剩口头说明。

| 论文主张 | 最小核查问题 | 代码或文档锚点 | 最小输出物 | 当前边界 |
| --- | --- | --- | --- | --- |
| 状态机治理不是口号，而是显式迁移约束 | 是否存在可检查的状态机控制器与测试入口 | `CtrlTaskStateMachine.pas`、`CtrlContractStateMachine.pas`、`tests/TestTaskStateMachine.pas` | `bin/dunitx-results.xml` | 当前不是形式化验证，只是可执行约束与测试证据 |
| 证据优先持久化能把主张绑到可审计对象 | 是否存在证据对象、审计记录与对应导出路径 | `CtrlEvidence.pas`、`CtrlAuditLogs.pas`、`eval/evidence_counts.csv` | `dunitx-results.xml`、`eval/evidence_counts.csv` | 当前更强调审计可追索，不声称所有证据类型都已完整外化 |
| 最小可复现评估路径可由外部执行 | 是否有稳定的复现文档、脚本与测试用例入口 | `docs/Replication.md`、`scripts/run_e2e_tests.ps1`、`tests/E2E_ProjectFlow.pas`、`tests/TestIntegrationE2E.pas` | `bin/dunitx-results.xml` | 依赖 Windows + PostgreSQL test DB，且测试具有 destructive 特征 |
| 表图与论文叙述可被复核 | 是否有从测试库导出表图数据的路径 | `scripts/export_eval_metrics.ps1`、`scripts/eval_exports.sql` | `eval/*.csv` | 当前只保证最小表图导出，不保证全部分析图都自动化 |

## 使用纪律

1. 论文主张若不在本表中找到对应输出物，不应在 supplement 中被写成“已复核”。
2. 若未来改换 venue，本表优先保留，附件命名与体例再单独调整。
3. 审稿人只跑最小路径时，至少应能核到 `状态机治理 + 证据链 + XML 输出` 三件事。
