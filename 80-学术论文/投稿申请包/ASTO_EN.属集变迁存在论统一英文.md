# Attribute-Set Transition Ontology (ASTO): A Unified Structural Grammar for Cross-Domain Application

## Abstract

Between philosophical foundations and domain-specific applications lies a structural gap: how do abstract ontological commitments translate into actionable constraints for engineering, governance, and decision-making? This paper introduces the Attribute-Set Transition Ontology (ASTO), the application bridge layer of Difference Monism (DM). ASTO provides a unified structural grammar — five state types, six temporal stages, seven action sequences, boundary conditions, and exception handling — that all downstream layers (ECET, TAT, ODD, COP, LMM) inherit as precoding before making domain-specific judgments. The paper presents the 1-5-6-7-1 spiral dynamics model and demonstrates ASTO's operation through two bridge examples: ASTO-to-ECET (urban governance) and ASTO-to-COP (AI system diagnosis).

---

## 1. The Problem ASTO Solves

Philosophical foundations answer "what exists and how." Engineering methods answer "what to do next." Between them is a gap: the same structural insight must be translated differently for governance, AI safety, organizational design, and software engineering. Without a shared structural grammar, each domain reinvents its own translation — and the connections between domains are lost.

ASTO fills this gap. It does not invent a new ontology. It compresses DM's ontological grammar (difference → structure → boundary → inheritance → emergence) into a unified precoding language that every downstream layer inherits.

## 2. Core Concepts

### Attribute (属)
An attribute is the identifiable unit of difference at a given observation point. Any attribute can itself be decomposed into further attribute-sets — this infinite regress is real and non-terminating, but not a flaw. Operational validity comes from perturbation purpose, not from attribute exhaustion.

### Set (集)
A set is the current organizational form of attributes under given observation conditions. It is not "the essence of a thing" but a snapshot of its current attribute configuration. The same attributes present differently under different perturbation conditions.

### Transition (变迁)
Transition is the reorganization of attribute-sets under perturbation. ASTO's core claim: **transitions inevitably occur, but the mode of intervention is optional. All transitions = attribute speed modulation (acceleration/deceleration), not creation from nothing.**

---

## 3. The 1-5-6-7-1 Spiral

ASTO's dynamic model is a spiral, not a linear path. It allows jumps, regressions, and parallel tracks.

### The One (一元): Current Existence Slice
At any moment, what an observer faces — a snapshot of the current attribute-set state. This is the starting point and the baseline to which the spiral returns.

### Five States (五态): Spatial Modalities
Attribute-sets can occupy five organizational states:

| State | Name | Characteristic | Downstream Use |
|-------|------|---------------|----------------|
| 1 | 自在 (In-itself) | Vague perception, patterns not yet identified | LMM field detection |
| 2 | 共识 (Consensus) | Verbally agreed, socially accepted patterns | LMM port engagement |
| 3 | 编码 (Encoded) | Formalized, patterns written as explicit rules | COP state encoding input |
| 4 | 物化 (Materialized) | Physically/executably implemented, rules hardened into tools | ODD sealing |
| 5 | 定向 (Self-Directed) | Self-evolving, contains meta-rules for when to change rules | ECET meta-governance |

### Six Stages (六阶): Temporal Phases
The rhythm of attribute-set change over time: Latent → Manifest → Diffusing → Peak → Decaying → Residual. COP uses these to determine signal completeness and freeze triggers.

### Seven Sequences (七序): Action Steps
Identify → Locate → Analyze → Design → Execute → Verify → Iterate. RT6 compresses these into its six-step engagement chain.

## 4. Boundary and Exception: ASTO's Safety Mechanisms

**Boundary** is the condition for structural identifiability — without boundary, you cannot tell where "this structure" begins and ends. Boundary status is precoded as: `clear | blurred | contested`.

**Exception** is what the structure currently cannot handle. Exceptions must be marked, not silently absorbed. FREEZE is triggered when: red-line violation, constraint conflict, irreversible subject impact, or unconfirmed critical sequence jumps.

## 5. Downstream Inheritance

| Layer | ASTO Precoding Provided | How Used |
|-------|------------------------|----------|
| ECET | Five states + boundary status | Determines when governance boundaries hold or distort |
| TAT | Five states + boundary + exception | Determines responsibility threshold triggers |
| ODD | Five states (materialized/self-directed) + seven sequences | Maps structural states to engineering gates |
| COP | Five states + six stages + boundary | Encodes structural state before diagnostic judgment |
| LMM | Five states (in-itself/consensus) + boundary | Provides structural context for frontline field detection |
| WSM | Five states + boundary + responsibility flow patterns | Compresses upstream encoding into frontline pattern labels |

## 6. Bridge Example 1: ASTO → ECET (Urban Bike-Share Governance)

A city faces excessive bike-share deployment. ASTO precoding reveals:
- Bike-share companies: state = 物化 (operations hardened into code)
- Government regulation: state = 共识→编码 transition (rules exist on paper but cannot be enforced)
- Public space governance: state = 自在 (vague dissatisfaction, no formed rules)

Boundary between government and companies: **blurred** — regulations say "reasonable deployment" but "reasonable" has no quantitative definition.

Without ASTO precoding, the policy response would be "stricter enforcement." With ASTO, ECET recognizes: the energy constraint (3 regulators for 500,000 bikes) makes enforcement impossible; the solution must transfer execution costs to companies (geo-fencing, quota auctions).

## 7. Bridge Example 2: ASTO → COP (AI Customer Service Upgrade Decision)

An e-commerce AI refund system shows rising return rates. Operations suspects the AI is "too lenient." Technology says the data shows no anomaly. ASTO precoding reveals the real problem:

- The AI system itself: state = 物化 (rules hardened into code)
- The refund rules: state = 共识 (oral spirit of "reasonable returns should be approved," not precisely encoded)
- Definition of "reasonable return": state = 自在 (each agent and customer understands it differently)

The boundary between "reasonable" and "unreasonable" returns is **blurred**. ASTO reveals this is not a technology problem — it's a consensus→encoding translation problem. The rules lost their boundary judgment when encoded into code. COP recommends: before upgrading the system, conduct a human audit of 100 randomly sampled approved returns to determine whether the problem is in the rules' encoding, not the AI's execution.

---

## 8. What ASTO Is Not

- Not a competing ontology parallel to DM
- Not a replacement for ECET's governance judgments or TAT's responsibility adjudication
- Not a "universal translator" — it can only translate structurally encoded constraints
- Not closed — the attribute-set infinite regress is real; ASTO operates only at the "currently identifiable" level

---

> **ASTO does not tell you what to do. It tells you what state the structure is in — so that ECET, TAT, ODD, COP, and LMM can do their jobs with the same structural map.**
