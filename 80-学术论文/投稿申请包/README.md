# 投稿申请包

> 定位：面向严肃期刊、会议或匿名送审的单篇申请包目录。
> 口径：当前已建立 `6` 个申请包，但成熟度并不相同。

---

## 一、与预印本发布包的分工

这里与 `../预印本发布包/` 不是重复关系，而是两条不同的外发线：

1. `预印本发布包`
   - 负责理论全本公开、征评占位与法源链展示。

2. `投稿申请包`
   - 负责单篇外投、匿名送审、投稿材料、补充材料与复现说明。

最短句：

**理论主链走预印本包，单篇论文走申请包。**

若要先看整个论文外发区的总说明，再回到：

- `../README.md`

---

## 二、一个合格申请包的最低结构

每个申请包至少应包含：

1. `README.md`
2. `paper/zh`
3. `paper/en`
4. `paper/blind`
5. `submission/`
6. `materials/`
7. `code/`

这套结构的目标不是“目录好看”，而是让一篇稿件能独立完成：

- 送审
- 匿名
- 投稿说明
- 补充材料
- 代码或复现边界说明

## 五、当前真正剩下的动作

申请包目录体系已经齐备，当前主要差额不再是“再建目录”，而是：

1. 锁定或确认唯一 venue
2. 完成匿名终核
3. 填实作者与平台元数据
4. 导出最终提交文件
5. 完成外部平台提交
