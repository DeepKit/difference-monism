# 投稿申请包

> 日期：2026-03-30
> 定位：面向顶刊 / 顶会 / 严肃期刊投稿的单篇申请包目录。

## 一、与预印本发布包的分工

这里与 `../预印本发布包/` 不是重复关系，而是两条不同的外发线：

1. `预印本发布包`
适合理论全本、公开征评、公开占位、全链路背景发布。

2. `投稿申请包`
适合单篇论文、专精尖投稿、匿名送审、配套材料与代码/复现说明组织。

最短句：

**理论最好走“全本预印本包”，论文最好走“单篇申请包”。**

## 二、一个合格申请包的最低结构

每个申请包应尽量是一个独立目录，至少包含以下 5 类内容：

1. `README.md`
一句话说明这篇论文是什么、当前状态如何、该目录怎么用。

2. `paper/`
放论文主文入口，尽量区分：
- `zh/` 中文版
- `en/` 英文版
- `blind/` 匿名送审版

3. `submission/`
放申请 / 投稿 / 送审相关说明：
- 申请指南
- 投稿检查清单
- 期刊或会议适配说明
- 投稿信 / 摘要版简介 / 分类信息

4. `materials/`
放配套资料：
- 图表
- 补充材料
- 附录入口
- 证据链、案例表、补充说明

5. `code/`
若有代码、复现脚本、artifact 包、仓库说明，应统一在这里说明；若无代码，也应明确写“本稿无代码依赖”。
