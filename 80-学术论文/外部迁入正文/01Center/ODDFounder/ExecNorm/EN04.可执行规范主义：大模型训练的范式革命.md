**From Alchemy to Chemistry: Deterministic Alignment for LLMs via Executable Norms**

**Author**: Yi Fu (ODDFounder)

## 摘要 (Abstract)

当前大模型（LLM）的对齐技术（如 RLHF）正深陷“不可解释”与“不可扩展”的泥潭。我们将训练过程比作古代的“炼丹术”：依赖黑箱（Reward Model）、依赖运气（随机梯度）、且严重依赖不可靠的人类主观反馈。

本文提出 **RLEN (Reinforcement Learning from Executable Norms)**，一种将**可执行规范主义**引入 AI 训练的全新范式。我们主张：**对齐的本质不是让模型“像人”，而是让模型“守规”**。通过将人类的价值观转化为形式化、可执行的代码或逻辑约束（Executable Norms），我们能够以**确定性**的计算信号替代模糊的人类反馈。

这不仅是工程效率的飞跃，更是 AI 治理哲学的回归：让人类从繁琐的标注劳役中解放出来，回归到**立法者（Legislator）**的崇高位置，通过制定“宪法”来规训智能体。

## 第一章：RLHF 的“黑箱困境”与哲学危机

### 1.1 “炼丹”的极限
RLHF（人类反馈强化学习）是当前的主流，但它本质上是一种**统计学上的逼近**。
*   **不稳定性**：标注员 A 说好，标注员 B 说坏。模型学到的是“平均值”，甚至是“偏见”。
*   **不可解释性**：当 Reward Model 给出一个负分时，工程师不知道是因为“逻辑错误”、“语气不对”还是“政治不正确”。
*   **对齐税 (Alignment Tax)**：为了安全，模型往往牺牲了智力。这是因为由于缺乏精确的边界，模型选择了“过度保守”。

### 1.2 人的异化
在 RLHF 体系中，人沦为了**“数据矿工”**。高智商的工程师被迫去标注低级的对话数据，这是对人类智能的浪费。
**哲学反思**：人类应当是规则的**制定者**，而不是规则的**数据源**。

## 第二章：RLEN——可执行规范主义的工程投射

### 2.1 核心定义
**RLEN (Reinforcement Learning from Executable Norms)** 是指：
在训练循环（Training Loop）中，使用**形式化、确定性、可自动执行的规范**（而非人类偏好模型）作为主要的奖励（Reward）来源。

**公式对比**：
*   **RLHF**: $Reward = P_{\theta}(Good | Output, Human)$ —— *概率的、模糊的*
*   **RLEN**: $Reward = Norm_{code}(Output) ? +1 : -\infty$ —— *确定的、二元的*

### 2.2 规范的工程形态
给工程师看什么叫“规范”：
*   **代码级规范**：`assert(compile(output) == success)` —— 代码必须能跑。
*   **结构级规范**：`assert(json.validate(output, schema))` —— JSON 必须合法。
*   **事实级规范**：`assert(kg.verify(output))` —— 必须符合知识图谱事实。
*   **安全级规范**：`assert(safety_filter(output))` —— 不得包含违禁词。

## 第三章：RLEN 的三阶段工程实战

### 3.1 Pre-training 阶段：数据的“宪法化清洗”
*   **痛点**：Common Crawl 数据集里充满了偏见、暴力和垃圾。
*   **RLEN 解法**：用可执行规范作为过滤器。
    *   *“凡是不符合逻辑一致性的文本，丢弃。”*
    *   *“凡是包含仇恨言论的文本，丢弃。”*
*   **哲学意义**：**垃圾进，垃圾出 (GIGO) -> 规范进，精品出 (NIGO)**。我们在源头为模型注入了“底线”。

### 3.2 SFT 阶段：合成数据的“流水线生产”
*   **痛点**：高质量 SFT 数据极其稀缺且昂贵。
*   **RLEN 解法**：**Constitutional Data Synthesis (宪法数据合成)**。
    1.  让 Teacher Model 生成 100 个答案。
    2.  用 Executable Norms 自动评分。
    3.  只保留评分满分的答案作为 Student Model 的训练数据。
*   **价值**：零成本构建百万级高质量数据集。

### 3.3 RL 阶段：在线对齐的“确定性梯度”
*   **痛点**：Reward Model 被破解（Reward Hacking）。
*   **RLEN 解法**：将规范直接封装为 Reward Function。
    *   模型生成：“1+1=3”
    *   规范（Python）：`return -100` (逻辑错误)
    *   模型生成：“1+1=2”
    *   规范（Python）：`return +10`
*   **效果**：模型迅速收敛。因为**真理是不需要“概率”的**。

## 第四章：实战案例——训练一个“严谨的代码助手”

让我们通过一个具体案例，看 RLEN 如何吊打 RLHF。

**目标**：训练一个只写正确 Python 代码的模型。

*   **RLHF 路径**：
    1.  雇佣 100 个程序员。
    2.  看模型生成的代码，打分（1-5分）。
    3.  **问题**：程序员也会看走眼；程序员很贵；反馈很慢。

*   **RLEN 路径**：
    1.  **定义规范**：
        *   `Norm 1`: 代码必须通过 Pylint 检查。
        *   `Norm 2`: 代码必须通过 Unit Test。
        *   `Norm 3`: 圈复杂度（Cyclomatic Complexity）必须小于 10。
    2.  **训练**：
        *   模型生成代码 -> 扔进 Sandbox 跑规范 -> 自动计算 Reward。
    3.  **结果**：
        *   **速度**：一秒钟跑 1000 个 Epoch。
        *   **质量**：模型学会了“不仅要写对，还要写得优雅（符合 Norm 3）”。

## 第五章：从黑箱到白箱——对齐的可解释性革命

### 5.1 为什么 RLEN 更安全？
*   **RLHF**：我们不知道模型为什么变好了。可能是因为它真的学好了，也可能是它学会了骗标注员。
*   **RLEN**：我们清楚地知道模型通过了哪些规范。如果模型出错了，我们只需要**Debug 规范**。
*   **结论**：RLEN 将 AI 对齐从“心理学问题”变成了**“软件工程问题”**。

### 5.2 哲学升华：机器的“法治”
RLEN 的终极愿景，是在机器内部建立一个**“法治社会”**。
*   Prompt 是“行政命令”。
*   Executable Norms 是“法律”。
*   Training Loop 是“执法过程”。

我们不再通过“调教”（Fine-tuning）来改变模型的性格，而是通过“立法”（Norms）来约束模型的行为。这是人类与超级智能共存的唯一可持续路径。
