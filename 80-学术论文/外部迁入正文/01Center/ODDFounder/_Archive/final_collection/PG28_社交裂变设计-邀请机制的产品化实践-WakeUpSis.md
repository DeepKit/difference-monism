## 摘要

工具类产品最难的是**留存**和**传播**。用完即走是工具的宿命。为了打破这个宿命，WakeUpSis (相公) 设计了一套**基于"第三方视角"的社交裂变机制**：邀请闺蜜/家人填问卷。这不仅是为了增长，更是为了补全数据拼图。本文解析这套机制的心理学原理与产品实现。

---

## 一、为什么用户愿意分享？

在相亲这个隐私场景下，让用户分享链接是很难的。除非——**分享能带来巨大的价值**。

### 1.1 确认感 (Validation)
"当局者迷，旁观者清"。用户不确定自己看人准不准，急需闺蜜的参谋。
*   *话术*："我看不准他，集美们帮我掌掌眼！"

### 1.2 虚荣心与自我探索 (Narcissism)
用户都想知道"别人眼里的我"是什么样的。
*   *话术*："在你眼里，我是那种'恋爱脑'吗？快来匿名评价我！"

### 1.3 共同话题 (Social Currency)
吐槽相亲对象，是闺蜜聚会永恒的话题。WakeUpSis 将这个话题**工具化**了。

---

## 二、裂变闭环设计

### 2.1 邀请流程
1.  **发起**：用户A 创建一个相亲档案（或自我画像）。
2.  **分享**：生成一张带有二维码的海报，或者一个小程序卡片。
    *   *文案*: "邀请你成为我的【恋爱军师】"
3.  **参与**：闺蜜B 点击进入。
    *   **无需注册**：直接开始填问卷（降低门槛）。
    *   **题目设计**：有趣、犀利、带梗（"她遇到渣男的概率是？A. 0% B. 100%"）。
4.  **转化**：闺蜜B 填完后，看到结果："原来我对A的看法是这样... 我也想测测自己！" -> **点击创建自己的档案**。

### 2.2 数据价值
这个裂变不仅仅是为了拉新，更是为了**数据清洗**。
*   **自我问卷**：用户A说自己"脾气很好"。
*   **闺蜜问卷**：闺蜜B说A"一点就炸"。
*   **AI 结论**：用户A存在自我认知偏差（权重调整）。

这种**多视角交叉验证 (Cross-Validation)**，使得 WakeUpSis 的画像比单一视角的竞品准得多。

## 三、工程实现

### 3.1 邀请关系表
在 `TECH_DESIGN.md` 的数据库设计中：

```sql
CREATE TABLE invitations (
    id TEXT PRIMARY KEY,
    user_id TEXT REFERENCES users(id),  -- 发起人
    type TEXT,                          -- 'friend' or 'family'
    invitee_name TEXT,                  -- 闺蜜昵称
    answers JSON,                       -- 闺蜜填写的答案
    token TEXT UNIQUE                   -- 一次性访问令牌
);
```

### 3.2 权限控制
*   **一次性链接**：为了保护隐私，生成的邀请链接通常是一次性的或有时效的。
*   **匿名机制**：允许闺蜜选择匿名填写，鼓励说真话。

## 四、增长效果预估

假设每个用户平均有 3 个核心闺蜜。
*   K因子 (Viral Coefficient) = (每个用户发出的邀请数) * (转化率)。
*   如果转化率能做到 35%（即3个闺蜜里有1个也成了用户），那么 K > 1，产品将实现**指数级自增长**。

WakeUpSis 的产品特性天然适合这种裂变，因为**相亲本身就是一种群体性焦虑**，需要群体性抱团取暖。

---

## 五、总结

最好的裂变不是送红包，而是**顺应人性的需求**。
WakeUpSis 的邀请机制抓住了用户"渴望被理解"和"渴望被参谋"的心理，将社交关系链变成了产品的核心功能模块。
**产品即营销**，这就是增长黑客的最高境界。
