# 双环认知架构：AI如何像人类一样思考

> **作者**: Fuyi ( ODDFounder  fuyi.it@live.cn )

## 摘要

为什么大多数AI Agent只能处理短期任务，却无法积累长期经验？因为它们缺乏类似人类的"认知双循环"。本文深入解析WizAxis系统的核心引擎——**ReMe双环认知架构**。它模仿人类的"快思考"（System 1）和"慢思考"（System 2），实现了AI从"被动响应"到"主动进化"的质变。

---

## 一、AI的"失忆症"与"短视症"

目前的LLM应用普遍存在两个致命缺陷：
1.  **失忆症**：聊了一千句，换个Session就全忘了。
2.  **被动性**：只能针对你的Prompt做反应，不懂得反思和总结。

这就像一个永远在打临时工的实习生，不管做过多少项目，能力永远停留在第一天。

认知心理学家丹尼尔·卡尼曼在《思考，快与慢》中提出，人类的大脑有两套系统：
*   **System 1（快思考）**：直觉、无意识、快速反应。（例如：看到红灯踩刹车）
*   **System 2（慢思考）**：逻辑、有意识、深度分析。（例如：解复杂的数学题，或者反思今天的错误）

现有的RAG（检索增强生成）大多只是System 1的拙劣模仿——简单地把资料搜出来塞给AI。**WizAxis**引入了真正的System 2，构成了**双环认知架构**。

---

## 二、WizAxis ReMe双环架构详解

WizAxis的核心公式是：
> **WizAxis Cognition = (Fast Loop: Context Injection) + (Slow Loop: Active Refinement)**

### 2.1 Loop A: 快思考 (Fast Loop) —— 毫秒级直觉

**场景**：用户正在与AI对话，或者Router正在决定调用哪个工具。
**目标**：**< 200ms** 响应，注入高价值的"直觉"。

在这个循环中，WizAxis不进行复杂的推理，而是基于当前的Query，并行执行**混合检索（Hybrid Retrieval）**：
1.  **Semantic Search** (pgvector)："这看起来像上次处理过的数据库迁移任务。"
2.  **Keyword Search** (pg_trgm)："用户提到了 'Progee'，检索相关文档。"
3.  **Graph Traversal** (Recursive CTE)："当前是 'Coding' 阶段，且角色是 'Reviewer'，自动加载代码审查规范。"

**输出**：一组精简的、高相关性的Prompt片段（Context），直接注入到LLM的System Prompt中。

> **效果**：AI"本能地"知道了你的偏好和项目规范，无需你重复。

### 2.2 Loop B: 慢思考 (Slow Loop) —— 深度反思与进化

**场景**：后台异步处理，发生在用户对话结束之后。
**目标**：从海量日志中**提炼智慧**，实现自我进化。

这是大多数AI系统缺失的一环。WizAxis通过一个名为 **Refiner** 的后台Worker实现这一过程。

**Refiner工作流**：
1.  **观察 (Observe)**：监听系统产生的 Trace、Log 和用户反馈。
2.  **压缩 (Compress)**：将冗长的 Short-Term Memory（短期对话）压缩为 Episodic Memory（情景记忆）。
    *   *原始*："你好，把这个变量名改成驼峰...不对，首字母小写..."
    *   *压缩后*："用户偏好 camelCase 命名风格。"
3.  **挖掘 (Mine)**：从失败中学习。
    *   *现象*：最近5次调用 `search_tool` 都超时了。
    *   *提炼*：生成一条 Tool Memory —— "使用 `search_tool` 时建议设置 timeout > 10s。"
4.  **遗忘 (Forget)**：清理低价值信息。基于遗忘曲线，删除那些不再被访问的陈旧记忆，防止知识库"过胖"。

> **效果**：系统使用得越多，Refiner提炼的"直觉"就越准。今天的错误，明天就不会再犯。

## 三、架构落地的关键技术

WizAxis是如何在工程上实现这一点的？答案是 **All-in-PG**。

我们没有引入复杂的向量数据库或图数据库，而是利用 **PostgreSQL** 的强大插件生态，在一个数据库内实现了所有认知功能：

| 认知功能 | PostgreSQL技术栈 | 作用 |
| :--- | :--- | :--- |
| **语义记忆** | `pgvector` | 存储向量化知识，支持相似度检索 |
| **关键词记忆** | `pg_trgm` | 支持模糊匹配和关键词检索 |
| **关联记忆** | `Recursive CTE` | 处理知识图谱和层级关系 |
| **记忆队列** | `SKIP LOCKED` | 构建高并发的 Refiner 任务队列 |
| **权限隔离** | `RLS (Row Level Security)` | 确保记忆不泄露，严格的多租户隔离 |

这种极简架构（Minimalist Architecture）不仅降低了运维成本，更重要的是保证了**数据的一致性**——快思考读取的数据，正是慢思考刚刚提炼写入的数据。

---

## 四、为什么这很重要？

双环认知架构解决了AI落地的"最后一公里"问题：**经验积累**。

*   没有双环，AI只是一个查字典的机器人。
*   有了双环，AI变成了一个**有经验的专家**。它不仅知道书本上（文档）写了什么，还记得上次项目（情景）里踩过的坑，更懂得根据你的习惯（画像）调整自己的工作方式。

在WizAxis中，我们不追求模型参数有多大，我们追求**认知闭环有多快**。一个能快速从错误中学习的小模型，远比一个固步自封的大模型更有价值。

这就是AI像人类一样思考的秘密：**在行动中依靠直觉（Fast Loop），在休息时依靠反思（Slow Loop）。**
