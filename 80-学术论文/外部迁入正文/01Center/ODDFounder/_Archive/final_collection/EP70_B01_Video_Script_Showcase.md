# 脚本：B01 - 10分钟从零到CRUD API (演示视频)

> **时长**: 2-3分钟
> **风格**: 快节奏，录屏 + 字幕/旁白
> **BGM**: 轻快，科技感

---

## 0:00 - 0:15 [Intro]
**(画面)**: 黑屏，出现文字 "传统开发一个 CRUD 接口需要多久？"
**(画面)**: 快速闪过：写 Model，写 Controller，写 Service，写 SQL，写 Test，调试报错... 时钟飞速转动。
**(旁白/字幕)**: "1小时？2小时？"
**(画面)**: 切换到 Progee 界面。文字："用 Progee，只要 10 分钟。"

## 0:15 - 0:45 [Define: 定义契约]
**(画面)**: 用户在 Progee 输入框输入："我要一个图书管理 API，包含增删改查。"
**(画面)**: Progee 迅速生成 YAML 契约。
**(特写)**: 鼠标划过 Input/Output 定义。
**(旁白)**: "第一步，定义。我们不写代码，我们只告诉 AI 我们要什么。看，契约生成了。输入、输出、验收标准，一目了然。"
**(动作)**: 用户点击 "Approve Contract"。

## 0:45 - 1:30 [Execute: AI并发执行]
**(画面)**: 屏幕分裂成三个窗口。
*   左边：Architect Agent 正在生成 SQL (`CREATE TABLE books...`).
*   中间：Worker Agent 正在写 Python 代码 (`FastAPI router...`).
*   右边：QA Agent 正在写测试用例 (`test_create_book...`).
**(特效)**: 进度条飞速前进。
**(旁白)**: "第二步，执行。Progee 的虚拟员工们正在并行工作。架构师建表，工人写接口，测试员写用例。你只需要喝口咖啡。"

## 1:30 - 2:00 [Verify: 自动验收]
**(画面)**: 终端窗口自动弹出，运行 `pytest`。
**(画面)**: 一片绿色通过。
**(特写)**: 突然变红！"Error: Title cannot be empty."
**(画面)**: Worker Agent 自动检测到错误，修改代码，重新运行。
**(画面)**: 再次变绿。
**(旁白)**: "第三步，验证。不仅是写完，更要写对。看，AI 发现了 Bug，并且自己修好了它。"

## 2:00 - 2:30 [Seal & Result]
**(画面)**: 界面显示 "Task Completed"。
**(动作)**: 用户打开 Swagger UI，现场演示：创建一个 Book，查询列表。成功。
**(旁白)**: "第四步，封印。代码入库，功能上线。全过程不到 10 分钟。这就是 ODD 的力量。"

## 2:30 - 2:45 [Outro]
**(画面)**: Progee Logo。
**(字幕)**: "Progee: Don't code. Define."
