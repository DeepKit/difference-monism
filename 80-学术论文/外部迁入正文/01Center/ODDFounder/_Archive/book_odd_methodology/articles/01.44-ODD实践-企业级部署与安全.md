# ODD实践：企业级部署与安全 (Enterprise Deployment)

> **版本**: v2.0
> **基于**: Progee 2.0 部署指南
> **标签**: 私有化, 安全, 审计, 运维

---

## 1. 架构概览

Progee 企业版采用 **B/S 架构**（但在单机版中退化为 C/S 混合）。

*   **Backend**: Python FastAPI (处理业务逻辑、AI 调度)。
*   **Database**: PostgreSQL (业务数据) + Redis (缓存/队列) + pgvector (向量)。
*   **Gateway**: Nginx (SSL 卸载、负载均衡)。

---

## 2. 安全设计

### 2.1 知识熔断 (Knowledge Circuit Breaker)
在将 Prompt 发送给 LLM 之前，系统会经过一层 **DLP (Data Loss Prevention)** 过滤器。
*   **正则扫描**：检测 AK/SK、手机号、身份证。
*   **语义扫描**：使用轻量级本地模型检测敏感商业机密。
*   **熔断**：一旦发现，立即拦截请求，并记录审计日志。

### 2.2 物理隔离
支持对接 **Ollama** 或 **vLLM** 部署的私有模型（如 Llama 3）。
在这种模式下，**一根网线都不需要**。所有的推理、生成、存储都在企业内网完成。

---

## 3. 部署流程

我们提供了一键部署脚本 `deploy.sh`：

1.  **环境检查**：检查 Docker, NVIDIA Driver。
2.  **初始化 DB**：执行 SQL Migration，创建 Schema。
3.  **启动服务**：`docker-compose up -d`。
4.  **健康检查**：轮询 `/health` 接口直到服务就绪。

---

## 4. 总结

Progee 2.0 的部署方案证明了：**ODD 是可以被容器化、标准化的**。
企业不需要购买昂贵的 SaaS 服务，完全可以在自己的机房里搭建一套 AI 软件工厂，既享受了 AI 的效率，又保证了数据的绝对安全。
