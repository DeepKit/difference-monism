# 03.37 多步骤任务编排：可中断可恢复的长流程设计

## 摘要

生成一个视频需要经过：下载素材 -> 提取音频 -> 转录字幕 -> 翻译 -> 语音合成 -> 视频合成。这是一个典型的**长流程 (Long-Running Process)**。任何一步报错（如下载超时），整个任务都会失败。**Chain2VFactory** 设计了一套**可中断、可恢复、可重试**的任务编排引擎。本文解析这套引擎的实现。

---

## 一、流水线 (Pipeline) 模型

Chain2V 将视频生成过程抽象为一个 **DAG (有向无环图)**，但在最简单的场景下，它是一个线性 Pipeline。

```python
steps = [
    DownloadStep(),
    ExtractAudioStep(),
    TranscribeStep(),
    TranslateStep(),
    SynthesizeAudioStep(),
    RenderVideoStep()
]
```

---

## 二、状态检查点 (Checkpoint)

为了实现断点续跑，每一步执行**前**和**后**，都会进行状态快照。

### 2.1 工作区结构
每个任务有一个独立的 `workspace/task_{id}/` 目录。
*   `step_1_download/`
*   `step_2_extract/`
*   `state.json` (记录当前进度)

### 2.2 幂等性执行
`Runner` 在执行某一步之前，先检查 `state.json`。
*   如果 `step_1` 标记为 `completed`，且输出文件存在，直接**跳过**，复用结果。
*   这使得用户可以在任务失败后，修改配置（如换个翻译引擎），然后点"重试"，系统会自动跳过已完成的耗时步骤（如下载和转录）。

---

## 三、异常处理与人工干预

### 3.1 暂停与恢复
用户可以点击"暂停"。系统会在当前 Step 完成后停止下一以 Step 的启动，并将状态设为 `PAUSED`。
恢复时，直接读取 `state.json` 继续。

### 3.2 错误恢复
如果在"翻译"步骤报错（如 API 欠费），任务挂起。
用户充值后，点击"重试"。
系统从"翻译"步骤重新开始，而不是从头下载视频。

### 3.3 人工介入 (Human-in-the-loop)
Chain2V 允许用户在步骤之间插入**断点**。
*   例如：在"转录"完成后暂停。
*   用户打开生成的 `subtitles.srt`，手动修正错别字。
*   点击"继续"，系统使用修正后的字幕进行下一步渲染。

## 四、代码实现 (Python)

```python
class PipelineRunner:
    def run(self, task):
        for step in task.steps:
            if self.is_completed(step):
                continue
            
            try:
                step.execute()
                self.mark_completed(step)
            except Exception as e:
                self.mark_failed(step, e)
                raise # 停止执行
```

---

## 五、总结

长流程任务的核心不是"跑通"，而是**"容错"**。
Chain2V 的编排引擎通过**文件系统作为状态后端**，实现了极低成本的持久化和恢复机制。这种设计让复杂的视频生成任务变得像断点续传下载一样可靠。
