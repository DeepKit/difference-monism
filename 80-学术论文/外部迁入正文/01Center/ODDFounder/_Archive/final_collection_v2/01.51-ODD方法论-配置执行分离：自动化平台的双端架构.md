# 01.51 配置执行分离：自动化平台的双端架构

> **作者**: Fuyi ( ODDFounder  fuyi.it@live.cn )
> **日期**: 2026-01-11
> **标签**: 架构设计, C/S, 自动化, wyjx, 远程控制

---

## 摘要

传统的自动化工具通常是"编辑器+执行器"合体的单体应用。但这在批量挂机（工作室）场景下行不通。**wyjx** 采用了 **Config端 (管理)** + **Exec端 (执行)** 的分离架构。Config端负责写脚本、配任务；Exec端是一个极轻量的无头进程，负责在虚拟机/云手机里跑脚本。两者通过 TCP/WebSocket 通信。

---

## 一、为什么分离？

*   **资源占用**：执行端通常运行在资源受限的环境（如 1核2G 的虚拟机）。它不需要带一个庞大的 IDE。
*   **集中管理**：工作室有 100 台手机。需要在中控台（Config端）一键下发脚本到 100 台设备。
*   **稳定性**：执行端越简单，越不容易崩。

---

## 二、双端架构设计

### 2.1 Config 端 (Delphi)
*   **功能**：脚本编辑、图像抓取、任务编排、日志监控。
*   **数据**：存储在本地 SQLite 或 MySQL。
*   **操作**：用户点击"部署"，将 Task JSON 序列化并通过网络发送给 Exec 端。

### 2.2 Exec 端 (C++/Go/Delphi Console)
*   **功能**：接收指令、执行动作、回传日志。
*   **特点**：
    *   **Headless**: 没有界面，只有托盘图标或后台服务。
    *   **Lightweight**: 内存占用极低。
    *   **Hot Reload**: 接收到新配置，热加载，无需重启。

### 2.3 通信协议
自定义的二进制协议或 Protobuf。
*   `CMD_DEPLOY_TASK`: 下发任务。
*   `CMD_START/STOP`: 启停控制。
*   `EVENT_LOG`: 实时日志回传。
*   `EVENT_SCREENSHOT`: 异常截图回传。

---

## 三、开发调试体验

虽然是分离的，但 wyjx 提供了**远程调试**体验。
在 Config 端点击"截图"，Exec 端立即截取当前屏幕并传回。用户可以在 Config 端直接在截图上选点、取色，生成代码，然后实时同步过去。
这种 **Remote IDE** 的体验，抹平了物理距离。

---

## 四、总结

配置执行分离是自动化工具走向规模化的必经之路。
它体现了**控制面 (Control Plane)** 与 **数据面 (Data Plane)** 分离的现代架构思想。让写脚本的人坐办公室，让跑脚本的机器进机房。
