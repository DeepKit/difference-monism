# 01.53 API Key自动轮换：让AI工具永不断线

> **作者**: Fuyi ( ODDFounder  fuyi.it@live.cn )
> **日期**: 2026-01-11
> **标签**: CLI, 账号管理, 高可用, 自动化

---

## 摘要

当我们使用 Intermediate Model, OpenAI 等大模型 API 时，经常遇到两个问题：一是余额不足突然断线，二是并发限制导致请求失败。**cli-account-manager** 提供了一个系统级的解决方案：**API Key 自动轮换**。通过后台监控余额和请求状态，它能在毫秒级自动切换到备用账号，确保 AI 服务永不掉线。

---

## 一、为什么需要自动轮换？

对于重度 AI 用户或企业级应用：
*   **余额焦虑**：不想时刻盯着余额表，但又怕半夜跑批处理时没钱了。
*   **限流规避**：单个账号有 RPM (Request Per Minute) 限制，多账号轮询可以突破此限制。
*   **容灾**：某个 Key 被封号或异常时，系统应自动隔离并切换。

---

## 二、自动轮换架构

cli-account-manager 采用了一个后台守护进程（或定时任务）来维护账号池的状态。

### 2.1 核心组件 (`CtrlAutoSwitch.pas`)

1.  **Account Pool (账号池)**：存储所有账号的 Key、余额、状态。
2.  **Balance Monitor (余额监控)**：定时（如每10分钟）查询当前活跃账号的余额。
3.  **Strategy Engine (策略引擎)**：决定何时切换。
    *   *规则*：当 `CurrentBalance < Threshold` 时，触发切换。
4.  **Environment Injector (环境注入)**：这是最核心的魔法。它通过修改**用户级环境变量**，让所有依赖该变量的工具自动获取新 Key。

### 2.2 工作流程

1.  **Check**: 后台线程检测到当前 Key `sk-xxx...A` 余额仅剩 0.5 元（低于阈值 1.0 元）。
2.  **Select**: 从数据库中查找余额充足的备用 Key `sk-xxx...B`。
3.  **Switch**: 调用 Windows API `SetUserEnvironmentVariable('AI_API_KEY', 'sk-xxx...B')`。
4.  **Notify**: 弹出 Toast 通知用户 "已自动切换到账号 B，余额 50.0 元"。
5.  **Restore**: 如果用户正在运行终端，甚至可以自动向活跃终端发送指令刷新环境变量。

## 三、代码实现细节

### 3.1 环境变量注入

```pascal
function TWinAPIUtils.SetUserEnvVariable(const Name, Value: string): Boolean;
begin
  // 1. 写入注册表 HKCU\Environment
  Reg.WriteString(Name, Value);
  // 2. 广播 WM_SETTINGCHANGE 消息，通知所有顶层窗口
  SendMessageTimeout(HWND_BROADCAST, WM_SETTINGCHANGE, ...);
end;
```

这确保了新开的终端窗口会立即生效。

### 3.2 失败重试机制

在查询余额时，网络可能波动。`DoQueryWithRetry` 实现了指数退避策略：
*   第一次失败：等待 1s 重试。
*   第二次失败：等待 2s 重试。
*   第三次失败：标记该账号为"异常"，并尝试切换到下一个。

---

## 四、不仅仅是 Key

这套机制不仅用于 API Key，还可以用于：
*   **Git Token 轮换**。
*   **Proxy 节点轮换**。
*   **云服务 Access Key 轮换**。

cli-account-manager 将**配置管理**从应用层下沉到了**OS环境层**，实现了一种无侵入的、系统级的高可用方案。
