# 03.14 WebView2浏览器自动化：多LLM并行测试实战

> **作者**: Fuyi ( ODDFounder  fuyi.it@live.cn )
> **日期**: 2026-01-11
> **标签**: 自动化测试, WebView2, LLM, 并行处理, Touchstone

---

## 摘要

如何同时测试 Advanced Model, Claude-3, Gemini-1.5 的响应效果？传统的 API 测试无法覆盖"Web 独占"的功能（如某些模型的联网搜索）。**Touchstone** 采用 **WebView2 (Edge内核)** 技术，构建了一个**并行浏览器矩阵**。它能同时操控 8+ 个浏览器窗口，模拟人类操作，截取 LLM 的回复，并进行横向对比。本文揭秘这套高并发自动化系统的实现。

---

## 一、API 不够用，浏览器来凑

很多 AI 模型（尤其是新出的或 Web 版特有的功能）并没有第一时间开放 API，或者 API 价格昂贵。
直接自动化 Web 界面（Chat Interface）成为了获取最新模型能力的唯一途径。

挑战在于：**浏览器极其消耗资源**，且 DOM 结构经常变动。

---

## 二、Touchstone 的并行架构

Touchstone 没有使用 Selenium 或 Puppeteer（因为它们太重且配置麻烦），而是直接使用了 Windows 原生的 **WebView2控件**。

### 2.1 多实例管理 (`CtrlProvider.pas`)

*   **Provider 抽象**：每个 LLM 厂商（OpenAI, Google, Bing）被抽象为一个 `TProvider`。
*   **WebView 池**：系统启动时，根据启用的 Provider 数量，创建对应数量的 `TFrameBrowser` 实例。
*   **资源隔离**：每个 WebView 使用独立的 `UserDataFolder`，确保 Cookie、LocalStorage 互不干扰。这让我们可以在一个 App 里同时登录 8 个不同的 Google 账号（如果需要）。

### 2.2 注入与控制 (`ScriptJs`)

为了控制页面，Touchstone 向每个 WebView 注入了一段 `preload.js`。
*   **输入**：`setValue('.input-area', 'Hello AI')` + `click('.send-btn')`。
*   **输出**：通过 `MutationObserver` 监听回复区域的 DOM 变化。
*   **回传**：通过 `chrome.webview.postMessage` 将提取到的文本实时回传给 Delphi 主程序。

---

## 三、并行执行器 (`ModelTestResult.pas`)

当用户点击"开始测试"时：
1.  **Broadcast**: 主控向所有 WebView 广播 Prompt。
2.  **Parallel Execution**: 8 个浏览器同时开始工作（输入 -> 发送 -> 等待）。
3.  **Stream Capture**: Delphi 端实时接收每个模型的流式输出，并在统一的 Dashboard 上并排显示。
4.  **Completion**: 当某个模型输出结束标记时，记录耗时和 Token 数（估算）。

这比传统的"串行测试"快了 N 倍（N = 模型数量）。

---

## 四、技术难点：Cookie 持久化

Web 自动化最怕掉登录。
Touchstone 利用 WebView2 的特性，将 `UserDataFolder` 指向本地的 `data/profiles/{provider_id}` 目录。
只要用户登录过一次，Cookie 就会被持久化保存。下次启动 App，直接就是登录状态，无需任何自动化登录脚本。

---

## 五、总结

Touchstone 展示了 **WebView2** 在桌面自动化领域的强大潜力。
它比 Electron 更轻（共享系统 Edge 运行时），比 Selenium 更快（进程内通信）。
通过这套并行架构，我们不仅实现了对 LLM 的"赛马"评测，更为构建**基于 Web 的 AI Agent** 提供了一个轻量级的基础设施。
