# 03.02 多数据库兼容层设计：一套代码支持PG和SQLite

> **作者**: Fuyi ( ODDFounder  fuyi.it@live.cn )
> **日期**: 2026-01-11
> **标签**: 数据库适配, PostgreSQL, SQLite, ORM, 设计模式

---

## 摘要

UniBase 需要同时支持 **PostgreSQL**（生产环境）和 **SQLite**（单机版/测试环境）。然而，SQL 方言的差异（如分页、自增ID、日期函数）让跨库支持变得困难。**doQry** 引入了一个轻量级的 **Dialect Adapter (方言适配器)** 层，让一套代码能无缝运行在不同数据库上。

---

## 一、方言的巴别塔

*   **分页**: PG 用 `LIMIT x OFFSET y`，MSSQL (旧) 用 `TOP`，Oracle 用 `ROWNUM`。
*   **自增ID**: PG 用 `RETURNING id`，SQLite 用 `last_insert_rowid()`。
*   **字符串连接**: PG 用 `||`，SQL Server 用 `+`。

如果我们在业务代码里写 `if DBType = PG then ... else ...`，代码将无法维护。

---

## 二、适配器模式实现 (`uDoQryDialect.pas`)

我们定义了一个抽象基类 `TDoQryDialect`：

```pascal
type
  TDoQryDialect = class
    function GetLimitSql(Limit, Offset: Integer): string; virtual; abstract;
    function GetLastInsertIdSql: string; virtual; abstract;
    function NormalizeDate(DateStr: string): string; virtual; abstract;
  end;
```

### 2.1 PG 适配器
```pascal
function TPostgresDialect.GetLastInsertIdSql: string;
begin
  Result := 'RETURNING id'; // 拼接到 INSERT 语句末尾
end;
```

### 2.2 SQLite 适配器
```pascal
function TSQLiteDialect.GetLastInsertIdSql: string;
begin
  Result := '; SELECT last_insert_rowid() as id'; // 拼接到语句后作为第二条查询
end;
```

---

## 三、配置中的宏替换

在 `sql.json` 配置中，我们支持一种特殊的宏语法：

```json
{
  "sql": "SELECT * FROM users ORDER BY id {{LIMIT_OFFSET}}"
}
```

doQry 在运行时，会根据当前连接的数据库类型，调用对应的 Dialect 方法，将 `{{LIMIT_OFFSET}}` 替换为正确的 SQL 片段。

---

## 四、总结

通过**方言适配器**和**配置宏**，UniBase 实现了**Write Once, Run Anywhere**。
这使得我们可以轻松地推出 Progee 的**单机版**（基于 SQLite，零配置）和**企业版**（基于 PG，高性能），而无需维护两套代码库。
