# 03.73 C05: Progee Enterprise Features & Security Overview
## Progee 企业版：功能与安全白皮书

---

## 1. 企业级安全 (Enterprise Security)

### 1.1 私有化部署 (Self-Hosted)
*   Progee 支持 Docker/K8s 私有化部署。
*   数据完全掌握在企业手中，不经过 Progee 的云端。
*   支持 **Local LLM** (如 Intermediate Model, Intermediate Model 私有部署) 对接，实现物理隔离的 AI 开发。

### 1.2 细粒度权限控制 (RBAC & BPL)
*   基于 **Billing** 系统的 BPL (Business Policy Logic) 引擎。
*   支持精确到 **"部门-项目-功能-模型"** 的四级权限控制。
    *   *例*: "实习生只能使用 Basic Model，且只能读取自己参与的项目。"

### 1.3 审计与合规 (Audit & Compliance)
*   **全链路日志**: 记录每一次 Prompt 输入和 Code 输出。
*   **敏感词过滤**: 内置 **WizAxis 知识熔断** 机制，防止敏感数据（如 AK/SK、PII）被发送给 LLM。
*   **水印追踪**: 对生成的代码添加隐形水印，防止泄露。

---

## 2. 企业级集成 (Enterprise Integration)

### 2.1 统一认证 (SSO)
*   支持 LDAP, SAML, OIDC, OAuth2。
*   无缝对接 Active Directory, Okta, 飞书/钉钉扫码登录。

### 2.2 CI/CD 集成
*   Progee 不是孤岛。它可以作为 **Pipeline** 的一部分。
*   支持 GitHub Actions, GitLab CI, Jenkins 插件。
*   *场景*: 代码提交后，自动触发 Progee 的 Foreman Agent 进行 Code Review，并生成报告。

### 2.3 知识库对接 (Knowledge Connect)
*   支持索引企业内部的 Confluence, Notion, Jira, Wiki。
*   让 AI 真正理解企业的**业务背景**和**遗留代码**。

Progee 企业版不仅仅是一个工具，它是为企业量身定制的 **AI 研发中台**。它解决了企业在拥抱 AI 时的三大顾虑：**安全、管控、集成**。
