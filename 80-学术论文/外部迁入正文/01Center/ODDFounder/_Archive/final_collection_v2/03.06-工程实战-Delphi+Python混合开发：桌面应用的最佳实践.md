## 摘要

Delphi 开发桌面 GUI 无敌，但生态贫瘠；Python 生态无敌（AI, 音视频），但打包部署和 GUI 开发是短板。**Chain2VFactory** 探索出了一条 **Delphi (Frontend) + Python (Backend)** 的混合开发之路。通过标准输入输出（StdIO）管道通信，结合 JSON 协议，实现了两者的优势互补。

---

## 一、为什么混合？

*   **Delphi 的强项**：原生 PE 文件，启动快，界面响应流畅，强类型，易于分发（单文件）。
*   **Python 的强项**：拥有 PyTorch, FFmpeg, Requests 等无限的库。写业务逻辑极快。

**Chain2VFactory 的需求**：做一个视频生成工具。界面要好用（Delphi），底层要调用 AI 和 FFmpeg（Python）。
如果全用 Python (PyQt)，界面丑且卡，打包巨大。
如果全用 Delphi，没有现成的 AI 库。
**结论**：合体。

---

## 二、架构设计：前后端分离 (Desktop Ver.)

Chain2VFactory 采用了类似 Web 开发的"前后端分离"架构，只是 HTTP 换成了 **Pipe (管道)**。

### 2.1 架构图

```
[Delphi GUI]  <===(StdIO Pipe/JSON)===>  [Python CLI / Service]
(VGenApp.exe)                            (main_cli.py)
     │                                         │
     ├─ 渲染界面                               ├─ 调用 FFmpeg
     ├─ 响应用户操作                           ├─ 调用 LLM API
     └─ 显示进度条                             └─ 文件处理
```

### 2.2 通信协议

我们定义了一套简单的 JSON-RPC 风格协议。

*   **Request (Delphi -> Python)**:
    ```json
    {
      "command": "create_video",
      "params": {
        "text": "Hello World",
        "style": "anime"
      },
      "request_id": "101"
    }
    ```

*   **Response (Python -> Delphi)**:
    ```json
    {
      "request_id": "101",
      "status": "success",
      "data": { "output_path": "c:/video/1.mp4" }
    }
    ```

*   **Progress Event (Python -> Delphi)**:
    ```json
    {
      "type": "progress",
      "task_id": "101",
      "percentage": 45,
      "message": "Rendering frame 100/200..."
    }
    ```

## 三、实现细节

### 3.1 Python 端 (`main_cli.py`)

Python 端被设计为一个 CLI 程序，但也支持 `server` 模式。
在 Chain2V 中，我们更倾向于 **CLI 模式**：
Delphi 直接调用 `python.exe main_cli.py run --config config.json`。
Python 通过 `print()` 输出 JSON 到标准输出，Delphi 捕获这些输出。

> *优点*：生命周期简单。任务结束，Python 进程退出，资源释放。不存在内存泄漏。

### 3.2 Delphi 端 (`Services.FFmpeg.VideoEngine.pas`)

Delphi 端使用 `CreateProcess` 启动 Python，并创建 Pipe 读取 `StdOut`。
为了不卡死 UI，我们在 Delphi 中使用了 **TTask / TThread** 进行异步读取。

```pascal
// Delphi 伪代码
procedure TEngine.RunTask;
begin
  TTask.Run(procedure
    begin
      // 启动 Python 进程
      RunPythonCommand('main_cli.py ...');
      // 循环读取输出
      while ReadPipe(Line) do
      begin
        if IsJson(Line) then
          SyncToMainThread(ParseAndUpdateUI(Line));
      end;
    end);
end;
```

## 四、部署方案

这是混合开发最大的坑：**Python 环境怎么发给用户？**

Chain2VFactory 采用了 **嵌入式 Python (Embedded Python)** 方案。
1.  下载 Python Windows Embeddable Package。
2.  将所需的库（FFmpeg, Requests等）安装到 `site-packages`。
3.  将整个 Python 目录精简后（剔除文档、无用库），放在 Delphi 程序的 `bin/python` 子目录下。
4.  Delphi 启动时，调用相对路径的 `bin/python/python.exe`。

**结果**：用户不需要安装 Python，解压即用。整个包体积控制在可接受范围内（~100MB，主要取决于依赖库）。

## 五、总结

Delphi + Python 是一对被低估的组合。
*   Delphi 负责 **壳**（体验、分发、控制）。
*   Python 负责 **核**（生态、算力、逻辑）。

Chain2VFactory 证明了这种架构在开发 AI 桌面工具时的巨大优势：**既有原生软件的质感，又有 AI 时代的算力。**
