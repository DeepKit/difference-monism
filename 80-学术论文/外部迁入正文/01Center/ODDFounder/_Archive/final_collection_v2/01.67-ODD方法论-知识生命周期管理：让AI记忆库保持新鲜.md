# 01.67 知识生命周期管理：让AI记忆库保持新鲜

> 你的AI记忆库是不是越来越臃肿？知识也有保质期，过期的要清理。

---

## 一、记忆库的"垃圾堆"问题

AI系统用久了，你会发现：

```
第1个月：100条记忆，精准有效
第6个月：3000条记忆，开始混乱
第12个月：10000条记忆，检索变慢，输出质量下降
```

问题出在哪？

**知识只进不出，没有生命周期管理。**

就像你的电脑硬盘，如果只存不删，迟早会满。更糟的是，过时的文件会干扰你找到真正需要的东西。

## 二、知识的四种状态

我设计了一套知识生命周期模型：

```
┌─────────┐     验证通过     ┌─────────┐
│  试用   │ ───────────────▶ │  激活   │
│ (Trial) │                  │(Active) │
└────┬────┘                  └────┬────┘
     │                            │
     │ 验证失败                    │ 持续有效
     │                            │
     ▼                            ▼
┌─────────┐                  ┌─────────┐
│  废弃   │                  │  核心   │
│(Discard)│                  │ (Core)  │
└─────────┘                  └─────────┘
```

### 状态1：试用（Trial）

新学到的知识，还没经过验证：

```python
new_knowledge = {
    "content": "用户喜欢简洁回复",
    "status": "trial",
    "created_at": "2026-01-10",
    "usage_count": 0,
    "success_count": 0
}
```

**特点**：
- 可以使用，但优先级低
- 需要积累使用数据
- 可能被废弃

### 状态2：激活（Active）

经过验证，证明有效的知识：

```python
# 晋升条件
TRIAL_TO_ACTIVE = {
    "min_usage_count": 3,      # 至少使用3次
    "min_success_rate": 0.8,   # 成功率≥80%
    "min_age_hours": 24        # 至少存在24小时
}
```

**特点**：
- 正常优先级使用
- 持续监控效果
- 可能晋升或降级

### 状态3：核心（Core）

长期有效，高度可靠的知识：

```python
# 晋升条件
ACTIVE_TO_CORE = {
    "min_usage_count": 10,     # 至少使用10次
    "min_success_rate": 0.9,   # 成功率≥90%
    "min_age_days": 7          # 至少存在7天
}
```

**特点**：
- 最高优先级
- 几乎不会被遗忘
- 是系统的"骨干知识"

### 状态4：废弃（Discard）

验证失败或过时的知识：

```python
# 废弃条件
DISCARD_CONDITIONS = [
    "success_rate < 0.5",           # 成功率太低
    "days_since_use > 90",          # 90天未使用
    "explicit_user_rejection",       # 用户明确否定
    "contradicted_by_newer"          # 被新知识否定
]
```

## 三、自动晋升机制

```python
class PromotionChecker:
    def check_and_promote(self):
        # 检查所有试用知识
        for knowledge in get_knowledge_by_status("trial"):
            if self.should_promote_to_active(knowledge):
                knowledge.status = "active"
                knowledge.promoted_at = now()
                log(f"知识晋升: trial → active: {knowledge.id}")
        
        # 检查所有激活知识
        for knowledge in get_knowledge_by_status("active"):
            if self.should_promote_to_core(knowledge):
                knowledge.status = "core"
                knowledge.promoted_at = now()
                log(f"知识晋升: active → core: {knowledge.id}")
    
    def should_promote_to_active(self, k):
        return (
            k.usage_count >= 3 and
            k.success_rate >= 0.8 and
            k.age_hours >= 24
        )
    
    def should_promote_to_core(self, k):
        return (
            k.usage_count >= 10 and
            k.success_rate >= 0.9 and
            k.age_days >= 7
        )
```

## 四、自动降级机制

知识不是只升不降的：

```python
class DemotionChecker:
    def check_and_demote(self):
        for knowledge in get_all_active_knowledge():
            # 成功率下降，降级
            if knowledge.recent_success_rate < 0.6:
                self.demote(knowledge)
            
            # 长期未使用，降级
            if knowledge.days_since_use > 30:
                self.demote(knowledge)
    
    def demote(self, knowledge):
        if knowledge.status == "core":
            knowledge.status = "active"
        elif knowledge.status == "active":
            knowledge.status = "trial"
        
        log(f"知识降级: {knowledge.id}")
```

## 五、过期清理策略

```python
class KnowledgeCleaner:
    def clean(self):
        # 1. 清理废弃知识（保留30天后删除）
        discarded = get_knowledge_by_status("discard")
        for k in discarded:
            if k.discarded_days > 30:
                self.archive_and_delete(k)
        
        # 2. 清理长期未用的试用知识
        trials = get_knowledge_by_status("trial")
        for k in trials:
            if k.days_since_use > 14 and k.usage_count < 2:
                k.status = "discard"
        
        # 3. 合并重复知识
        self.merge_duplicates()
    
    def archive_and_delete(self, knowledge):
        # 先归档，再删除（可恢复）
        archive_store.save(knowledge)
        knowledge_store.delete(knowledge.id)
```

## 六、知识新鲜度评分

```python
def calculate_freshness(knowledge):
    """
    计算知识新鲜度（0-1）
    """
    # 时间衰减
    days_old = (now() - knowledge.last_used_at).days
    time_factor = math.exp(-0.05 * days_old)
    
    # 使用频率
    usage_factor = min(knowledge.monthly_usage / 10, 1.0)
    
    # 成功率
    success_factor = knowledge.success_rate
    
    # 综合评分
    freshness = (
        time_factor * 0.4 +
        usage_factor * 0.3 +
        success_factor * 0.3
    )
    
    return freshness
```

**新鲜度用途**：
- 检索排序：新鲜的排前面
- 清理决策：不新鲜的优先清理
- 注入优先级：新鲜的优先注入

## 七、矛盾知识处理

当新旧知识矛盾时：

```python
def handle_contradiction(old_knowledge, new_knowledge):
    """
    处理矛盾知识
    """
    # 比较可信度
    old_score = calculate_trust_score(old_knowledge)
    new_score = calculate_trust_score(new_knowledge)
    
    if new_score > old_score * 1.2:  # 新知识明显更可信
        old_knowledge.status = "discard"
        old_knowledge.discard_reason = f"被 {new_knowledge.id} 取代"
        return new_knowledge
    
    elif old_score > new_score * 1.2:  # 旧知识明显更可信
        new_knowledge.status = "discard"
        new_knowledge.discard_reason = "与已有知识矛盾且可信度低"
        return old_knowledge
    
    else:  # 难以判断，标记待人工审核
        flag_for_review(old_knowledge, new_knowledge)
        return old_knowledge  # 暂时保留旧的
```

## 八、实际效果

| 指标 | 无生命周期管理 | 有生命周期管理 |
|------|---------------|---------------|
| 知识库大小 | 持续膨胀 | 稳定可控 |
| 检索延迟 | 越来越慢 | 保持稳定 |
| 知识准确率 | 逐渐下降 | 持续提升 |
| 存储成本 | 线性增长 | 对数增长 |

## 九、最佳实践

1. **定期清理**：每天运行一次清理任务
2. **渐进晋升**：不要跳级，让知识慢慢证明自己
3. **快速降级**：发现问题立即降级，不要犹豫
4. **保留归档**：删除前先归档，以防万一
5. **监控指标**：关注知识库的健康度指标

## 十、总结

知识生命周期管理的核心思想：

> **知识不是越多越好，而是越准越好**

四个状态让知识有序流动：
- **试用**：新知识的观察期
- **激活**：验证通过的工作状态
- **核心**：高度可靠的骨干知识
- **废弃**：等待清理的过时知识

让你的AI记忆库像活水一样，保持新鲜、保持高效。
