# 🧾 Paper I

## Abstract & Contributions Template

### Abstract（论文一）

> Recent advances in large language models have significantly accelerated AI-assisted code generation. However, most development methodologies still assume human code review as the final line of defense. As AI systems increasingly act as primary producers, this assumption fails at scale, exposing a fundamental gap in how software artifacts are evaluated and trusted.
>
> This paper introduces **Output-Driven Development (ODD)**, an artifact-centric paradigm that reframes software production around the legitimacy of outputs rather than the correctness of code. In ODD, **contracts** define the admissible boundaries of artifacts, and trust is transferred from human inspection to systematic verification and auditable sealing.
>
> By defining artifact legitimacy as a first-class object, ODD provides a conceptual foundation for AI-dominant software production; questions of contract acquisition, enforcement effectiveness, and legitimacy evolution are deferred to follow-up studies.

### Contributions（论文一）

* We introduce **Output-Driven Development (ODD)**, an artifact-centric paradigm that reframes software quality as artifact legitimacy rather than code-centric correctness.
* We define **contracts as legitimacy boundaries** and clarify the shift of trust from human review to institutionalized verification and sealing.
* We articulate the scope and limits of this conceptual contribution and outline a research agenda for feasibility, enforcement, and evolution.

# 🧾 Paper II

## Abstract & Contributions Template

### Abstract（论文二）

> A central obstacle to deploying Output-Driven Development (ODD) in practice is not code generation, but **contract acquisition**: precise, machine-verifiable contracts are expensive to author, and human code review becomes a bottleneck once AI generation scales. This paper investigates how ODD can remain feasible for non-expert teams by minimizing human effort in both contract authoring and code inspection.
>
> We present an acquisition pipeline that combines clarity assessment, structured context injection (e.g., role-/stage-aware context organization), and multi-agent proposal generation to produce a small set of contract candidates. Humans no longer write specifications from scratch; instead, they select among candidates and arbitrate a few high-impact choices. Release decisions are evidence-based, relying on verification reports rather than line-by-line code review.
>
> Through empirical evaluation on real development tasks, we demonstrate that this interaction model significantly reduces human time while preserving auditable decision traceability.

### Contributions（论文二）

* We formulate **contract acquisition** as a first-class bottleneck for ODD adoption and propose a pipeline that converts open-ended authoring into bounded decision-making.
* We introduce clarity assessment and structured context injection to externalize ambiguities and inject implicit constraints before generation.
* We reframe human involvement from code review to **evidence-based arbitration**, supported by progressive trust and risk-aware escalation.

### Abstract（支线）

> As AI agents increasingly act as primary producers, the dominant cost and failure mode in AI-assisted software development shifts from code generation to **context management**: repeated LLM calls amplify token costs, uncontrolled context injection causes noise and boundary violations, and long tool outputs undermine traceability and auditability.
>
> This paper introduces **Context Engineering** as a supporting infrastructure for Output-Driven Development (ODD). We present an Intelligence Officer that pre-queries databases and indices, assembles role-/stage-aware layered context under explicit token budgets, and externalizes long outputs as Evidence objects referenced by IDs. We further propose a layered context schema (e.g., a 17-layer stack), hard boundary enforcement, version consistency, and dual-stream memory to enable safe reuse and continuous improvement.
>
> Through system-level evaluation (token efficiency, latency, rework reduction, and ablation studies), we show how context engineering makes ODD workflows cheaper, safer, and more auditable, while keeping the mainline contributions focused on legitimacy definition (Paper I), feasibility (Paper II), enforcement (Paper III), and evolution (Paper IV).

### Contributions（支线）

* We formulate **context** as a first-class engineering object with cost, security, and auditability constraints, and position it as a key bottleneck in AI-dominant production.
* We propose an **Intelligence Officer** architecture for one-shot context assembly (pre-query + trim + inject) with hard boundary enforcement and token budget control.
* We introduce Evidence-first handling of long outputs and a layered context schema (e.g., 17 layers) with role-/stage-aware visibility.
* We propose a dual-stream memory pipeline and quality metrics to support safe, evolvable knowledge reuse.
