# 双引擎SVG渲染：Skia静态+CEF动画的最佳组合

> **作者**: ODDFounder (基于 SVGThing 实践)
> **日期**: 2026-01-11
> **标签**: SVG, 渲染引擎, Skia, CEF, Delphi, 图形技术

---

## 摘要

SVG 渲染是一个深坑。标准的 `TSVG` 组件往往不支持复杂的滤镜和动画。为了在桌面应用中实现完美的 SVG 预览和导出，**SVGThing** 创造性地采用了 **双引擎架构**：用 **Skia4Delphi** 处理高性能静态渲染，用 **CEF (Chromium Embedded Framework)** 处理复杂的 CSS/SMIL 动画渲染。本文揭秘这套"动静分离"的混合架构。

---

## 一、SVG 渲染的痛点

SVG 不仅仅是矢量图，它实际上是一个**文档**。
*   **静态部分**：Path, Rect, Fill, Stroke。这部分容易渲染。
*   **动态部分**：CSS Animation, SMIL (`<animate>`), JavaScript。这部分极难渲染。

传统的 Delphi SVG 库（如 RiverSoftAVG 或 Skia4Delphi）对静态支持很好，但对动态支持几乎为零。
想要完美支持动画，你实际上需要一个**浏览器内核**。

---

## 二、双引擎架构设计

SVGThing 没有试图重新发明浏览器，而是直接集成了浏览器。

### 2.1 静态引擎：Skia (Skia4Delphi)
*   **角色**：主力渲染器。
*   **场景**：列表预览、缩略图生成、静态导出（PNG/JPG）。
*   **优势**：**极快**。毫秒级渲染，内存占用低。
*   **实现**：直接将 SVG XML 喂给 `TSkSvg`，绘制到 Canvas 上。

### 2.2 动态引擎：CEF (Chromium)
*   **角色**：特种兵。
*   **场景**：动画预览、动图导出（GIF/WebP/MP4）。
*   **优势**：**兼容性无敌**。支持所有现代 CSS3 动画和 SVG 特性（因为它就是 Chrome）。
*   **缺点**：重，启动慢。

---

## 三、CEF 离屏渲染黑科技 (SvgEngine.CEF.pas)

为了用 CEF 导出 GIF，我们不能只是把它显示出来，我们需要**逐帧捕获**。

### 3.1 隐藏窗体 (Hidden Form)
我们创建了一个不可见的 `TCEFHost` 窗体：
```pascal
constructor TCEFHost.CreateHidden(AOwner: TComponent);
begin
  inherited CreateNew(AOwner);
  BorderStyle := bsNone;
  SetBounds(0, 0, 512, 512); // 视口大小
  // ...初始化 Chromium
end;
```

### 3.2 HTML 包裹
我们将 SVG 代码包裹在一个极其简单的 HTML 中，以控制视口和背景：
```pascal
function BuildHtml: string;
begin
  Result := '<!DOCTYPE html><html><style>svg{width:100%;height:100%;}</style>' +
            '<body>' + FSvgText + '</body></html>';
end;
```

### 3.3 逐帧截图 (Frame Capture)
这是最关键的一步。为了生成 GIF，我们需要每隔 33ms（30fps）截取一张图。
*   **同步难题**：浏览器是异步渲染的。
*   **解决方案**：`CaptureBitmap` 方法等待 `OnPaint` 事件或使用 `TakeSnapshot` API。
*   **时间控制**：虽然不能精准控制浏览器的内部时钟，但通过精确的 `Sleep` 和 `Application.ProcessMessages` 循环，我们可以近似地捕获时间轴上的帧。

```pascal
for I := 0 to FrameCount - 1 do
begin
  // 等待下一帧时间点
  WaitUntil(NextTick);
  // 截图
  Bitmap := Host.CaptureBitmap;
  // 添加到 GIF
  GifEncoder.AddFrame(Bitmap);
end;
```

## 四、智能路由

在 `ISvgRenderer.pas` 中，我们定义了统一接口。系统会根据 SVG 内容自动选择引擎：

```pascal
if ContainsAnimation(SvgText) then
  Renderer := TCEFSvgRenderer.Create // 有动画 -> 用 CEF
else
  Renderer := TSkiaSvgRenderer.Create; // 只有静态 -> 用 Skia
```

这种策略既保证了**性能**（90% 的图标是静态的，用 Skia 秒开），又保证了**能力**（遇到复杂动效，切到 CEF 完美呈现）。

---

## 五、总结

SVGThing 的双引擎架构是**工程实用主义**的胜利。
*   不死磕单一技术栈的局限性。
*   利用 Skia 的快，弥补 CEF 的重。
*   利用 CEF 的全，弥补 Skia 的缺。

这种**混合架构**思路，对于处理复杂的富媒体应用开发具有普遍的参考价值。
