# 配置驱动的SQL执行：告别硬编码SQL的优雅方案

> **作者**: ODDFounder (基于 UniBase/doQry)
> **日期**: 2026-01-11
> **标签**: Delphi, SQL, 配置驱动, ORM, UniBase

---

## 摘要

硬编码 SQL (`SQL.Text := 'SELECT ...'`) 是维护的噩梦。ORM 虽然方便，但往往牺牲了 SQL 的灵活性和性能。UniBase 的 **doQry** 模块提供了一种"第三条道路"：**配置驱动的 SQL 执行**。将 SQL 逻辑从代码中剥离，存入配置（或数据库），通过 `ProcName` 和 `JSON Params` 进行调用，实现了代码与数据的完美解耦。

---

## 一、硬编码 SQL 的痛点

在 Delphi (或任何语言) 开发中，我们经常看到这样的代码：

```pascal
Query.SQL.Text := 'SELECT * FROM users WHERE age > ' + IntToStr(Age);
if Name <> '' then
  Query.SQL.Add(' AND name LIKE ' + QuotedStr('%' + Name + '%'));
```

**问题**：
1.  **SQL 注入风险**：拼接字符串是万恶之源。
2.  **维护困难**：SQL 散落在 Pas 文件各处，修改 SQL 需要重新编译程序。
3.  **可读性差**：业务逻辑和数据访问逻辑混杂。

---

## 二、doQry 的设计理念

doQry 的核心思想是：**代码只负责"发起调用"，配置负责"具体执行"。**

### 2.1 核心 API

在 `uDoQry.pas` 中，我们可以看到极其精简的 API：

```pascal
// 1. 查询
function DoQryExecSelect(const Proc: string; const ParamsJson: string; var Data: TClientDataSet; const Ctx: TDoQryContext): Integer;

// 2. 执行命令 (Insert/Update/Delete)
function DoQryExecNonQuery(const Proc: string; const ParamsJson: string; const Ctx: TDoQryContext): Integer;

// 3. 事务
DoQryRunInTx(Ctx, procedure
  begin
    DoQryExecNonQuery('CreateUser', ...);
    DoQryExecNonQuery('AddLog', ...);
  end);
```

### 2.2 调用方式

开发者不再写 SQL，而是调用 **过程名 (Proc)**：

```pascal
// 代码中
DoQryExecSelect('GetUserByAge', '{"age": 18, "active": true}', DataSet, Ctx);
```

### 2.3 配置层 (Configuration)

具体的 SQL 存储在外部配置中（可以是文件、数据库表或 KV 存储）：

```json
// 配置中 (GetUserByAge)
{
  "sql": "SELECT * FROM users WHERE age > :age AND is_active = :active",
  "params": {
    "age": "integer",
    "active": "boolean"
  }
}
```

## 三、技术实现细节

### 3.1 JSON 参数绑定

`uDoQry` 内部使用了 `uDoQryJsonParams`。它会自动解析传入的 JSON 字符串，将其映射到 FireDAC 的 `TFDParam`。
这意味着开发者只需要构造一个 JSON 对象，无需手动一个个 `ParamByName('xxx').AsInteger := ...`。

### 3.2 上下文与追踪 (Correlation ID)

`TDoQryContext` 包含了一个 `CorrelationId`。
```pascal
function DoQryMakeContext(..., const CorrelationId: string): TDoQryContext;
```
这使得每一个 SQL 执行都可以被追踪。在分布式系统或微服务中，我们可以通过这个 ID 串联起整个调用链。

### 3.3 事务管理

`uDoQryTxManager` 封装了复杂的事务逻辑。通过 `DoQryRunInTx` 接受一个匿名方法，自动处理 `StartTransaction`, `Commit`, `Rollback`。这利用了 Delphi 的匿名函数特性，让事务代码写起来像 Python 的 `with` 语句一样优雅。

---

## 四、优势总结

1.  **安全性**：强制参数化查询，彻底杜绝 SQL 注入。
2.  **热更新**：SQL 逻辑在配置里。线上出现慢查询，DBA 可以直接修改配置优化 SQL，而不需要重新发布 EXE。
3.  **标准化**：所有数据库操作都通过统一的入口，便于做日志审计、性能监控和权限控制。
4.  **AI 友好**：AI 非常擅长生成 JSON 格式的配置。ODD 模式下，AI 只需要生成 SQL 配置条目，而不需要去改动 Delphi 核心代码。

**doQry** 是 UniBase 的数据访问基石，它证明了：**优雅的封装不是为了屏蔽底层，而是为了更好地管理底层。**
