# 多格式导出引擎：HTML/PDF/PPT的统一方案

> **作者**: ODDFounder (基于 TheLot 实践)
> **日期**: 2026-01-11
> **标签**: 文档导出, HTML, PDF, PPTX, TheLot, Delphi

---

## 摘要

**TheLot (案场)** 是一个 AI 辅助演示文稿生成工具。用户希望看到的是网页（HTML），分享给客户的是链接，但打印需要 PDF，汇报需要 PPT。如何用一套数据源，生成三种完全不同格式的产出物？TheLot 设计了一个**基于中间模型 (Model-Centric)** 的导出引擎。本文解析这套引擎的架构与实现。

---

## 一、数据源：单一真理 (Single Source of Truth)

在 `ModelCase.pas` 中，我们定义了演示文稿的核心数据结构：
*   **TCase**: 整个课件。
*   **TPage**: 单页幻灯片（包含 Title, Body, ImageUrl, Notes）。

无论导出什么格式，数据源永远只有这一个。

---

## 二、导出策略

### 2.1 HTML 导出 (`THTMLExporter`)
这是最简单的，因为 Web 是原生的展示层。
*   **模板引擎**：读取 HTML 模板文件。
*   **数据填充**：将 `TCase` 数据注入到 JS 变量或 DOM 结构中。
*   **交互**：生成的 HTML 包含完整的 Reveal.js 或 Swiper 逻辑，支持翻页、动画。

### 2.2 PDF 导出 (`TPDFExporter`)
PDF 是**打印机语言**，需要精确的排版。
TheLot 没有使用 HTML 转 PDF（容易排版错乱），而是使用了 **Skia PDF Backend**。
*   **绘制**：复用屏幕渲染逻辑（`RenderPageToCanvas`）。
*   **输出**：将 Canvas 的绘制指令重定向到 `TSkPDFDocument`。
*   **优势**：所见即所得。屏幕上长什么样，PDF 里就长什么样，连字体渲染都一致。

### 2.3 PPTX 导出 (`TPPTXExporter`)
这是最难的。PPTX 本质上是一堆 XML 的压缩包（OpenXML）。
TheLot 并没有调用 PowerPoint COM 接口（需要安装 Office），而是直接**生成 XML**。
*   **结构**：创建 `[Content_Types].xml`, `presentation.xml`, `slide1.xml` 等。
*   **映射**：将 `TPage.Title` 映射到 PPT 的标题占位符，将图片映射到 Image Shape。
*   **打包**：使用 Zip 库将这些 XML 打包成 `.pptx` 文件。

---

## 三、架构优势

这套**多态导出 (Polymorphic Export)** 架构的优势在于：
1.  **解耦**：数据层不关心展示层。
2.  **原生性**：每种格式都利用了其原生特性（HTML 的交互、PDF 的矢量、PPT 的可编辑）。
3.  **零依赖**：不需要用户安装 Chrome（转PDF）或 Office（转PPT），所有生成逻辑都内置在 App 中。

---

## 四、总结

文档导出的本质是**数据结构的转换**。
TheLot 通过将内容抽象为 `TCase` 模型，实现了对不同输出格式的**适配器模式 (Adapter Pattern)**。这不仅是一个技术选择，更是产品体验的保证——让用户的内容在任何载体上都能完美呈现。
