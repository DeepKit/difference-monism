# DM 代表作外投执行清单

## 使用范围

本清单不再处理正文扩写，只处理 `P04 / P09` 的外投执行。

## 一、期刊模板

1. 为 `P04` 选定唯一目标期刊。
2. 为 `P09` 选定唯一目标期刊。
3. 按目标期刊统一 `标题页 / 摘要长度 / 关键词数 / 标题级次 / 表注格式 / 参考文献格式`。

## 二、匿名送审

1. 使用本目录中的两份 `匿名送审版` 作为送审底本。
2. 再检查一遍作者识别信息、项目名、仓库名与内部命名痕迹。
3. 若投稿系统要求单文件或 PDF，再从匿名版导出，不直接从工作稿导出。

## 三、逐条核引

### P04

1. 核对关键理论文献是否与正文主张一一对应。
2. 核对 `表 1 / 表 2` 的每一条结论句是否都能回到正文或注释。
3. 核对 `DOC/MCS / 麻醉 / 裂脑 / no-report / 盲视` 五组案例的核心引注是否精确。
4. 对所有失败线触发句做一次“删一句是否更稳”的清理。

### P09

1. 核对法律文本条款是否精确。
2. 核对判例名称、卷页与结论句是否精确。
3. 核对官方报告的结论归因、建议与页位是否精确。
4. 核对二手短引是否只承担背景和解释，不替代一手事实链。

## 四、送审材料

1. 准备 `投稿信`。
2. 准备 `摘要版简介`。
3. 准备 `关键词` 与 `学科分类`。
4. 准备 `作者信息版` 与 `匿名版` 两套文件名。
5. 准备 `若被要求补充材料时的最小回应模板`。

## 五、退修纪律

1. `表达误读`：只改写句子，不扩母题。
2. `核引不足`：只补引注，不顺手增论证。
3. `结构性反对`：只下调结论强度，不反向把体系重新写大。
4. `新扩张冲动`：一律延后，直到当前一轮送审结束。

## 最短结论

当前最值钱的动作不是再写正文，而是把 `送审清稿` 变成 `可提交文件`。
