# ODD v1.0 公开征评包

> **作者 / Author**：Yi Fu（付毅，ODDFounder，fuyi.it@live.cn）
>
> **公开边界**：`主论文 + 英文主论文 + 白皮书 + 发布说明`
>
> **文件名说明**：包内保留的 `RM.ODD.工程方法论.md` 属于历史命名阶段的路径保留，用于兼容既有引用；当前对外主品牌与层级口径仍以 `DM -> ASTO -> ECET -> TAT -> ODD` 为准。

本目录为 ODD 的首轮外发包，适用于公开征评、技术传播与平台补充上传。

ODD 与前几层不同的一点是：其主论文已经公开发布并具备 DOI。因此本包的功能更偏向：

- 统一手工上传材料
- 明确首轮公开边界
- 为平台补充提供主论文之外的支撑文

若平台只允许一个主文，建议上传：

1. `论文/论文-学术版.md`

如平台更适合英文主文，可使用：

2. `Paper_01_ODD_Core_English_2026-01-16.md`

若平台支持主文 + 附件，建议顺序：

1. `ODD.预印本发布说明.v1.0.md`
2. `论文/论文-学术版.md`
3. `Paper_01_ODD_Core_English_2026-01-16.md`
4. `论文/白皮书.md`
5. `RM.ODD.工程方法论.md`
6. `PUBLISH_READINESS.md`

对外口径固定为：

> **ODD 是建立在 DM、ASTO、ECET、TAT 之上的工程方法层。它当前最适合以“已公开主论文 + 公开征评支撑文档组”的方式发布，不应被包装为已经替代全部传统软件工程方法或完成全领域大规模经验验证的终局标准。**

补充纪律：

- 不把工程仓库中的缓存、数据库、样例库和工具脚本一并上传
- 不把“减少对逐行 code review 的依赖”误读成“人类不再承担责任”
