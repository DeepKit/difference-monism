# ODD.P99.4 案例、指标与实证证据索引

> **公开口径**：`工具层文档 / 公开征评版`

---

## 一、本文解决什么问题

`ODD` 当前已经有：

- 主论文
- 白皮书
- 完整规格
- 导读层
- 大量实证案例与样例代码

真正缺的是：

1. 哪些数字主张可以公开说
2. 这些主张来自哪里
3. 它们当前属于什么证据强度
4. 哪些只是初步结果，不能升格成稳定结论

本文就是把这四件事压成索引。

---

## 二、证据强度分级

| 级别 | 名称 | 当前含义 |
|------|------|---------|
| `E0` | 结构一致性 | 文本、协议、对象模型内部自洽 |
| `E1` | 概念验证 | 单案例、样例代码、受控实验、局部演示 |
| `E2` | 初步实证 | 有明示方法、统计、对照或多样本，但范围仍有限 |
| `E3` | 跨样本支持 | 多案例、多任务或较稳定复核支撑 |
| `E4` | 跨场景稳健支持 | 多领域、多团队、可重复的外部稳健支持 |

当前纪律：

- `E0 / E1` 不得包装成“已完成方法验证”
- `E2` 可以写成“初步实证结果”
- `E3 / E4` 才接近“较强经验支持”

## 四、公开数字主张登记格式

任何需要出现在公开导读、白皮书或投稿补充材料里的数字主张，建议登记为：

```yaml
claim:
  id: ""
  statement: ""
  strength: E1 | E2 | E3 | E4
  source_ref: ""
  method_ref: ""
  scope_limit: ""
  replacement_rule: ""
```

## 五、当前公开表达规则

### 可以直接说的

- `ODD` 已有主论文 DOI
- `ODD` 已形成完整的契约、门禁、证据、封存对象模型
- `ODD` 已有软件与创作方向的案例入口

### 只能降级说的

- 某些效率或质量改进数字
- 某些跨任务统计结论
- 某些对传统方法的优越性结论

更稳表达：

- `初步实证结果`
- `当前案例显示`
- `概念验证级支持`
- `仍需更广泛复核`
