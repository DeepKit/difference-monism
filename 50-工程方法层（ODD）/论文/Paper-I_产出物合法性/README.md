## 核心贡献

1. **概念重构**：将软件工程的控制单元从代码转向产出物（Artifact）
2. **基础公理**：定义 ODD 的五条核心公理，区别于代码中心方法论
3. **责任重分配**：人类从代码作者变为契约仲裁者
4. **验证基础**：提出基于变异测试的分层验证框架

## 章节结构

1. Introduction（背景、核心问题、范式转换）
2. Core Concepts（产出物、契约、变异测试、封存、五步循环）
3. Comparison（vs TDD/BDD/DbC/MDE/Prompt Engineering/HITL/Formal Methods）
4. Implications（责任重分配、渐进信任、无透明性治理）
5. Limitations and Future Work（ODD 论文系列路线图）
6. Conclusion
