# ODD：产出物驱动开发

## 意向
ODD（Outcome-Driven Development，产出物驱动开发）是一套工程方法论。
它的核心认知：**代码是负债，产出物才是资产。**

代码需要维护、会腐烂、可能有错——它本身是成本。只有经过约定、验证、封存的产出物，才是真正可信赖、可复用的资产。
ODD 的目标就是把开发工作从“写代码”转向“生产可验证的资产”。

## 你可能遇到过这些问题

- “这个需求到底要做成什么样？”——没人说得清，做完了才发现不对。
- “这个功能谁测过了？”——口头说测了，但找不到任何记录。
- “上线后出了 bug，不知道是哪步出的问题”——无法追溯。
- “代码被悄悄改了，没人知道”——产出物没有保护。
- “AI 生成的代码质量参差不齐，没有统一的验收标准”——人和 AI 协作时尤其明显。

## ODD 怎么解决

- **约定先行**：开工前把验收条件写明白，不会“做完才发现不对”。
- **证据说话**：每次检查都留下可查证的记录，不靠口头承诺。
- **门禁拦截**：不达标的产出物进不了下一步，问题在早期就被拦住。
- **封存保护**：通过验证的产出物锁定归档，可追溯、不可篡改。
- **人机通用**：不管是人写代码还是 AI 生成代码，都走同一套流程，同一套标准。

## 机制
ODD 的最小机制是“约定→执行→验证→封存”。

## 四个动作

```
约定 → 执行 → 验证 → 封存
```

1. **约定**：动手之前，先写清楚"做什么、怎样算做好了"。这叫契约。
2. **执行**：按契约干活，产出具体的东西（代码、文档、配置……）。这叫产出物。
3. **验证**：拿产出物对照契约，检查是否达标。检查点叫门禁，检查结果叫证据。
4. **封存**：验证通过的产出物锁定归档，不可篡改。这叫封存。

## 规范
最小规范就是三条底线，不满足则不准开工/放行/封存。

## 三条底线

- **没有契约不开工**：不清楚要做什么，就不要开始做。
- **没有证据不放行**：说"做完了"不够，要拿得出证据。
- **没有封存不算完**：通过验证的东西必须锁定，防止被悄悄改掉。

## 实践
从 L1 起步，按风险升级，不必一开始就用最严格流程。

## 复杂度可以选

不是所有项目都需要最严格的流程。ODD 分三个等级，按需选用：

- **L1 轻量**：一个人写小工具。自动测试过了就行。
- **L2 标准**：团队协作。加上人工审查和质量检查。
- **L3 严格**：关键系统或 AI 协作。加上对抗测试、交叉审查、不可变审计。

从 L1 起步，需要时再升级。

## 蜂巢式开发：ODD 的结构优势

传统开发的产出是一堆文件，它们之间的关系在人的脑子里。ODD 不一样。

ODD 的产出物是原子化的——每个产出物都是自包含的、经过验证的最小单元，像一个个封好的蜂巢格。
这些原子产出物通过管道组合，擢升为新的产出物；新产出物又可以继续组合，形成活的网状结构。
整个网络由功能树统一管理——你可以精确定位任何一个产出物，追溯它的来源、依赖和变动影响。

```
原子产出物 ─┐
                ├─ 管道组合 ─→ 新产出物 ─┐
原子产出物 ─┘                           ├─ 管道组合 ─→ 更大的产出物
                    原子产出物 ─────────┘
                          ↑
                    功能树统一管理
```

结构上的好处：
- **可复用**：封存过的产出物可以直接被其他管道引用，不用重做。
- **可替换**：某个产出物有问题，替换它不影响其他已封存的部分。
- **可追溯**：从任何一个终态产出物，沿功能树向下追溯到每一个原子单元。
- **可生长**：新功能 = 新的组合管道，不破坏已有结构。

生产上的好处：

每个产出物有清晰的契约边界，所以不同的人（或 AI）可以同时建造不同的格子，互不干扰。

```
车间 A ──契约:用户登录── → 产出物 A（封存）──┐
车间 B ──契约:权限管理── → 产出物 B（封存）──├── 管道组合 → 更大的产出物
车间 C ──契约:审计日志── → 产出物 C（封存）──┘
```

- **可并行**：多个车间同时开工，各自封存各自的产出物，互不干扰。
- **可隔离**：一个车间出故障，其他车间不受影响，契约回队列由新车间接手。
- **可伸缩**：产能不够？加车间。产出物是标准化的，加车间就是加产能。
- **可交叉审核**：A 车间造的东西，由 B 车间来检——避免“自己检查自己”。

这就是“蜂巢式开发”——每个格子独立封装、验证完整；格子可以同时建造；格子之间通过结构连接，整体自然生长。

## 核心词汇表

- **契约（Contract）**：对一件工作的明确约定，包含验收条件。
- **产出物（Artifact）**：工作的具体产出，如代码文件、API 文档、测试报告。
- **门禁（Gate）**：检查点，产出物必须通过门禁才能进入下一阶段。
- **证据（Evidence）**：门禁检查的结果记录，证明产出物是否达标。
- **封存（Seal）**：将通过验证的产出物及其证据锁定，不可修改。
- **管道（Pipeline）**：有契约的组装线，把已封存的产出物组合擢升为新产出物。
- **功能树（Functional Tree）**：业务功能索引，管理产出物网络的定位、追溯和影响分析。
- **车间（Workshop）**：隔离的执行环境，一个契约绑定一个车间，多车间可并行工作。
