# LMM 核心单文件入口版

> 说明：本文档把 `LMM` 压缩为一份面向公共阅读的单文件入口。它只保留当前主链下稳定成立的公共内容，不再沿用历史上的旧综合封装写法。

---

## 一、LMM 的当前定位

`LMM` 当前不是独立宇宙式的方法总论，而是主链中的现实前线协议层：

```text
DM -> ASTO -> ECET -> COP -> LMM -> TAT -> ODD
```

各层最小分工：

- `DM` 提供对象边界与失败优先意识
- `ASTO` 提供桥接语法与变迁纪律
- `ECET` 提供治理边界
- `COP` 提供主体状态诊断与分流编码
- `LMM` 提供显影、弱测量、港口接引与责任转介节律
- `TAT` 提供高影响责任门槛
- `ODD` 提供承诺留痕、封存与回滚

---

## 二、LMM 只稳定新增五类协议

1. `场域探测协议`
2. `问题显影协议`
3. `弱测量与物理接近协议`
4. `港口接引与责任转介协议`
5. `回流与自消解协议`

只要某份文档无法明确落在这五类里，它就很可能不是 `LMM` 当前公共主链该承担的东西。

---

## 三、最小工作链

`LMM` 的最小工作链是：

```text
BOS -> COP -> 显影 -> 弱测量 -> 港口接引 -> TAT -> ODD
```

解释如下：

1. `BOS` 决定这个问题场值不值得进入
2. `COP` 决定这个主体当前处于什么状态
3. `LMM` 决定怎样显影、怎样低阻接引
4. `TAT` 决定高影响责任门槛是否成立
5. `ODD` 决定正式承诺如何被记录、封存与回滚

当前固定纪律：

- `BOS >= 45` 才能进入公共主链正式立塔
- `30-44` 只属于灰区补证与私有测试
- `BOS` 评场，不评人
- `LMM` 不替代 `COP`、`TAT`、`ODD`

## 四、五条核心命题

### 1. 显影优先

在高噪声场中，问题先被看见，比方案先被推出更重要。

### 2. 弱测量优先

在主体尚未自识别之前，低阻动作通常比强表态更稳。

### 3. 港口后置

港口不是开放收纳器，而是责任边界开始明确的入口。

### 4. 责任前置转介

高影响承接必须回引 `TAT` 与 `ODD`，不能由 `LMM` 单层吞下。

### 5. 止血优先

在真实漏损场景中，先阻断结构性损耗，比空泛增长叙事更构成价值。

---

## 五、LMM 不处理什么

当前 `LMM` 不应被写成以下任一种东西：

1. 独立的本体论总论
2. 主体状态的最终诊断引擎
3. 高影响责任裁决层
4. 承诺验证与审计封存系统
5. 私有产品、报价与回流逻辑的公开总称

## 七、证据与升级纪律

`LMM` 当前最稳的公开主张，应主要停在三层：

1. 定义与边界
2. 协议与失败条件
3. 有限匿名个案支持

若要继续升级，必须经过：

1. 匿名化
2. 结构化记录
3. 失败案例处理
4. 第三方复核或跨案对照

也就是说，`LMM` 当前比继续扩写更重要的任务，是把每一类主张对应到清楚的证据接口。

## 九、最短结论

`LMM` 的核心，不是让前线“更会卖”，而是把高责任场景里的前线顺序压成一套可转介、可收边界、可接审计的协议系统。
