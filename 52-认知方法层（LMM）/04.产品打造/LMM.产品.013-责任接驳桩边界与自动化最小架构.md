# LMM.产品.013 — 责任接驳桩边界与自动化最小架构

> **版本**：v1.0 | 日期：2026-04-02
> **属性**：04.产品打造 — `责任滑逸CT` 的有界处方前门
> **定位**：不解决全部组织问题，只补第一段最危险责任断链
> **关系**：诊断模板见 `LMM.产品.011`，主规格见 `LMM.产品.007`

---

## 一、一句话定位

> **责任接驳桩不是总改造，而是给最危险责任断链先立一根不再继续滑的桩。**

---

## 二、本产品必须做什么

1. 只接 `责任滑逸CT` 已确认的首要断点
2. 明确第一责任承接人是谁
3. 明确其是否拥有叫停权、预算否决权、回滚触发权
4. 明确第一段闭环的触发条件、禁止动作、升级条件
5. 把新边界最小物化到文档、流程、权限、审计对象或系统字段中
6. 给出复审入口与回滚接口

---

## 三、本产品明确不做什么

1. 不承诺全量组织改革
2. 不替代 CEO 全面重画组织架构
3. 不替代 `TAT` 做责任终裁、补偿、申诉与冻结授权
4. 不替代 `ODD` 完成正式契约编译、验证、封存与回滚
5. 不把一段接驳件伪装成企业长期治理总方案

---

## 四、何时必须升级到 `TAT / ODD`

出现以下任一情形，`责任接驳桩` 必须停止单独推进：

1. 责任承接人与实际权力错位
2. 需要冻结、申诉、补偿、召回或正式否决
3. 需要把新边界写入合同、制度、代码、审计流程
4. 继续推进将造成高不可逆损害
5. `triage_status` 为 `FREEZE / REFER`

最短纪律：

> **凡是碰到“谁有权决定最终责任”与“如何把新边界正式写进去”，都不再只靠接驳桩本身解决。**

---

## 五、最小交付清单

每次 `责任接驳桩` 至少应交付：

| 交付物 | 用途 |
|:---|:---|
| `first_closure_target` | 第一段要补的责任闭环 |
| `responsible_owner` | 第一责任承接人 |
| `stop_right_map` | 叫停、否决、回滚权映射 |
| `boundary_patch` | 新边界最小补丁 |
| `prohibited_actions` | 接驳完成前禁止继续的动作 |
| `review_entry` | 复审入口 |
| `rollback_entry` | 回滚入口 |
| `upgrade_route` | 升级到 `TAT / ODD / Harness` 的接口 |

---

## 六、自动化最小架构

### A. 可自动化

1. 灯塔内容页
2. 弱测量与自测问答
3. 首诊第一页采样表
4. 材料提交、校验、脱敏提醒
5. 损耗粗测、结构字段预填
6. `COP` 预编码草稿
7. 首诊报告模板生成
8. 事件日志、证据索引、版本归档

### B. 必须人工守门

1. 责任最终确认
2. `FREEZE / REFER` 放行
3. 是否进入 `TAT / ODD`
4. 接驳件签发
5. 回滚、补偿、申诉、冻结与否决
6. 高影响收费与正式承接确认

### C. 最小流水线

```text
灯塔显影
-> 弱测量
-> 责任滑逸CT第一页
-> 材料提交
-> COP预编码
-> 人工复核
-> 责任滑逸CT报告
-> 是否进入责任接驳桩
-> TAT/ODD/Harness
```

## 七、网站最小页面集合

若后续做全自动前门，最小页面只需以下六类：

1. `问题显影页`
2. `弱测量自测页`
3. `责任滑逸CT第一页`
4. `材料提交页`
5. `首诊结果页`
6. `接驳申请页`

原则只有一条：

> **先把责任滑逸看清，再决定是否值得下单接驳；不允许用网站把高风险裁决自动卖出去。**

> **责任滑逸CT负责看清“哪里在滑”，责任接驳桩负责立住“第一根不再滑的责任桩”。**
