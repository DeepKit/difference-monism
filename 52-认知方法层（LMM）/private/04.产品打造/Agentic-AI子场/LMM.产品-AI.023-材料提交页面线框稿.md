---

## 1. 总体版式

最稳版式为：

`统一单列主表单 + 顶部进度条 + 底部固定操作区`

统一原因：

- 四步像同一个流程，不像四个独立产品页
- 用户知道自己正在进入诊断准备，而不是被引导购买
- 每一步都可复用同一页面骨架

固定结构：

- `顶部`：进度条 + 当前步骤标题 + 一句说明
- `中部`：当前步骤核心输入区
- `底部固定区`：上一步 / 保存草稿 / 下一步或提交

## 2. 四步线框

### 第1步：先确认这次看的对象

#### 上半区

- 进度条：`1 / 4`
- 标题：`先确认这次看的对象`
- 说明：`本次只收一条工作流与一个明确场景。`
- 可选摘要：`AI.011 已确认对象摘要`

#### 中段

- `工作流名称`
- `所属场景`
- `当前状态`
- `系统范围`
- `一句话说明`

#### 底部固定区

- `保存草稿`
- `下一步：确认边界`

### 第2步：再确认这次只看什么

#### 上半区

- 进度条：`2 / 4`
- 标题：`再确认这次只看什么`
- 说明：`本次只收一条工作流、一个明确场景。`

#### 中段

- `首要问题`
- `本次最想回答的 1-3 个问题`
- `最近一次异常或担忧场景`
- `本次纳入范围`
- `本次不纳入范围`
- `是否涉及权限 / 外部调用 / 自动执行`

#### 底部固定区

- `上一步`
- `保存草稿`
- `下一步：提交材料`

### 第3步：把首诊所需材料交清

#### 上半区

- 进度条：`3 / 4`
- 标题：`把首诊所需材料交清`
- 说明：`请先上传最小证据包，不必一开始提交全部资料。`
- 提示：`可上传类型 / 最少必需项`

#### 中段

按卡片分组：

- `流程图 / 步骤清单`
- `系统架构图`
- `日志样本`
- `权限说明`
- `接口说明 / API 文档`
- `异常案例`
- `复盘纪要 / 问题记录`
- `补充备注`

#### 底部固定区

- `上一步`
- `保存草稿`
- `下一步：确认对接人`

### 第4步：确认对接人与提交

#### 上半区

- 进度条：`4 / 4`
- 标题：`确认对接人与提交`
- 说明：`请确认本次提交的责任接口与后续沟通方式。`
- 摘要：`本次提交对象 / 边界 / 材料概览 / 责任接口`

#### 中段

- `业务 Owner`
- `技术 / AI Owner`
- `联系人`
- `角色`
- `联系方式`
- `可沟通时间`
- `补件通知方式`
- `最终确认`

#### 底部固定区

- `上一步`
- `返回修改`
- 主按钮：`提交进入核对`

## 3. 成功页线框

### 标题

`材料已收到，进入首诊核对`

### 上半区

- 状态提示：`已收到对象、边界与材料`

### 中段

正文分三块：

- `我们已收到你本次提交的对象、边界与材料。`
- `接下来会先核对材料完整性与本次首诊边界是否清楚。`
- `如材料齐全，我们会发出首诊准备确认；如仍有缺口，会先向你发补件提醒。`

### 底部固定区

- `查看已提交内容`
- `补充材料`
- `返回总览`

成功页禁用动作：

- `立即咨询`
- `预约销售`
- `获取报价`
