# LMM.产品.015 — 入口分流、前卡与样本提交链

> **版本**：v1.1 | 日期：2026-04-16
> **属性**：04.产品打造 — 入口分流与样本提交总门
> **定位**：把原 `前卡 / 样本申请页 / 会后回流弱测量自测页` 收成一篇统一入口链文档，用于把“责任滑逸问题是否值得进入正式首诊”压成可分流、可补证、可提交的前门
> **关系**：正式首诊模板见 `LMM.产品.011`

---

## 一、一句话定位

> **这不是正式首诊，而是一条把“共鸣回流 -> 前卡判断 -> 样本提交”压成一页的入口分流链。**

它只回答一件事：

`这次事件，值不值得进入正式样本提交与 CT 首诊。`

---

## 二、三段入口链

本文件固定把前门收成三段：

1. `回流弱测量`
2. `责任滑逸前卡`
3. `样本正式提交`

三段关系是：

`先判断这是不是我的问题 -> 再把最近一次掉球钉成单链 -> 再提交最小样本`

---

## 三、适用位置

本文件适合放在以下位置之后：

- 行业论坛演讲页
- 灯塔文章末尾
- 白皮书摘要页
- 会后回放页
- 案例摘要页
- 事件型入口页面

它不适合：

- 单独作为首页
- 直接代替正式首诊
- 作为报价预约页

---

## 四、第一段：回流弱测量七问

固定七问：

1. 最近 `60` 天内，你们是否发生过停线、返工、延期、赔付或重大补位？
2. 这件事能否压成一条明确的动作链？
3. 你能否写出谁拍板、谁执行、谁承担后果？
4. 你能否指出最先失真的交接点？
5. 目前是否至少留有两类最小证据？
6. 老板或最终后果承担者是否在场，或至少可被上送？
7. 如果今天只补一个洞，你是否知道最先该补哪一段？

### 回流结果分流

- `R1 继续照明`：只感到有共鸣，还没有真实事件
- `R2 补证观察`：有事件，但证据不足或角色链不清
- `R3 可做前卡`：有事件、有单链、已能初步说清角色链
- `R4 可做前卡并准备样本`：事件、角色、损耗、证据、上送条件都更成熟

## 五、第二段：责任滑逸前卡

### 适用

- 最近 `30-60` 天内刚发生过停线、返工、延期、赔付、验收争议、售后升级或多方互推责任
- 问题能够压成一条交付链，而不是整个组织泛问题
- 至少能指出谁拍板、谁执行、谁承担后果中的两项

### 不适用

- 只是一般抱怨，没有具体事件
- 只想讨论大而化之的组织优化
- 事件已进入重大诉讼、刑责或不可公开合规调查
- 只想直接谈合作，不愿意说明事实与边界

### 前卡首屏三问

1. 最近一次最严重的掉球，发生在哪条链？
2. 当时谁拍板、谁执行、谁承担最终后果？
3. 这次事件留下了什么最小证据？

只要这三句答不出来，就不进入强判断。

### 前卡固定字段

| 字段 | 说明 | 最低纪律 |
|:---|:---|:---|
| `event_title` | 本次事件名 | 必须是具体事件，不得写成泛主题 |
| `chain_name` | 最危险动作链 | 必须单链 |
| `event_time` | 最近一次事件时间 | 至少精确到月/日 |
| `event_place` | 事件发生环节 | 例如验收、实施、售后、复盘 |
| `decision_owner` | 谁拍板 | 角色优先，最好带姓名 |
| `execution_owner` | 谁执行 | 角色优先，最好带姓名 |
| `consequence_owner` | 谁承担后果 | 若无法写出，默认高风险 |
| `loss_signal` | 当前最直观损耗 | 至少写工时、金额、延期、客户影响之一 |
| `evidence_types` | 现有证据类型 | 至少两类才建议进样本提交 |
| `boss_presence` | 老板是否在场或可上送 | 必填 |
| `in_scope` | 本次只看什么 | 必填 |
| `out_of_scope` | 本次明确不看什么 | 必填 |

### 前卡三档判定

#### `A. 进入样本提交`

- 有具体事件
- 有单链
- 有至少两类最小证据
- 老板在场或能上送到老板

#### `B. 补证后再看`

- 有事件
- 有单链
- 但证据不足，或后果承担者暂时不清

#### `C. 退出当前入口`

- 没有具体事件
- 没有单链
- 只有概念性兴趣

## 六、第三段：样本正式提交

### 开场文案

建议固定文案：

> 如果你最近一次最严重的停线、返工、延期、赔付或责任互推，还说不清到底卡在哪条链，  
> 可以先把样本交进来。  
> 我们先看事实，不先卖方案。

### 固定模块

#### 模块 1：样本基本信息

- `sample_title`
- `industry_band`
- `company_size`
- `role_of_submitter`
- `can_reach_boss`

#### 模块 2：事件链说明

- 最近一次事件发生时间
- 发生在哪条链
- 当时谁拍板
- 谁执行
- 谁承担后果
- 当前最明显损耗是什么

#### 模块 3：证据清单

最低要求：至少两类。

可多选：

- 合同/责任分工文档
- 流程图/里程碑图
- 异常记录/返工记录
- 赔付/延期/投诉记录
- 会议纪要/工单/聊天指令
- 日志/抓包/系统记录
- 复盘材料

#### 模块 4：边界确认

- 本次只看一条链
- 本次不等于全量组织诊断
- 如证据不足，可能只得到补证建议
- 高风险事项可能被冻结或转介

### 样本提交输出状态

- `S-A 进入CT候选`
- `S-B 补证候选`
- `S-C 不受理`

## 七、页面建议结构

### 屏一：问题钉子

> 你不是来做一份泛管理问卷。  
> 你是在把最近一次真实掉球，压成一张可复核的责任前卡。

### 屏二：回流弱测量

- 七问弱测量
- 输出 `R1 / R2 / R3 / R4`

### 屏三：前卡三问录入

- 掉球发生在哪条链
- 谁拍板 / 谁执行 / 谁担后果
- 留下了什么证据

### 屏四：样本正式提交

- 时间
- 链路
- 角色
- 损耗
- 附件与证据
- 边界确认

### 屏五：最小判断

- `继续照明`
- `补证观察`
- `进入样本提交`
- `暂不进入`

---

## 八、严格边界

本入口链禁止：

- 写成组织成熟度问卷
- 写成报价预约页
- 写成“留下电话我来帮你解决”的推进页
- 在前门阶段承诺完整解决方案
- 把 `R3 / R4` 自动解释成成交信号
- 自动承诺受理或自动生成正式责任裁决

本入口链允许：

- 低阻邮箱/微信留存，用于回传结果
- 转入正式样本提交
- 输出补证建议

## 十一、页面文案母本

以下内容吸收原 `LMM.产品.023`，作为本文件的可上线页面文案附录。

原则是：

`015` 负责入口结构，

这里负责把同一入口结构压成页面标题、字段提示、校验语、按钮语与成功页文案。

### 11.1 总定位

本附录不是新产品，

而是把已有前门件变成：

- 可上线表单文案
- 可直接给设计/开发的页面文案
- 可统一复用的提示语与拦截语

一句话压缩：

> **这里不是再定义流程，而是让 `015` 真的能长成页面。**

### 11.2 统一写作纪律

前门页面统一遵守四条：

1. 先收事实，不先收预算
2. 先收一条链，不先收全组织背景
3. 先做边界确认，不先做高承诺动作
4. 先说“我们先看事实”，不说“我们马上帮你解决”

统一禁用语：

- `免费诊断你所有问题`
- `立即预约专家`
- `马上获取方案`
- `留下电话立刻联系`

统一可用语：

- `先把最近一次掉球说清`
- `先交一条链，不交全组织`
- `先看事实，再决定是否进入CT`
- `如证据不足，先补证，不强进`

### 11.3 页面一：责任滑逸CT首诊前卡

#### 页面标题与副标题

- 标题：
  `把最近一次责任滑逸，钉到同一页`
- 副标题：
  `先写清哪条链掉球、谁拍板、谁执行、谁担后果，再决定是否值得进入样本申请。`

#### 顶部提示条

建议文案：

> 这不是组织成熟度问卷。  
> 本页只看最近一次最危险的掉球事件。

#### 步骤条

最稳为 `3步`：

1. `先写事件`
2. `再写角色与损耗`
3. `确认证据与边界`

#### 第一步字段文案

- 字段：`本次事件名`
  - 占位语：
    `例如：机器人联调后二次停线争议`
- 字段：`最近一次掉球发生在哪条链`
  - 占位语：
    `例如：验收 -> 联调 -> 现场恢复`
- 字段：`事件发生时间`
  - 占位语：
    `至少精确到月/日`
- 字段：`发生环节`
  - 选项建议：
    - `验收`
    - `实施`
    - `售后`
    - `升级`
    - `复盘`
    - `其他`

#### 第二步字段文案

- 字段：`谁拍板`
  - 占位语：
    `先写角色，再写姓名`
- 字段：`谁执行`
  - 占位语：
    `先写角色，再写姓名`
- 字段：`谁承担最终后果`
  - 占位语：
    `若写不出，说明这里已经高风险`
- 字段：`当前最直观损耗`
  - 提示语：
    `至少写一项：工时 / 金额 / 延期 / 客户影响`

#### 第三步字段文案

- 字段：`你现在已有的最小证据`
  - 多选项：
    - `流程图 / 责任分工`
    - `异常记录 / 日志`
    - `返工 / 赔付 / 延期记录`
    - `会议纪要 / 工单 / 指令痕迹`
    - `复盘材料`
- 字段：`老板是否在场或能被上送`
  - 选项：
    - `老板在场`
    - `能直接上送`
    - `暂时不能`
- 字段：`本次只看什么`
  - 占位语：
    `例如：只看机器人联调接口绕行，不看整体组织改造`
- 字段：`本次明确不看什么`
  - 占位语：
    `例如：不看绩效体系与整体架构`

#### 前卡按钮文案

- 上一步：
  `返回上一页`
- 保存：
  `保存这张前卡`
- 下一步：
  `继续把事件写清`
- 提交：
  `提交前卡，获取初步判断`

#### 前卡校验提示语

- 未写事件名：
  `请先写清这次具体掉球事件，不要写成泛主题。`
- 未写链路：
  `请先压成一条链，否则还不适合进入前卡。`
- 未写角色：
  `至少先写出拍板者、执行者、担后果者中的两项。`
- 未写边界：
  `请先说明本次只看什么、不看什么。`

#### 前卡结果页文案

##### 结果 A：可进入样本申请

- 标题：
  `这条链已具备进入样本申请的基础`
- 正文：
  `你已经写清了一次真实事件、最危险动作链，以及初步角色链。下一步请补齐样本与最小证据。`
- 按钮：
  `进入样本申请`

##### 结果 B：需补证

- 标题：
  `这条链已显影，但还需要先补证`
- 正文：
  `当前事件已明确，但最小证据或后果承担者仍不够清。先补材料，再决定是否进入样本申请。`

  `查看补证清单`

- 标题：
  `当前还不适合进入样本申请`
- 正文：
  `你现在更像是在描述一般性的组织烦乱，而不是一条已显影的责任滑逸链。建议先回到灯塔内容或回流自测。`
- 按钮：
  `回到回流自测`

### 11.4 页面二：样本申请页

#### 页面标题与副标题

- 标题：
  `提交一条真实责任滑逸样本`
- 副标题：
  `请只交最近一次最危险的责任滑逸事件，不交全组织，不交泛问题。`

#### 顶部提示条

建议文案：

> 我们先收一条真实样本，  
> 再判断它是否值得进入 `责任滑逸CT`。

#### 步骤条

最稳为 `4步`：

1. `先交样本信息`
2. `再交事件链`
3. `再交最小证据`
4. `最后确认边界`

#### 第一步字段文案

- 字段：`样本标题`
  - 占位语：
    `例如：机器人验收后接口绕行争议`
- 字段：`行业带`
  - 选项：
    - `机器人 / 自动化系统集成`
    - `专用设备制造 / 产线升级`
    - `工业软件 / 企业AI集成`
    - `信息技术服务 / 系统实施`
    - `其他`
- 字段：`公司规模`
  - 选项：
    - `30-50`
    - `50-100`
    - `100-300`
    - `300-500`
    - `500+`
- 字段：`提交人角色`
  - 占位语：
    `例如：交付经理 / 工程负责人 / CIO / 总经理`

#### 第二步字段文案

- 字段：`最近一次事件发生了什么`
  - 占位语：
    `用100-300字写清：什么动作继续放行、结果怎么掉了`
- 字段：`谁拍板 / 谁执行 / 谁承担后果`
  - 占位语：
    `按三行写，不要合成一句话`
- 字段：`本次最明显损耗`
  - 占位语：
    `例如：停线6小时 / 返工2轮 / 客户延收 / 赔付风险`
- 字段：`老板是否在场或能被上送`
  - 选项：
    - `老板在场`
    - `能直接上送`
    - `不能`

#### 第三步字段文案

- 区块标题：
  `先交最小证据，不必一开始交全部`
- 上传提示：
  `至少两类材料，才建议进入CT候选。`
- 支持材料：
  - `合同/责任分工`
  - `流程图/里程碑`
  - `异常记录/日志`
  - `延期/赔付/客诉记录`
  - `会议纪要/工单/聊天痕迹`
  - `复盘材料`
- 上传说明：
  `可脱敏；若暂时无法上传，可先写明现有材料名称。`

#### 第四步字段文案

- 字段：`本次只纳入什么`
  - 占位语：
    `例如：只看这次接口绕行，不看整体项目绩效`
- 字段：`本次明确不纳入什么`
  - 占位语：
    `例如：不纳入供应商谈判与组织架构重画`
- 勾选项 1：
  `我确认本次只提交一条链，而不是全组织总诊断`
- 勾选项 2：
  `我确认如证据不足，可能只收到补证建议`
- 勾选项 3：
  `我确认高风险事项可能被冻结或转介，不自动进入执行`

#### 样本页按钮文案

- 上一步：
  `返回上一页`
- 保存：
  `保存样本草稿`
- 下一步：
  `继续补齐样本`
- 提交：
  `提交样本，进入核对`

#### 样本页校验提示语

- 没有单链：
  `请先把问题压成一条链，再提交样本。`
- 没有事件：
  `请先写最近一次真实事件，而不是泛泛描述。`
- 没有最小证据：
  `请至少补两类最小证据，再进入CT候选。`
- 没有边界说明：
  `请先写清本次纳入与不纳入范围。`

#### 样本提交成功页

- 标题：
  `样本已收到，进入CT候选核对`
- 正文：
  `我们会先核对：这是不是一条单链样本、证据是否足够、边界是否清楚。`
- 说明：
  `你将收到以下三种结果之一：进入CT候选 / 补证候选 / 暂不受理。`
- 按钮：
  - `查看已提交样本`
  - `继续补充材料`
  - `返回总览`

成功页禁止出现：

- `立即咨询`
- `立即购买`
- `获取报价`
