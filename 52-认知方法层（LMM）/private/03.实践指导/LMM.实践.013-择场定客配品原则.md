# LMM.实践.013 — 择场定客配品原则与执行清单总门

> **版本**：v1.3 | 日期：2026-04-16
> **定位**：`LMM` 前线从 `问题场识别 -> 首要客体锁定 -> 分层承接配置` 的统一执行母文
> **核心逻辑**：蓝海不是找出来的，而是在责任真空层中，通过结构诊断、客体识别与承接设计生成的。

---

## 0. 总原则

`LMM` 不以既有产品倒推市场，

而以问题场识别为前提，

经由优先客体识别，

再完成产品配置。

压成最短版就是：

`先定场，后定人，再定物。`

或者：

`先定问题与责任，再定客户，最后定产品。`

这意味着：

- 产品不是起点
- 客户也不是起点
- 起点是 `真实存在、持续损耗、责任未闭合` 的问题场

补充纪律：

- `BOS` 只是筛查工具，不是市场规模预测器
- `Kairos` 信号决定何时靠近，不是日历决定何时轰炸
- `弱测量` 优先于强追问
- 前端最小链条应为：`灯塔内容 -> 自测 -> 港口承接`

---

## 1. 蓝海的结构性定义：三元真空

在 `LMM` 中，

所谓蓝海，不是“没人做”的市场，

也不是“听起来很大”的新概念，

而是一个可被责任闭合系统稳定进入的 `结构缝隙`。

一个值得动刀的蓝海场域，至少应同时具备以下三个物理特征：

### 1.1 认知刚性 `R`

行业存在极强的路径依赖，

受众防御结构厚重，

默认反应是：

- `没办法`
- `只能这样`
- `一直都是这么做`

### 1.2 损耗强度 `L`

对象在旧结构中存在高频、高能耗的黑洞损耗，

并持续支付隐性税收，例如：

- 时间损耗
- 资金损耗
- 精力损耗
- 注意力损耗
- 协同损耗

### 1.3 责任真空 `V`

现有供应商只提供：

- 信息
- 工具
- 产品
- 话术

却拒绝承接后果。

也就是说，

- 没有人对最终结果负责
- 没有人对失败路径负责
- 没有人对误配后果负责

---

## 2. 蓝海的收紧定义

本文件给出正式收紧定义：

`蓝海市场，是指一种尚未被主流语言充分命名、尚未被既有产品有效承接，但已存在持续结构张力，并可被某一责任闭合系统稳定进入的问题场。`

所谓蓝海至少应满足四个条件：

1. `真实张力`
2. `解释失灵`
3. `低占位`
4. `可闭合承接`

如果没有第四条，

即使前三条都成立，

也不构成 `LMM` 意义上的蓝海。

## 3. 探测阶段：寻找结构缝隙

在进入任何市场前，

必须先做结构探测，

而不是先做产品想象。

### 3.1 别扭感审计

优先寻找对象最常说下面这些话的环节：

- `没办法`
- `只能这样`
- `先凑合`
- `一直都这么搞`
- `大家都这样`

### 3.2 损耗量化

至少量化下面四项中的两项：

- 每周损耗多少时间
- 每月损耗多少金钱
- 每天损耗多少精力
- 每次协同造成多少延误或返工

### 3.3 信号捕捉

优先定位处于：

- `流变阶`
- `脉冲阶`

的个体。

### 3.4 弱测量与 Kairos 纪律

优先观察：

- 多次回看同一问题内容
- 主动追问具体步骤
- 收藏、转发、抄送同事
- 下载确认单、访问预约页
- 愿意提交最小样本数据

## 4. LMM-BOS 结构评分模型

本文件建议使用：

`LMM-BOS Scorecard`

采用 `1-5` 分制。

### 4.1 评分维度

| 维度 | 指标 | 评分标准（1 → 5） |
|------|------|-------------------|
| `R` | 认知刚性 | `1 = 开放尝试新工具；5 = 强路径依赖，行业惯性极强` |
| `L` | 损耗强度 | `1 = 低频低损耗；5 = 高频高损耗，痛点每天发生` |
| `V` | 责任真空 | `1 = 有明确售后或结果分担；5 = 完全无责任承接，客户自负` |
| `T` | 相变触发性 | `1 = 极难演示变化；5 = 可在 3 分钟内制造明显相变感` |

### 4.2 判定公式

`Blue Ocean Score = (R + L + V) × T`

### 4.3 评分标准

- `>= 45`：核心蓝海
- `30-44`：灰区测试市场
- `< 30`：暂时放弃

### 4.4 使用提醒

- `BOS >= 45` 才值得优先建塔
- `30-44` 只代表“可继续补证”，不代表“已通过正式择场”
- `T` 低而总分高，不构成立即进场理由
- 无证据支撑的高分，应按低一级处理

## 5. 择场定客配品的六段流程

### 第一步：择场

先定义要进入的 `问题场`，

回答三件事：

- 这里的旧结构为什么持续存在
- 这里的损耗为什么持续发生
- 这里的责任为什么没有被妥善闭合

### 第二步：定客

不是找所有潜在客户，

而是找最先会被触发的一小群人。

优先顺序通常是：

- 最痛
- 最急
- 最愿付费
- 最容易形成案例

若场域是组织型问题场，

还应进一步区分：

- `首要痛点客体`
- `守门角色 / 上送节点`
- `最终拍板者`

### 第三步：场景与支付验证

验证三件事：

- 该客体是否真实持续受困
- 该客体是否有立即解决意愿
- 该客体是否愿意为改变付费

### 第四步：价值主张成形

至少回答：

- 你到底在解决什么
- 为什么旧方法接不住
- 为什么你的进入方式不同
- 为什么值得更高价格

### 第五步：配品

此时才决定承接器形态。

并应至少区分三层：

- `灯塔内容层`
- `自测 / 弱测量层`
- `港口承接层`

### 第六步：首单闭环与标准化

先形成第一批真实承接闭合与交付，

再标准化：

- 交付流程
- 承接路径
- 关键话术
- 案例证据
- 升级路径
- 回流结构

---

## 6. 与传统产品打法的根本差别

### 传统路线

`先有产品 -> 再找市场 -> 再教育客户`

问题在于：

- 容易功能自恋
- 容易反向编造痛点
- 容易扩大适用边界，最后责任外部化

### LMM 路线

`先择场 -> 再定客 -> 再配品`

本质是：

- 先确认 `该不该介入`
- 再确认 `该为谁介入`
- 最后确认 `用什么介入`

---

## 7. Stop Rule：撤退判定

`不能闭合责任，就不能点灯。`

若出现以下任一情况，必须立即停止或暂停进入：

- 不可验证
- 责任失重
- 节律错位
- 能量耗尽
- 支付失真
- 位阶错位

---

## 8. 一票否决项

以下情形直接判定为不符合 `LMM` 的择场定客配品原则：

- `先有产品，再反向编造痛点`
- `把蓝海理解成主观想象，而不是可验证的问题场`
- `未验证客户支付意愿，就直接做重产品`
- `没有承接能力，却先用市场叙事放大需求`
- `无法定义失败条件，却继续扩大干预`
- `前台尚未完成照明，就直接推首诊说明或正式承接文书`

## 9. 执行清单总门

从本轮起，

`择场 / 定客 / 配品` 三张执行清单收纳在本文件内部，

不再平行拆成三个独立文件。

更稳的使用顺序是：

1. 先读本文件前九节的原则层
2. 再进入第十节择场清单
3. 再进入第十一节定客清单
4. 最后进入第十二节配品清单

## 10. 执行清单 A：择场检查

### 10.1 使用原则

本清单只判定一件事：

`这个问题场值不值得进，能不能进。`

在使用前，必须先完成 `LMM.实践.009` 的基础门禁项：

- `触发定义`
- `失败判定`
- `补偿机制`
- `责任主体`

### 10.2 定义层

- `问题场名称`
- `一句话定义`
- `暂定进入理由`

### 10.3 成立层检查项

#### 结构成立

- [ ] `问题场可命名`
- [ ] `持续张力成立`
- [ ] `认知刚性成立`
- [ ] `损耗强度成立`
- [ ] `责任真空成立`

#### 市场成立

- [ ] `旧解释失灵`
- [ ] `低占位成立`
- [ ] `可切口性成立`
- [ ] `可触达性成立`

#### 承接成立

- [ ] `可证据化`
- [ ] `可闭合承接`
- [ ] `值得进入`

### 10.4 最低通过条件

- `问题场一句话定义` 已写清
- `R / L / V` 三项同时成立
- `BOS >= 45`
- 无一票否决项触发

### 10.5 输出层

- `进入的问题场名称`
- `核心矛盾一句话`
- `进入理由`
- `暂不进入边界`
- `首切场景`
- `BOS 分数`
- `是否为灰区测试还是正式放行`

### 10.6 最短结论

`择场不是找看上去大的地方，而是找真实张力、责任真空和最小承接能力同时成立的地方。`

## 11. 执行清单 B：定客检查

### 11.1 使用原则

本清单只判定一件事：

`先服务谁，才能形成第一闭环。`

它不追求大市场画像，

而只寻找：

- 第一批最先觉醒的人
- 第一批最愿进入正式承接的人
- 第一批最容易形成案例的人

### 11.2 定义层

- `首要客体名称`
- `身份 + 情境`
- `他正在承受的核心矛盾`
- `为什么是现在，而不是以后`

若属于组织型场域，再补：

- `守门角色 / 上送节点`
- `最终拍板者`
- `首个物理接近动作`

### 11.3 成立层检查项

#### 张力成立

- [ ] `客体承受核心矛盾`
- [ ] `紧迫度成立`
- [ ] `高频触发成立`
- [ ] `Kairos 信号成立`

#### 承接条件成立

- [ ] `授权链清楚`
- [ ] `正式承接意愿成立`
- [ ] `资源条件现实`
- [ ] `可复制性成立`

#### 组织拓扑成立

- [ ] `首要痛点客体清楚`
- [ ] `上送节点清楚`
- [ ] `最终拍板者清楚`

#### 进入成立

- [ ] `可触达性成立`
- [ ] `可理解性成立`
- [ ] `可承接性成立`
- [ ] `可案例化成立`

### 11.4 最低通过条件

- `身份 + 情境` 写得清
- 授权链清楚
- 存在真实正式承接意愿
- 存在现实可用的触点
- 无一票否决项触发

### 11.5 输出层

- `首要客体`
- `上送节点 / 守门角色`
- `最终拍板者`
- `次级客体`
- `暂缓客体`
- `首批触点`
- `首个弱测量动作`
- `预估正式承接理由`
- `预估主要异议`

### 11.6 最短结论

`定客不是找谁都可能感兴趣的人，而是找最先会痛、最先会进入正式承接、最先能形成证据，并能把问题送进决策链的人。`

## 12. 执行清单 C：配品检查

### 12.1 使用原则

本清单只判定一件事：

`该用什么承接器进入。`

这里的“产品”可以是：

- 咨询包
- 诊断服务
- 顾问包
- 软件
- 工具包
- 训练营
- 白皮书

原则只有一句：

`产品形态服从问题场，不让问题场服从既有产品。`

### 12.2 定义层

- `主产品形态`
- `它只承担哪一个主任务`
- `它服务哪个首要客体`
- `它对应哪个首切场景`
- `成功交付意味着什么`
- `它属于哪一层`
- `它之前的上一步是什么`
- `它之后的下一步是什么`

### 12.3 成立层检查项

#### 形态成立

- [ ] `单一主任务成立`
- [ ] `客体匹配成立`
- [ ] `价值表达成立`
- [ ] `位阶归属成立`

#### 交付成立

- [ ] `交付边界清楚`
- [ ] `承诺边界清楚`
- [ ] `失败判定清楚`
- [ ] `补偿与退出路径存在`
- [ ] `首单可完成`

#### 节奏成立

- [ ] `前后步骤清楚`
- [ ] `弱测量友好`
- [ ] `物理接近动作清楚`

#### 承接资源成立

- [ ] `资源点可接受`
- [ ] `承接动作匹配`
- [ ] `可标准化`
- [ ] `可回流总盘`

### 12.4 最低通过条件

- `主产品形态` 已定义
- `单一主任务` 已写清
- `成功交付` 可定义
- `失败判定` 可定义
- 至少有一个客户愿意按该形态和价格真实付费
- 无一票否决项触发

### 12.5 输出层

- `主产品形态`
- `所属层级`
- `辅助产品形态`
- `明确不做的形态`
- `价格带`
- `交付边界`
- `失败判定`
- `补偿与退出路径`
- `上一步 / 下一步`
- `首单验证动作`

### 12.6 最短结论

`配品不是决定你想推出什么，而是决定用什么分层承接器，在正确节奏上把问题场和首要客体接住。`

## 13. 当前最稳原则条文

本文件建议将原则压成如下正式条文：

`先识别真实而未被妥善承接的问题场，再确认应服务的人群与责任边界，最后设计可承接后果的产品；不得以既有产品反向制造需求。`

其短版表达保留为：

- `先定场，后定人，再定物。`
- `先定问题与责任，再定客户，最后定产品。`

## 14. 最短结论

把本文件压成最短版，只留下面八句：

- 蓝海不是凭空找出来的，而是在责任真空层中被切开的
- 真正的起点不是产品，而是 `问题场`
- 蓝海的三个核心指标是：`认知刚性 + 损耗强度 + 责任真空`
- 进入任何市场前，都应先做 `LMM-BOS` 评分
- 前端最小链条应是：`灯塔内容 -> 自测 -> 港口承接`
- 产品只是问题场的承接器，不是方法论起点
- 不能闭合责任，就不能点灯
- `LMM` 的前端顺序是：`先择场，后定客，再配品`
