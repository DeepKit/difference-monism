# WSM.004 · 与 COP 及 LMM 的接口规范

> 定位：定义 WSM 作为前线结构识别子框架，与 LMM 和 COP 的协作关系、触发条件与转接纪律。
> 作者：Yi Fu（付毅，ODDFounder）

---

## 一、WSM 在 LMM 内部的角色

WSM 是 LMM 的内部二级理论，不是独立层。它的分工是：

```
LMM 负责：寻域 → 确路 → 定人 → 灯塔 → 自测 → 港口 → 首次责任闭合
WSM 负责：在上述流程中，当问题涉及"看不清的结构困局"时，
          提供五局识别、责任流向分析与转接建议
```

WSM 的使用场景是：**LMM 已经接住了人，但发现这个人说不清"我到底卡在哪"**——这时 WSM 提供一个快速的结构化诊断语言。

---

## 二、LMM → WSM 触发条件

WSM 不在 LMM 的每一步都介入。它只在以下信号出现时被调用：

| 触发信号 | LMM 阶段 | WSM 介入方式 |
|----------|---------|-------------|
| 用户反复描述"我不知道为什么总是这样" | 自测阶段 | 用五局识别卡做结构显影 |
| 用户描述的问题在旧框架下解释不通 | 定人/港口 | 切换为责任流向视角重新解读 |
| 用户所在组织的问题被怀疑是结构性的 | 港口/闭合 | 判局后建议是否转 COP 或 TAT |
| 用户已明确表示"不是能力问题但就是动不了" | 自测/港口 | 这是 WSM 最强的适用信号 |

### 不触发 WSM 的情况

- 问题明确属于个人技能提升（→ RT6 直接承接）
- 问题已进入 COP 诊断管线（→ 由 COP 统一处理，不重复判局）
- 紧急危机需要立即冻结（→ 直接触发 COP FREEZE，不走过场判局）

---

## 三、WSM → COP 转接规则

WSM 判局后，根据严重程度和持续性决定是否转 COP：

### 直接转 COP 的条件

| 条件 | 示例 |
|------|------|
| 用户正承受高不可逆后果 | 已被降职、被开除、被法律追诉 |
| 局型长期反复（>6 个月）且持续恶化 | 同一模式的第三次以上重复 |
| 涉及多角色且需要正式诊断 | 不只是一个人卡住，是整条责任链断裂 |
| 前线判断置信度低 | 判局结果标记为"低置信度" |

### 转 COP 时携带的信息

```yaml
wsm_to_cop:
  wsm_diagnosis:
    primary_pattern: ""     # 主局
    secondary_pattern: ""   # 次局
    confidence: high | medium | low
  key_signals: []           # 支撑判断的关键信号
  duration: ""              # 局型持续时长
  escalation_reason: ""     # 转 COP 的原因
  recommended_cop_action: FREEZE | MIXED | UNKNOWN  # 建议但不绑定
```

### 不转 COP 的情况

- 局型初现、干预后已改善 → 继续 LMM/WSM 前线跟进
- 用户明确要求不上报 → 尊重（除非涉及他人安全或法律红线）
- 通过 WSM 的最低动作已经止血 → 标记为"已止流，观察中"

---

## 四、WSM → TAT 转接条件

WSM 不直接转 TAT。所有 WSM→TAT 的转接必须经过 COP 或 LMM 的港口承接后，由它们判断是否已触及 TAT 的责任门槛。

这是因为：WSM 是前线识别工具，它的判局结论不应直接触发责任裁决。中间必须经过至少一级复核（LMM 港口或 COP 分诊）。

### 例外：紧急冻结直通

如果前线识别到以下任一信号，WSM 可建议直接冻结并上报：

1. 用户声称其人身安全受到组织结构的威胁
2. 用户正在被强迫承担违法后果
3. 责任失衡已导致明确的医疗诊断（如过劳、抑郁诊断）

在这种情况下，WSM 输出 `EMERGENCY_FREEZE`，不经完整判局流程，直通 COP 或人工复审。

---

## 五、WSM 与 LMM 的反馈回流

WSM 的判局结果应回写到 LMM 的港口记录中，形成闭环：

```
LMM 港口接住人 → WSM 判局 → LMM 给出结构视角 →
用户尝试最低动作 → 反馈效果 → WSM 调整判断
```

如果最低动作无效：

- 重新判局（可能是次局被低估了）
- 如果两次判局后仍然无效 → 升级到 COP

如果最低动作有效：

- LMM 港口记录闭合
- WSM 将此例标记为"正例"，未来类似信号可加速判断

## 六、越界纪律

| 越界行为 | 为什么危险 | 正确做法 |
|----------|-----------|---------|
| WSM 判局后直接断定"这就是 TAT 责任问题" | 前线识别工具不应做最终裁决 | 转 COP 或 LMM 港口复核 |
| WSM 被用来给同事贴标签 | 会恶化关系、简化人的处境 | WSM 语言只能用于自诊或委托诊断 |
| WSM 判局替代 COP 的冻结判定 | COP 的 FREEZE 有自己的触发标准 | WSM 判局只是 COP 的输入之一 |
| WSM 判局结果直接对外公开 | 判局结论未脱敏，可能伤害当事人 | 对外只能讲模式，不能讲具体人名/事件 |

---

## 七、一句话

> WSM 是 LMM 的眼——帮前线看见结构。但它不是法官，冻结和裁决要交给 COP 和 TAT。
