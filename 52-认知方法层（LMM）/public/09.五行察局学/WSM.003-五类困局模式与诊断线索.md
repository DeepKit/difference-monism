# WSM.003 · 五类困局模式与诊断线索

> 定位：对五种责任流向失衡模式做深度展开——每种局型的形成机制、恶化路径、常见变体、诊断 checklist。
> 作者：Yi Fu（付毅，ODDFounder）

---

## 水局 · 责任下滑

### 形成机制

水局的核心动力不是"有人推卸责任"，而是**责任没有在结构上被接住**。当一件事的主责方未被明确，或虽有名义主责但实际不承接，责任就会沿着最小阻力路径滑到最能扛的人身上。

### 恶化路径

```
主责模糊 → 默认沉默 → 最靠近的人顶上 →
第一次被默认为"本来就是你的事" →
第二次连问都不问就直接流向你 →
系统形成"找他就对了"的默认路径
```

### 常见变体

| 变体 | 特征 |
|------|------|
| 缝隙型 | 两部门之间的灰色地带，谁的都不是 |
| 缺席型 | 名义上有主责，但那个人始终不在场 |
| 层级滑落型 | 上层不做决定，决策压力和后果一起下滑 |

### 诊断 Checklist

- [ ] 有明确的主责方吗？你说得出来吗？
- [ ] 接手时有正式的交接或委托吗？
- [ ] 如果有人问"这件事谁负责"，答案是你还是别人？
- [ ] 你接这个任务是因为你是最合适的人，还是因为只有你在？
- [ ] 如果明天你请假，这件事有人接吗？

## 木局 · 责任蔓生

### 形成机制

木局的动力不是"被欺负"，而是**善良被结构吸收**。一次临时的"我帮你"，因为做得好、做得快，变成了"你做最合适"，再变成"这不就是你的事吗"。

木局最隐蔽的伤害是：**扩张过程中没有人是恶意的**——所有人都真心觉得"你确实最合适"。

### 恶化路径

```
一次临时补位 → 结果不错 → 下次自然又找你 →
范围从"那个模块"变成"那类问题" →
你的能力边界被默认为无限 →
退出变得"不负责任"
```

### 常见变体

| 变体 | 特征 |
|------|------|
| 能力型蔓生 | 因为你做得好，所以越来越多 |
| 速度型蔓生 | 因为你做得快，所以紧急的都找你 |
| 好人型蔓生 | 因为你不拒绝，所以默认你会接 |

### 诊断 Checklist

- [ ] 有没有一个任务是"先顶着"但顶了超过三个月的？
- [ ] "帮忙"和"主责"的边界在你的工作里清晰吗？
- [ ] 你的职责范围在最近半年是扩大了还是缩小了？
- [ ] 有没有人说"这个你比较熟你先做"但没说何时正式交给你？
- [ ] 如果要退出一项任务，你知道该交给谁吗？

## 火局 · 责任引爆

### 形成机制

火局的前提是**平时不定责**。日常运转中，谁决策、谁执行、谁批准是模糊的。一旦出事，追责的本能是找"离事故最近的那个人"——而那个人往往不是决策者，是执行者。

火局的爆发是瞬间的，但火局的**引信在平时就埋好了**。

### 恶化路径

```
日常不定责 → 决策链无留痕 → 事故发生 →
追责本能找最近可追的人 → 执行位被点燃 →
事后补口径、补流程 → 但下次仍不定责
```

### 常见变体

| 变体 | 特征 |
|------|------|
| 事后追认型 | 出事后才补"这是谁的责任" |
| 链条断裂型 | 中间审批环节有人签名但无人真正审过 |
| 甩锅竞技型 | 多部门互推，最后停在最不会推的人身上 |

### 诊断 Checklist

- [ ] 关键决策有书面记录吗？（不只是口头的）
- [ ] 你知道谁批准了这个方案吗？
- [ ] 如果这件事出错，第一个被找的人是谁——你估计是决策者还是执行者？
- [ ] 近半年有没有发生过"事后补签字/补口径"的情况？
- [ ] "出了事我负责"这句话在你们环境中，是事实还是安慰？

## 土局 · 责任沉积

### 形成机制

土局的本质不是某次任务分配的问题，而是**系统缺少清理机制**。低可见度的杂务、历史遗留问题、灰区任务——这些没有"完成"标志的东西，会在一段时间后变成承压点的永久附着物。

土局不像火局那样爆发，它更像缓慢堆积——三年后才发现自己成了系统的缓冲垫。

### 恶化路径

```
灰区任务产生 → 无人认领 → 默认归到能处理的人 →
新的灰区任务继续产生 → 旧的不清，新的追加 →
承压点从"做这些事的人"变成"系统缓冲垫"
```

### 常见变体

| 变体 | 特征 |
|------|------|
| 杂物堆积型 | 大量低可见度杂务集中在一人身上 |
| 历史遗留型 | 前人留下的没人敢动的老系统/老流程 |
| 情感劳动型 | 情绪安抚、矛盾调解、团队维护等不被计量但持续消耗的工作 |

### 诊断 Checklist

- [ ] 你手头有没有超过三件"不知道该归谁"的任务？
- [ ] 你的杂务有没有被正式计入工作量？
- [ ] 这些杂务有没有定期清理、交接或消解的机制？
- [ ] 如果有人问"你每天都在忙什么"，你能列出一份清晰的任务清单吗？
- [ ] 最近一次有人说"这些杂事你先顶着"是什么时候？有退出时间吗？

## 金局 · 权责割裂

### 形成机制

金局是五种局型中结构暴力最强的。它的本质不是"能力不足"，而是**结果责任与决策权的结构性分离**——你承担"事情没办好"的全部后果，但你没有决定"事情怎么办"的权力。

这在组织中通常表现为：资源审批在上、执行压力在下、后果归属在执行位。

### 恶化路径

```
组织赋予目标责任 → 但没有同步赋予资源/决策权 →
执行位自己想办法凑资源 → 结果不理想时被追责 →
"你怎么没做好？"← 不提"你根本没有决定权"
```

### 常见变体

| 变体 | 特征 |
|------|------|
| 资源型割裂 | 有目标没资源，被要求"自己想办法" |
| 决策型割裂 | 执行中的关键选择需要上级批准，但上级不承担延误后果 |
| 承诺型割裂 | 被要求对客户/外部做承诺，但内部资源随时可能被抽走 |

### 诊断 Checklist

- [ ] 在对你最重要的那项任务上，你有权决定关键的资源分配吗？
- [ ] 你的绩效考核依据是"结果"还是"你控制范围内的产出"？
- [ ] 如果有人需要为这个项目的失败负责，是你还是给你定目标的人？
- [ ] 你最近有没有遇到过"被要求对结果负责，但关键决定不通过你"的情况？
- [ ] "我先批一下"——这句话在你的工作流中是常态还是例外？

## 五局交叉识别

现实中最常见的不是单局型，而是**交叉局型**：

| 常见交叉 | 机制 |
|----------|------|
| 水+木 | 责任从模糊边界滑过来，因为你能做而蔓生 |
| 水+火 | 平时不定责，出事时滑到你身上再引爆 |
| 木+土 | 临时补位长成长期职责，再逐渐沉积为系统缓冲垫 |
| 土+金 | 杂务堆积的同时，核心任务上权责割裂 |
| 火+金 | 平时有责无权，出事时第一个被点燃 |

交叉局型处理原则：**先止当前正在造成最大损失的那个流向。**
