# WSM.002 · 判局方法与结构识别框架

> 定位：WSM 的方法论核心——前线人员如何从模糊感受出发，通过结构化步骤识别当前处于哪种责任流向失衡。
> 作者：Yi Fu（付毅，ODDFounder）

---

## 一、判局不是算命

判局的核心纪律是：**不是"我觉得你是什么局"，而是"当前信号更匹配哪种流向模式"**。

判局不要求判断者"理解全部组织架构"或"掌握完整上下游理论"。它只要求判断者能做三件事：
1. 看信号（发生了什么）
2. 比模式（更接近哪种局型）
3. 做最小动作（先止流）

---

## 二、判局四步法

### 第一步：定位承压点

先不问"为什么"，先问"现在谁在承受后果"。

```
问题：这件事最终压在谁身上？
信号：谁在加班？谁在被追问？谁在补锅？谁在说"我来"？
输出：一个或多个承压点名称/角色
```

这一步只需要观察，不需要分析。承压点可以是自己，也可以是观察到的他人。

### 第二步：追溯流向

从承压点往前追溯：责任是怎么流到这里的。

```
问题：这个责任是怎么落到承压点身上的？
信号：
  - 是没人接所以默认落到这里？（→水局线索）
  - 是因为上次做得好所以继续追加？（→木局线索）
  - 是出了事被点名？（→火局线索）
  - 是一直堆在这里没清过？（→土局线索）
  - 是要负责但没权决定？（→金局线索）
```

### 第三步：比对准入条件

每种局型有最低准入条件。满足准入条件才能标注该局型。

| 局型 | 最低准入条件 |
|------|-------------|
| 水局 | 承压点之上存在一个未明确的主责方，且无人打断默认承接 |
| 木局 | 存在至少一次临时补位被转化为长期职责的证据 |
| 火局 | 承压点在近期事故/追责中是第一个被点名的执行位 |
| 土局 | 同一节点上存在至少两类不同性质的杂务长期堆积 |
| 金局 | 在至少一项任务上，结果责任与决策权明确不在同一节点 |

### 第四步：标注主局与次局

```
输出格式：
  主局：X局（命中 N 条准入条件）
  次局：Y局（命中 M 条准入条件，但主要矛盾在 X）
  置信度：高/中/低
```

若两个局型命中数接近，优先以"当前正在造成最大损失"的局型为主局。

---

## 三、前线最短判局协议

当没有时间完整走四步时，使用九题快速判局：

1. 这件事有明确的主责人吗？没有 → 水局线索
2. 你接这个任务时有人正式交接过吗？没有 → 水局线索
3. 你上次帮忙做的事变成你的固定职责了吗？是 → 木局线索
4. 有人说过"这个你比较熟你先做"吗？有 → 木局线索
5. 最近一次出事，第一个被找的是你而不是决策者吗？是 → 火局线索
6. 这件事的决策过程有书面记录吗？没有 → 火局线索
7. 你手头有没有三件以上"不知道该归谁"的任务？有 → 土局线索
8. 有没有人说过"这些杂事你先顶着"？有 → 土局线索
9. 你被要求对结果负责，但无权决定关键资源或方向？是 → 金局线索

每条"是"计 1 分。某局型累计 2 分 → 触发该局型标注。

## 四、判局失败模式

| 失败模式 | 表现 | 修正 |
|----------|------|------|
| 过早归因 | 看到一个信号就下结论 | 至少命中 2 条准入条件才标注 |
| 人格化解释 | "他就是这种人所以被压" | 回到流向追溯——看结构不看性格 |
| 单局型盲区 | 标注了主局就忽略了次局 | 双局型标注是默认要求 |
| 过度自信 | 低信息量下标高置信度 | 信息不足时标注"低置信度"并建议补充观察 |
| 只判不动 | 识别了局型但没有最低动作 | 每个判局必须跟一个最低动作 |

## 五、弱测量：在不完整信息下做判断

现实前线很少能拿到完整信息。弱测量是指在信息不完整的情况下，用可观测的间接信号做方向性判断。

### 弱测量的三条纪律

1. **只基于可观测信号，不基于推测**：信号是"他连续三周在周末补这个模块"，不是"他一定很痛苦"
2. **标注信号来源和可靠性**：是自己观察的？对方说的？第三方转述的？
3. **低可靠性信号不得单独支撑判断**：至少需要一条高可靠性信号或两条中可靠性信号

### 弱测量信号卡

```yaml
weak_measurement:
  signal: ""              # 可观测信号
  source: self | direct | indirect  # 来源
  reliability: high | medium | low
  points_to: ""           # 指向哪个局型
  confidence: high | medium | low
```

## 六、判局与ASTO前置编码的关系

在进行判局之前，理想情况下应先继承 ASTO 的前置编码：

| ASTO 字段 | 判局前要问的问题 |
|-----------|---------------|
| 五态 | 当前结构处于什么状态？（稳态/扰动/转换/崩塌/重建） |
| 六阶 | 问题处于哪个阶段？（潜伏/显性/扩散/峰值/衰减/残留） |
| 边界 | 边界清楚还是模糊？ |
| 例外 | 这是正常情况还是例外情况？ |

如果来不及做完整 ASTO 编码，至少回答一个问题：

> **这个问题是突发的（急），还是长期存在的（慢）？**

突发 → 优先看火局/水局。长期 → 优先看木局/土局/金局。
