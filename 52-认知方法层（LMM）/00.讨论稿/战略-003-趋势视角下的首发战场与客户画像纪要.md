## 一、这轮为什么要改判法

本轮裁决：

> **行业选择不能再按“某家大公司利润很高”来判断。**

原因很简单：

1. 单个龙头公司利润高，不等于新咨询品牌容易进入。
2. 老板会不会买单，取决于他是否处在持续流血、责任滑逸、且能快速看到证据的场域。
3. 真正该看的，是：

`趋势强度 + 行业利润质量 + 责任复杂度 + 老板决策权 + 证据可见性 + 进入难度`

因此这轮不再以“一个企业一个企业查利润”为主，

而改为：

> **先看趋势报告里的重复关键词，再用官方行业数据和 LMM/COP 的责任逻辑做二次筛选。**

## 二、趋势报告里反复出现的关键词

### 1. 刘润线索

从 `2025` 年度演讲及相关传播里，最值得记住的不是某个单一行业，

而是以下关键词：

- `大迁徙`
- `生态位干旱化`
- `品类迁徙`
- `能力迁徙`
- `智能迁徙`

对老板线的启发是：

> **企业不是单纯缺流量，而是旧能力、旧组织和新场景发生错配。**

这意味着：

- 交付链会变长
- 接口会变多
- 老老板法则会失效
- 责任滑逸更容易出现

### 2. 吴晓波线索

`2025-12-28` 的《AI闪耀中国》一类材料中，反复出现的关键词是：

- `场景化 AI`
- `智能体`
- `具身智能`
- `工业 5.0`
- `灯塔工厂`
- `AI 进入真实产业`

对老板线的启发是：

> **真正持续扩张的，不是抽象 AI 概念，而是 AI/自动化进入真实交付链后的系统重组。**

一旦 AI、设备、软件、人和服务开始一起交付，

责任滑逸就会从“说不清”变成“赔得起、拖不起、停不起”的问题。

### 3. 香帅线索

香帅团队在 `2025-11-08` 左右发布的出海竞争力材料里，

更稳定的关键词是：

- `第二增长曲线`
- `技术密集型`
- `价值链两端攀爬`
- `产能出海`

对老板线的启发是：

> **一批企业正在从“卖产品”转向“卖系统、卖能力、卖交付”，这会极大抬高责任边界复杂度。**

## 三、官方行业数据给出的硬信号

本轮只保留与战场筛选有关的硬信号：

1. 软件和信息技术服务业：
   - `2025` 年业务收入 `154831` 亿元
   - 同比增长 `13.2%`
   - 利润总额 `18848` 亿元
   - 同比增长 `7.3%`
   - 其中信息技术服务收入同比增长 `14.7%`

2. 机器人行业：
   - `2025` 年前三季度营收同比增长 `29.5%`

3. 工业利润细分：
   - `2025` 年一季度专用设备制造业利润同比增长 `14.2%`
   - 医疗仪器设备及器械制造利润同比增长 `12.5%`
   - 航空航天器及设备制造利润同比增长 `23.9%`

这些数据说明：

- 产业智能化升级是真趋势
- 高技术制造和连接层服务都有钱
- 但“有钱”与“适合首发”不是一回事

## 四、主持人总裁决

### 1. 最值得打的不是单一行业，而是一条产业带

当前最稳的首发战场，不应被写成：

- 只打医疗
- 只打软件
- 只打机器人

而应写成：

> **产业智能化升级中的连接层产业带**

它主要包括：

1. `机器人 / 工业自动化系统集成`
2. `专用设备制造 / 电子工业专用设备 / 产线升级`
3. `工业软件 / 企业 AI 集成 / 系统实施 / 信息技术服务`

因为这些场域同时满足：

- 趋势在向上
- 老板有预算
- 交付链复杂
- 责任滑逸高发
- 损耗易量化
- 新品牌也更可能切入试点

### 2. 当前三层排序

#### 第一优先：机器人 / 自动化系统集成 + 专用设备制造

这是当前最稳的 **首发主战场**。

原因：

- 趋势热
- 预算真实
- 责任滑逸最容易表现为停线、延期、返工、赔付
- 证据最好拿
- 老板更容易理解“责任断点”不是空话

#### 第二优先：工业软件 / 企业 AI 集成 / 系统实施 / 信息技术服务

这是当前最稳的 **次战场**。

原因：

- 软件与信息技术服务总体增长稳定
- “买了很多系统但没有结果”是高频痛感
- 很适合 `责任滑逸CT`

但它不是第一优先，

因为这类客户更容易把你理解成：

- 流程咨询
- IT 交付顾问
- 数字化优化建议方

区分度比设备与自动化交付场更难打穿。

#### 第三优先：高端医疗器械制造

这是当前最稳的 **高价值后备战场**。

原因：

- 利润和预算质量高
- 合规、追溯、质控要求高
- 责任闭环价值大

但它不宜做首发，

因为进入门槛、合规要求和采购周期都更重。

### 3. 暂不建议首发的行业

以下行业虽然有钱，当前却不宜优先做首发：

1. `航空航天 / 军工链`
2. `纯半导体大厂 / 超大型 IC 龙头`
3. `纯金融机构`

原因：

- 审批长
- 监管重
- 保密强
- 新品牌信任门槛高
- 试点很难做到轻量、快速、可复核

## 五、首发客户画像 v0.1

### 1. 公司画像

当前最美好顾客，不是“行业龙头”，

而更像：

- 员工规模约 `50-500`
- 已经进入多部门、多角色、多系统协作
- 正在同时使用 `ERP / CRM / MES / WMS / OA / AI 工具 / 项目系统` 中的若干项
- 有设备、软件、实施、交付、售后中的至少两段以上责任交接
- 老板已经开始觉得“钱花了很多，但结果没出来”

### 2. 老板画像

最稳的对象不是“最会讲话的老板”，

而是：

- 创始人
- 总经理
- 事业部一号位
- 能拍板预算，也要承担交付后果的人

他/她通常会说：

- `为什么每个人都说自己在负责，但事情还是掉了？`
- `系统买了、设备上了、人也加了，怎么还是拖？`
- `到底哪一段最先该补？`

### 3. 必须同时出现的五个信号

只有以下五个信号同时出现，才值得优先靠港：

1. 有持续流血
2. 有多角色交接
3. 有责任真空
4. 有最小证据
5. 老板在场

### 4. 暂不进入的对象

以下对象即使看起来大，也不建议先打：

1. 只想听概念、不愿给事实的人
2. 责任人不在场，只让中层接待的人
3. 完全没有证据链，却只想要一份背书的人
4. 明明是全局组织改造，却伪装成单点诊断的人

## 六、对后续咨询的直接影响

本轮之后，

老板线咨询不应再先问：

- 我们要不要先定义产品名？
- 要不要先做网站？
- 要不要先铺全行业？

而应先问：

1. 第一批样本先从哪一类连接层企业拿？
2. 哪一类责任滑逸最容易被量化成停线、延期、返工、赔付？
3. 第一批灯塔内容应该打哪一类老板的“别扭感”？
4. 首发战场里，哪个单病种最容易做出首个样板案例？
