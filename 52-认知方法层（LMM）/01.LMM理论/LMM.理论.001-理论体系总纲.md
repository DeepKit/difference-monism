## 一、理论总判：前线协议，而非经营技巧

灯塔方法（LMM）不是关于“如何放大流量或推进交易”的技巧集合，而是关于**“在高噪声问题场中，如何显影结构断口、识别弱测量信号、组织港口接引，并把高影响问题送入责任闭合与审计链”**的前线协议。

在整个上游链条中：

- `DM` 提供研究纲领与最低本体承诺。
- `ASTO` 提供跨层桥接语法。
- `ECET` 提供治理边界与文明级约束。
- `COP` 负责主体状态诊断与分流编码。
- `TAT` 负责高影响责任门槛与闭门责任裁决。
- `ODD` 负责承诺、证据、封存与审计实现。
- `LMM` 只负责现实前线的显影、弱测量、港口接引、正式承接与责任回流节律。

补充总纲纪律：

- `LMM` 的目标不是扩大触达量，而是降低错配、误承接与责任悬空。
- 前端最小合法链条不是“内容 -> 线索 -> 推进”，而是：
  `灯塔内容 -> 自测/弱测量 -> 港口接引 -> 正式承接 -> 责任闭合记录`
- 自动化可以承担前置显影、筛查与节律管理，但不得替代首次正式承接与责任说明。

## 二、分层发布与隔离结构

为确保公共理论主链、私有承接层与私有回流层不互相倒灌，本体系执行严格的分层隔离：

| 模块 | 身份 | 发布位阶 | 允许内容 | 禁止回灌 |
|:---:|:---|:---|:---|:---|
| **01.LMM理论** | 公共理论主链 | 可外发 | 核心命题、术语、证伪条件、失败线 | 私有定价、承接条款、经营数据 |
| **02.LMM定位理论** | 公共探测层 | 可外发 | 场域探测、BOS 纪律、寻域与确路 | 个案话术、私有正式承接经验 |
| **03.实践指导** | 公共协议层 | 条件外发 | 显影、弱测量、港口接引、责任转介模板 | 定价脚本、回款动作、经营指标 |
| **04.产品打造** | 私有承接层 | 内部/定向外发 | 表单、首诊包、承接件、交付封装 | 冒充公共法源或理论证据 |
| **05.理论变现** | 私有回流层 | 仅内部 | 定价、对价、复靠、回流与退款规则 | 直接充当公共理论结论 |
| **06.理论发售(隔离区)** | 隔离档案层 | 极低 | 隔离式经营材料、历史封存、发售方案 | 倒灌到公共主链 |
| **08.扩展应用** | 扩展应用层 | 低 | 自动化实验、部署母本、应用扩展 | 冒充主链入口 |
| **07.LMM研究** | 公共研究接口层 | 入口可外发 | 研究总述、摘要、案例复核入口 | 私有回流数据直接当理论证据 |

> 分层边界与发布纪律以 `LMM.P99.2.公共-私有-隔离发布边界表.md` 为准。

### 关于 `011` 的唯一合法身份

`011`

不是产品规格书，

也不是私有回流指南。

它在 `01` 库中的唯一合法身份是：

`受限附录（Restricted Appendix）`

即：

- 只允许从 `原理 -> 实现类型` 做单向追溯说明
- 不允许把 `04/05` 的私有条款、定价逻辑、回流动作反压回 `01`
- 不得在 `011` 中出现需要频繁变动的私有编号、对价条款与沟通脚本

压成一句话：

`011 只是一张法源到实现类型的只读索引，不是穿墙指南。`

## 三、术语公约 (Term Mapping)

为保持公共主链的克制感与可审阅性，在 `01-03` 与 `07` 中执行以下术语强制：

| 旧经营语境 | 公共主链强制口径 | 核心语义 |
|:---|:---|:---|
| 客户 / 消费者 | **认知主体 / 责任节点** | 强调其作为系统中的反应单位 |
| 销售 / 成交 | **正式承接 / 责任闭合动作** | 强调风险与权属的物理转移 |
| 卖货 / 产品 | **干预模块 / 承接件** | 强调对系统摩擦的切除功能 |
| 话术 / 宣传语 | **触点协议 / 认知探针** | 强调对认知相变的触发手段 |
| 流量 / 寻客 | **触点动能 / 问题场进入** | 强调对系统流变信号的识别 |
| 变现 / 收钱 | **私有回流 / 风险对价回收** | 强调价值交换与责任补偿平衡 |

## 五、03库当前主链补注

自 `2026-03-31` 起，

03库前端执行顺序统一理解为：

`择场 -> 定客 -> 配品 -> 灯塔内容 -> 自测 -> 港口接引 -> 正式承接 -> 责任闭合`

其中：

- `灯塔内容` 负责显影，不负责推进对价动作
- `自测` 负责弱测量，不负责强逼表态
- `港口接引` 只承接已经出现物理接近或明确问题窗口的主体
- 超出前线权限后，主体状态回引 `COP`，高影响责任门槛回引 `TAT`，承诺与证据封存回引 `ODD`

若后续出现派生文件与此链条冲突，

以 `LMM.理论.015` 与相关真本文件为准。
