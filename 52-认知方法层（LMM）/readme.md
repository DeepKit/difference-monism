# 灯塔营销方法论 (LMM) - 资产总目录

> **更新时间**：2026-03-31
> **状态**：本库已完成第一轮工程整编；当前公共主链按 `01-03 + 07` 阅读，`04-05` 为私有承接与回流层，`06` 为隔离层，`08` 为扩展应用层，`09` 为 `LMM` 内部新增的二级理论层。双 `06` 过渡期已经结束。

补充定位：

**LMM 不是独立于主链理论之外的经营技巧总称，而是整套理论在现实前线用于显影、接引、承接与责任闭合的协议层。**

进一步的跨层边界固定为：

- `BOS` 只评估问题场是否值得进入，不评估具体主体的结构状态。
- `COP` 负责主体状态诊断、分流状态与升级判断。
- `LMM` 负责问题显影、弱测量、港口接引与责任回流节律。
- `TAT` 负责高影响责任门槛与闭门责任裁决。
- `ODD` 负责承诺、证据、封存与审计实现。

## DM 风险防火墙

### 最小依赖

即使不先接受 `DM` 的强版本，`LMM` 至少仍可建立在以下弱前提上：

1. 现实中存在可被显影的问题场
2. 主体会表现出可观测的损耗、断链与责任真空
3. 前线承接需要弱测量、接引、转介与首次责任接近
4. 首次责任闭合之后，必须把高影响动作交给更正式的协议层

### `DM` 受争议时如何阅读本层

若 `DM` 的强本体论或强现象学承诺受争议，
`LMM` 当前最稳的降级阅读方式是：

- 先把它读成前线 intake / handoff 方法
- 先把它读成现实显影与责任接近协议
- 不把它继续写成只有接受 `DM` 才成立的经营神学

### 本层可独立成立的部分

- 寻域、确路、定人
- 灯塔、自测、港口接引
- 首次责任接近与责任回流节律
- 公共主链与私有承接层的边界纪律

### 本层仍强依赖 `DM` 的部分

- 与 `DM` 的强现象学解释链直接绑定的句子
- 把 `LMM` 的全部有效性写成 `DM` 唯一前线落地方式的说法

### 本层防火墙句

> **即使 `DM` 受压，`LMM` 仍可作为现实前线的显影、接引与首次责任接近协议成立；本层首先由能否把问题场看清、把责任移交做实来站住，而不是只由上游哲学叙事站住。**

## 核心分工准则与目录结构

绝不允许跨目录混用概念。每个目录负责解决一个特定层级的唯一问题，一切以“入库判准”作为边界。

### 01.LMM理论 (核心本体)
**前缀符**：`LMM.理论.xxx`
**入库判准**：这是否是不随客户改变而改变的底层定律与红线？
**核心动作**：法源（为什么能做） + 机制（如何相变、网树降维） + 边界（责任闭合的红线）。
> *注：只有这里产出的东西才叫“法源”。这是整个 LMM 体系的最高解释层，一旦定稿，前线无权反压篡改。*

### 02.LMM定位理论 (雷达与导航)
**前缀符**：`LMM.定位.xxx`
**入库判准**：这是否清晰地回答了域、路、人的坐标？
**核心动作**：寻域（找结构性流血区） + 确路（识别有效接近路径） + 定人（识别处于真实损耗中的责任节点）。
> *注：01 讲物理定律，02 用雷达捕捉问题场。在这里，LMM 的定位只做寻域、确路、定人。*

### 03.实践指导 (单兵战术动作)
**前缀符**：`LMM.实践.xxx`
**入库判准**：这篇指南是否能直接指导执行者去“建塔、打光、接引”？
**核心动作**：立塔 + 发光（发触点高频脉冲） + 接引。
> *注：这里保留的是协议型操作模板，不是私有经营手册。*

### 04.产品打造 (产品与交付件)
**前缀符**：`LMM.产品.xxx`
**入库判准**：这东西能否被直接拿在手里、装在系统里，或者拿去填表？
**核心动作**：将 LMM 思想物化成前台自测件、港口承接件、首诊大纲、软件许可（2KeyRun）、AI探测器。

### 05.理论变现 (承接与回流)
**前缀符**：`LMM.变现.xxx`
**入库判准**：这个策略或话术，能不能把责任闭合、对价回流与后续动作说清？
**核心动作**：港口承接路径设计、报价逻辑、回流设计、R=1 / R-Eval 合同闭环。
> *注：这里是私有回流层，不属于首轮公共理论主包。*

### 06.理论发售(隔离区)
**前缀符**：`LMM.发售.xxx`
**入库判准**：这到底是为了卖 B2B 干预服务，还是为了教授别人如何靠理论发售赚钱？
**核心动作**：这是隔离区，专门封存“理论发售”方案，避免其混入 `01-05` 主战线。

### 07.LMM研究 (学术研究接口)
**前缀符**：`LMM.研究.xxx`、`Case.xxx`
**入库判准**：这是否属于理论对外发表、案例证据、复核、匿名化、盲评与投稿执行的研究接口？
**核心动作**：研究总纲 + 对外理论总述 + 案例论文 + 方法论文 + 案例证据链 + 投稿执行。
> *注：自 2026-03-29 起，`07.LMM研究` 的执行主位统一放在 `D:/_Progs/.BetterCiv/05_学术型/07.LMM研究/`。`一元论` 中仅保留理论入口与必要索引，不再把案例补证、盲评、投稿推进材料作为主维护位。*

### 08.扩展应用 (实验与部署层)
**前缀符**：`LMM.扩展.xxx`
**入库判准**：这是否属于自动化部署、应用实验或单人创始人工作流，而非 `LMM` 公共理论主链？
**核心动作**：承载自动化实验、部署母本、扩展应用与场景化试装稿。
> *注：`08` 不构成公共理论主链，只作为扩展应用与实验层存在。*

### 09.五行察局学 (二级理论与结构识别子框架)
**前缀符**：`WSM.xxx`
**入库判准**：这是否属于把 `COP / TAT` 的责任流向判断，压成前线可调用的结构识别语言？
**核心动作**：识别责任如何沿最小阻力路径流动、失衡、堆积、引爆与割裂，并把模糊感受转成可复述、可判断、可行动的结构语言。
> *注：`09` 不是独立于 `LMM` 的新总层，也不是公开主链之外的散件区；它是 `LMM` 内部新增的二级理论线。上承 `COP / TAT` 的诊断与责任逻辑，下接 `LMM` 的前线显影、产品转译与现实承接。*

当前 `09` 的优先优化方向固定为：

- 用 `ASTO` 统一其状态、节律、边界与例外语言
- 用 `COP` 固定其冻结、未知与转介纪律
- 用 `TAT` 固定其责任闭合与权责割裂判断边界
- 把输出压成前线短句、标签、判别树与弱测量指标

## 纪律红线 (圆桌总宪章)

1. **更新权限的阶梯**：前线执行侧可修改 `04` 和 `05`，专家层才能调整 `02` 和 `03`，只有最高指挥官拥有修改 `01` 法源的权限。
2. **严禁混淆“定位”概念**：任何人再试图“用 LMM 定位去操纵用户心智”，即视为对本体系法则的背叛。LMM 的定位只做：**寻域、确路、定人**。
3. **研究执行主位单一**：`07.LMM研究` 的案例、补证、匿名化、盲评与投稿执行材料，以 `.BetterCiv/05_学术型/07.LMM研究/` 为唯一执行主位；`一元论` 不再双头维护。
4. **收入必须回流主链**：任何由 LMM 形成的产品收入，原则上都应服务于理论传播、现实部署、案例积累与学术推进，不得脱离主链单独漂移。
