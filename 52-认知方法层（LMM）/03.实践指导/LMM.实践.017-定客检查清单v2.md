## 0. 使用原则

本清单只判定一件事：

`先服务谁，才能形成第一闭环。`

它不追求大市场画像，

而只寻找：

- 第一批最先觉醒的人
- 第一批最愿付费的人
- 第一批最容易形成案例的人

在使用本清单前，

必须已有一个通过 `LMM.实践.015-择场检查清单v2` 的问题场。

补充纪律：

- 不只问“谁会买”，还要问“谁最痛、谁能上送、谁会拍板”。
- 组织型问题场中，`首要痛点客体` 与 `最终拍板者` 往往不是同一个人。
- 前端触达优先寻找出现 `Kairos` 信号的人，而不是名义上最有权的人。

## 2. 成立层检查项

### 2.1 张力成立

- [ ] `客体承受核心矛盾`：他真实持续受困于该问题场
- [ ] `紧迫度成立`：问题已逼近决策点，不是纯兴趣
- [ ] `高频触发成立`：该矛盾会反复出现，而非偶然一次
- [ ] `Kairos 信号成立`：已出现追问、转发、下载、自测、样本提交等弱测量信号

### 2.2 商业成立

- [ ] `付款链清楚`：能分清付款人、决策人、使用者
- [ ] `支付意愿成立`：该客体愿意优先为此问题花钱
- [ ] `预算现实`：预算、价格承受力与我方定位匹配
- [ ] `可复制性成立`：不是孤例，后续可扩成第二批、第三批

### 2.3 组织拓扑成立

- [ ] `首要痛点客体清楚`：谁最先感受到损耗，已能写清
- [ ] `上送节点清楚`：谁能把问题安全送进决策链，已能写清
- [ ] `最终拍板者清楚`：谁真正拥有批准或否决权，已能写清

### 2.4 进入成立

- [ ] `可触达性成立`：存在稳定触点、内容入口、社群入口或关系入口
- [ ] `可理解性成立`：该客体能听懂我方价值表达
- [ ] `可承接性成立`：我方对这类客体的结果风险可承担
- [ ] `可案例化成立`：成交后容易形成案例、证词或样板

---

## 3. 最低通过条件

满足下面五条，才算通过：

- `身份 + 情境` 写得清
- 付款链清楚
- 存在真实支付意愿
- 存在现实可用的触点
- 无一票否决项触发

若属于组织型问题场，

建议再加一条：

- `首要痛点客体 / 上送节点 / 最终拍板者` 至少已有最小拓扑判断

建议补充判断：

- 若客体`痛但不付钱`，放弃作为首批
- 若客体`会付钱但极难触达`，转为次级客体
- 若客体`会触达也会付钱但案例价值低`，谨慎进入
- 若客体`最痛但无法上送`，不宜单独作为首批入口

---

## 4. 一票否决项

出现以下任一情况，直接停止：

- [ ] 只有兴趣，没有真实痛感
- [ ] 有痛感，但无付款权或决策权
- [ ] 需要过长教育期，短期无法形成闭环
- [ ] 客体过宽，无法明确首批承接对象
- [ ] 风险或补偿义务超出我方责任上限
- [ ] 即使成交，也无法形成案例或复制路径
- [ ] 组织型场域中，连守门角色和拍板者都分不清
- [ ] 尚无任何弱测量信号，却试图直接推进高承诺动作

## 7. 最短结论

把本清单压成一句话：

`定客不是找谁都能买的人，而是找最先会痛、最先会付、最先能形成证据，并能把问题送进决策链的人。`
