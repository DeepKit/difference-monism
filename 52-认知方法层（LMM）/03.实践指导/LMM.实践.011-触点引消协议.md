# LMM.实践.011 — 触点引消协议 (LPE Protocol)

> **版本**：v1.4 (Kairos + Weak Measurement Edition) | 日期：2026-03-27
> **属性**：03.实践指导 触点执行引擎
> **定位**：LMM 核心执行引擎。

---

## 0. 入场判定 (Gate) [强制执行项]
出现以下任一信号，方可执行干预协议：
* **认知裂缝**：受众表达明确的不满或操作卡顿。
* **结构损耗**：受众反复执行可被自动化的低效行为。
* **相变窗口**：受众主动询问替代方案（场域进入流变阶）。

### 0.1 五类 Kairos 信号

为避免把所有触达都做成时间驱动的“定时轰炸”，

本协议补充五类高优先级 Kairos 信号：

1. **张力溢出**：多次访问同页、深夜浏览、连续停留。
2. **追问**：主动提出具体问题，而非泛泛点赞。
3. **锚定**：收藏、转发、抄送同事或向上级上送。
4. **降级询问**：从“你是谁”转为“这一步怎么做”。
5. **物理接近**：访问预约页、下载确认单、提交样本或日志。

## 1. 属性调适 (Intervention)
* **原则**：严禁在受众的“秩序阶”进行大规模扰动。
* **动作**：只在受众认知的“流变阶”或“脉冲阶”入场，注入跨阈值信号，加速受众从旧稳态向新稳态的转换。

### 1.1 弱测量纪律

- 不直接逼问“你准备好改变了吗”。
- 优先使用自测表、低阻探针、样本提交、预约动作等弱测量手段判断状态。
- 对秩序阶主体，优先保持低频照明；对流变阶主体，方可提高信号密度。

## 2. 扰动模板与网树降维 (Reduction)
* **原则**：禁止展示复杂的“网状原理”。
* **动作**：通过高能量的“认知切片”，为受众拎出一棵“可爬的树”。确保该路径具备极低的认知损耗和极强的行动指向性。

### 2.1 四拍扰动模板

前台灯塔内容，

优先按下列四拍生成：

1. **结构损耗陈述**：先给出可验证的损耗或断口。
2. **时间锚定的下行威胁**：说明若继续不处理，将在何时何处恶化。
3. **结构归因**：指出这是系统构型问题，而不是主体品质问题。
4. **身份召唤 + R=1 信号**：说明谁有资格改变，以及我方承担什么边界内风险。

### 2.2 树状纪律

- 每个触点只推进一步。
- 每一步只给一个主动作。
- 不提供横向“你也许还想看”的发散出口。

## 3. 节律控制与语言维护

### 3.1 事件驱动优先于时间驱动

- 触点节律优先由 Kairos 信号触发，而不是按第 N 天固定发送。
- 持续发光会迅速降级为背景噪音；不必要的频率会损害信号对比度。

### 3.2 阻尼机制

- 若主体出现“接近 -> 退出 -> 再接近”的振荡，不应继续加压。
- 正确动作是降频、沉默、等待下一次高质量信号，而不是提高扰动强度。

### 3.3 语言半衰期

- 同一套扰动语言会因重复曝光而老化。
- 当前台语言老化时，应沿以下顺序下潜更新：
  `现象层 -> 结构层 -> 成因层`
- 已完成相变主体提供的新语言，应优先作为下一轮灯塔材料来源。

## 4. 引消与归元 (Extinguish & Seal)
* **引消**：利用证据化事实证明旧结构已不可持续，消除受众的回退路径。
* **归元**：干预完成后，迅速撤出营销能量。确保受众进入由“责任锚定”支撑的新稳态，并实现结构的自我持存。
* **退出**：对不匹配者降频退出；对已完成承接者转入稀疏维护或毕业序列。

---
**执行准则**：LMM 不从解释堆叠开始，而从`Kairos识别 -> 弱测量 -> 四拍扰动 -> 单步前进 -> 及时退出`开始。
