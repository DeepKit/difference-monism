> - [LMM.实践.013 择场定客配品原则](LMM.实践.013-择场定客配品原则.md)

## 0. 使用原则

本清单只判定一件事：

`这个问题场值不值得进，能不能进。`

在使用本清单前，

必须先完成 `LMM.实践.009-责任闭合规范` 的基础门禁项：

- `触发定义`
- `失败判定`
- `补偿机制`
- `责任主体`

若四项中任一缺失，

本清单直接停止。

若本次拟进入的是高风险、高影响或高争议问题场，

还应同步预想是否需要 `R-Eval` 客观校验，

否则不宜贸然建塔。

## 2. 成立层检查项

### 2.1 结构成立

- [ ] `问题场可命名`：能用一句话说清“这里到底出了什么结构性问题”
- [ ] `持续张力成立`：问题反复发生，不是热点波动或偶发情绪
- [ ] `认知刚性成立`：受众存在明显路径依赖与“只能这样”的默认防御
- [ ] `损耗强度成立`：至少可量化两类隐性税收，如时间/资金/精力/返工
- [ ] `责任真空成立`：现有方案只卖信息或产品，不承接后果

### 2.2 市场成立

- [ ] `旧解释失灵`：现有主流理论、话术或产品在该问题场中存在清晰断口
- [ ] `低占位成立`：尚未出现强势承接者牢固占据该位置
- [ ] `可切口性成立`：存在一个低阻进入的首切场景或首切动作
- [ ] `可触达性成立`：存在现实可用的触点、内容入口或关系入口

### 2.3 承接成立

- [ ] `可证据化`：问题、损耗、失败都能被观察、复盘或量化
- [ ] `可闭合承接`：我方具备最小责任承接能力，不只是会描述问题
- [ ] `值得进入`：该问题场能支撑正式承接、升级或长期资产沉淀

---

## 3. 最低通过条件

满足下面四条，才算通过：

- `问题场一句话定义` 已写清
- `R / L / V` 三项同时成立
- `BOS >= 45`
- 无一票否决项触发

补充建议：

- `30-44`：只允许灰区补证与私有测试，不构成正式建塔许可
- `>= 45`：方可进入公共主链的正式择场通过态
- `< 30`：暂缓或放弃

---

## 4. 一票否决项

出现以下任一情况，直接停止：

- [ ] 问题场靠概念包装而不是现实张力成立
- [ ] `R / L / V` 三项中任一项明显不成立
- [ ] 无法定义失败判定条件
- [ ] 无责任主体或补偿能力
- [ ] 场太大太散，无法切出首切场景
- [ ] 我方没有最小承接能力，却试图先行点灯

---
