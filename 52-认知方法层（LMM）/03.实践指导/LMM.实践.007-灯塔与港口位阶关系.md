# LMM.实践.007 — 灯塔与港口位阶关系 (LMM Lighthouse-Harbor Order)

> **版本**：v1.2 | 日期：2026-03-27
> **定位**：纠正 LMM 中“灯塔”与“推销/承接”的位阶关系

---

## 0. 总裁决

`灯塔不是推销。`

灯塔的第一任务不是成交，

而是：

`先照亮真实问题场，让对方先看见自己原本看不见的断口。`

成交、报价、服务包，

都只能是：

`港口承接`

不是灯塔本体。

---

## 1. 灯塔与推销的根本差别

### 推销

起点是：

`我有什么要卖`

结构是：

`产品 -> 说服 -> 成交`

### 灯塔

起点是：

`那里有一个真实但未被看清的问题场`

结构是：

`照明 -> 自识别 -> 主动趋近 -> 承接`

因此：

- 推销是财政动作
- 灯塔是结构显影

推销关心：

`怎么把东西卖出去`

灯塔关心：

`怎么让该来的人先看见自己身处何种危险水域`

---

## 2. 位阶顺序

在 LMM 中，

正确顺序应是：

`灯塔 -> 触点 -> 自测 -> 港口承接 -> 成交`

补充纪律：

- `灯塔` 负责照亮问题，不负责催促成交。
- `自测` 是弱测量过滤器，不是报价入口。
- `港口` 只承接已经完成初步自识别、并出现物理接近动作的人。

### 2.1 灯塔

定位：

`场域照明器`

任务：

- 命名问题
- 暴露断点
- 建立判断标准
- 触发高质量自我识别，而不是直接索取表单

### 2.2 触点

定位：

`灯塔的外化节点`

任务：

- 让目标对象接触到那束光
- 让问题第一次进入其注意力系统
- 捕捉 `Kairos` 信号，而不是机械轰炸

### 2.3 自测

定位：

`轻诊断过滤器`

任务：

- 让对方先判断“这是不是我的问题”
- 筛出真正适配者
- 用低阻动作完成弱测量，避免强逼表态

### 2.4 港口承接

定位：

`责任闭合入口`

任务：

- 让已被照亮、已完成自识别的人有地方靠岸
- 承接对话、首诊、报价、边界确认
- 锁定责任附件、失败判定与退出机制

### 2.5 成交

定位：

`港口中的一种交换动作`

任务：

- 完成财政闭合

但要强调：

`成交不是灯塔成功的唯一判据。`

灯塔成功也可能表现为：

- 靠近
- 咨询
- 转介绍
- 加入
- 共建

---

## 3. 什么是灯塔内容

灯塔内容不是服务介绍页，

而是那种能让目标对象产生结构性识别的内容。

它还应满足以下三条：

- 优先使用 `四拍扰动模板`
- 每篇只推进一个主动作
- 看完后最理想的反应不是“你真会卖”，而是“这就是我正在漏血的地方”

例如：

- `为什么你的 Agent 工作流最危险的不是模型错，而是责任链断裂`
- `为什么平台日志很多，团队仍然查不清谁该负责`
- `哪三类工作流最容易在上线后出现“可运行但不可治理”`

灯塔内容的判断标准不是：

`看完后会不会立刻购买`

而是：

`看完后会不会第一次准确识别出：这就是我正在面对的问题。`

更进一步的优先信号是：

- 下载自测表
- 访问确认单
- 向同事或上级转发
- 提交样本或请求具体说明

---

## 4. 什么是港口承接

港口承接不是灯塔，

而是：

`灯塔照亮之后，给对方的靠岸方式`

它可以包括：

- 对话申请
- 首诊说明
- 确认单 / 责任附件
- 报价单
- 责任边界说明
- `R-Eval` 物理校验说明

但要强调：

- `自测表` 更接近前置过滤器，不是港口的主文书
- 港口不是“所有线索都欢迎”，而是“只承接已接近靠岸的人”

一句话压缩：

`灯塔负责发光，港口负责接船。`

---

## 5. 当前体系中的纠偏

在此前几轮讨论中，

最主要的错位是：

`把首诊包和报价单推到了前台。`

这是不对的。

因为：

- 首诊包是承接器，不是灯塔
- 报价单是港口文书，不是前台主话语

正确理解应是：

- `自测包` 不属于港口主文书
- 它只在对方已被照亮之后，承担灯塔与港口之间的弱测量过滤任务

更精确地说：

- `自测包` 是前台过滤层
- `首诊包/确认单/报价单` 才属于真正的港口承接层

---

## 6. 最稳的新公式

### 静态公式

`场 = 问题结构`

`灯塔 = 结构显影`

`承接器 = 责任闭合入口`

`销售 = 承接器中的一种交换动作`

### 动态公式

`真实问题场 -> 灯塔显影 -> 自我识别 -> 主动趋近 -> 港口承接 -> 交易/合作/共建`

若写成最小执行链，则是：

`灯塔内容 -> 触点 -> 自测 -> 确认单/首诊 -> 责任闭合`

压成一句纲领式表达：

`先点亮，不先推销；先让人看见自己所处之场，再让其自主进入承接。`

---

## 7. 最短结论

把本文件压成最短版，只留下面六句：

- `灯塔不是推销`
- `灯塔先于销售，且高于销售`
- `灯塔负责让问题显影`
- `港口负责让已被照亮的人靠岸`
- `报价单是港口文书，不是灯塔内容`
- `正确顺序是：灯塔 -> 触点 -> 自测 -> 港口承接 -> 成交`
