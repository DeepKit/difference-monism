## 核心原则：先看这张表，再打开任何文件

```
作战阶段         → 应用工具文件
─────────────────────────────────────────────────────────────────
[位阶判定]       LMM.实践.007  灯塔与港口位阶关系
                  → 确认"我的动作是在点灯、做自测，还是已经进入港口承接"

[场域探测]       LMM.定位.007  蓝海场域探测记分卡 (BOS)
                  → 确认"这个场域值不值得进"
                  LMM.实践.015  择场检查清单v2
                  → 逐项核对场域、R-Min 与是否需要 R-Eval

[客体锁定]       LMM.实践.017  定客检查清单v2
                  → 确认"谁最痛、谁能上送、谁会拍板"

[产品配置]       LMM.实践.013  择场定客配品原则（原理）
                  LMM.实践.019  配品检查清单v2
                  → 确认"灯塔层/自测层/港口层该如何分层承接"

[整体战术]       LMM.实践.005  核心战术执行蓝图
                  → 立塔/发光/接引的完整流程总览

[触点机制]       LMM.实践.011  触点引消协议 (LPE Protocol)
                  → 何时入场、如何弱测量、如何四拍扰动、何时撤退

[责任门禁]       LMM.实践.009  责任闭合规范 (R-Min Spec)
                  → 任何正式承接动作前的强制检查清单；高争议场景同步预想 R-Eval
─────────────────────────────────────────────────────────────────
```

## 快速判定树

**你目前的任务是什么？**

```
不确定该进哪个场 ──→ 读 LMM.实践.015（择场检查清单v2）
已选定场，找谁突破 ──→ 读 LMM.实践.017（定客检查清单）
已找到客，怎么分层承接 ──→ 读 LMM.实践.019（配品检查清单）
要发前台内容/触点   ──→ 先读 LMM.实践.007，再读 LMM.实践.011
要发进入说明/正式承接文书 ──→ 先读 LMM.实践.009（责任门禁），确认闭合后才能出手
感觉动作顺序错位          ──→ 回读 LMM.实践.007（灯塔vs港口位阶）
```
