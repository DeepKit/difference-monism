# LMM.实践.031 — R-Eval协议模板库

> **版本**：v1.0 | 日期：2026-03-27
> **定位**：为高争议、高风险、高影响承接场景提供最小可用的 `R-Eval` 客观校验模板。
> **用途**：把 `R-Eval` 从原则说明推进为可直接填写、可直接挂到正式进入文书里的协议骨架。

---

## 一、使用目的

本文件只解决一件事：

`当项目需要 R-Eval 时，应该怎么写，才不退化成主观条款。`

它不是：

- 法务合同全文
- 定价单
- 私有商务报价单

它是：

`R-Eval 的最小模板库`

---

## 二、模板使用纪律

每个 `R-Eval` 模板至少要写清四件事：

1. `样本范围`
2. `时间窗口`
3. `误差阈值`
4. `执行方式`

若四项中任一项写不清，

则不应声称“本次已配置 R-Eval”。

补充纪律：

- `R-Eval` 只处理约定边界内的争议
- 不处理无限扩张责任
- 不替代项目整体合同
- 不自动适用于所有场景

---

## 三、通用空白骨架

以下骨架适用于任何需要客观校验的场景：

### 通用模板

```
【R-Eval 客观校验条款】

1. 校验对象：
本次仅校验：____________________

2. 样本范围：
本次样本锁定为：____________________

3. 时间窗口：
样本抽取时间窗口为：____________________

4. 误差阈值：
若我方判断与样本比对误差 > ______%，则判为失败；
若误差 <= ______%，则判为成立。

5. 执行方式：
由 ________ 提供样本，
由 ________ 见证，
按 ________ 方式记录。

6. 触发后动作：
若判失败，则执行：____________________
若判成立，则执行：____________________
```

---

## 四、模板 A：诊断型 R-Eval

适用场景：

- 首诊
- 结构诊断
- 责任断点识别

### 模板 A

```
【诊断型 R-Eval 条款】

本次 R-Eval 仅校验：
我方对约定范围内关键断点或损耗位置的识别是否与真实样本一致。

样本范围：
双方在首诊前共同锁定的 2-3 个关键流程样本 / 工单 / 任务片段。

时间窗口：
以 ________ 至 ________ 期间产生的真实样本为准。

误差阈值：
若我方对断点位置、损耗来源或责任断层的判断误差 > ______%，则判失败；
若误差 <= ______%，则判诊断成立。

执行方式：
由客户方提供原始样本，
双方现场或远程共同抽样，
并将比对过程形成书面记录或屏幕录制记录。

失败后动作：
退还本次诊断费用 / 提供一次复盘说明会 / 其他：________
```

适配文件：

- `LMM.产品.007`
- `LMM.实践.023`

## 五、模板 B：AI工作流型 R-Eval

适用场景：

- Agentic-AI 工作流
- 日志断链
- 权限边界或异常处理判断

### 模板 B

```
【AI工作流型 R-Eval 条款】

本次 R-Eval 仅校验：
我方对指定 AI 工作流中责任断链、异常断层或关键风险点的识别是否与真实日志样本一致。

样本范围：
从约定工作流中随机抽取 ______ 条真实异常日志 / 执行记录 / 工单片段。

时间窗口：
样本限定于 ________ 至 ________ 的真实运行窗口内。

误差阈值：
若我方报告指出的断层率、异常来源或责任交接点，与样本比对误差 > ______%，则判失败；
若误差 <= ______%，则判成立。

执行方式：
由客户方在见证下从原始系统中调取日志，
双方按预设字段逐条核验，
并将核验过程形成记录。

失败后动作：
退还本次首诊费用 / 提供 30 分钟解释会 / 其他：________
```

适配文件：

- `LMM.产品-AI.031`

## 六、模板 C：组织责任型 R-Eval

适用场景：

- 组织责任界面
- 权责断点
- 多角色协同失配

### 模板 C

```
【组织责任型 R-Eval 条款】

本次 R-Eval 仅校验：
我方对约定业务环节中责任断点、协同失配点与责任空白节点的识别是否与真实流程样本一致。

样本范围：
双方事前锁定的 ______ 个业务流程片段 / 决策链片段 / 协同事件。

时间窗口：
以 ________ 至 ________ 期间的真实业务记录为准。

误差阈值：
若我方对责任归属断点的判断与样本比对误差 > ______%，则判失败；
若误差 <= ______%，则判成立。

执行方式：
由客户方提供流程记录、会议记录、工单流转或角色访谈材料，
双方按预设节点逐项核验。

失败后动作：
退还本次费用 / 降级为一般咨询说明 / 其他：________
```

适配文件：

- `LMM.产品.007`

## 七、禁止写法

以下写法默认不合格：

- `客户不满意即可退款`
- `如感觉没效果则退费`
- `若甲方认为判断不准，则视为失败`
- `校验标准由双方后续再协商`

这些写法的问题是：

- 全部主观化
- 全部可漂移
- 全部事后争夺解释权

---

## 八、选择哪种模板

| 场景 | 优先模板 |
|:---|:---|
| 诊断准确性争议 | 模板 A |
| AI工作流断链争议 | 模板 B |
| 组织责任界面争议 | 模板 C |

若仍不确定，

优先问：

`最终争议会落在什么样本上？`

样本类型决定模板类型。
