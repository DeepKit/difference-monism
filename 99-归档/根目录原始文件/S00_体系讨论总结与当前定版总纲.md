# S00｜体系讨论总结与当前定版总纲

> 用途：汇总本阶段关于 LMM / COP / RT6 / WSM / WSH 的讨论成果，形成当前统一口径、边界、命名、主链、产品关系与后续优先事项。
> 状态：当前定版草稿
> 性质：内部总纲

---

## 一、总览：当前已经定下来的大结构

本阶段讨论的结果，不是零散观点，而是一个逐步清晰的上中下三层结构：

### 1. 上位总体系

**复杂问题显影—跃迁—承接体系**

这是当前对 LMM / COP / RT6 的统一上位命名。

其含义是：
- 处理的不是单一品类营销问题，而是**复杂问题如何被看见、被承认、被进入处理、被持续调整**的问题。
- 它不是“卖理论的理论”，而是一个**通用的复杂问题处理总体系**。

### 2. 三个主干体系

在上位总体系之下，保留三套彼此衔接、但职责清楚的理论体系：

- **LMM**：前线显影理论
- **COP**：分诊与跃迁理论
- **RT6**：承接与处方理论

不建议把三者合并成一个“大理论”，原因是：
- 合并后边界会糊
- 会把“为什么成立”和“怎么做”混在一起
- 以后很难升级、修订与产品化

正确关系是：

> 三理论，不合并；一总纲，要统一。

补充当前定版：

> **若严格区分位阶，则 `LMM / COP` 更偏理论与机制主干，`RT6` 更偏方法学主干；三者共同构成当前的显影—跃迁—承接主链。**

### 3. 两个并列主产品

在具体产品层，当前明确保留两个并列主产品：

- **WSM（五行察局学）**
- **WSH（五行和悦论）**

它们不是一主一辅，也不是同一个入口的两个分支，而是：

> **同一类母客群之下的两个并列泊位型主产品。**

它们共享母客群，但处理的是不同问题层。

## 二、LMM / COP / RT6 各自解决什么问题

### 1. LMM：前线显影理论

LMM 解决的不是“普通营销”问题，而是：

> **在高噪声现实中，如何找到真正的问题场、照亮断口、把正确的人引到正确入口。**

它关注：
- 立塔位置
- 发光方式
- 问题场定义
- 弱测量
- 港口接引
- 责任闭合前的前线秩序

它不等于传统定位理论。
当前统一口径是：

> **传统定位理论的目标是占据心智；LMM 的目标是接管责任。**

### 2. COP：分诊与跃迁理论

COP 解决的是：

> **用户为什么会从“看见问题”走到“主动进入处理”。**

它关注：
- 属集张力
- 旧框架与新框架
- 解释权迁移
- 认知相变
- 冷共鸣 / 热共鸣
- 入泊成立的条件

它的核心命题已经定为：

> **承接的本质，是把用户对自己处境的解释权，从旧框架迁移到新框架。解释权迁移完成，入泊自然发生。**

### 3. RT6：承接与处方理论

RT6 当前统一命名为：

- **中文名**：共鸣与跃迁六步法
- **英文名**：Resonance and Transition Six-Step Method
- **缩写**：RT6

它不是营销战术，也不只是认知方法。
它更准确的归类是：

> **通用承接方法学**
> 或者
> **复杂问题进入处理后的通用承接骨架**

它解决的是：
- 症状如何进入结构化陈述
- 诊断和处方如何分层
- 总方与细方如何切开
- 付费点如何放置
- 复诊与换方如何成立

RT6 与 COP 的关系已定为：

> **COP 讲相变，RT6 讲处方。**

更完整一点：
- COP 是承接机制理论
- RT6 是承接机制的通用操作骨架

再压一句当前定版口径：

> **LMM 负责显影，COP 负责分诊与跃迁，RT6 负责承接与处方。**

### 4. 与 TAT / ODD / Harness 的关系

`LMM / COP / RT6` 不单独构成完整治理执行链。  
它们之后若进入高影响动作面，还要继续接：

- `TAT`：责任门槛与复审
- `ODD`：工程编译、门禁、证据、封存
- `Harness`：真实执行、阻断、留痕、回滚

因此当前应同时区分两条链：

> **理论/产品链：`LMM -> COP -> RT6`**

> **治理/执行链：`LMM -> COP -> TAT -> ODD -> Harness`**

再补一条边界：

> **`Harness` 有用，但不属于理论母体；它是运行时基础设施概念，而不是新的上位理论。**

## 三、WSM / WSH 的关系：同母客群，双泊位主产品

### 1. 母客群

当前统一母客群口径为：

> **长期承担结果、持续推进生活与工作、但正在发生结构失血或自我偏移的人。**

这个母客群不按性别先切，而按状态切：
- 高责任
- 长期能扛
- 不低行动力
- 已经在现实里承担很多
- 但开始出现失血、失准、偏移、主权下降或方向模糊

### 2. WSM 解决什么

WSM 的统一定义已经收口为：

> **WSM（五行察局学）解决的是：你为什么会在错误结构中不断失血（当前）。**

它处理的是：
- 责任如何流
- 责任如何失衡
- 为什么责任最后落到你身上
- 当前主局 / 次局 / 承压点 / 最低动作

### 3. WSH 解决什么

WSH 的统一定义已经收口为：

> **WSH（五行和悦论）解决的是：你如何实现年度最好目标靶心，发现月度偏移并回到自己。**

它处理的是：
- 首测阶段：当前不舒服与当下偏移
- 入泊后：年度靶心建立
- 后续：月度纠偏与成长轨迹系统

### 4. 二者关系

当前定版关系是：

> **WSM 和 WSH 不是两类客户，而是同一类高责任用户的两种入口系统。**

更短一点：
- WSM 是结构止血系统
- WSH 是年度回正系统

### 5. 前台关系

前台不共用入口，不共用问卷，不做统一分流问卷。
当前统一原则是：

> **前台双泊位、双界面、双问卷；后台同港口、可桥接、可互导。**

桥接不发生在首屏，不发生在首问卷，而发生在：
- 结果页
- 报告建议
- 后续升级路径
- 用户生命周期管理中

## 四、双塔系统：立塔、发光、承接的当前定版

### 1. 总原则

两座塔建在同一片母海，但不建在同一个礁石上。

即：
- 同一类高责任母客群
- 两种不同断口
- 两种不同泊位

### 2. WSM 的塔

#### 发光定义

**背锅共鸣之光**

#### 承接问题

**你背的是什么样的锅？**

#### 核心逻辑

WSM 前台不是先讲“责任流向”，而是先打中用户最原生、最具体的体验词：
- 背锅
- 怎么又落到我头上
- 为什么每次都轮到我扛
- 为什么我总在补锅

然后再往下收进：
- 主局 / 次局
- 承压点
- 最低止血动作

当前内部操作口令可记为：

> **WSM：先共鸣“背锅”，再识别“锅型”。**

### 3. WSH 的塔

#### 发光定义

**不舒服共鸣之光**

#### 承接问题

**你具体的不舒服的共同表达是什么？**

#### 核心逻辑

WSH 前台不是先讲年度靶心，也不是先讲成长，而是先接住用户此刻最真实的“不舒服”：
- 累
- 堵
- 散
- 乱
- 没意义
- 一直在做，但越来越不像自己

然后再往下收进：
- 当前主重心
- 当前回归入口
- 本周动作

当前内部操作口令可记为：

> **WSH：先共鸣“不舒服”，再识别“当前重心与回归入口”。**

### 4. 双塔总句

当前最短定版为：

> **立双塔，发双光，进双泊。**
> **塔建在不同礁石上，光照不同断口，泊位各自纯接引。**

## 五、WSH 当前最重要的产品链定版

### 1. 首测命名修正

已明确：

> **没有年度靶心，就不能严格说“本月偏移”。**

因此，WSH 首次测评的正式口径应为：

- **当下偏移报告**
- 或 当前状态偏移报告
- 或 回归预看报告

其中最推荐的当前正式名称是：

> **当下偏移报告**

### 2. WSH 首发顺序

当前定版主链为：

1. **不舒服共鸣之光**
2. **入泊问卷**
3. **当下偏移报告（免费 CT）**
4. **询问是否进入处理**
5. **给总方（免费）**
6. **小步回正包（收费药方）**
7. **复诊与换方**
8. **年度靶心建立**
9. **月度纠偏与成长轨迹系统**

### 3. 免费与收费边界

当前明确：

#### 免费
- 当下偏移报告 = CT 诊断
- 总方免费

#### 收费
- 小步回正包 = 药方

当前统一口径为：

> **CT 送，药方卖。**

### 4. 收费点为什么放这里

原因是：
- 免费层负责让用户看见“我现在偏在哪里”
- 收费层负责让用户得到“我现在从哪里开始回来，以及具体怎么回来”

所以：
- 免费卖看清
- 收费卖执行

### 5. WSH 的第二港口

已明确：

> **年度靶心建立不是第一束光，而是第二港口。**

顺序上：
- 先识别当下偏移
- 再小步回正
- 再建立年度靶心
- 再进入月度纠偏与长期轨迹

## 七、当前关于市场与变现的结论

### 1. WSM / WSH 不建议强绑同卖

当前统一结论：

> **同一母客群，双入口获客，强桥接导流，统一后端留存。**

不要：
- 一开始打包卖两套理论
- 让用户首屏理解两套理论关系

应该：
- 用户先按主问题进入
- 再在后续结果和路径里桥接另一系统

### 2. WSH 的优先逻辑

针对 C 端自动化方向，当前最稳的顺序是：
- 先从“不舒服”进入
- 先给免费 CT（当下偏移报告）
- 再卖“小步回正包”
- 再升级到年度靶心与月度纠偏

### 3. B端方向的补充定位

WSH 已明确不仅是 C 端成长系统，也可以作为 B 端系统存在：

> **对个人：WSH 是成长轨迹系统。**
> **对有场域的 B 端：WSH 是客户回流系统。**

也即：
- 对 C 端卖回正
- 对 B 端卖回流机制

### 4. 先 C 还是先 B

当前最稳判断不是纯二选一，而是：

> **小C验证，B端收钱。**

也就是：
- 用极小规模 C 端样本验证机制
- 再把机制卖给小 B 端场域

如果一定二选一，现阶段更偏：
- 主路径偏小 B 端
- 但要配一个最小 C 样本池做证据
