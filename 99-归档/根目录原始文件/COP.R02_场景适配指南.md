# COP.R02｜场景适配指南
**承接协议理论 · 场景适配指南**
版本：V1.0 | 状态：草稿 | 日期：2026

---

## 使用说明

COP的核心机制（相变、五节点序列、两阶段共鸣）在所有场景中保持不变。

本文档解决的是：**同一套机制，在不同媒介和场景中如何调整具体的执行方式。**

---

## 一、场景适配的三个维度

任何承接场景，都需要在以下三个维度做适配：

**维度一：接触点的物理形式**
触发器在哪里呈现？文章中段、落地页首屏、对话回应、还是视频时间点？

**维度二：触发器的投放位置**
在整个内容或对话流程中，触发器出现在哪个节点？

**维度三：相变信号的定义**
在这个场景里，什么样的用户行为代表相变发生了？

---

## 二、场景适配总表

| 场景 | 接触点物理形式 | 情境激活段位置 | 触发器投放位置 | 命名锚定位置 | 主要相变信号 |
|------|--------------|--------------|--------------|------------|------------|
| 知乎长文 | 文章中段 | 文章前三分之一 | 第一识认后 | 文章中后段 | 评论自述/收藏 |
| 落地页 | 首屏之后第二屏 | 首屏 | 滚动至中段 | 中段底部 | 继续滚动/点击详情 |
| 咨询初次对话 | 用户自述完毕后 | 倾听阶段 | 第一次追问之后 | 对话尾段 | 用户主动深问 |

## 三、各场景详细适配说明

### 场景A：知乎长文

**场景特征：**
- 用户主动搜索进入，已有一定属集张力
- 阅读时间弹性大，可承载复杂触发器
- 评论区是相变信号的主要观测点

**适配要点：**

入口设计：
- 标题必须是痛点识别型，非好奇心型
- 副标题或开头第一句话承接标题，继续加深痛点识别
- 禁止标题党（制造好奇心但让有效用户感到被骗）

结构布局：
```
开篇（前10%）：情境激活段
              ↓ 用户进入情境
前中段（10-40%）：内容铺垫，建立"他懂这个话题"的信任
              ↓ 用户完成第一识认
中段（40-60%）：触发器五节点展开
              ↓ 相变发生
中后段（60-80%）：命名锚定
              ↓ 入泊完成
后段（80-100%）：可展开部分框架内容（用户已入泊，不再是销售）
```

相变信号观测：
- 强信号：评论中用户描述自己的具体经历
- 中信号：收藏（区别于点赞）
- 假阳性：泛点赞

**知乎特殊注意：**
- 知乎评论区会暴露触发效果，热评如果是"说得好"（冷共鸣）说明未触发热共鸣
- 热评如果是用户描述自己的具体处境，说明热共鸣触发成功

---

### 场景B：落地页（goodmem.cn产品页）

**场景特征：**
- 用户通常从知乎文章跳转进入，已完成第一识认
- 停留意愿不稳定，需要快速进入情境
- 页面结构决定触发器投放位置

**适配要点：**

首屏设计（情境激活段）：
```
首屏不是产品介绍，是情境确认
目标：用户看到首屏，感到"我来对地方了"
内容：一句话处境陈述 + 一个具体细节
禁止：价格、功能列表、用户评价（这些在用户入泊后才有效）
```

第二屏（触发器投放）：
```
NODE_1-NODE_3 在第二屏完整展开
用户在这里完成热共鸣
设计原则：文字为主，不被图片分散注意力
```

中段（命名锚定+入泊）：
```
NODE_4-NODE_5 在中段完成
命名锚定后，提供"了解这个名字"的路径
路径目的地：框架简介（2-3句话），不是购买页面
```

后段（入泊后内容）：
```
用户已入泊，此时可以展示：
- 产品是什么（简短）
- 产品能做什么（针对相变后的需求）
- 价格（简洁）
- 购买入口（清晰，不强迫）
```

相变信号观测：
- 强信号：用户滚动至底部并点击购买入口
- 中信号：停留时长超过3分钟
- 假阳性：页面加载后短暂停留（可能只是等待）

**落地页特殊注意：**
- 从知乎跳转来的用户已完成第一识认，落地页的首屏情境激活可以更简短
- 落地页的触发器应针对与知乎文章不同的原材料——不要重复知乎已说过的内容

### 场景C：咨询初次对话

**场景特征：**
- 实时互动，用户张力通常已经很高
- 触发器是口头的，不是书面的
- 相变信号是对话中的行为变化

**适配要点：**

倾听阶段（情境激活段）：
```
不要急着介绍框架
让用户把自己的处境说完
用"还有呢？""那时候发生了什么？"引导
目的：让用户自己把张力激活
```

第一次追问后（触发器投放时机）：
```
用户说完之后，有一个短暂的停顿
这是触发器的投放时机
用户在等你的反应，防御是开着的
```

口头触发器结构：
```
NODE_1（简短）："你说的那次[具体细节]，我有印象。"
NODE_2（精准）："你当时选择了[具体行为]，对吧？"
NODE_3（核心）："那不是[旧归因]。是因为[结构性描述]。
                  在那个位置上，任何人都会[同样结果]。"
NODE_4（引导）："你想想，不只是那一次，对吗？"
               （停顿，让用户自己说）
NODE_5（命名）："这个处境有一个名字。"
               （停顿，看用户反应）
```

相变信号观测：
- 强信号：用户主动说出其他类似经历（NODE_4的验证自发完成）
- 强信号：用户追问"这个名字是什么？"
- 中信号：用户沉默一段时间后说"对，好几次都是这样"
- 假阳性：用户礼貌性的"嗯，有道理"

**咨询特殊注意：**
- 口头NODE_3不需要像书面那样完整，核心一句话即可
- 用户的自发反应比计划更重要——如果用户在NODE_2后自发开始讲更多，不要打断
- 命名锚定后的停顿尤为重要，不要填满沉默

---

### 场景D：社群内容

**场景特征：**
- 用户已在社群中，有一定信任基础
- 触发器更多出现在评论互动中，不是主帖
- 传播性强，相变后用户会主动扩散

**适配要点：**

主帖（情境激活段）：
```
主帖提供一个具体的处境描述或案例
不在主帖放完整触发器——让用户先反应
观察哪些用户在评论中表达共鸣（第一识认信号）
```

评论区触发器（真正的触发器投放位置）：
```
当用户在评论区表达共鸣（"我也有这种感觉"）时
回复这个评论，展开NODE_2-NODE_5
这是最精准的触发器投放——用户已经激活，张力最高
```

相变信号观测：
- 强信号：用户主动@特定的人"你看看这个，说的就是咱们"
- 强信号：用户把帖子截图转发到其他社群
- 中信号：用户在评论中详细描述自己的经历
- 假阳性：泛转发（"分享"按钮点击，无个人描述）

**社群特殊注意：**
- 社群中用户已有信任基础，NODE_1的细节精度要求可以略低
- 优先回复描述了具体经历的评论，不是只说"赞"的评论
- 命名锚定后提供的路径必须是社群内可访问的，减少跳出摩擦

---

### 场景E：WSM系列文章（五行察局）

**场景特征：**
- 故事性强，用户通过人物经历激活张力
- 相变发生在读者识别"这个人物的处境就是我的处境"的时刻
- 连续性内容，前一篇的相变效果影响后一篇的入泊率

**适配要点：**

故事开篇（情境激活段）：
```
通过主角的具体处境描述激活读者张力
主角的困境必须是读者经历过的，不是虚构的
细节精度要求最高——读者在人物身上看到自己
```

主角关键困境节点（触发器投放位置）：
```
主角面临"怎么做都是错"的时刻
这是读者张力最高的时刻
在主角的选择之后（NODE_2位置），投放NODE_3（结构揭示）
不是叙事者来说，而是通过剧情本身揭示结构
```

章节末尾（命名锚定）：
```
结尾处给主角的处境命名（剧情内的命名）
读者感到"这个名字也是我处境的名字"
追读下一篇的驱动力：想知道这个名字意味着什么，主角会怎么面对
```

相变信号观测：
- 强信号：读者评论"写的就是我"并附上具体经历
- 强信号：读者主动寻找下一篇
- 中信号：在对应内容页面停留时间显著长于平均

**WSM特殊注意：**
- 故事形式让NODE_2（行为命名）更自然——主角的行为本身就是命名
- NODE_3（结构揭示）不能破坏叙事节奏，需融入故事情节中
- 多篇连载中，每篇都应有独立的完整触发器，不依赖前篇

## 四、新场景适配方法

遇到未在本文档中列出的新场景，按以下步骤适配：

```
Step 1：确定媒介特性
  - 用户进入时的张力状态如何？（高/中/低）
  - 内容是单次消费还是连续消费？
  - 互动是实时的还是异步的？

Step 2：定位情境激活段位置
  - 用户张力高：激活段可短（甚至省略）
  - 用户张力低：激活段必须在触发器前完整出现

Step 3：定位触发器投放位置
  - 原则：在用户完成第一次自我识认之后
  - 识别第一识认时刻：用户从被动浏览变为主动专注的节点

Step 4：定位命名锚定位置
  - 通常在内容或对话的60-75%处
  - 之后留出足够空间让用户完成主动跨越

Step 5：定义该场景的相变信号
  - 主动行为优先（评论、转发、追问、点击）
  - 排除被动行为（点赞、泛阅读）
  - 明确假阳性
```
