## 六、推进步骤

### Step 1：固定边界

- 统一按 `11` 条活跃论文线统计当前外发面板。
- 不再把 `COP / Harness / WSM / BetterCiv` 混进独立论文线。

### Step 2：维持公开法源线

- `DM / ASTO / ECET / TAT / ODD` 按预印本发布包继续管理。
- 公开法源链只按 `DM -> ASTO -> ECET -> TAT -> ODD` 理解。

### Step 3：推进当前直接投稿线

- `DM-P00 / DM-P04 / DM-P09 / Progee-System-Paper` 进入逐包执行：
  - venue
  - 匿名终核
  - 元数据
  - 导出
  - 提交

### Step 4：继续压缩待定线

- `LMM-理论总述` 继续拆稿，不把 `WSM` 单独拉成新论文线。
- `ExecNorm` 先做唯一主稿裁决，再进入投稿流程。

### Step 5：执行侧单向留痕

- 案例、补证、盲评、平台动作统一回 `.BetterCiv/05_学术型`。
- `一元论` 只保留法源主位、申请包与总导航。

### Step 6：完成外部提交

- 剩余真正不能由仓库内文档替代的事项，只剩：
  - 作者元数据确认
  - 平台账号与确权
  - 实际点击提交

## 七、最短结论

当前论文体系已经从“散文件推进”切换为“两条外发线并行”：

- 预印本发布线负责公开法源
- 单篇申请包线负责真实投稿

当前最应该推进的，不是继续扩目录，而是把 `DM-P00 / DM-P04 / DM-P09 / Progee-System-Paper` 四条线真正推过外部提交门槛。
