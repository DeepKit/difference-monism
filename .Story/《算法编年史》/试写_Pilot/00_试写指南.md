# 《算法编年史》试写指南
# Pilot Writing Guide v1.0

## 试写目标

**目的**：验证整个写作体系的可行性
- 测试素材植入流程
- 验证线索系统的有效性
- 确认ASTO契约方法论

**字数**：约70,000字（每集平均3500字）

## 写作工作流

### 每集写作步骤

```
步骤1：准备（40分钟）
├── 读场景契约
├── 查线索总表
├── 研究素材
└── 写场景细纲

步骤2：写作（2-3小时）
├── 场景1：开头钩子（500-800字）
├── 场景2：发展（800-1000字）
├── 场景3：转折/高潮（1000-1200字）
└── 场景4：结尾（500-800字）

步骤3：自检（30分钟）
├── 17层检查（前5层）
├── 字数检查
├── 线索标记检查
└── 快速修订
```

### 使用文档的顺序

1. **写作前**：
   - 场景契约（第X集详细规划）
   - 素材档案（如WAR-001）
   - 线索映射表（前100集）

2. **写作中**：
   - 写作指南（每集规划模板）
   - 素材维度表（快速参考）

3. **写作后**：
   - 17层检查清单
   - 线索追踪系统（更新状态）

## 第1集特殊准备

### 第1集的重要性
- **第一 impression**：决定读者是否继续
- **世界观建立**：需要清晰但不冗长
- **主角引入**：萨尔贡的首次登场
- **核心谜题**：六十一点横的首次出现
- **钩子设计**：必须强力

### 第1集写作要点

#### 开头（前500字）
- **场景**：河边泥板作坊
- **动作**：萨尔贡无意识敲击泥板
- **冲突**：学徒的困惑和恐惧
- **钩子**：瞳孔闪光 + 61个凹点

#### 关键元素植入
1. **主角**：萨尔贡（13岁，奴隶，异常者）
2. **世界观**：公元前1750年，巴比伦
3. **核心谜题**：六十一点横（CLUE-SC-001）
4. **主线1**：信使传承（瞳孔闪光 CLUE-MS-001）
5. **主线2**：编译器对抗（所罗门之门伏笔）
6. **素材**：WAR-001（长平之战-数据欺骗，隐喻）
7. **钩子**："你敲的是什么？"

#### 结尾设计
- **悬念**：学徒准备告发
- **埋下**：CLUE-BINARY-01, CLUE-PUPIL-01, CLUE-OLDMAN-01
- **预告**：第2集学徒的内心挣扎

## 常见问题与解决

### Q1：卡文了怎么办？
**解决**：
1. 回顾场景契约，确认核心目标
2. 查看素材档案，寻找灵感
3. 写场景细纲，拆分小步骤
4. 先写对话，再补描写
5. 允许自己先写烂版本，再修订

### Q2：线索太多记不住？
**解决**：
1. 只关注本集契约中的线索
2. 使用线索映射表快速查询
3. 写作时打开线索追踪系统
4. 不要一次引入超过3个新线索

### Q3：素材植入生硬？
**解决**：
1. 先写情节，再植入素材
2. 用人物对话引入素材
3. 通过场景氛围暗示素材
4. 不必每个素材都明确说明

### Q4：字数不够？
**解决**：
1. 增加感官描写
2. 增加人物内心独白
3. 增加场景细节
4. 增加对话回合
5. 放慢节奏，增加张力

### Q5：字数超标？
**解决**：
1. 删除冗余形容词
2. 合并重复信息
3. 加快场景转换
4. 精简对话
5. 删除与主线无关的支线

## 完成标准

**数量标准**：
- [ ] 20集全部完成

- [ ] 通过17层检查（前10层必须通过）

- 素材植入是否自然？

**3. 系统评估**
- 17层检查是否必要？
- 线索追踪是否好用？
- 素材库是否充分？

**4. 调整建议**
- 哪些流程需要优化？
- 哪些文档需要补充？
- 是否继续写下去？
