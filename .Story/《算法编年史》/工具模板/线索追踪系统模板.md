# 版本：Excel/Notion 数据库模板 v1.0

## 使用说明

此模板用于追踪5000集的线索状态，确保：
1. 线索不出现遗漏
2. 伏笔按时回收
3. 逻辑保持一致

**工具选择**：
- **Excel**：适合单机，功能强大
- **Notion**：适合协作，可视化好
- **Airtable**：适合高级用户，自动化强

## 自动化建议

### Notion自动化

1. **当集状态变为"已完成"**：
   - 自动更新线索总览表的出现次数
   - 自动检查伏笔回收

2. **当伏笔超过"计划回收集"**：
   - 发送提醒（需要回收伏笔了）
   - 标红显示

3. **每日统计**：
   - 已完成集数
   - 各线索出现次数
   - 未回收伏笔数量

### Excel公式示例

**统计主线出现次数**：
```excel
=COUNTIF('集数线索分配表'!F:F,"算法进化")
```

**检查伏笔是否超期**：
```excel
=IF(TODAY()>DATE(2026,12,31),"超期","正常")
```

## 使用流程

### 写作前

1. 在"集数线索分配表"中规划本集
2. 确定使用哪些主线、副线、素材
3. 确定埋下哪些伏笔

### 写作中

1. 参考规划，但可灵活调整
2. 记录实际使用的线索和素材
3. 标记临时新增的伏笔

### 写作后

1. 更新"集数线索分配表"的实际内容
2. 更新"线索总览表"的状态和次数
3. 更新"伏笔回收追踪表"
4. 运行一致性检查
5. 标记为"已完成"

### 审核时

1. 检查"一致性检查表"是否有问题
2. 检查伏笔是否按时回收
3. 检查线索出现频率是否合理
4. 通过后标记为"已审核"

## 示例：完整工作流程

**第1天：规划第1-10集**
1. 在"集数线索分配表"中创建10行
2. 填写每集的主线、副线、素材
3. 标记第1集埋下伏笔CLUE-BINARY-01

**第2-11天：写作**
1. 每天写1集
2. 实际写作可能与规划不同
3. 每天更新表格的实际内容

**第12天：审核**
1. 检查"伏笔回收追踪表"（确认埋下）
2. 检查"线索总览表"（更新状态）
3. 运行"一致性检查"
4. 标记10集为"已完成"

**第13天：规划第11-20集**
1. 重复上述流程
