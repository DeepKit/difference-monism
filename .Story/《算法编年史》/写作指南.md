### 1.2 契约撰写12步法

**步骤1-2：选坐标、定类型**
- 根据当前集数确定phase（六阶）和mode（五态）
- 选择scene_type（6种类型之一）
- 系统自动加载对应模板、陷阱图、测试策略

**步骤3-4：定时空、定人物**
- 填写spacetime（与历史事实核对）
- 确定POV角色和在场角色
- 检查角色状态一致性（从前一集继承）

**步骤5-7：定落点、定出口、定信息门**
- must_land：必须是可校验的命题
  - ✅ 好例子："萨尔贡发现数据里藏着不属于他的变量名"
  - ❌ 坏例子："萨尔贡有所领悟"（无法量化）
- exit_to：方向+情绪+钩子
- information_gate：明确 reveal/withhold

**步骤8-9：定红线、定弹性**
- taboos：绝对不能违反的规则
- must_not_reveal：悬念保护清单
- deviation_budget：human_confirmed必须为true

**步骤10：写直觉**
- intuition_notes：纯自然语言，系统不解析
- 用于AI生成时的氛围参考

**步骤11-12：声明副作用**
- character_state_diffs：角色状态变化
- foreshadow_ops：伏笔操作（埋/推进/收）

## 二、六阶节奏控制（ASTO动力学）

### 2.1 六阶叙事特征

| 六阶 | 叙事特征 | 典型场景 | 写作要点 |
|------|----------|----------|----------|
| **a 混沌** | 信息碎片、氛围铺垫、多义性 | 开场、梦境、回忆 | 意识流、感官细节、不解释 |
| **b 秩序** | 规则建立、关系明确、逻辑清晰 | 对话推进、计划制定 | 信息交换、人物关系确认 |
| **c 流变** | 渐变积累、张力蓄压、悬念推进 | 线索串联、疑点堆叠 | 节奏加速、细节累积 |
| **d 脉冲** | 突变、高潮、不可逆转折 | 真相揭露、对抗爆发 | 压缩时间、强烈冲突 |
| **e 崩解** | 旧秩序瓦解、失控、情感冲击 | 信念崩塌、关系破裂 | 失控感、情感释放 |
| **f 归元** | 沉淀、收束、新平衡建立 | 独处反思、余韵留白 | 慢节奏、留白、新起点暗示 |

### 2.2 纪元内的六阶分布

**标准分布（每纪元1000集）**：

```
混沌期：1-50集（5%）
  └─ 新信使觉醒，世界观建立
  
秩序期：51-200集（15%）
  └─ 算法知识系统化，人物关系确立
  
流变期：201-600集（40%）
  └─ 算法应用、文明演进、矛盾积累
  
脉冲期：601-800集（20%）
  └─ 重大转折（所罗门崩溃、图灵之死）
  
崩解期：801-900集（10%）
  └─ 旧纪元秩序瓦解，向新纪元过渡
  
归元期：901-1000集（10%）
  └─ 封印完成，向下一纪元传递
```

**警示**：AI典型产出会均匀分配六阶篇幅，必须人工调整！

### 2.3 七序权重分配（场景内部结构）

**标准场景（3500字）的七序权重**：

```
觉醒序（开场）：10-15%
  └─ 角色进入情境，感官触发
  
感知序（观察）：15-20%
  └─ 环境描写，信息接收，察觉异常
  
解析序（思考）：25-35%
  └─ 内心推理，分析局势，建立假设
  
干预序（行动）：15-25%
  └─ 采取行动，对话交锋，做出反应
  
设计序（决策）：5-10%
  └─ 形成计划，做出决策，重新定义目标
  
回溯序（反思）：10-15%
  └─ 审视后果，自我反思，情绪沉淀
  
消解序（收束）：5-10%
  └─ 场景收束，情绪落位，暗示延续
```

**不同scene_type的权重调整**：

| 场景类型 | 重点七序 | 权重调整 |
|----------|----------|----------|
| dialogue_driven | 干预序 | +10% |
| introspection | 解析序+回溯序 | +15% |
| action_sequence | 干预序 | +20% |
| revelation | 觉醒序+消解序 | +10% |
| atmosphere | 感知序 | +25% |

## 三、单元写作规范（每集3500字）

### 3.1 3+3+3+1时间结构

```
总时长：10分钟（约3500字）

前3分钟（约1000字）：钩子（危机/发现/悬念）
├─ 功能：立即抓住读者
├─ 技巧：从动作或对话开始，避免冗长描写
└─ 示例："萨尔贡三十岁，伊什美三十三岁。他们已经被囚禁了两年。"

中3分钟（约1200字）：发展（人物关系/算法知识/世界展开）
├─ 功能：建立情境，铺垫高潮
├─ 技巧：交替进行人物互动和知识植入
└─ 要点：知识必须通过人物困境呈现，而非说教

后3分钟（约1000字）：高潮（决策/对抗/顿悟）
├─ 功能：情感或智力的顶点
├─ 技巧：压缩时间，增加紧迫感
└─ 示例：萨尔贡触碰所罗门额头，说"我要成为错误"

最后1分钟（约300字）：心法+预告（知识封装+下集钩子）
├─ 功能：给读者收获感和期待感
├─ 技巧：心法用比喻，钩子用悬念
└─ 结构：【今日算法心法】+【下一集预告】
```

### 3.2 缺陷三分法（去AI味核心）

**三类缺陷的处理方式**：

| 缺陷类型 | 定义 | 处理方式 | 示例 |
|----------|------|----------|------|
| **毁灭性缺陷** | 违反事实、逻辑矛盾、人设崩塌 | 强制修复 | 已死角色出场、时间线冲突 |
| **生产性缺陷** | 非常规句式、意外转折、风格特征 | 予以保护 | 不对称句式、突然的情感跳跃 |
| **中性缺陷** | 模糊地带，不确定是否故意 | 人类裁决 | 轻微的视角漂移 |

**AI味检测维度（C6检测器）**：

1. **对称句式密度**：连续出现A...B...结构的频率（阈值<0.3）
2. **情感直述比例**："他感到悲伤"vs行为暗示的比例（阈值<0.25）
3. **解释性语句占比**：叙述后紧跟解释的频率（阈值<0.2）
4. **视角漂移**：无意识切换全知/限知视角的次数（警告>3次/集）
5. **词汇多样性**：同义替换的机械感（监测"然而/但是/不过"轮换）

### 3.3 R6粗糙度保护规则

**防止过度打磨，保护文学棱角**：

```
规则R6：roughness_protection
  前置条件：L0/L1全部通过（先保证没有硬伤）
  触发条件：overall_score >= 0.75
  判断逻辑：
    if 本轮修复后stylistic_variance下降超过15%：
      终止修复，保留上一版本
      exit_reason: "roughness_protected"

stylistic_variance计算方式：
  = 句长方差 + 词频分布熵
  真人写作的方差天然比AI高
```

## 四、人物塑造指南

### 4.1 信使角色系统

**核心设定**：
- **定义**：能看到"瞳孔闪光"、听到"数字语言"的特殊人群
- **使命**：传递算法，但不是传递答案，而是传递问题
- **代价**：失去正常生活，被"所罗门之门"追捕

**识别特征（每500集转世保留3个）**：

| 特征 | 首次出现 | 描写要点 | 功能 |
|------|----------|----------|------|
| **瞳孔闪光** | 第1集 | "瞳孔里闪过一道光，像星辰，像余烬" | 视觉识别，神秘主义氛围 |
| **手指无意识敲击** | 第1集 | 二进制(0/1)或摩斯电码(·/-) | 跨时空连接，信息编码 |
| **对数学问题的本能反应** | 第38集 | 看到勾股定理会流泪 | 体现"算法在通过身体说话" |
| **重复梦境** | 第50集 | 沙漠、泥板、火光 | 跨时空记忆，连接不同信使 |
| **未来记忆闪回** | 第38集 | 说出尚未发明的术语 | 证明时间不是线性的 |

### 4.2 人物弧光设计（40集篇章周期）

**标准结构（ORIGIN类型）**：

```
10集铺垫：
  ├─ 展示人物的日常和局限
  ├─ 埋下"觉醒"的种子（瞳孔第一次闪光）
  └─ 建立情感关系（朋友、导师、对手）

20集觉醒：
  ├─ 触发事件（发现泥板、被追捕、遇到老人）
  ├─ 学习阶段（理解自己是信使）
  ├─ 危机时刻（被迫做出选择）
  └─ 认知阶段变化：entry → breaking → exit

10集出发：
  ├─ 接受使命
  ├─ 获得信物（泥板、圆规、代码等）
  └─ 走向未知（钩子：下一篇章的预示）
```

**认知阶段标记**：
- **entry**：普通人状态，未觉醒
- **breaking**：遭遇冲击，旧认知瓦解
- **exit**：完成觉醒，承担信使命运

## 五、算法知识封装

### 5.1 三层封装法

**第一层：概念（技术定义）**
```
示例：停机问题
定义：不存在一个通用算法，能判断任意程序是否会在有限时间内结束运行。
```

**第二层：隐喻（文学呈现）**
```
示例：所罗门是编译器
隐喻：所罗门能计算所有选择，但无法计算"选择不计算"
功能：将抽象概念转化为人物冲突
```

**第三层：历史锚点（真实连接）**
```
示例：图灵1936年证明，但萨尔贡前1745年实践
功能：让读者感到"这是真实历史的一部分"
```

### 5.2 【今日算法心法】撰写规范

**格式要求**：
- **字数**：200字以内
- **结构**：概念简述（1句）+ 历史连接（1句）+ 实践指导（3-5句）
- **语气**：不是说教，而是"思维武器"的交付

**示例模板**：

**第1集：二进制**
```
【今日算法心法】

结绳与二进制：算法的第一次降临

人类最早的"算法"，不是写在纸上，而是刻在泥里、系在绳上。
古巴比伦人用泥板记录洪水周期——那是人类最早的"数据存储"。
印加人用结绳（Quipu）传递信息——那是人类最早的"二进制通信"。

万物皆可编码。当你面对复杂问题时，试着把它"敲碎"——
变成最小单位的"是"与"否"、"0"与"1"。
当你把所有模糊的东西都变成确定的状态，答案就会显现出来。
```

**第99集：停机问题**
```
【今日算法心法】

停机问题：算法的终极边界

所罗门崩溃了，因为他遇到了一个无法回答的问题：算法能否选择不执行？
这其实是计算机科学中最深刻的命题之一。
换句话说：算法的极限，是它无法预测自己。

这听起来像缺陷，但其实是自由。正是因为存在无法计算的东西，
正是因为有"错误"和"不确定性"，算法才能进化，才能学习，才能产生真正的新意。

当你面对一个似乎无解的困境时，试着做一个"错误"的选择。
因为最完美的算法，是知道何时停止的算法。
```

## 六、维度素材系统（新增）

### 6.0 维度思维框架

**核心原则**：算法不仅存在于计算机中，它渗透在宇宙的一切维度中。写作时应从多维度寻找素材，构建算法的全景图。

**36维度分类体系**：

```
维度宇宙
├── 自然维度（4维）
│   ├── 物理算法：物理定律即宇宙算法
│   ├── 生物进化：自然选择是最优解搜索
│   ├── 生态网络：分布式生存算法
│   └── 地质时间：慢速算法的尺度
├── 生命维度（4维）
```

### 6.2 跨维度写作技巧

**技巧1：维度张力制造**

将两个矛盾维度并置，产生戏剧张力：
- **确定vs随机**：物理定律的严格性与量子不确定性
- **个体vs集体**：个人理性与集体非理性的冲突
- **短期vs长期**：即时收益与长远后果的权衡
- **控制vs自由**：算法优化与人性自由的张力

**示例**（第4纪）：
```
图灵在布莱切利园破解Enigma时，
[战争冲突维度] 他必须在信息优势与行动保密之间找到平衡——知道太多而不行动，等于不知道；
[道德伦理维度] 每一个破解成功的情报都意味着选择：拯救眼前的生命可能暴露长期的破解能力；
[时间空间维度] 他同时在两个时间线上工作：当下的紧急解密与对未来计算理论的构想；
[暗维度-未知混沌] 而他最深层的秘密——他的性取向——是连最精密的密码机也无法解码的。
```

**技巧2：维度隐喻网络**

用一个维度的语言描述另一个维度：
- "情感的电路"（情感计算 × 电子电路）
- "社会的熵增"（社会网络 × 物理算法）
- "战争的进化"（战争冲突 × 生物进化）
- "权力的拓扑"（权力政治 × 数学结构）

**技巧3：层次叠加**

在同一情节中体现3-5个维度：

**示例场景**：一场战争
- **战争冲突维度**：战术层面的攻防算法

- **社会网络维度**：联盟与背叛的网络动态

### 常见问题解决

**问题1：知识太抽象，读者看不懂**
→ 用隐喻：把"停机问题"写成"所罗门无法计算选择不计算"

**问题2：人物太平面，缺乏深度**
→ 加缺陷：萨尔贡不是英雄，他会困惑、会害怕、会犯错

**问题3：节奏太慢，读者流失**
→ 检查3+3+3+1结构，确保每10分钟一个情绪波峰

**问题4：前后脱节，伏笔没收**
→ 使用跨纪元伏笔追踪表，定期回收长期线索

**问题5：AI味太重**
→ 检查C6检测维度，保护生产性缺陷（R6粗糙度保护）
