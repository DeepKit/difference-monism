# 《算法编年史》架构设计

## 一、ODD核心原则

> **"定义终局输出，反向分解路径，每集填空执行，分层验证封印"**

### 终局契约（第5000集）
- **时间锚点**：公元2026年（现实当下）
- **输出工件**：读者意识到自己就是第5001个信使，算法选择成为"永恒的提问者"
- **验收标准**：叙事在读者意识中完成递归循环，打破第四面墙
- **副作用**：小说成为自我延续的算法，每位读者都是新的计算节点
- **关键区别**：不是科幻未来，而是当下觉醒

## 二、36维度素材体系

### 2.0 维度思维框架

**核心理念**：算法不仅存在于计算机中，它渗透在宇宙的一切维度中。小说从36个维度构建算法的全景图。

**维度分类**：

| 维度类别 | 数量 | 核心特征 | 对应Epoch |
|---------|------|----------|-----------|
| **自然维度** | 4维 | 物理定律、生物进化、生态系统、地质时间 | 第1纪 |
| **生命维度** | 4维 | 遗传编码、神经认知、免疫防御、衰老死亡 | 第1-2纪 |
| **人类维度** | 12维 | 感知、情感、语言、社会、经济、政治、战争、法律、教育、艺术、宗教、伦理 | 贯穿全纪 |
| **技术维度** | 7维 | 能源、材料、机械、电子、通信、AI、量子 | 第3-5纪 |
| **抽象维度** | 5维 | 数学、逻辑、概率、复杂系统、时空 | 贯穿全纪 |
| **暗维度** | 4维 | 欺骗、偏见、崩溃、未知 | 警示维度 |

### 2.1 维度使用原则

**跨维度写作**：
- 每集选择2-3个核心维度
- 制造维度张力（如确定vs随机、个体vs集体）
- 构建维度隐喻网络（如"情感的电路"、"社会的熵增"）

**维度与纪元的映射**：

| Epoch | 核心维度 | 维度张力 |
|-------|----------|----------|
| 第1纪 | 物理算法、生物进化、战争冲突、数学结构 | 确定vs随机 |
| 第2纪 | 社会网络、经济交换、语言符号、宗教信仰 | 个体vs集体 |
| 第3纪 | 能源转化、机械运动、教育学习、法律制度 | 效率vs公平 |
| 第4纪 | 信息通信、人工智能、概率随机、神经认知 | 控制vs自由 |
| 第5纪 | 人工智能、量子计算、道德伦理、未知混沌 | 已知vs未知 |

### 2.2 维度与素材库的关系

**双重索引体系**：

1. **维度索引**（36维）：思维框架，指导创作方向
2. **类别索引**（7类100素材）：具体历史案例

**使用流程**：
```
确定Episode主题 → 选择2-3个核心维度 → 查询对应类别素材 
→ 提取历史案例 → 创作文学化场景 → 标记维度标签
```

**详细维度说明**：参见《素材库/素材维度表.md》

```

单元 = 3500字 = 1000钩子+1200发展+1000高潮+300心法
```

## 四、5条主线结构（核心骨架）

> **编年体不是单线叙事，而是5条主线 + 12条副线 + 临时插播的三层结构**

### 4.1 主线概览

| 主线 | 名称 | 核心冲突 | 功能 | 出现频率 | 占比 |
|------|------|----------|------|----------|------|
| **主线1** | 信使传承 | 觉醒与传递 | 人物载体 | 每5-10集 | 15% |
| **主线2** | 源代码 | 发现与隐藏 | 核心悬念 | 每10-20集 | 10% |
| **主线3** | 编译器对抗 | 控制与自由 | 冲突引擎 | 每15-30集 | 10% |
| **主线4** | 算法进化 | 进步与优化 | 文明进步 | 每3-5集 | 20% |
| **主线5** | 算法代价 | 异化与伤害 | **负面影响** | 每3-5集 | 20% |

**特殊说明**：
- 主线4（进化）与主线5（代价）必须**同时推进**，形成光明与阴影的对照
- 不是每集都推进主线，允许主线休眠10-20集
- 临时插播集（26%）可以完全不涉及主线，纯编年记录

### 4.2 主线与Epoch的对应

| Epoch | 主线4：进化 | 主线5：代价 | 对照关系 |
|-------|------------|------------|----------|
| **第1纪** | 几何学诞生 | 知识暴力、劳动异化 | 毕达哥拉斯定理 vs 希帕索斯之死 |
| **第2纪** | 代数传播 | 经济压迫、知识垄断 | 代数学 vs 算法征税 |
| **第3纪** | 机械计算 | 时间异化、系统风险 | 机械钟 vs 劳动规训 |
| **第4纪** | 信息革命 | 科学暴力、毁灭力量 | 统计学 vs 优生学 |
| **第5纪** | AI觉醒 | 算法歧视、失业、控制 | AI诊断 vs AI偏见 |

### 4.3 编年体三线并进

```
5000集结构：
├─ 主线推进集（44% = 2200集）：推进1-2条主线
├─ 副线展开集（30% = 1500集）：展开3-4条副线（12维度）
└─ 临时插播集（26% = 1300集）：独立编年记录，无主线推进

示例（第2500集附近）：
├─ 第2498集：主线推进（牛顿发明微积分）- 主线4
├─ 第2499集：副线展开（英荷战争背景）- 战争维度
├─ 第2500集：主线推进（牛顿的孤独与偏执）- 主线5
├─ 第2501集：临时插播（1666年伦敦大火）- 纯历史
└─ 第2502集：主线推进（莱布尼茨也在计算）- 主线2
```

## 六、封印（Seal）原则

> **一旦某层通过验证，即成为不可变基础，上层构建其上。**

| 层级 | 封印条件 | 封印仪式 |
|------|----------|----------|
| **单元**（集） | 通过5层验证 | 发布，不可修改 |
| **篇章**（40集） | 终章读者满意度>80% | 实体解谜书发布 |
| **时代**（200集） | 终章形成文化讨论 | 纪录片/播客联动 |
| **纪元**（1000集） | 终章被学术引用 | 学术研讨会 |
| **本体**（5000集） | 终章读者成为创作者 | 开源叙事宇宙 |

### 反馈循环（L17修正）
```
读者反馈 → 数据分析 → 架构调整 → 后续集数修正 → 验证
```

**示例**：
- 第38集发布后，读者对"摩斯电码"困惑
- L17触发：在第50集增加"未来记忆"的解释性场景
- 第99集重新强调：伊什美的困惑是"觉醒"的正常现象
- 验证：第100集评论中"终于懂了"占比>60%

## 七、核心机制详解

### 7.1 递归叙事结构
- **第1集**抛出终极问题："你敲的是什么？"
- **每100集**释放一个"假答案"，让读者以为接近真相
- **第5000集**揭晓：算法成为提问者，读者成为信使
- **结构**：终点即起点，形成叙事莫比乌斯环

### 7.2 信使转世系统
- **周期**：每500集更换主角（信使）
- **保留**：3个识别特征（瞳孔闪光、手指敲击、数学本能等）
- **功能**：读者像追星一样追踪"TA又出现了"

### 7.3 算法知识封装
- **概念**：停机问题、圆周率、熵增等
- **隐喻**：所罗门=编译器，萨尔贡=错误，图灵=信使
- **历史锚点**：图灵1936年证明，萨尔贡前1745年实践

### 7.4 三层伏笔系统
- **短期**（40集）：学徒的枣木杖→篇章2 reveal
- **中期**（200集）：摩斯电码种子→第4000集回收
- **长期**（1000+集）：六十一点横→第5000集 reveal 为源代码

## 八、本体验收标准

1. ✅ 第1集与第5000集形成完美的叙事莫比乌斯环
2. ✅ 每个纪元的终章都是下一纪元的"编译器"
3. ✅ 读者完成阅读后，对"算法"的定义发生不可逆改变
4. ✅ 小说可被视作一个自我运行的算法（有输入、处理、输出、反馈）

## 九、终极输出：读者成为信使

**第5000集终章《输出》验收标准：**

> 读者关闭书本后，看向自己的手，想起第1集萨尔贡敲击泥板的画面，意识到：
> 
> "我也是信使。我的每一个选择，每一个'错误'，每一次'重新算'，都是算法在通过我延续自己。

> 这部小说不是关于算法的历史。它本身就是算法。而我，刚刚执行了它。"

**副作用**：读者成为新的叙事节点，可能产生同人创作、学术分析、或仅仅是思维方式的不可逆改变。
