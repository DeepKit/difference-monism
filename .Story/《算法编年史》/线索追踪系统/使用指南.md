# 线索追踪系统使用指南
# 《算法编年史》线索管理系统 v1.0

## 系统概览

本系统用于管理《算法编年史》5000集中的所有线索，确保：
1. 伏笔不会遗漏
2. 回收时机恰当
3. 线索逻辑一致
4. 长程线索有效管理

## 文件说明

### 1. 00_线索总表.md
**用途**：全局线索数据库
**内容**：
- 所有主线线索（5条线）
- 50位信使的标识
- 关键物品、概念、意象线索
- 线索状态跟踪

**使用场景**：
- 查找特定线索
- 了解线索层次结构
- 检查线索分布

### 2. 01_前100集线索映射.md
**用途**：集数-线索交叉索引
**内容**：
- 第1-100集每集出现的线索
- 埋下(B)/回收(R)标记
- 本集钩子
- 回收的线索

**使用场景**：
- 写作时查看本集该出现什么线索
- 检查线索一致性
- 规划回收时机

### 3. 02_线索回收提醒.md
**用途**：防止线索遗漏
**内容**：
- 紧急回收提醒（🔴）
- 注意回收（🟡）
- 长程线索跟踪（>500集）
- 第5000集回收总表

**使用场景**：
- 每周检查待回收线索
- 长程线索进度跟踪
- 第5000集规划

### 4. CLUE-XXXX_线索名称.md
**用途**：重要线索的详细档案
**内容**：
- 线索的完整生命周期
- 埋下、发展、回收的详细描述
- 关联线索
- 回收规则

**使用场景**：
- 深入理解核心线索
- 确保回收质量
- 传递给其他写作者

## 使用流程

### 写作前

1. **查看线索映射表**
   - 打开 `01_前100集线索映射.md`
   - 找到当前写作的集数
   - 查看应该出现的线索

2. **检查回收提醒**
   - 打开 `02_线索回收提醒.md`
   - 查看是否有紧急回收的线索
   - 确认回收方式

3. **查看线索档案**（如需要）
   - 如果是核心线索回收
   - 查看详细档案确保一致性

### 写作中

1. **埋下新线索**
   - 在草稿中标记 `CLUE-XXX(B)`
   - 记录在线索总表中
   - 更新线索映射表

2. **回收旧线索**
   - 在草稿中标记 `CLUE-XXX(R)`
   - 确保回收符合原始设定
   - 记录回收效果

3. **保持活跃**
   - 活跃线索在文中保持存在感
   - 定期提及但不推进

### 写作后

1. **更新线索状态**
   - 在线索总表中更新状态
   - 在线索映射表中记录

2. **检查一致性**
   - 确保没有逻辑矛盾
   - 检查线索间隔合理

3. **设置回收提醒**
   - 对新埋下的线索设置提醒
   - 更新长程线索进度

## 线索编号规则

### 主线线索（L1）

```
CLUE-[主线缩写]-[序号]

CLUE-MS-001 = 信使传承线-第1个线索
CLUE-SC-001 = 源代码线-第1个线索
CLUE-CC-001 = 编译器对抗线-第1个线索
CLUE-AE-001 = 算法进化线-第1个线索
CLUE-AC-001 = 算法代价线-第1个线索
```

### 支线线索（L2）

```
CLUE-SUB-[纪号]-[支线索号]-[序号]

CLUE-SUB-1-001-001 = 第1纪-第1条支线索-第1个节点
```

### 物品线索（L3）

```
CLUE-ITEM-[序号]

CLUE-ITEM-001 = 物品线索第1个
```

### 概念线索（L4）

```
CLUE-CON-[序号]

CLUE-CON-001 = 概念线索第1个
```

### 意象线索（L5）

```
CLUE-IMA-[序号]

CLUE-IMA-001 = 意象线索第1个
```

## 线索状态标记

| 标记 | 含义 | 使用场景 |
|-----|------|---------|
| (B) | Buried - 已埋下 | 首次出现时标记 |
| (R) | Recycled - 已回收 | 回收时标记 |
| (A) | Active - 活跃中 | 已埋下但未回收，持续存在 |
| (P) | Pending - 待使用 | 计划中但尚未出现 |
| (X) | Abandoned - 废弃 | 不再使用的线索 |

## 最佳实践

### 埋下线索时

1. **明确回收计划**
   - 埋下时就想好何时回收
   - 记录在回收提醒系统中

2. **保持一致性**
   - 埋下时的描述与回收时一致
   - 如有变化需有解释

3. **控制数量**
   - 每集埋下线索不超过3个
   - 长程线索不超过10个同时活跃

### 回收线索时

1. **提前提醒**
   - 回收前3-5集提及该线索
   - 让读者回忆起来

2. **情感冲击**
   - 回收时要有情感或认知冲击
   - 不仅是解释，更是揭示

3. **新线索接替**
   - 回收旧线索时埋下新线索
   - 保持悬念连续

### 长程线索管理

1. **定期提醒**
   - 每100-200集轻微提及
   - 保持读者记忆

2. **渐进揭示**
   - 不要一次性揭示全部
   - 分层次逐步揭示

3. **终极回收**
   - 第5000集回收所有核心线索
   - 达到最大情感冲击

## 常见问题

### Q: 如何处理线索冲突？
A: 优先保证主线线索，支线线索可调整。记录所有修改。

### Q: 忘记回收线索怎么办？
A: 使用回收提醒系统定期检查。如已遗漏，可在后续集数补救性回收或改为废弃。

### Q: 线索太多记不住？
A: 只关注当前100集内的线索。长程线索使用自动提醒。

### Q: 读者没注意到线索？
A: 回收前增加提醒，但不要过于明显。接受部分读者会遗漏。
