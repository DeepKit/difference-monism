# 算法类型：行政算法 / 税收系统 / 中央集权

## 权力算法逻辑

```
输入：土地面积、人口数量、粮食产量
约束：皇权合法性 + 军事维稳成本
算法：分级征税 × 户籍管理 → 帝国运转
输出：财政盈余 / 军队规模 / 官僚层级
```

## 历史事件

**汉代算赋制度（公元前200年）**
- 算法核心：按人头征税，15岁以上每人120钱
- 权力逻辑：把人口转化为数字，国家第一次用算法管理人
- 代价：逃籍、隐户、豪强兼并

**罗马行省税收算法（公元1世纪）**
- 算法核心：普查（Census）→ 分级征税
- 权力逻辑：算法是帝国扩张的基础设施
- 代价：行省官员腐败，算法被权力扭曲

**唐代均田制崩溃（公元780年）**
- 算法核心：两税法取代均田制
- 权力逻辑：当土地算法失效，帝国开始瓦解
- 代价：藩镇割据，算法权威分裂
