# 素材编号：WAR-002
# 素材名称：韩信·明修栈道暗度陈仓
# 算法类型：分布式系统的CAP定理 / 信息不对称利用

## 核心算法逻辑

```
约束条件：信息传输不可靠（敌方侦察网络）
决策：牺牲可用性（示弱）换取最终一致性（突袭成功）
```

## 历史背景

- **时间**：公元前206年
- **地点**：陈仓（今陕西宝鸡）
- **关键数据**：
  - 栈道：明线，故意烧毁修复（时间成本O(n)）
  - 陈仓小道：暗线，奇袭（时间成本O(1)）

## 算法拆解

### 系统假设
- 章邯的侦察系统 = 分布式节点
- 带宽限制：只能同时监控有限通道
- 延迟：情报传递需要数日

### 算法设计
1. **资源分配**：80%资源投入栈道（噪音）
2. **真实路径**：20%资源走陈仓小道（信号）
3. **一致性策略**：延迟暴露真实意图，直到无法逆转

### CAP权衡
- **Consistency（一致性）**：章邯对汉军位置的认知
- **Availability（可用性）**：章邯需要随时应对可能进攻
- **Partition tolerance（分区容错）**：侦察网络的不完整性

韩信选择：容忍暂时不一致（章邯不知道真实进攻点），换取最终一致性（突袭时敌方无法响应）。
