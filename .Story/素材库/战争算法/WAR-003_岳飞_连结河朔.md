# 素材编号：WAR-003
# 素材名称：岳飞·连结河朔
# 算法类型：分布式网络 / P2P拓扑 / 容错系统

## 核心算法逻辑

```
网络结构：去中心化P2P（义军节点）
容错机制：单点失效不影响整体
攻击模式：分布式骚扰，非中心化决战
```

## 历史背景

- **时间**：1134-1140年
- **地点**：黄河以北沦陷区
- **网络规模**：数十万义军，数千个据点
- **中心节点**：岳飞（名义统帅，非实际控制）

## 算法拆解

### 网络拓扑对比

| 类型 | 宋军传统 | 连结河朔 |
|------|---------|---------|
| 结构 | 树状（中心-分支） | 网状（P2P） |
| 单点失效 | 灾难性（主帅阵亡=系统崩溃） | 自愈（节点动态替换） |
| 信息传递 | 层级传递，延迟高 | 多路径广播，冗余高 |
| 资源分配 | 集中调度 | 本地自治 |

### 拜占庭容错
- 义军中有叛徒（拜占庭节点）
- 岳飞设计"忠诚度共识算法"：多次小规模合作测试
- 无法通过测试的节点被隔离

### 攻击策略
- **金军算法**：寻找中心节点斩首
- **岳飞算法**：没有中心节点可斩
- 金军被迫O(n)逐个清除 → 成本过高放弃
