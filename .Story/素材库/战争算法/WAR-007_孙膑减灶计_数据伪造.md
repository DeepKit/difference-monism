# 素材名称：孙膑减灶计·数据伪造
# 算法类型：数据投毒 / 虚假信号注入 / 模型欺骗

## 核心算法逻辑

```
输入：敌方庞涓的追击决策模型
操作：伪造"兵力减少"的数据（减灶）
输出：敌方模型学习到错误模式 → 过度自信追击 → 陷阱
```

## 历史背景

- **时间**：公元前341年
- **地点**：马陵道
- **关键数据**：
  - Day 1：10万灶
  - Day 3：5万灶
  - Day 5：3万灶
- **结果**：庞涓自杀，魏军覆灭

## 算法拆解

### 数据投毒（Data Poisoning）

**庞涓的模型假设**：
- 灶数 ∝ 兵力（正相关）
- 兵力↓ → 军心涣散 → 可追击

**孙膑的投毒**：
- 故意减少灶数（虚假标签）
- 庞涓的模型被"训练"错误
- **过拟合**：看到局部趋势（灶数减少），忽略全局（陷阱）

### 特征工程攻击
- 庞涓选择的特征（灶数）= 可伪造的
- 更好的特征（侦察实际兵力）= 成本高，未采用
- 攻击者利用了防御者的"特征懒惰"

### 反馈循环
1. 庞涓看到减灶 → 确信齐军溃散
2. 加速追击 → 进入伏击圈
3. 模型自我强化（确认偏见）
