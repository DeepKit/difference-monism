## 历史背景

### 三条并行的算法战线

**1. 密码战线（1945~1991）**
- 美国NSA vs 苏联KGB：密码算法成为国家核心机密
- VENONA计划：美国破译苏联外交电报，算法决定间谍命运
- 一次性密码本（OTP）：理论上无法破解的算法，双方都在寻找对方的漏洞
- 核心算法逻辑：**信息熵最大化 = 无法预测 = 无法破解**

**2. 核弹计算线（1945~1960）**
- 曼哈顿计划遗产：ENIAC最初用途是计算炮弹弹道
- 氢弹设计需要蒙特卡洛模拟：冯·诺依曼架构的直接驱动力
- 苏联第一颗原子弹（1949）：依赖从美国窃取的计算方法
- 核心算法逻辑：**蒙特卡洛方法 = 用随机性逼近确定性**

**3. 网络起源线（1969）**
- ARPANET诞生：为防止核打击摧毁单一通信节点
- 分布式路由算法：没有中心，信息自动绕过损毁节点
- 核心算法逻辑：**去中心化路由 = 反脆弱网络拓扑**

## 算法拆解

### 密码算法的本质
```
明文 + 密钥 → 密文
密文 + 密钥 → 明文
破解 = 在不知道密钥的情况下，从密文还原明文
```

冷战密码战的核心矛盾：
- 密钥越长越安全，但传递密钥本身就是风险
- 一次性密码本理论完美，但实际操作中苏联重复使用密钥 → VENONA破译成功

### 蒙特卡洛方法
```
问题：氢弹内部中子碰撞路径无法用方程精确求解
解法：模拟10万次随机碰撞，统计平均结果
本质：用大量随机样本的统计规律，逼近确定性答案
```

### ARPANET路由算法
```
节点A → 节点B（路径1：正常）
节点A → 节点C → 节点B（路径2：节点B被摧毁时自动切换）
核心：每个节点只知道"下一跳"，不需要知道全局路径
```

## 可植入场景

### 场景A：NSA地下室，1952年
密码分析员盯着一列乱码，手边是厚厚的频率分析表。
他们不知道自己正在破译的电报，将导致一个间谍网络的覆灭。
算法在这里是武器，也是绞索。

### 场景B：洛斯阿拉莫斯，1949年
冯·诺依曼站在ENIAC前，输入蒙特卡洛参数。
机器轰鸣，打印纸缓缓吐出一列数字。
这列数字将决定氢弹能否引爆——算法第一次触碰了人类的生死边界。

### 场景C：ARPANET节点，1969年10月29日
第一条消息从UCLA发往斯坦福：`LO`
系统崩溃，只传出两个字母。
但这两个字母，是互联网的第一声啼哭。
