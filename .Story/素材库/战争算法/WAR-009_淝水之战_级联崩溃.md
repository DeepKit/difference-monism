# 算法类型：级联失效 / 正反馈循环 / 雪崩效应

## 核心算法逻辑

```
初始扰动：小股部队后撤（假消息"秦军败了"）
放大机制：信息在密集网络中传播
级联效应：恐慌 → 踩踏 → 全面崩溃
输出：90万大军自我毁灭
```

## 历史背景

- **时间**：公元383年
- **地点**：淝水（今安徽）
- **兵力对比**：前秦90万 vs 东晋8万
- **关键事件**：朱序（降将）在阵后大喊"秦军败了"

## 算法拆解

### 级联失效（Cascading Failure）

**网络结构**：
- 秦军 = 密集连接网络（信息传播快）
- 多民族混编 = 信任度低（脆弱性高）

**触发点**：
- 小范围后撤（正常战术调整）
- 被误解为"败退"（朱序的恶意传播）

**正反馈**：
```
恐慌 → 逃跑 → 更多人看到逃跑 → 更恐慌 → ...
```

### 临界阈值
- 系统处于亚稳态（90万人协调困难）
- 微小扰动 > 临界阈值 → 相变（有序→混乱）

### 复杂系统的脆弱性
- 规模越大 ≠ 越强
- 超过协调复杂度阈值，规模成为劣势

**第5纪·时代25（AI对齐）**
