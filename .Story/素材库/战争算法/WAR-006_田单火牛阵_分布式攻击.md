# 素材名称：田单火牛阵·分布式攻击
# 算法类型：并行计算 / 蜂群智能 / 去中心化战术

## 核心算法逻辑

```
输入：1000头牛 + 火把 + 夜间
算法：每只牛 = 独立攻击单元
输出：并行冲击 → 敌方阵型崩溃
```

## 历史背景

- **时间**：公元前279年
- **地点**：即墨（齐国）
- **创新点**：非人类作战单元（牛）
- **结果**：燕军溃败，收复70城

## 算法拆解

### 蜂群智能（Swarm Intelligence）

**单个牛的规则**：
1. 尾巴着火（疼痛驱动）
2. 向前狂奔（最短路径逃离）
3. 遇到障碍物转向（简单避障）

** emergent行为**：
- 无中央控制
- 群体形成"洪流"效应
- 敌方无法预测个体轨迹

### 并行 vs 串行

| 类型 | 传统步兵冲锋 | 火牛阵 |
|------|-------------|--------|
| 控制 | 中心指挥（串行） | 分布式（并行） |
| 单点失效 | 指挥官阵亡=崩溃 | 个体损失不影响整体 |
| 可预测性 | 高（阵型固定） | 低（混沌运动） |
| 心理冲击 | 中等 | 极高（非理性恐惧） |

### 噪声利用
- 牛的嘶鸣、火光的闪烁 = 感官过载
- 敌方决策系统崩溃（无法处理异常输入）
