# 素材名称：罗马税收算法·精确掠夺
# 维度：算法代价（COST-003）

## 核心事件

```
时间：公元前1世纪-公元5世纪（第2纪）
地点：罗马帝国全境
算法：人口普查 + 财产评估 + 统一税率
代价：农民破产、饥荒、起义
```

## 算法拆解

### 罗马的财政算法

**人口普查（Census）**：
- 定期统计人口、财产、奴隶数量
- 精确到每个成年男性
- 算法的基础：数据收集

**税率计算**：
- 根据财产等级确定税率
- 统一标准，减少腐败（理论上）
- 实际：地方官员额外征收

### 代价产生

**精确性的暴力**：
- 算法不考虑天灾、疾病、个人困境
- 精确计算"应该缴纳"的税额
- 无法缴纳者：卖身为奴或逃亡

**系统性饥荒**：
- 税收拿走生存所需的最低粮食
- 农民被迫卖掉种子粮
- 下一年歉收 → 饥荒

## 可植入场景

**第2纪·第1200集**
- 罗马税收官使用算盘计算税额
- 农民的绝望
- 信使花拉子米的前世（作为被征税者）

**第5000集回收**：
- "算法的精确，可以成为压迫的精确。"
- "数据不知道同情，但收集数据的人应该知道。"
