# 素材编号：ECO-002
# 素材名称：交子·信用算法
# 算法类型：信任网络 / 信用创造 / 分布式共识

## 核心算法逻辑

```
前提：金属货币不便携带（高交易成本）
创新：纸币 = 信用凭证（承诺兑换金属）
关键：如何维持"信用"不崩溃（共识机制）
```

## 历史背景

- **时间**：北宋（1023年官方发行）
- **地点**：四川（铁钱区）
- **创新**：世界最早纸币
- **失败**：南宋超发导致通胀

## 算法拆解

### 信用共识机制

**正常情况**：
- 交子 = 存款凭证（100%准备金）
- 持票人可随时兑换 → 维持信任

**信用扩张**（危险操作）：
- 发行交子 > 准备金
- 只要"挤兑"不发生，系统可持续
- **博弈论**：信任是自我实现的预言

### 准备金率算法

**最优准备金率**：
- 太高：流动性不足，经济停滞
- 太低：挤兑风险，系统崩溃
- 黄金比例：历史经验值（约20-30%）

**南宋的失败**：
- 战争压力 → 超发交子（准备金率<10%）
- 信心崩溃 → 挤兑 → 恶性通胀
- 最早的**货币危机算法**
