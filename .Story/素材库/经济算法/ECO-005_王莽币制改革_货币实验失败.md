# 算法类型：货币实验 / 过度创新 / 系统崩溃

## 核心算法逻辑

```
目标：解决贫富分化、土地兼并
方法：复古改制（井田制、新货币）
问题：脱离现实，执行混乱
输出：恶性通胀、社会崩溃、新朝灭亡
```

## 历史背景

- **时间**：公元7-23年
- **改革**：四次币制改革，28种货币并行
- **结果**：经济崩溃，农民起义，王莽被杀

## 算法拆解

### 复杂系统的脆弱性

**过度优化**：
- 理想模型（复古周礼）≠ 现实可行性
- 货币系统需要共识（信任）
- 频繁改变规则 → 信任崩溃

**执行困境**：
- 政策在纸面上完美
- 执行层腐败、套利
- **算法正确 ≠ 系统正确**

### 历史教训

**改革算法的设计原则**：
1. 渐进 > 激进（小步快跑）
2. 尊重现有路径依赖
3. 考虑执行层激励相容

**王莽的失败**：
- 违反全部三条原则
- 算法天才，系统白痴
