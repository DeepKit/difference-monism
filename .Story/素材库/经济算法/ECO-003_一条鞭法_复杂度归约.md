# 素材编号：ECO-003
# 素材名称：一条鞭法·复杂度归约
# 算法类型：降维算法 / 复杂度归约 / 标准化

## 核心算法逻辑

```
输入：复杂的赋役体系（里甲、均徭、杂泛）
问题：O(n²)复杂度，执行成本高，腐败点多
算法：归约为单一银两计算（O(n)）
输出：简化征收，提高效率
```

## 历史背景

- **时间**：1581年（张居正改革）
- **地点**：全国推行
- **核心**：赋役合并，计亩征银
- **影响**：明朝中兴，但触动既得利益

## 算法拆解

### 复杂度分析

**旧系统**（里甲制）：
- 维度1：田赋（按亩）
- 维度2：徭役（按丁）
- 维度3：杂税（各种项目）
- 交互项：各维度间换算（复杂函数）
- **复杂度**：O(n × m × k)

**新系统**（一条鞭）：
- 单一维度：白银
- **复杂度**：O(n)
- 优化效果：执行成本指数级下降

### 标准化收益

**信息成本降低**：
- 无需记录复杂的徭役折算
- 银两可分割、可储存、可比较

**监督成本降低**：
- 腐败节点减少（简化流程）
- 审计更容易（单一指标）

### 副作用

**维度丢失**：
- 徭役的社会功能（公共工程、国防）被货币化
- 长期：公共品供给不足

**白银依赖**：
- 中国不产银，依赖进口
- 银荒时经济崩溃（明末）
