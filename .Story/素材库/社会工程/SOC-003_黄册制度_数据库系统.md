# 素材编号：SOC-003
# 素材名称：黄册制度·数据库系统
# 算法类型：分布式数据库 / 索引结构 / 查询优化

## 核心算法逻辑

```
数据：全国人口、土地、赋役信息
存储：分层分布式（中央-省-府-县）
索引：里甲制（110户为一里）
查询：通过层级索引快速定位
```

## 历史背景

- **时间**：明洪武年间（1381年建立）
- **规模**：全国人口约6000万
- **更新频率**：每10年（大造）
- **存储地点**：南京后湖黄册库

## 算法拆解

### 数据模型

**Schema设计**：
```
户（Household）
├── 户主信息（姓名、年龄、职业）
├── 土地（位置、面积、等级）
├── 人口（丁男、妇女、老幼）
└── 赋役（应缴、已缴、欠缴）
```

**关系型数据**：
- 里（110户）→ 甲（10户）→ 户
- 层级关系，便于聚合查询

### 分布式存储

**数据分片**（Sharding）：
- 中央：汇总数据（聚合统计）
- 地方：详细数据（原始记录）
- 备份：黄册库（冷备份，档案留存）

**一致性策略**：
- 最终一致性（10年同步一次）
- 牺牲实时性，换取存储效率

### 查询优化

**索引结构**：
- 里甲制 = B+树索引
- 通过里→甲→户的层级，O(log n)查询

**缓存机制**：
- 地方衙门保存常用数据
- 减少中央查询压力
