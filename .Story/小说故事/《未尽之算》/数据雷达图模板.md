# 《未尽之算》单章数据雷达图 (发布前自查表)

> **使用说明**：
> 每次发布新章前，请根据文章内容，对以下六个维度进行打分 (0-10)。
> 目标：总分 > 45，且无短板 (<6分)。

| 维度 | 定义 | 评分标准 (0-10) | 自评分 | 优化动作 (若 < 6分) |
| :--- | :--- | :--- | :--- | :--- |
| **打开率 (Open Rate)** | 标题与Hook的吸引力 | 0=无感；10=看了标题必点 | [ ] | 优化前300字，让“系统”更早登场。 |
| **收藏率 (Save Rate)** | 干货/考据的密度 | 0=纯水文；10=教科书级 | [ ] | 增加“冷知识”或“数据成本”描述。 |
| **付费率 (Conversion)** | 卡点的悬念强度 | 0=无聊；10=不看睡不着 | [ ] | 强化“反杀”的具体细节和狠度。 |
| **完读率 (Completion)** | 节奏的紧凑度 | 0=读两行就退；10=一口气读完 | [ ] | 删除所有形容词，增加对话密度。 |
| **评论量 (Interaction)** | 结尾引战的撕裂度 | 0=无人说话；10=评论区打架 | [ ] | 检查引战问题是否没有标准答案。 |
| **意义感 (Significance)** | 哲学内核的渗透度 | 0=纯爽文；10=引发思考 | [ ] | 检查金句是否能脱离故事独立存在。 |

---

## 雷达图可视化 (ASCII版)

```
       打开率 (10)
          |
意义感(10)--+--收藏率(10)
    |     |     |
评论量(10)--+--付费率(10)
          |
       完读率 (10)
```
