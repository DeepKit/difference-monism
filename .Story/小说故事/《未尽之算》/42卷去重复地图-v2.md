# 《未尽之算》42卷去重复地图 v2（Anti-Fatigue Architecture）

> 目标：确保1008集不会变成“重复刷副本”。
> 规则：相邻两卷至少在 **2个维度** 上不同；同一部内案型不连续重复。

---

## 维度定义
### 维度A：案型（Case Type）
- 漏洞案：规则自相矛盾
- 责任链案：谁负责？如何防止责任漂白
- 不可逆案：隐私/退出权/不可逆伤害（触发熔断）
- 合规陷阱案：表面合规的暴力（公文/合同/话术做刀）
- 资源稀缺案：床位/粮食/算力/排队（冷数学做凶器）
- 舆论案：系统用“讲述权”反杀

### 维度B：对手形态（Antagonist Shape）
- NPC：中层脚本机器
- 无脸：模型/指标
- 链条：外包/分布式无人负责
- 法律：监管/法院/合法暴力
- 叙事：公关/媒体/现实扭曲
- 自适应：会学习你、会升级补丁

### 维度C：标准升级（Standard vX.Y）
- 这是“裁决栈/审计板”的版本升级记录：每卷只升级一条。

### 维度D：证据增量（Missing Piece）
- 每卷必须交付一个“拼图碎片”，推动总谜题。

### 维度E：结局类型（Outcome）
- 胜利 / 惨胜 / 平局 / 失败 / 反转（你以为救人，其实在喂系统）

### 维度F：呈现形式（Form Factor）
- 传统叙述 / 邮件链拼贴 / 听证会质询 / SOP培训材料 / 法庭记录 / 报表&日志 / 时间跳切 / 红act vs unredacted
