# 《未尽之算》ASTO公理简表（小说创作版）

> 目标：给写作团队一个“可用的公理词典”。
> 原则：只写**一句话定义 + 小说化用法提示**，不引入哲学术语堆。

---

## 公理 -1：语言的极限（Linguistic Limitation）
**一句话**：地图永远不是领土；任何描述都是有损压缩。
**小说化用法**：当角色以为“写在条款里就等于现实”，用此公理提醒“文本≠生命”。

## 公理 0：环境熵增（Environmental Entropy Increase）
**一句话**：默认状态是崩塌；不维护就会变坏。
**小说化用法**：资源稀缺、系统老化、供应链断裂、疫情失控等“自然下滑”。

## 公理 1：结构性稳态（Structural Homeostasis）
**一句话**：存在就是在噪声海里维持一个低熵小岛。
**小说化用法**：制度/公司为了稳态会吃人；“人被当燃料”不是阴谋，是稳态代价。

## 公理 2：属性分层（Attribute Stratification）
**一句话**：硬约束在底层，软规则在上层；别用上层幻想对抗底层物理。
**小说化用法**：距离、床位、粮食、水、电、带宽、算力、交通这类“物理红线”。

## 公理 3：合规传递（Compliance Transmission / Impedance）
**一句话**：行为会走最小阻抗路径；合规是“更省事”的那条路才会发生。
**小说化用法**：流程拖死、责任漂白、外包甩锅、指标驱动的“最省事作恶”。

## 公理 4：扰动场（Motility Field）
**一句话**：不要试图改变水流，去改河道；真正的控制是“场”的设计。
**小说化用法**：平台、规则、激励、审计、工具链——把人变成自动执行脚本。

## 公理 5：结构自指（Structural Self-Reference / Tech Debt）
**一句话**：结构会自我增殖并带来维护利息；每一次修补都在喂养未来的怪物。
**小说化用法**：补丁越打越臃肿、合规越来越复杂、解释权越来越集中。

## 公理 6：规范跃迁（Normative Transition）
**一句话**：当旧规则的维护成本大于重写成本，跃迁是唯一生路。
**小说化用法**：制度崩解前的“硬重构”、版本升级、系统重启、旧秩序的死亡。

## 公理 7：不可约性（Irreducibility）
**一句话**：有些东西算不出来（良知、体验、爱、羞耻）；只能由人承担裁决。
**小说化用法**：算法无法替代的“人类裁决层”，以及它被系统驯化/消灭的过程。

## 公理 8：实践回路（Practice Loop）
**一句话**：结构是否有效，不看自洽，看能否在现实里闭环验证。
**小说化用法**：制度在现实里“跑不动”、指标反噬、试点失败；以及系统如何学习。

## 公理 9：内部张力（Internal Tension）
**一句话**：矛盾不是异常，是系统维持稳定与演化的张力来源。
**小说化用法**：主矛盾/次矛盾切换；“你以为解决了，实际只是换了一个矛盾”。

## 公理 10：观察者效应（Observer Effect / Settlement）
**一句话**：观察就是结算；结算就是锁定。
**小说化用法**：一旦被记录/评分/入库，就被系统“定性”；身份认证、画像、标签化。

## 公理 11：自由（Freedom）
**一句话**：自由不是无约束，而是在不可触碰边界内引入不可预测扰动的能力。
**小说化用法**：人做“非最优选择”、拒绝路径、退出权；以及自由带来的责任与代价。

## 公理 12：悖论熔断（Paradox Circuit-Breaker）
**一句话**：当“必须做”撞上“绝不能做”，系统必须熔断并上交人类裁决。
**小说化用法**：电车难题、救一人/救一群、隐私红线、不可逆伤害——触发熔断。

## 公理 13：存在连续性（Existential Continuity）
**一句话**：身份靠关键标识链的连续性，不靠材料堆砌；时间不可逆。
**小说化用法**：特修斯之船、数字遗产与AI复刻、记忆篡改、版本更新、历史循环证据。
