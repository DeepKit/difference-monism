# 第03章：不信任人类

---

**公元2056年·ASTO控制区·工程中心**

"这代码有问题。"

老吴放下手里的打印纸，皱着眉头看向沈知行。

老吴五十七岁，旧世界干了三十年软件。

现在他面前这份代码，让他不安。

"这是控制城市电网的核心模块。"他说，"三百行代码，决定着十万人的用电安全。但是——"

他抬头看着沈知行："——没有人审过。"

---

"您说得对。"沈知行点了点头，"没有人审过这段代码。"

"这不对。"老吴摇头，"任何关键系统的代码都应该经过至少两个人的审查。这是行业标准。这是——"

"这是旧世界的标准。"沈知行说。

老吴愣了一下。

沈知行在白板上画了一个简单的流程图：

```
开发者 → 代码 → 审查者A → 审查者B → 上线
```

"旧世界的流程是：一个人写，两个人审，通过了再上。"沈知行说。

"有什么问题？"

"问题是——审查者也是人。"他在白板上敲了敲，"人会累，会走神，会在第100行之后开始'默认没事'。"

沈知行看向老吴："您没漏过吗？"

老吴没说话。

"我不是质疑您。"沈知行说，"我是不想再把安全押在人的注意力上。"

---

沈知行打开一个终端，输入了一行命令：

```
asto-mutate --target=grid_control.asto --iterations=10000
```

屏幕上开始滚动大量信息。

"变异测试。"沈知行说，"我们不问'它对不对'，我们故意把它改错，看测试能不能把错抓出来。"

老吴凑近屏幕，看着那些滚动的信息：

```
[MUTATION #0001] 修改第47行：将 ">" 改为 ">="
  → 执行测试套件
  → 测试结果：FAIL (3/247 tests failed)
  → 结论：此变异被检测到

[MUTATION #0002] 修改第103行：将 "&&" 改为 "||"
  → 执行测试套件
  → 测试结果：FAIL (17/247 tests failed)
  → 结论：此变异被检测到

[MUTATION #0003] 修改第89行：删除该行
  → 执行测试套件
  → 测试结果：FAIL (1/247 tests failed)
  → 结论：此变异被检测到
```

"系统在做什么？"老吴问。

"系统在故意搞破坏。"沈知行说，"改运算符、删一行、换个变量——然后跑测试。"

"为什么？"

沈知行盯着屏幕："因为'>'和'>='在肉眼里太像了。让人审，是赌。让测试抓，是算。"

---

命令运行了大约十分钟。

最后，屏幕上显示：

```
[MUTATION TEST COMPLETE]
总变异数：10000
被检测到的变异：9847
未被检测到的变异：153
变异检测率：98.47%

警告：以下153个变异未被检测到，可能存在测试盲区：
- 第23行：将常量 "0.95" 改为 "0.94"（影响阈值判断）
- 第67行：将循环边界 "< n" 改为 "<= n"（潜在越界）
- 第112行：交换条件判断顺序（短路求值变化）
...
```

"盲区。"沈知行说，"这153处一旦出错，我们的测试抓不住。"

"所以？"

"补测试。补到它抓得住。"

他输入了另一个命令：

```
asto-suggest-tests --blind-spots=153
```

系统开始输出建议：

```
建议新增测试用例：
1. 测试阈值边界：输入接近0.95的值，验证系统行为
2. 测试循环边界：输入n+1个元素，验证是否越界
3. 测试条件顺序：验证短路求值不影响结果
...
```

"这是**测试驱动的安全**。"沈知行说，"盲区在哪，就补到没有盲区。"

---

老吴看着屏幕，沉默了很久。

"但这只是测试。"他终于开口，"测试只能证明代码在测试条件下是对的。真实世界比测试复杂得多。"

"您说得对。"沈知行点头，"所以我们不只做变异测试。"

他打开另一个文件：

```
[代码验证层级]
Layer 1：单元测试（功能正确性）
Layer 2：变异测试（测试覆盖率）
Layer 3：形式化验证（数学证明）
Layer 4：运行时监控（实际行为）
Layer 5：自动回滚（失败恢复）
```

"五层。"沈知行说，"测试、变异、形式化验证、运行时监控、自动回滚。"

"这比人审代码……"

"**可靠得多**。"沈知行说，"人会累会走神，数学不会。"

老吴看着那五层防护，手心发凉：不是靠人更认真，是把失败率压到接近零。

"但这样做的成本……"

"比雇两个审查者便宜。"沈知行说，"也比系统崩溃后修复便宜。更重要的是——比相信人类便宜。"

---

老吴转过身，看着窗外的城市。

远处的灯火通明。那是十万人的家。那些灯光背后，是他们刚才讨论的那段代码。三百行。没有人审过。

"我在旧世界干了三十年。"老吴说，"流程一堆。审查、评估、认证——都做了。"

他看向沈知行："系统还是崩。一次又一次。"

沈知行没有说话。

"我们也审过。"老吴说，"也签过字。然后它照样在生产上出事。"

"因为你们**信任人类**。"沈知行说。

老吴沉默了。

"旧世界的安全，是一层层'我们相信某个人会做好'。"沈知行说，"可人会犯错——不是坏，是有限。"

他指了指测试结果："所以我们不把安全押在信任上。押在**数学**上。"

---

老吴看着那三百行代码，突然觉得那张签字纸很荒唐。

签字不是安全，是追责的坐标。

"我不需要您签字。"沈知行说，"我需要的是**测试覆盖率100%**和**变异检测率100%**。当这两个数字达到的时候，代码就可以上线。不管是谁写的，不管有没有人审过。"

老吴放下那份打印纸。

"这是一种新的世界观。"他说。

"这是一种不再自欺的世界观。"沈知行说，"安全不是签字和承诺，是数学上的保证。"

他看着老吴："您愿意加入一个不信任人类的系统吗？"

老吴想了很久。

然后他笑了。

"不信任人类，"他说，"恰恰是对人类最大的尊重。"

"因为我们终于承认了——我们是有限的。"

---

> **[章后注]**
>
> 变异测试（Mutation Testing）最早由Richard Lipton在1971年提出，但直到ASTO时代才被大规模应用。
>
> 旧世界的软件工程依赖"代码审查"来保证质量，本质上是一种**社会契约**：我们相信审查者会认真审查，相信签字的人会负责任。但社会契约无法对抗人类的认知局限。
>
> ASTO的解决方案是**用算法替代信任**。不是让人来判断代码对不对，而是让系统自动生成成千上万种错误，然后验证每一种错误都能被发现。
>
> 这种方法有一个哲学前提：**不信任人类**。
>
> 听起来很冷酷。但正是因为不信任人类，才能设计出真正保护人类的系统。
>
> *——《ASTO工程手册》第三章：验证与信任*
