# 第02章：功能树

---

**公元2055年·废墟·Day 47**

沈知行站在一块巨大的白板前。

白板是从一个倒塌的学校里搬来的，表面有几道裂痕，但还能用。上面密密麻麻写满了字——不是文字，是一棵树。

一棵倒过来的树。

根在最上面，写着一行字：

```
[ROOT] 重建城市供水系统
```

往下延伸，是三个主分支：

```
├─ [1.0] 水源获取
├─ [2.0] 水处理
└─ [3.0] 配送网络
```

每个主分支又分裂成更多的子分支。子分支下面还有子分支。分支越来越细，越来越具体，直到末端变成了一个个小方块。

每个小方块里都是一个可以独立完成的任务。

林维数了数："多少个？"

"3847个。"沈知行说。

"三千八百四十七个任务？"罗南瞪大了眼睛，"就为了供水？"

"准确地说，是3847个**原子任务**。"沈知行用手指沿着树干向下滑动，"每个任务都足够小、足够独立：一个人一天做完，也不必等别人。"

---

**一小时前**

会议室——如果一个用帆布搭起来的棚子可以叫会议室的话——里坐着十七个人。

这是过去一个月里加入他们的幸存者。有工程师，有教师，有厨师，有学生。有人带来了技术，有人带来了体力，有人什么都没有，只带来了饥饿和恐惧。

"我们需要重建供水系统。"林维说，"整个城市的。"

沉默。

一个年轻人举起手："怎么做？"

在旧世界，这个问题会拖出一串会议和审批，最后变成一份没人看的"可行性报告"。

但沈知行的回答不一样。

他打开平板电脑，投影到帆布墙上：

```
[PROJECT DEFINITION]
项目名称：城市供水系统重建
最终目标：为10万人口提供稳定的饮用水供应
验收标准：
  - 覆盖率 ≥ 95%
  - 水质符合WHO标准
  - 日均供水量 ≥ 200升/人
  - 系统可用性 ≥ 99%
时限：365天
```

"这是目标。"他说，"接下来我们要做的，是把这个目标**分解**。"

---

"什么叫分解？"有人问。

沈知行没解释。他画了个大圆，写上"供水系统"："需要什么？"

人群开始报：水源、水处理、管道、泵站、储水。

每报一个，他就拉出一条分支。然后继续往下追问：水源怎么来？取水要什么？电力在哪？直到每条线都细到半天能做完。

三个小时后，帆布墙上挂着一棵倒立的树。

"功能树。"沈知行说，"叶子就是任务。"

---

罗南盯着那棵树，表情复杂。

"3847个任务。"他重复了一遍，"我们只有17个人。"

沈知行点头："摊到365天，每人每天不到一件。前提是任务足够小。"

"听起来不多。"

"不多，但不能大。"

他放大了树的一个末端：

```
[TASK 2.3.4.7]
任务描述：测试3号泵站的出水压力
输入：3号泵站已安装完成
输出：压力测试报告
所需资源：压力表×1，记录本×1
预估时间：2小时
依赖任务：2.3.4.6（3号泵站安装）
被依赖任务：2.3.4.8（3号泵站验收）
```

"这就是一个**原子任务**。"他说，"半天能做完，写得足够清楚：输入、输出、资源、依赖都在这里。前置任务一过，就能立刻开始。"

林维明白了："所以整个项目被切成了3847个这样的小块。"

"对。"沈知行说，"谁都能直接去领一片叶子：没人占的就拿，做完就交。"

"这……"一个老工程师犹豫了一下，"这不会乱吗？"

"不会。"沈知行指着树上的连线，"因为每个任务的**依赖关系**是明确的。你不能在安装泵站之前测试泵站，系统不会让你领取那个任务。但你可以在等待泵站安装的时候，去做其他不依赖泵站的任务。"

"并行。"老工程师说。

"对。**最大化并行**。"沈知行说，"旧世界慢，不是任务多，是大家都在等。我们先把等待砍掉。"

---

系统"上线"。

所谓上线，就是一份共享文档：3847个任务，三种状态——未开始、进行中、已完成。

谁都能认领，做完就提交，状态跟着走。

第一个被认领的任务是：

```
[TASK 1.1.1.1]
任务描述：勘探A区地下水位
输入：无
输出：地下水位测量报告
所需资源：简易测水杆×1，卷尺×1
预估时间：4小时
依赖任务：无
被依赖任务：1.1.1.2（A区水质初步评估）
```

认领者是一个叫小刘的年轻人。他之前是个外卖员，对水利工程一无所知。

但他能看懂任务描述。他知道需要什么工具，需要做什么，需要输出什么。

四个小时后，他提交了一份手写的报告：A区地下水位深度约12米。

报告被验收。任务状态变成"已完成"。

下一个任务——"A区水质初步评估"——自动变成了"可认领"状态。

---

一周后，进度条显示：已完成任务147个，占总数的3.8%。

听起来不多。但一周前，这个数字是0。

更重要的是，没人"管理"进度，也没人每天开会决定"今天做什么"。

每个人早上起来，打开文档，认领一个自己能做的任务，做完，提交。就这么简单。

有人一天做三个，有人三天做一个。有人挑简单的，有人啃困难的。

都行——系统只认结果：做完就是做完。不用汇报，不用解释，不用等谁点头。

"这很奇怪。"林维有一天晚上对沈知行说，"我……没有人在领导，但事情在推进。"

"因为顺序写在依赖里。"沈知行说，"你不用等人指挥，看一眼就知道该做什么。"

"那偷懒呢？"

"他可以偷懒。"沈知行说，"只不过进度条不会替他撒谎。"

---

**两个月后**

进度条：已完成任务1203个，占总数的31.3%。

第一条临时管道铺设完成。

干净的水第一次从管道里流出来的时候，所有人都沉默了。

那不是什么壮观的场景。只是一根生锈的铁管，插在一个用油桶改装的临时水箱上，流出细细的水柱。

但那不是奇迹，是系统：3847个任务里，有1203个被做完了，它就自己流出来。

罗南接了一杯水，喝了一口。

"甜的。"他说。

所有人都笑了。
