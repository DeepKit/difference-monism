# 《断骨之伞》写作指南（优化修正版）

**核心任务（修正）**
构建一个**值得生活、也经得起拒绝**的黄金时代；
它可以高效运行，但**不要求所有人持续配合**。

> 注：这是文明的目标，不是系统的 KPI。

---

## 0. 术语澄清（防混淆声明）

*   **2.0 / 3.0**
    指**社会—技术治理范式**的版本差异，而非小说分卷或章节序号。
*   **“稳态”**
    不等于冻结、均衡或最优解，
    而是一个**可持续运行、允许失败、具备纠错与退出机制的日常工程状态**。

> 写作中避免把“稳态”当作完美状态，它更接近“长期可修”。

---

## 1. 基调与美学 (Tone & Aesthetics)

### 1.1 从“冰蓝”到“暖白”，但阴影必须存在

*   **2.0 视觉语汇**：
    冰风暴、蓝光屏幕、机柜、红色警示（应急 / 对抗 / 非日常）
*   **3.0 视觉语汇**：
    **自然光、木纹、琥珀晶体、暖白界面（修复 / 日常 / 可停留）**

**描写原则（收敛版）：**

*   **多写**：
    光的变化、材料的老化、身体的触感、植物的节律、城市的呼吸
*   **少写**：
    炫技式 UI、泛科技感口号、无具体对象的“先进”

**关键约束（补丁）：**

> 暖白不是消除痛苦，
> 而是**让痛苦被看见、被照料、但不被抹平**。
> （参见 [ASTO.Topic: 痛苦私有性](../../papers/ASTO.Topic.Pain_Privatization.md)）

必须写出阴影：

*   谁因优化而更舒适
*   谁因此被忽略、被延后、被默认“不重要”

### 1.2 平静技术 (Calm Technology)：像氧气，但必须“可拒绝”

**总原则：**
技术应当降低认知负担，但**不能替人承担意志**。

**规范（精炼版）：**

1.  **界面退后**：信息嵌入环境，但不制造“你必须注意到我”的压力
2.  **默认省力**：自动化为默认，但**手动永远存在**
3.  **不读心**：预测必须征询，生理信号不能直接等价为指令
4.  **可退出**：任何自动化都必须
    *   可关闭
    *   可查看
    *   可追责

**示例**：
*   **正确**：“他觉得冷，椅背先亮了一下微光，像在问‘要不要？’他点了点头，热才慢慢上来。”
*   **错误**：“他一冷椅子立刻发热。”（读者潜台词：那我想要的/不想要的，谁说了算？）

## 2. 核心词汇表 (Vocabulary)

**使用原则（新增）：**

> 官方语言越“干净”，人物私语越可以保留旧词与情绪词。
> 这是制度与生活的自然张力，不是世界观漏洞。

| 旧词汇 | 3.0 专用词汇 | 备注 |
| :--- | :--- | :--- |
| **AI / 人工智能** | **伴侣系统 (Companion) / 辅助体 (Auxiliary)** | 二选一为主词。别两套并行太久。 |
| **钱 / 货币** | **贡献值 (Credits)** | 注：需体现贡献计算、申诉、防垄断机制。 |
| **公司 / 部门** | **协作组 (Guild) / 节点 (Node)** | 强调液态、平等的组织关系。 |
| **老板 / 经理** | **协调人 (Coordinator)** | 协调人也受规则约束，可被罢免。 |
| **警察 / 法官** | **裁决者 (Arbiter)** | 裁决流程需公开、留痕、可复核。 |
| **穷人 / 底层** | **低能量期 / 需要补给的人** | 避免“待充能者”这种电池化标签。 |
| **死刑** | **强制离线 (Forced Offline)** | 极罕见。需明确罪名边界、审理与救济程序。 |

**总注补丁：**

*   所有 3.0 词汇，**都必须在剧情中至少出现一次“被质疑 / 被挑战 / 被修正”的场景**
*   否则该词汇会自动变成口号

## 3. 叙事策略 (Narrative Strategy)

### 3.1 “点针式”技术植入

> 技术出现的密度，应当与人物的生活密度一致；
> 如果技术比生活更抢镜，说明写多了。

不要大段解释原理（那是论文的事）。把技术原理磨碎，撒进日常行为里。
*   **错误**：“Q代计算机利用量子纠缠原理进行超光速计算……”
*   **正确**：“许冬看着那块琥珀色的晶体，里面的光流像呼吸一样忽明忽暗。他知道，那每一次闪烁，都是系统在几亿个平行宇宙里做了一次选择。”

### 3.2 冲突不是“你死我活”，但必须付出代价

**稳态冲突三源（微调措辞）：**

1.  **灰度冲突**
    规则是否允许例外？
    例外的代价由谁承担？是否可被回溯？
2.  **公平冲突**
    算法对谁不友好？
    修正之后，谁会失去原本的优势？
3.  **隐私冲突**
    照护与预测的边界在哪里？
    何时“善意”开始变成默认监护？

> 稳态的张力，来自**没有明显坏人，但每个选择都有人受影响**。

### 3.3 微观视角与反证机制

**宗旨**：宏大的系统，是为了守护微小的瞬间。

**操作约束**：
> 每一个“被照护的瞬间”，
> 都必须在同章或相邻章节中，
> 显示一次**系统的边界、迟疑或成本**。

## 4. 人物状态 (Character State)

### 4.1 许冬（建设者）

*   **状态**：温和，但不是没脾气；稳，但有疲惫的夜晚。
*   **台词风格**：“我们修的是路，不是人。”
*   **底线**：他清楚自己设计的不是“最好的人生”，而是“允许更多人不被系统决定的人生”。

### 4.2 沈知行（守护者）

*   **状态**：隐退的智者。
*   **防神化**：他不是“系统之外的人”，而是**愿意在系统失败时，仍然承担责任的人**。这份责任不是权力，因此可以、也必须被质疑。

### 4.3 普通人（受益者）

*   **状态**：**从容**。
*   **定义**：
    > 从容，来自“知道自己不是流程的一部分”。

## 6. 风险控制与去神化 (Risk Control)

> **核心原则**：如果 ASTO 消失了，这个故事依然必须成立。
> **如果只有系统在推动剧情，那就是失败。**

### 6.1 压制系统的“神性”
*   **禁忌**：禁止出现“一行代码秒杀反派”、“一个按钮瞬间拯救世界”。
*   **要求**：多写**延迟**、**误伤**、**无效操作**。
*   **示例**：
    *   *错误*：沈知行输入代码，系统立刻停止。
    *   *正确*：沈知行输入代码，系统卡顿了三秒，风扇狂转，报错两次，最后才勉强停下。

### 6.2 悲剧密度稀释
*   不要章章见血。读者会疲劳。
*   每隔 3-4 个单元，必须插入一个**“无事发生”**的章节。
*   让系统看起来“暂时是对的”，甚至“有点无聊”。这种无聊是暴风雨前的宁静。

### 6.3 角色去神化（低级时刻）
*   许冬和沈知行不能永远是“智者”和“圣徒”。
*   **必须给他们安排低级的情绪**：
    *   **嫉妒**：“凭什么他能休息，我要加班？”
    *   **犹豫**：“如果不按这个按钮，我也许能升职。”
    *   **吵架**：为了琐事（比如谁喝了最后一杯咖啡）而发火，而不是为了哲学。

### 6.4 国际视野：抄错作业
*   外部世界不是只会等待救援的难民。
*   他们会模仿 ASTO，但因为缺乏核心伦理（Phil），他们会**抄错作业**。
*   **情节植入**：某国引进了 ASTO 的“贡献值系统”，但去掉了“申诉机制”，结果导致了新型奴隶制。

### 6.5 终章的微裂缝
*   严禁“道德完美闭环”。
*   在最光明的时刻，必须留下一道**无法解释的阴影**。
*   比如：孩子出生时，系统无故离线 3 秒。没人知道为什么，也没人能修复。这暗示了不可控的未来。
