# 《未尽之算》北美版翻译指南 v2.2

## 1. 核心战略：保留骨架，转译叙事 (Chinese Skeleton + Localization)
**原则**：不替换背景，不西化人物。保留中国特有的技术官僚体系和宋代历史镜像，通过“文化桥梁”让北美读者产生共鸣。

## 2. 战略性本土化处理
| 元素 | 处理方式 (英文正文) | 目的 |
| :--- | :--- | :--- |
| **房贷压力** | 保留数字 + 解释：*"2.8 million yuan—roughly $400,000, two decades of an average salary"* | 建立压迫感的直观体感 |
| **社会救助** | 译为 *welfare algorithm* 或 *public health ledger* | 强调系统性与非人性 |
| **仵作 (Wuzuo)** | 译为 *wuzuo (coroner)*；类比：*"11th-century CSI"* | 建立职业认知的文化桥梁 |
| **开封府** | 译为 *Kaifeng Prefecture*；解释：*"the Song Dynasty's data center"* | 解释其官僚/计算职能 |
| **算盘 (Suanpan)** | 译为 *suanpan (abacus)*；解释：*"ancient calculator of merchants"* | 强化“前现代工具对抗算法”的隐喻 |
| **道德熔断** | 译为 *Moral Circuit Breaker* | 对应北美“制动/熔断”概念 |
| **信用分** | 译为 *Credit Score* | 严禁翻译为 reputation 或 trust score |

## 3. v2.2 叙事深度与技术排版规范

#### 1) 禁止性翻译清单 (Strictly Prohibited)
- **禁止过度类比**：严禁将系统类比为 Amazon, Google, Meta 或 IRS, SSA。保持其独特的中式赛博孤立感。
- **去人格化**：`System` 严禁描写为 `sentient`（有感情的）、`evil`（邪恶的）或 `malicious`（有恶意的）。它应当是无差别的数学逻辑。

#### 2) 黑色幽默标准 (Black Humor Standard)
- **核心准则**：`Humor should sound tired, not clever.` (幽默感应听起来疲惫，而非机灵。)
- **来源**：来源于底层职业比喻、机械冷漠与人类习惯的碰撞。禁止俏皮话（witty banter）。

#### 3) 系统日志视觉约束 (Visual Constraints for System Logs)
- **排版要求**：使用全大写（ALL CAPS）或等宽字体感（如 ` ` ）。
- **语调**：永远比正文更冷、更短、更机械。禁止使用修辞。

#### 4) “未尽之算”并置策略 (Title Strategy)
- **语言断裂感**：在灵魂冲突章节，允许 `Wei Jin Zhi Suan — The Unfinished Calculation` 并置。使其从词汇演变为压迫性符号。

## 4. 格式规范
- **[SYSTEM_LOG]** / **【系统指令】**: 保持纯中文。内容翻译仅出现在英文正文的引用中。
- **[SYSTEM_NOTE]**: 英文，用于向读者解释特定的文化背景。
- **文件名与目录**: 必须保持原始中文。
