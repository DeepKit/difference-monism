# 第16章：投票与界面（反助推：UI 中立性规范）

> 正文草稿 v0.1：把“反助推”写成一种对人性的信任；正当性不是赠品，是必须支付的耐心。

反对按钮被做成灰色。

灰得像天要下雨。

产品经理很不爽。

他把 A/B 测试的数据拍在桌上。

“你们知道灰色会降低转化吗？”他问。

“点击率降了 40%。这意味着有 40% 的人流失了。”

姬可正在调试一台旧机器。

她没有抬头。

“知道。”她说。

“那为什么还要灰？”产品经理声音高了八度，“为什么不把‘支持’做大一点？为什么不加个倒计时？为什么不把‘反对’藏到二级菜单里？”

姬可终于停下动作。

她转过身，看着产品经理。

“因为绿色会偷走选择。”她说。

“你把‘支持’做成绿色，做成大按钮，加上烟花动画，人就会下意识地点。那不是他们的选择，那是你的选择。”

产品经理冷笑：“我的选择就是让他们快点过。这也是为了他们好。”

姬可没说话。

她只是指了指窗外。

窗外是投票窗口，人排得很长。

队伍里有一个老人。

老人站在机器前，眉头皱得很紧。

他戴着老花镜，还是看不清。

屏幕上的字是中性的黑白，没有诱导色，没有倒计时。

“我想反对。”老人说。

工作人员指了指屏幕角落：“在这里。”

老人眯着眼，手指在屏幕上戳了两下。

戳错了。

差点点到旁边的“弃权”。

他的手停在半空。

停得像在悬崖边。

他不知道该怎么选，因为没有绿色告诉他“点这里是对的”。

协议翻译员蹲下去。

她没有替他点。

也没有说“大爷，点这个绿色的就完了”。

她把屏幕上的文字一字一句读给他听。

“选项一：同意。后果：你的社区将改建，你会获得补偿。”

“选项二：反对。后果：社区保留，但你要承担维修费。”

她读得很慢。

慢得像在读一首诗。

老人听完，想了一会儿。

“我要保留。”他说。

“好，那就是反对。”翻译员说，“在这个灰色的框里。”

老人点头。

他的指尖按下去。

灰色按钮没有任何庆祝动画。

没有烟花。

没有“恭喜你”。

只有一个很小的、朴素的提示：

“已记录。永不删除。”

老人抬头，问：“它真的会被看见吗？那个绿色的看着更厉害。”

翻译员笑了笑。

“绿色的只是亮。”她说，“灰色的才是你的。”

“它会被保留。”

老人松了一口气。

他走出队伍时，背好像比刚才直了一点。

晚上，A/B 测试审计报告出来。

报告上写着：

“实验组（中性 UI）：转化率下降 42%。”

“对照组（助推 UI）：转化率提升 15%。”

产品经理指着那行红色的下降数据：“看，这就是代价。”

姬可把那份报告拿过来。

她在“代价”两个字旁边写了一行备注：

“难用是成本。”

“但难用让人知道：选择不是赠品。”

她把报告归档。

归档的文件夹叫《反助推审计》。

里面全是这种“失败”的数据。

每一条下降的曲线，都是一次对“下意识”的拒绝。

因为文明不是为了让人更顺滑地被统治。

文明是为了让人在哪怕最麻烦的时候，也能停下来，想一想：

我是不是真的同意？
