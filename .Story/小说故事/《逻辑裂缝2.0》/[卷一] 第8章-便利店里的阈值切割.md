# 第8章 便利店里的阈值切割

**2027年8月19日，中午12点，全家福便利店。**

午高峰。便利店里挤满了来买盒饭的白领。
小雪站在收银台前，手里拿着扫描枪，“滴滴”声不绝于耳。
她的脑子里，另一个“滴滴”声也在响。那是Titan的人力资源管理系统在倒计时。
**“当前时段客流量激增，建议结账速度：15秒/人。当前平均速度：18秒/人。请加快速度，否则将扣除绩效分。”**

小雪深吸了一口气。
15秒。
这意味着她不能和顾客有任何多余的交流，不能笑，甚至不能眨眼。
她必须像一台机器一样运转。

“我要这个，还要那个关东煮，两个萝卜一个丸子，多加点汤。”一个戴着工牌的女白领指着关东煮的锅子。
小雪手脚麻利地夹起萝卜和丸子，装进杯子里，舀了一勺汤。
“还要多加点汤！这汤怎么这么少？”女白领不满意了。

“不好意思，杯子就这么大，再加就洒了。”小雪解释道。
“我就要多加点！你这人怎么这么死板？”女白领提高了声音，后面排队的人开始探头探脑。

小雪看了一眼屏幕上的倒计时。
**14秒……15秒……16秒……**
红色的警告灯亮了。

如果是以前，小雪会慌，会道歉，会手忙脚乱地再舀一勺汤，然后洒在桌子上，然后更慌。
但今天，她想起了陈易凡说的**“阈值切割”**。
那是她昨晚在网上查到的ASTO理论。
**“当系统的输入超过处理阈值时，必须果断切割低价值的冗余请求，以保护核心进程。”**

在这个场景里：
核心进程 = 维持结账队伍的流动。
冗余请求 = 女白领无理取闹的“多加汤”。
阈值 = 15秒。

现在已经超标了。
必须切割。

小雪没有再解释。她甚至没有再看女白领一眼。
她直接盖上关东煮的盖子，扫码，报数：
“一共25块8。微信还是支付宝？”

女白领愣住了。她没想到这个平时唯唯诺诺的小店员今天这么硬气。
“你……”
“后面还有很多人在排队。”小雪指了指长长的队伍，“如果您对服务不满意，可以在APP上投诉我。工号9527。下一位。”

女白领张了张嘴，最后什么也没说，愤愤地刷了码，拿着杯子走了。

小雪的心脏跳得很快。
她以前最怕被投诉。投诉一次扣50块，相当于白干半天。
但今天，她突然觉得没那么怕了。
因为她掌握了一种**武器**。
一种叫做“逻辑”的武器。

她用这把刀，切掉了那个女白领的无理取闹，也切掉了自己心里的恐惧。
虽然Titan的系统依然在扣她的分（因为刚才那一单超时了）。
但在她自己的系统里，她觉得她赢了。

她看了一眼门外。
阳光很刺眼。
也许，哪怕是在这个被算法精确控制的便利店里，也能找到一点点属于自己的“暂存区”吧。

（第8章 完）
