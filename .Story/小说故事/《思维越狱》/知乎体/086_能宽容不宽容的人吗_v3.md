# 086_能宽容不宽容的人吗

> **本章进度**：卷八·人群 (Ep.086/109)
> **核心痛点**：网络暴力 / 无限宽容悖论
> **CS隐喻**：防御性编程 (Defensive Programming) / SQL 注入防护 / 安全边界

## 1. Hook：被杠精围攻
“气死我了！真是气死我了！”

苏心把手机狠狠摔在沙发上，眼圈通红。

“我只是发了个帖子，说‘我觉得这部电影挺好看的’。结果那个人追着我骂了三天！他说我是‘文盲’、‘洗地狗’、‘收了钱的营销号’。我拉黑了一个号，他又换个小号来骂，还把我的照片挂到其他论坛去网暴！”

林哲看了看她的手机屏幕：“这人明显就是个职业杠精，现实生活不如意来网上发泄的。你别理他不就行了？”

“我本来也是这么想的。”苏心委屈得直掉眼泪，“我想着做人要大度，要宽容，不要跟这种烂人计较。结果我的忍让反而让他更嚣张了！现在连我的朋友都被他骂了。我是不是太软弱了？是不是好人就该被枪指着？”

## 2. Trap：宽容悖论
“你这不是软弱，是逻辑错误。”

叶清捡起苏心的手机，冷静地翻看那些恶毒的评论。

“你在试图对一个**破坏规则的人**使用**遵守规则的策略**。这在哲学上叫**宽容悖论**（Paradox of Tolerance）。”

“卡尔·波普尔认为：如果一个社会对‘不宽容’（Intolerance）也无限宽容，那么‘不宽容’最终会消灭‘宽容’，并摧毁这个社会。”

“简单说：如果你允许流氓在你的花园里随地大小便，因为你要‘宽容’，那么最后你的花园会变成粪坑，而你会失去花园。”

叶清看着苏心：“你所谓的‘大度’，是在给病毒提供温床。”

## 3. Turn：防御性编程与 SQL 注入
“在编程领域，我们从不讲‘大度’，我们讲**防御性编程**（Defensive Programming）。”

叶清打开电脑，指着一段代码。

“一个好的系统，确实应该允许用户输入各种各样的数据（多样性）。但是，如果用户输入的是一段**恶意代码**，比如 SQL 注入指令 `DROP TABLE Users`（删除用户表），系统该怎么办？”

“难道系统要‘宽容’地执行这段代码，然后把自己删了吗？”

“当然不！”叶清敲击键盘，“系统必须立刻**拦截**、**报错**，甚至直接**封禁**这个 IP。”

“那个杠精，就是网络社会里的 SQL 注入攻击。他不是来交流观点的，他是来**破坏交流环境**的。他利用你的‘宽容’（输入框没有过滤机制），试图摧毁你的心理防线（数据库）。”

“对付这种人，任何解释、讲理、忍让，都是在允许恶意代码运行。唯一的解法是：**防火墙拦截**（Block）。”

## 4. Action：拉黑是正义的
“可是……拉黑他显得我很心虚啊。”苏心还是有点纠结，“显得我骂不过他一样。”

“这是什么奇怪的逻辑？”叶清皱眉，“你看到路边有条疯狗在叫，你会为了证明你不心虚，趴在地上跟它对叫吗？”

叶清拿过苏心的手机，操作了几下。

1.  **截图取证**（保留日志 Log）。
2.  **点击举报**（发送 Abuse Report）。
3.  **全平台拉黑**（加入 Blacklist）。
4.  **开启评论权限设置**（开启防火墙，仅允许好友评论）。

“好了，世界清静了。”叶清把手机还给苏心。

“记住，你的社交媒体是你的**私有服务器**。你有权决定谁能访问，谁不能。拉黑不是心虚，是**系统维护**。清除垃圾数据，是为了让正常的交流（有效数据）能跑得更流畅。”

“不要对试图毁灭你的人讲道德。那是上帝的事。你的任务是送他去见上帝——或者至少送他进小黑屋。”

## 5. Outro：有牙齿的善良
看着干干净净的评论区，苏心长舒了一口气。

“你说得对。我以前总觉得‘好人’就该是温顺的绵羊。但现在看来，没有牙齿的善良，就是软弱。”

“善良必须带有锋芒。”叶清补充道，“宽容是有边界的。这个边界就是：**我宽容一切，除了‘不宽容’本身**。”

“下次再遇到这种 `SQL 注入`，”叶清冷冷地说，“别废话，直接回车执行 `Block`。那是你作为系统管理员的最高权力。”

林哲在一旁鼓掌：“说得好！以后谁敢杠我，我就当他是病毒，直接杀毒软件伺候！”

“但前提是，”叶清瞥了林哲一眼，“你自己别当那个病毒。”
