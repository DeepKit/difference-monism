# 公司规定“所有不穿工服的人都得扣钱”，那制定规则的人穿不穿？

> **本章进度**：卷四·话术 (Ep.038/109)
> **核心痛点**：职场双标 / 霸王条款 / 规则漏洞
> **CS隐喻**：权限组 (Permission Group) / Root 权限 / 规则集冲突 (Rule Conflict)

---

## 1. 炸毛时刻 (The Hook)

周二上午 10:00。
林哲在兼职的公司（一家教育机构）前台，气得满脸通红。

“凭什么扣我钱？”林哲指着考勤表。

前台小妹一脸无辜：“林老师，公司规定，上班时间必须穿工服。你昨天没穿，所以扣 200。”

“可我不算正式员工啊！我是兼职讲师！”林哲据理力争，“而且，昨天是周末，我是来补课的，当时老板也在，他也没穿工服啊！为什么他不扣钱？”

前台小妹翻了个白眼：“老板是制定规则的人，你是执行规则的人。这能一样吗？”

林哲气结：“这不公平！如果‘所有人’都得穿，那‘所有人’就包括老板。如果老板不穿，那这个‘所有人’的定义就是假的。你们这是**双标**！”

“林老师，您别跟我杠。”前台小妹把考勤表一摔，“反正规定就是这样，爱干不干。”

林哲陷入了**“规则逻辑死循环”**：
他试图用逻辑去挑战权力的边界，却发现权力根本不讲逻辑。

## 2. 鬼打墙 (The Trap)

回到公寓，林哲还在愤愤不平。

“苏心，这在集合论上，叫——**理发师悖论 (Barber Paradox)**。”

“罗素说：村里有个理发师，他立下规矩：‘我只给村里所有**不给自己刮脸**的人刮脸。’
那么问题来了：理发师给不给自己刮脸？
如果他给自己刮，那他就是‘给自己刮脸的人’，根据规矩，他不该给自己刮。
如果他不给自己刮，那他就是‘不给自己刮脸的人’，根据规矩，他必须给自己刮。”

林哲指着前台小妹的话：“老板规定‘所有人都要穿工服’。老板也是‘人’，所以老板也在这个集合里。但他不穿。这说明‘所有人’这个集合出现了悖论。这个规则是**自相矛盾**的！”

苏心听得头大：“林哲，你跟资本家讲集合论？你是不是读书读傻了？”

## 3. 降维打击 (The Turn)

**“不是悖论。是权限组不同。”**

叶清的声音从沙发上传来。她正在配置服务器的防火墙规则。

“什么权限？”林哲问。

叶清转过身，看了一眼林哲那件被扣了 200 块的衬衫。

“林哲，你试图把管理员 (`Admin`) 强行拉进用户组 (`User Group`)，系统当然会报错。”

叶清打开电脑，调出一个权限管理界面：
“在 Linux 系统里，有一个超级用户叫 **Root**。Root 用户拥有一切权限，包括制定规则、修改规则、甚至删除系统本身。”

“老板就是 Root。你就是普通 User。”

“规则文件 `/etc/rules.conf` 里写着：`User must wear uniform`。
但这行规则是对 `User` 组生效的。Root 用户不受此限制。”

叶清轻声道：
“你觉得‘双标’，是因为你以为大家都在同一个 `Flat Hierarchy`（扁平层级）里。
但在工程上，系统是**分层**的。
**规则制定者 (Rule Maker) 永远在规则执行者 (Rule Executor) 的上层。**”

“你试图用 User 的逻辑去约束 Root 的行为，这叫 **越权访问 (Privilege Escalation)**。系统不仅会拒绝，还会记录你的违规操作（扣钱）。”

“理发师悖论之所以无解，是因为罗素把理发师也放进了‘村民’的集合里。但在现实中，理发师是**规则的上帝**。他想给自己刮就刮，不想刮就不刮。规矩是写给村民看的。”

## 4. 物理行动 (The Action)

林哲愣住了。
“所以我只能忍着？被他白扣钱？”

“如果你想继续在这个系统里运行，是的。”叶清点头，“除非……”

“除非什么？”

“除非你**切换系统**，或者**获取 Root 权限**。”

叶清合上电脑：
“林哲，你是个兼职讲师。你的核心竞争力是**讲课**，不是**穿工服**。”
“如果这个系统的规则 (`Rule Set`) 严重影响了你的核心进程（扣钱导致心情不好，讲课质量下降），那么这个系统对你来说就是**不兼容 (Incompatible)** 的。”

“与其在那儿跟前台小妹辩论集合论，不如做个 **Cost-Benefit Analysis (成本收益分析)**。”

“为了这 200 块，你浪费了半天时间生气，还可能影响明天的课。值得吗？”

“不值得。”林哲摇头。

“那就执行 **Exit()** 操作。”叶清说，“辞职。换一家不双标的机构。或者自己干（成为 Root）。”

林哲想了想。
他看着手里那张扣款单。
确实，为了这点破事消耗情绪，太不划算了。

“好。”林哲站起来，“我不伺候了。我去辞职。”

“等等。”叶清叫住他，“辞职前，利用一下 **Sudo 权限**。”

“什么？”

“你不是有很多学生喜欢你吗？走之前，把你的微信号留给学生。告诉他们：‘老师要换个地方讲课了，想听的加我。’”

“这叫**流量迁移 (Traffic Migration)**。既然老板不遵守规则，那你也可以利用规则漏洞，把属于你的资源（学生）带走。”

林哲眼睛亮了。
“叶清，你真黑。”

“这不叫黑。”叶清笑了，“这叫**资源回收**。”

## 5. 尾声与引战 (The Outro)

周三。
林哲辞职了。走的时候，他带走了 50 个学生家长的微信。
老板气急败坏，指责他“不讲武德”。
林哲只回了一句：“我是兼职，合同里没写竞业协议。这叫**合规操作**。”

回到公寓，林哲心情大好，请大家吃烧烤。

“叶清，”林哲举杯，“我发现工程思维比哲学好用。哲学只会让我纠结，工程思维让我**止损**。”

叶清喝着啤酒。
“哲学负责解释世界，工程负责**改造世界**。”
“当规则不合理时，别试图修补规则。**重构系统**才是正解。”

她看了一眼那个**缺角的算盘**。
算盘的规则是死的。珠子只能上下拨动。
但人是活的。
如果你不喜欢这个算盘，那就**把算盘砸了**。
