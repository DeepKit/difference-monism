# 找工作要经验，有经验才能找工作，这Bug谁写的？室友：这是死锁，得暴力破解

> **本章进度**：卷四·话术 (Ep.045/109)
> **核心痛点**：就业歧视 / 应届生困境 / 规则死循环
> **CS隐喻**：死锁 (Deadlock) / 注入攻击 (Injection) / 开挂 (Cheat) / 循环等待

---

## 1. 炸毛时刻 (The Hook)

周三下午 2:00。
苏心正在帮林哲修改简历。
林哲（虽然辞了兼职，但还得找全职）看着招聘网站上的要求，脸都绿了。

职位：初级讲师。
要求：**应届毕业生，且有 3 年以上相关工作经验。**

“这是人话吗？”林哲气得拍桌子，“应届生怎么可能有 3 年经验？我有 3 年经验我就不是应届生了！这就像是‘想吃鸡必须先有蛋，想有蛋必须先有鸡’。这让我怎么投？”

苏心也无语：“可能他们想要的是……在娘胎里就开始实习的天才？”

不仅是这一条。几乎所有的好工作，都陷入了这种怪圈：
想进大厂 -> 需要大厂背书。
想有背书 -> 需要先进大厂。

林哲把鼠标一摔：“我不找了！这规则就是用来玩我的！这是一个**逻辑闭环**，我这种守规矩的人根本进不去！”

他陷入了**“资格悖论”**：
被要求拥有“只有通过被录取才能获得的资格”来获取被录取的资格。

## 2. 鬼打墙 (The Trap)

苏心叹了口气，虽然她也觉得离谱，但还得安慰林哲。

“林哲，这在逻辑学上，叫——**第22条军规 (Catch-22)**。”

“约瑟夫·海勒的小说里写道：疯子可以免于飞行任务，但必须由本人提出申请。但如果你能提出申请，说明你神智清醒（没疯），所以你必须去飞。”
“如果你疯了，你可以不飞，但你不知道申请。如果你没疯，你可以申请，但申请会被驳回。”

“总之，无论你疯没疯，你都得飞。这就是**第22条军规**——一种**不仅无法通过，而且会自我证成的逻辑陷阱**。”

林哲指着屏幕：“这个招聘要求就是第22条军规。它设立了一个**不可能满足的条件**，目的可能根本不是为了招人，而是为了——”

“为了什么？”

“为了证明‘我不招你是有理由的’。或者为了内部萝卜坑。”林哲绝望地说，“在这样的规则体系下，诚实的人注定是输家。”

## 3. 降维打击 (The Turn)

**“那就不诚实。注入攻击。”**

叶清的声音从门口传来。她手里拿着一瓶快乐水，刚从外面回来。

“什么攻击？”林哲问。

叶清走过来，看了一眼那个“应届生+3年经验”。

“林哲，在操作系统里，这叫 **Deadlock (死锁)**。”
“资源 A（工作）需要资源 B（经验），资源 B 需要资源 A。系统卡住了，进入了**循环等待 (Circular Wait)**。”

“在正规流程里，死锁是无解的。除非……”

叶清喝了一口快乐水：
“除非你引入一个 **External Intervention (外部干预)**，或者使用 **Cheat Code (作弊码)**。”

“在这个招聘里，HR 写的规则是一个 **Bug**。但这个 Bug 同时也暴露了一个**漏洞 (Vulnerability)**。”

“什么漏洞？”

“**Input Validation (输入验证) 的缺失。**”

叶清指着屏幕：
“系统要求‘3年经验’。它会去社保局查你的流水吗？不会。它会去你前公司做背调吗？初级岗位通常不会。”
“它唯一的验证方式，就是看你的 **Resume (简历)** —— 这是一段 **User Input (用户输入)**。”

“只要你的简历里写了‘3年经验’，你就通过了第一层 **Filter (过滤器)**。”

林哲惊呆了：“你是让我……造假？”

“不。造假是 `fraud`，会被封号。”叶清摇摇手指，“我让你做的是 **SQL Injection (SQL注入)**。”

“利用规则的模糊性，注入一段**精心构造的真实代码**，来绕过检查。”

叶清拿过林哲的简历：
“你虽然没全职工作过，但你兼职了 3 年，对吧？”
“把‘兼职讲师’改成‘签约讲师’（你是签了合同的）。”
“把‘帮导师带学生’改成‘负责教学团队管理’（你确实带了）。”
“把‘写论文’改成‘深度内容研发’。”

“看，”叶清敲下回车，“**3 年经验，Get。**”

“你没有撒谎。你只是把你的**非结构化数据 (Unstructured Data)**，按照对方的**接口规范 (Interface Spec)** 进行了**重构 (Refactoring)**。”

“对方的规则是流氓逻辑（第22条军规）。对付流氓逻辑，唯一的办法就是**利用逻辑漏洞，进行暴力破解 (Brute Force)**。”

## 4. 物理行动 (The Action)

林哲看着被叶清改过的简历。
依然是他的经历，但换了一套“互联网黑话”之后，瞬间变得高大上了。
甚至连他自己都觉得：卧槽，我原来这么牛逼？

“这……真的行吗？”林哲还有点虚，“面试的时候不会露馅吗？”

“面试是**运行时 (Runtime)**。”叶清说，“你现在要解决的是**编译时 (Compile Time)** 的报错。”
“先把简历投递进去，拿到面试机会。只要进了面试，那就是人与人的博弈，不再是死板的规则卡死。”

“而且，”叶清拍了拍他的肩膀，“有经验的人都在忙着干活，没空投简历。能投简历的，要么是想跳槽的（不稳定），要么是像你这样的应届生（便宜）。”
“HR 写那个要求，只是为了**过滤掉老实人**。”

“**别当那个被过滤掉的 `Exception`。当那个注入成功的 `Payload`。**”

林哲深吸一口气。
他想起了之前被前台小妹扣钱的经历。
老实人确实没糖吃。
在这个充满 Bug 的世界里，学会一点**黑客思维 (Hacker Mindset)**，是生存必备技能。

“好。”林哲点击“投递”，“注入开始。”

## 5. 尾声与引战 (The Outro)

周五。
林哲收到了面试通知。
HR 在电话里说：“我看你简历经验很丰富啊，虽然是应届生，但比很多社招的都强。”
林哲忍住笑，淡定地回：“是的，我一直很注重实战。”

挂了电话，林哲对叶清说：“我觉得，那个写 Bug 的 HR，可能也在等一个能破解 Bug 的人。”

叶清正在玩游戏，开了个“锁血挂”。
“**规则是给守规矩的人制定的。**”她淡淡地说，“对于黑客来说，规则只是**建议**。”

她看了一眼那个**缺角的算盘**。
算盘的边框限制了珠子的位移。
但如果把算盘拆了，珠子就能滚到任何地方。
**Break the Loop.**
