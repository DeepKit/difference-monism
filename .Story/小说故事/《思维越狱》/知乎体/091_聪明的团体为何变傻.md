# 091_聪明的团体为何变傻

> **本章进度**：卷八·人群 (Ep.091/109)
> **核心痛点**：群体决策失败 / 不敢反对
> **CS隐喻**：回音室效应 (Echo Chamber) / Error 屏蔽 / 红队 (Red Team)

## 1. Hook：一群博士的弱智方案
“完了，项目黄了。公司亏了五百万。”

林哲垂头丧气地回到家，把公文包扔在地上。

“怎么会？”苏心惊讶道，“那个项目不是你们组最顶尖的团队做的吗？三个博士，五个硕士，全是名校毕业。这么豪华的阵容，怎么会搞砸？”

“就是因为阵容太豪华了。”林哲叹气，“开会讨论方案的时候，其实我也觉得那个技术路线有点冒险。但是我看李博士在点头，王总监在叫好，我想人家是大神，肯定比我懂。于是我也跟着点头，还补充了几句赞美的话。”

“结果事后复盘才知道，李博士也觉得有问题，但他看王总监支持，不敢反驳。王总监其实也不懂，他是看李博士点头了，以为技术上可行。最后，我们一群高智商的人，像傻瓜一样互相点赞，通过了一个显而易见的**垃圾方案**。”

## 2. Trap：群体迷思
“精彩。”叶清坐在转椅上转了一圈，“这就是经典的**群体迷思**（Groupthink）。”

“当一个团体具有高凝聚力、且与外界隔离时，成员为了维护团体的和谐与一致性，会不自觉地压抑自己的反对意见，忽视潜在的风险。”

“你们不是在开会，你们是在搞**宗教崇拜**。你们的神不是‘真理’，而是‘和谐’。每个人都在自我审查，生怕自己的一句‘我有异议’会破坏这种虚假的团结氛围。”

叶清轻声道：“这就好比泰坦尼克号的船员们聚在一起，每个人都看到了冰山，但每个人都想：‘船长都没慌，我慌什么？’最后大家一起唱着歌撞了上去。”

## 3. Turn：回音室与错误处理
“在信息论里，这叫**回音室效应**（Echo Chamber）。”

叶清打开电脑，模拟了一个闭环网络。

“当一个系统屏蔽了所有的**负反馈**（Negative Feedback），只允许正反馈（点赞）流通时，信号会被不断放大，直到失真。”

“你们的会议室就是一个没有 `Error Handling`（错误处理）机制的系统。当有人提出一个 Bug（风险）时，正常的系统应该捕获它（Catch）、分析它、修复它。而你们的系统选择了**吞噬它**（Ignore），甚至把提出 Bug 的人当作 Bug 处理掉。”

“一个不允许报错的系统，注定是脆弱的。因为它把所有的纠错机会都浪费在了维持‘正常运行’的假象上。”

## 4. Action：引入红队
“那该怎么办？以后开会专门安排个人抬杠？”林哲问。

“没错。你需要引入**红队**（Red Team）。”

叶清解释道：“在网络安全演习中，红队负责扮演攻击者，专门找系统的漏洞，试图黑进系统。而蓝队负责防守。”

“如果不引入外部的红队，仅仅靠蓝队自己自查，是永远查不出大问题的。因为蓝队有思维盲区，而且蓝队不想让自己难堪。”

“下次开会，你们应该设立一个**魔鬼代言人**（Devil's Advocate）的角色。这个角色的任务就是：无论方案多完美，都要强行挑出三个毛病。如果不挑出来，就不许散会。”

“把‘找茬’变成一种**制度化的任务**，而不是个人的挑衅。这样大家才能放下心理包袱，畅所欲言。”

## 5. Outro：为了真理，破坏气氛
林哲若有所思：“所以，那个在会上让大家下不来台的人，其实是救世主？”

“对。他是那个打破回音室玻璃的人。”

叶清拍了拍林哲的肩膀：“下次再遇到那种‘一团和气’的会议，记得做那个讨厌的红队。哪怕被大家翻白眼，至少你保住了那五百万。”

“或者，”叶清补充道，“你也可以匿名发一封邮件。毕竟，在这个充满了虚荣心的系统里，**匿名**有时候是唯一的真话通道。”

林哲握紧了拳头：“好，下次我就当那个魔鬼代言人！为了公司的五百万！”

苏心在旁边泼冷水：“得了吧，你连在群里说‘我不去爬山’都不敢，还敢当面怼博士？”

林哲脸红了：“那……那我就负责给那个怼博士的人鼓掌！这也是一种贡献！”
