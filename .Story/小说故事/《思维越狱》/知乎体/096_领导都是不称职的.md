# 096_领导都是不称职的

> **本章进度**：卷八·人群 (Ep.096/109)
> **核心痛点**：领导无能 / 晋升陷阱
> **CS隐喻**：版本回滚失败 (Rollback Failure) / 降级机制缺失 / Bug 版本卡死

## 1. Hook：草包领导
“我受不了了，我要辞职！”

苏心一进门，就把包扔在地上，气得直跺脚。

“那个新来的王总监，简直就是个草包！他根本不懂业务，只会开会、做 PPT、喊口号。今天他居然让我把一个明明需要三天的需求，在一个小时内做完。我跟他解释技术难度，他根本不听，就一句话：‘那是你的问题，我只要结果。’”

“这种人到底是怎么当上总监的？难道公司高层都瞎了吗？还是说他是老板的亲戚？”

林哲在旁边叹气：“天下乌鸦一般黑。我之前的那个领导也是，技术一窍不通，抢功第一名。我觉得公司越大，这种草包领导就越多。”

“这不科学啊。”苏心很不解，“按理说，能升职的人应该是能力强的啊。为什么升上去的反而是一群废物？”

## 2. Trap：彼得原理
“这很科学。”叶清放下手中的书，书名是《彼得原理》。

“劳伦斯·彼得在 1969 年就提出了这个定律：**在一个等级制度中，每个员工都会趋向于上升到他所不胜任的职位**。”

“什么意思？”苏心问。

“假设你是一个优秀的技术员（Level 1）。因为你技术好，表现出色，公司把你提拔成了小组长（Level 2）。如果你的管理能力也不错，公司又把你提拔成了部门经理（Level 3）。”

“但是，”叶清画了一个阶梯图，“到了 Level 3，需要的不是技术能力，而是战略眼光和政治手腕。如果你恰好不具备这些能力，你的表现就会变差。于是，你就**停止升职**了。”

“结果就是：你被卡在了 Level 3 这个你**无法胜任**的位置上，直到退休。而那个还在 Level 1 的优秀新人，还在继续往上爬，直到他也爬到那个他不胜任的位置。”

叶清总结道：“所以，随着时间的推移，每个职位最终都会被一个**不胜任的员工**所占据。这就是为什么领导看起来都是草包。”

## 3. Turn：版本回滚失败
“这在软件发布里，叫**版本升级失败**。”

叶清打开电脑，演示了一个发布流程。

“王总监在 V1.0 版本（销售员）时，运行得非常稳定，业绩全公司第一。于是系统决定给他升级到 V2.0 版本（销售总监）。”

“但是，V2.0 需要的依赖库（管理技能）和 V1.0 完全不同。升级后，系统出现了严重的兼容性错误（不胜任）。”

“在正常的软件开发中，如果新版本崩溃了，我们会立刻执行**回滚**（Rollback），让他退回到 V1.0。但在职场里，这是**不允许**的。”

“为什么？”苏心问。

“因为面子。”叶清摊手，“让一个总监降职回去做销售员，他会觉得丢脸，公司也会觉得打脸。所以，大家只能硬着头皮让他继续在 V2.0 这个满是 Bug 的版本上运行。他痛苦，下属也痛苦。”

## 4. Action：理解无能，寻找补丁
“那我们这些下属岂不是死定了？”

“也不一定。既然你知道他是个 Bug 版本，你就可以给他打**补丁**（Patch）。”

叶清给出了生存指南。

“不要试图去‘教会’他怎么当领导，也不要跟他硬刚。你要做的是**填补他的缺失模块**。”

“比如，他不懂技术，只会催进度。那你就不要跟他讲技术细节，而是给他提供三个**可视化的选项**：
*   A 方案：1 小时做完，但有 50% 概率崩溃。
*   B 方案：3 小时做完，稳定。
*   C 方案：砍掉一半功能，1 小时做完。”

“把‘技术问题’转化成‘选择题’。这样既满足了他‘做决定’的控制欲，又保护了你的技术底线。”

叶清看着苏心：“记住，你的任务不是为了证明他有多蠢，而是为了**让项目活下去**。有时候，哄着一个草包领导把路走通，也是一种高级能力。”

## 5. Outro：不要盲目升级
苏心若有所思地点点头：“所以我应该把他当成一个智障儿童来哄？”

“可以这么理解。同情他吧，他也是系统错误的受害者。”

叶清合上书。

“另外，这个原理也给你们提了个醒：**不要盲目追求升职**。”

“如果你在当前的位置上干得很开心，也很擅长，那就不要为了面子或那点涨薪，强行去爬那个更高的梯子。否则，你也会变成那个被下属骂的草包。”

“在自己擅长的领域做到极致，比在一个不擅长的领域当领导，要幸福得多。”

林哲在一旁举手：“那我不想升职，只想涨工资怎么办？”

叶清白了他一眼：“那是另外一个付费课程了。”

**【下集预告】**
苏心发现，虽然公司里有草包领导，但大家似乎都很忙。明明半天能干完的活，非要拖到下班。
而且部门人越多，大家反而越忙。
叶清：这是帕金森定律。工作会自动膨胀，直到填满所有可用的时间。
