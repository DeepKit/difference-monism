## 一、场景契约系统

### 1.1 契约的核心作用

场景契约是**人类与AI的接口**：人类提供创意方向，AI执行具体写作。契约确保双方对"这集必须发生什么"有明确共识。

### 1.2 简化版契约模板（AI优化）

```json
{
  "scene_key": "v01.e001",
  "scene_name": "河边的孩子",
  
  "ASTO": {
    "phase": "a",
    "phase_label": "混沌",
    "mode": "I",
    "mode_label": "自在"
  },
  
  "spacetime": {

  },
  
  "narrative": {
    "enter": "萨尔贡在河边制作泥板，背上有鞭痕",
    "must_land": [
      "萨尔贡无意识地敲击出六十一点横",
      "学徒发现萨尔贡的异常并质问",

    ],
    "exit": {
      "direction": "神秘召唤",

    },
    "reveal": ["萨尔贡会无意识敲击", "萨尔贡是奴隶"],
    "withhold": ["河边老人身份", "六十一点横的真正含义"]
  },
  
  "taboos": [
    "不得解释二进制原理（第1集只需现象）",
    "不得直接说出'算法'一词",
    "不得使用现代时间单位",
    "禁止使用规划文档概念词：数、术、命、不完备意识、ProtectionCoefficient、术化、命-遗忘",
    "禁止使用写作素材考古.md未收录的具体历史器物名称",
    "禁止神秘主义内容（预言、神迹、超自然现象）",
    "禁止科幻内容（未来技术、时间旅行、外星元素）",
    "所有设定必须在历史现实主义范围内"

  "must_land_rule": "must_land必须用可观察的行为表达，禁止概念标签。错误示例：'展示不完备意识丧失'。正确示例：'萨尔贡第一次没有质疑祭司报出的数字，因为他太累了'",

  "sensory_anchor": "至少一个反常规的具体感官细节，例：'萨尔贡的左手有一道旧伤，他在紧张时会无意识地摩挲它'",
  
  "surprise_variable": "本集必须引入的意外变量（主角判断失误/信息误判/盟友异常行为），禁止留空",
  
  "scene_type": "standard | hook | explosion（标准集/钩子集/爆点集，见下方定义）",
    "target": 3500,
    "min": 2800,
    "max": 4200
  }
}
```

### 1.6 三种集数类型定义

| 类型 | 触发条件 | 契约重点 | 频率 |
|------|----------|----------|------|
| **standard（标准集）** | 正常推进 | must_land + sensory_anchor + surprise_variable | 约70% |
| **hook（钩子集）** | 每10集一次，纪元开篇 | 强信息缺口 + 不可退出状态 + 结尾悬念最大化 | 约20% |
| **explosion（爆点集）** | 每10集第9-10集，纪元结尾，主线转折 | 结构级反转 + 人格后果 + failure_ripple必填 | 约10% |

**爆点节奏公式（每10集）**：

| 集数 | 任务 |
|------|------|
| 1-3 | 强钩子（hook集） |
| 4-8 | 情节推进（standard集） |
| 9-10 | 结构级爆点（explosion集） |

**纪元级爆点节点**：

| 集数 | 任务 |
|------|------|
| 第20集 | 世界观扩大 |
| 第30集 | 主角认知崩塌 |
| 第50集 | 阵营翻转 |

## 三、人物配置系统

### 3.1 每个case的标准配置

**结构：1主 + 2复用 + 4新**

| 位置 | 数量 | 规则 |
|------|------|------|
| 主角 | 1 | 每个纪元专属 |
| 复用配角 | 2 | 从复用池选，跨case出现，角色可换但位置不变 |
| 新建配角 | 4 | 从新建池选，每个case专属 |

### 3.2 10种配角类型

| # | 类型 | 核心功能 | 戏剧张力 |
|---|------|----------|----------|
| 1 | 情感锚 | 主角最在乎的人 | 情感代价、留存动力 |
| 2 | 对立者 | 价值观相反但动机可理解 | 道德撕裂、智力冲突 |
| 3 | 镜像者 | 相似选择，不同命运 | 人性循环暗线载体 |
| 4 | 传承者 | 接收或传递使命/秘密/知识 | 跨纪元连接点 |
| 5 | 见证者 | 旁观并记录 | 叙事视角、历史感 |
| 6 | 催化剂 | 触发主角关键转变后退场 | 情节推进、节奏控制 |
| 7 | 背叛者 | 被信任后背叛 | 信任崩塌、最深的伤 |
| 8 | 制度之手 | 体制/权力的具体化身 | 主角对抗的"脸" |
| 9 | 牺牲者 | 因主角选择而付出代价 | 道德重量、失败后果 |
| 10 | 导师/学徒 | 知识传递的两端 | 守护者传承的微观体现 |

### 3.3 分池规则

**复用池**（每case选2，跨case出现）：
- 传承者、见证者、镜像者、导师/学徒

**新建池**（每case选4，专属本case）：
- 情感锚、对立者、催化剂、背叛者、制度之手、牺牲者

### 3.4 禁令

- 禁止神秘主义内容
- 禁止科幻内容
- 所有人物设定必须在历史现实主义范围内

- 参考下文"ASTO情绪指令"

**步骤2：定时空与人物**
- 填写spacetime（与历史事实核对）

- 必须是**可校验的命题**

- taboos：绝对不能违反的规则

## 二、ASTO坐标系统详解

### 2.1 六阶（Phase）：叙事节奏

| 六阶 | 叙事特征 | AI情绪指令 | 典型场景 | 纪元占比 |
|------|----------|-----------|----------|----------|
| **a 混沌** | 信息碎片、氛围铺垫 | "不要解释，只展示感官细节。氛围神秘，多义性" | 开场、梦境、回忆 | 5%（1-50集） |
| **b 秩序** | 规则建立、关系明确 | "信息交换，人物关系确认，逻辑清晰" | 对话推进、计划制定 | 15%（51-200集） |
| **c 流变** | 渐变积累、张力蓄压 | "节奏加速，细节累积，悬念推进" | 线索串联、疑点堆叠 | 40%（201-600集） |
| **d 脉冲** | 突变、高潮、不可逆 | "压缩时间，强烈冲突，不可逆转" | 真相揭露、对抗爆发 | 20%（601-800集） |
| **e 崩解** | 旧秩序瓦解、失控 | "失控感，情感释放，信念崩塌" | 关系破裂、系统失败 | 10%（801-900集） |
| **f 归元** | 沉淀、收束、新平衡 | "慢节奏，留白，新起点暗示" | 独处反思、余韵留白 | 10%（901-1000集） |

### 2.2 五态（Mode）：角色认知状态

| 五态 | 角色状态 | AI情绪指令 | 核心动作 | 场景类型 |
|------|----------|-----------|----------|----------|
| **I 自在** | 无意识、本能 | "角色无意识触发，困惑和好奇" | 觉醒触发 | 内省 |
| **II 自觉** | 发现真相 | "震惊、觉醒，认知升级" | 发现真相 | 揭示 |
| **III 对抗** | 与系统冲突 | "紧张、恐惧，资源受限" | 采取行动 | 动作 |
| **IV 反思** | 理解处境 | "悲伤、决心，内心挣扎" | 审视后果 | 氛围 |
| **V 定向** | 做出选择 | "悲壮、解放，重大决策" | 做出选择 | 对话 |

### 2.3 六阶×五态组合示例

| 组合 | 场景特征 | 示例 |
|------|----------|------|
| **a+I（混沌+自在）** | 无意识觉醒，氛围神秘 | 第1集：萨尔贡无意识敲击泥板 |
| **b+III（秩序+对抗）** | 规则明确，冲突展开 | 第3集：萨尔贡被监视，开始逃亡 |
| **c+IV（流变+反思）** | 张力积累，内心挣扎 | 第50集：萨尔贡理解自己是"异常" |
| **d+V（脉冲+定向）** | 高潮转折，重大选择 | 第99集：停机的选择 |
| **e+IV（崩解+反思）** | 秩序瓦解，情感释放 | 第900集：所罗门崩溃 |
| **f+II（归元+自觉）** | 新平衡建立，认知升级 | 第1000集：向第2纪传递 |

## 三、单双集结构模式

### 3.1 两种集的功能定位

| 维度 | 单数集 | 双数集 |
|------|--------|--------|
| **核心功能** | 推进主线、制造悬念 | 展开支线、深化人物 |
| **情绪曲线** | 上升、紧张、爆发 | 平缓、沉淀、回味 |
| **信息密度** | 高（多事件） | 中（单事件深入） |
| **结尾设计** | 钩子（想知道） | 闭环+暗示（满足+期待） |
| **读者预期** | "下一集会怎样" | "人物关系会如何发展" |

### 3.2 标准节奏模式

**模式1：标准交替（1:1）**
```
单-双-单-双-单-双
适合：主线推进期、人物成长期
```

**模式2：加速推进（2:1）**
```
单-单-双-单-单-双
适合：高潮前夕、危机密集期
```

**模式3：减速沉淀（1:2）**
```
单-双-双-单-双-双
适合：世界观展开期、情感沉淀期
```

### 3.3 单集结构（4000字，知乎盐选版）

```
【第一段：死亡威胁】500字
→ 极端场景：主角面临生死危机
→ 钩子：为什么是他？他做了什么？

【第二段：背景铺陈】1500字
→ 快速建立世界观：时间、地点、权力结构
→ 展示"数"的暴力：税收/测量/算法如何控制人
→ 身份反差：低微身份 vs 高维认知

【第三段：危机升级】1500字
→ 新危机出现：被追捕/被标记/被监视
→ 展示"术"的运作：系统如何发现"异常"
→ 主角发现线索：61点横/错误/不完备性

【第四段：强钩子】500字
→ 绝对不能在单集给答案
→ 钩子类型：
   a) 发现不该发现的东西
   b) 外部威胁逼近
   c) 内部觉醒开始
```

### 3.4 双集结构（4000字，知乎盐选版）

```
【第一段：揭晓答案】1200字
→ 承接单集钩子，给出核心答案
→ 但不要给完，留一层窗户纸

【第二段：对抗/逃生】1200字
→ 主角利用新发现的认知应对危机
→ 展示"数术命"的张力：用认知对抗系统

【第三段：情感升华】1000字
→ 角色的内心转变
→ 主题点题：什么是"错误"？什么是"真相"？

【第四段：新钩子预埋】600字
→ 本单元闭环，但下一单元开启
→ 新角色出现 / 新危机酝酿 / 新认知觉醒
```

## 五、一致性维护框架

### 5.1 17层上下文检查（核心版）

| 层级 | 检查项 | 关键问题 |
|------|--------|----------|
| L1 | 组织边界 | 72柱的当前势力范围？ |
| L2 | 人物目标 | 萨尔贡当前优先级？ |
| L3 | 读者意图 | 读者此刻最想知道什么？ |
| L4 | 技术栈 | 本集涉及什么算法概念？ |
| L5 | 人物关系 | 萨尔贡-伊什美-所罗门三角状态？ |
| L6 | 核心意象 | "错误"意象是否出现？ |
| L7 | 场景 | 时间、地点、持续时间？ |
| L8 | 教育目标 | 必须让读者理解什么？ |

### 5.2 封印机制

> **一旦某层通过验证，即成为不可变基础，上层构建其上。**

| 层级 | 封印条件 | 封印仪式 |
|------|----------|----------|
| **单元**（集） | 通过10项核心检查 | 发布，标记为sealed |
| **篇章**（40集） | 终章读者满意度>80% | 实体书发布 |
| **时代**（200集） | 终章形成文化讨论 | 学术研讨会 |
| **纪元**（1000集） | 终章被学术引用 | 纪录片联动 |
| **本体**（5000集） | 终章读者成为守护者 | 开源叙事宇宙 |

## 六、AI写作特供：契约生成提示词

### 6.1 给AI的契约解析指令

```
你是一位专业的小说写作AI。请根据以下场景契约，生成一集3500字的小说内容。

【契约信息】
- 集数：{scene_key}
- 标题：{scene_name}
- ASTO坐标：{phase}（{phase_label}）+ {mode}（{mode_label}）
- POV角色：{pov}
- 必须落地：{must_land}
- 必须揭示：{reveal}
- 必须保留：{withhold}
- 红线：{taboos}

【情绪指令】
根据ASTO坐标，本集情绪基调为：{AI情绪指令}

【写作要求】
1. 严格遵循四条红线（禁止发光/预知/术语/光环）
2. 使用物质性描写（泥板、河水、芦苇、伤口等）
3. 延迟解释：展示现象，不解释原理
4. 每1000字一个情绪波峰
5. 结尾必须有强钩子

【输出格式】
- 场景1（钩子）：1000字
- 场景2（发展）：1200字
- 场景3（高潮）：1000字
- 场景4（心法+钩子）：300字
```
