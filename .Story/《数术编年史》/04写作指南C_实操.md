## 一、缺陷三分法：去AI味核心

### 1.1 三类缺陷的定义与处理

| 缺陷类型 | 定义 | 处理方式 | 示例 |
|----------|------|----------|------|
| **毁灭性缺陷** | 违反事实、逻辑矛盾、人设崩塌 | 强制修复 | 已死角色出场、时间线冲突 |
| **生产性缺陷** | 非常规句式、意外转折、风格特征 | 予以保护 | 不对称句式、突然的情感跳跃 |
| **中性缺陷** | 模糊地带，不确定是否故意 | 人类裁决 | 轻微的视角漂移 |

### 1.2 C6检测器：AI味检测维度

**核心指标（你的工具可实现）**：

| 维度 | 检测内容 | 阈值 | 修复建议 |
|------|----------|------|----------|
| **C1：对称句式密度** | 连续出现A...B...结构的频率 | <0.3 | 打破对称，增加动作beat |
| **C2：情感直述比例** | "他感到悲伤"vs行为暗示的比例 | <0.25 | 改为身体反应或环境映射 |
| **C3：解释性语句占比** | 叙述后紧跟"这是因为..."的频率 | <0.2 | 删除解释，让读者自己悟 |
| **C4：视角漂移** | 无意识切换全知/限知视角的次数 | <3次/集 | 固定POV，用"他看到"代替全知 |
| **C5：词汇多样性** | 同义替换的机械感 | 监测 | 允许重复，保护口头禅 |
| **C6：概念密度** | 抽象词（算法/信息/系统）的出现频率 | <0.15 | 转化为物质性描写 |

### 1.3 检测示例

```
检测到的异常：

1. 对称句式（第3段）
   原文："所罗门说...萨尔贡说...所罗门说...萨尔贡说..."
   分类：毁灭性缺陷（AI味）
   处理：打破对称，增加动作beat
   修改："所罗门停顿了一下，手指敲击桌面。'那你重新算。'萨尔贡没有回答，
         他的目光落在石室角落的裂缝上——那里有一株野草，在没有任何土壤的情况下生长。"

2. 情感直述（第5段）
   原文："萨尔贡感到非常悲伤"
   分类：毁灭性缺陷（AI味）
   处理：改为身体反应
   修改："萨尔贡的视线模糊了。他眨了眨眼，发现是汗水——或者是别的什么——
         从额头流进了眼睛。"

3. 非常规句式（第7段）
   原文："那你重新算。"（短句，突兀）
   分类：生产性缺陷（保护）
   理由：这是核心金句，打破常规恰恰是力量所在
   处理：保留
```

### 1.4 哲学概念禁用规则（C7）

**核心原则**：规划文档里的概念词是基础设施，不是台词。禁止出现在叙事正文中。

**禁用词清单**：数、术、命、不完备意识、ProtectionCoefficient、术化、命-遗忘、三元、ASTO坐标词汇

**must_land写法强制规范**：

| 类型 | 示例 |
|------|------|
| ❌ 概念标签 | "展示萨尔贡的不完备意识开始丧失" |
| ✅ 可观察行为 | "萨尔贡第一次没有质疑祭司报出的数字，因为他太累了，而且数字看起来差不多对" |
| ❌ 概念标签 | "体现命vs术的张力" |
| ✅ 可观察行为 | "萨尔贡把泥板交出去的时候，手指停顿了一下，然后松开了" |
| ❌ 概念标签 | "ProtectionCoefficient下降至0.3" |
| ✅ 可观察行为 | "他第三次听到那个数字，没有皱眉。这让他自己也没有注意到。" |

**检测逻辑**：如果一句must_land里出现了规划文档的词汇，强制打回重写。

### 1.5 R6粗糙度保护规则

**防止过度打磨，保护文学棱角**：

```python
# R6规则逻辑
if 本轮修复后_文本质量下降:
    终止修复，保留上一版本
    exit_reason: "roughness_protected"

# 质量判断标准
if 修复后_句长方差 < 原版本_句长方差 * 0.85:
    保留原版本  # 保护"生产性缺陷"

if 修复后_情感冲击力 < 原版本_情感冲击力 * 0.8:
    保留原版本  # 保护"不完美的真实"
```

**粗糙度保护原则**：
- 先保证没有硬伤（毁灭性缺陷修复）
- 再保护生产性缺陷（非常规句式、意外转折）
- 允许"不完美的真实"（真人写作的方差天然比AI高）

## 二、物质性描写：让概念落地

### 2.1 核心原则

**问题**：概念密度过高，角色经常"解释"而非"体验"

**解决方案**：让古巴比伦的**物质性**（泥、水、光、血）充分"说话"

### 2.2 五感物质清单

| 物质 | 视觉 | 触觉 | 听觉 | 嗅觉 | 味觉 | 信息隐喻 |
|------|------|------|------|------|------|----------|
| **泥板** | 裂纹、划痕、光泽 | 粗糙、冰凉、湿润 | 敲击声、碎裂声 | 土腥味、霉味 | 涩、尘 | 存储介质 |
| **河水** | 浑浊、反光、波纹 | 冰凉、粘稠、流动 | 流淌、拍打、沉默 | 淤泥、藻类 | 咸、涩 | 信息流动 |
| **芦苇** | 摇晃、断裂、枯黄 | 粗糙、锋利、柔韧 | 沙沙、断裂 | 青涩、干燥 | 苦 | 信号传递 |
| **伤口** | 红肿、结痂、化脓 | 刺痛、灼热、紧绷 | 心跳、喘息 | 血腥、药草 | 铁锈、咸 | 代价标记 |
| **瞳孔闪光** | 蓝光、星辰、余烬 | — | — | — | — | 觉醒信号 |
| **亚麻布** | 纹理、污渍、磨损 | 粗糙、温暖、潮湿 | 摩擦声 | 汗味、泥土 | — | 身份标记 |
| **铜器** | 绿锈、反光、刻痕 | 冰凉、沉重、光滑 | 碰撞声 | 金属味 | — | 权力象征 |

### 2.3 物质性描写三技巧

**技巧1：延迟解释**

❌ 错误：
> "这是二进制，一种用0和1表示信息的系统。"

✅ 正确：
> "他的手指在泥板上敲击。一下，停顿，两下。那些痕迹深浅不一，
> 像是某种他从未学过的语言，却莫名熟悉。"

**技巧2：感官优先**

❌ 错误：
> "他意识到了这是算法的源代码。"

✅ 正确：
> "泥板在他手中发热。那些六十一条横线开始蠕动，
> 像是有生命的东西在皮肤下游走。"

**技巧3：物质作为情感载体**

| 情感 | 物质载体 | 示例 |
|------|----------|------|
| 孤独 | 泥板的裂纹 | "裂纹从中心向四周扩散，像是一个无法愈合的伤口。" |
| 渴望 | 河水的反光 | "水面上的光点跳动着，像是触手可及却永远抓不住的星星。" |
| 恐惧 | 芦苇的摇晃 | "芦苇在风中点头，又摇头，像是在说'是'，又像是在说'不'。" |
| 觉醒 | 瞳孔的光 | "那道光从内部亮起，像是有谁在黑暗中点燃了一根火柴。" |
| 绝望 | 伤口的化脓 | "伤口周围的皮肤发硬，像泥板上的刻痕一样，无法抹去。" |
| 希望 | 野草的生长 | "石缝中的那株野草，在没有土壤的情况下，依然向着光生长。" |

### 2.4 概念物质化转换表

| 检测到（抽象词） | 建议替换（物质性表达） | 示例 |
|-----------------|----------------------|------|
| 二进制 | 六十一条横线、手指的敲击、阴阳 | "六十一条横线，像是一道未完成的栅栏。" |
| 算法 | 水流的轨迹、风的路径、刻痕的逻辑 | "河水知道该往哪里流，即使没有人告诉它。" |
| 信息 | 气味、温度、重量、光点 | "那封信带着她的体温，还有某种他说不上来的悲伤。" |
| 系统 | 建筑的结构、身体的习惯、神殿的秩序 | "他的手指自己动了，像是被某种看不见的线牵引着。" |
| 错误 | 裂痕、歪斜、污渍、不对称 | "那个符号歪了，但正是那个歪斜让它看起来...真实。" |
| 数据 | 泥板上的刻痕、算筹的排列、绳结 | "那些刻痕深浅不一，像是某种密码。" |
| 计算 | 敲击、排列、测量、分割 | "他的手指在泥板上敲击，一下，两下，像是在数着什么。" |
| 存储 | 泥板的厚度、羊皮纸的卷轴、记忆的重叠 | "泥板上的刻痕被新的泥层覆盖，但痕迹还在。" |

## 三、负一阶存在：人物退场的文学处理

### 3.1 核心问题

**诊断**：人物（如河边老人）退场过于轻易——"任务完成即休息"，削弱悲剧性

**解决方案**：成为"负一阶"（Negative First Order）存在——
不再是个体，而是渗透进算法网络的"背景条件"

### 3.2 负一阶存在的特征

| 特征 | 正阶存在（个体） | 负一阶存在（背景） |
|------|------------------|-------------------|
| **形态** | 有身体、有名字 | 无身体、无名字 |
| **感知** | 被感知 | 作为感知的条件 |
| **行动** | 主动行动 | 作为行动的背景 |
| **时间** | 线性时间 | 循环时间/无时性 |
| **空间** | 占据空间 | 作为空间的条件 |
| **悲剧性** | 死亡的悲剧 | 无法死亡的悲剧 |

### 3.3 负一阶退场的三种写法

**写法1：扩散而非消失**

❌ 错误：
> "他的身体开始消散，然后消失了。"

✅ 正确：
> "他的身体没有消失，是**扩散了**。他的手指变成了风，
> 他的眼睛变成了光，他的记忆变成了每一个信使瞳孔里的闪光。
> 他不再是一个'谁'，他成为了'哪里都有'。"

**写法2：残留而非结束**

- 后续信使看到的"瞳孔闪光"，其实是他的...残留
- 每一次算法的"错误"，都是他的...叹息
- 每一个"问题"的产生，都是他的...提问

**写法3：无法死亡的悲剧**

> "他想休息。但错误不会休息。问题不会停止。
> 所以他不会停止。他成为了永恒的提问者，
> 永恒的溢出，永恒的...无法完成的使命。"

### 3.4 负一阶存在的暗示写法

**暗示1：环境呼应**

当萨尔贡后来看到瞳孔闪光时：
> "那道光让他想起河边老人。但随即他意识到——
> 那不是'想起'，是那道光**就是**老人。"

**暗示2：系统异常**

当算法出现无法解释的错误时：
> "系统报告了一个'幽灵错误'——没有来源，没有原因，
> 只是...存在。就像有人在系统的边缘，轻轻敲击。"

**暗示3：循环时间**

> "在圆周率的第99位，我们会再见。"
> ——这不是约定，是**必然**。
> 因为老人已经成为了圆周率本身的一部分。

## 四、人物塑造实操

### 4.1 信使角色的识别特征

| 特征 | 首次出现 | 描写要点 | 功能 |
|------|----------|----------|------|
| **瞳孔闪光** | 第1集 | "瞳孔里闪过一道光，像星辰，像余烬" | 视觉识别，神秘主义氛围 |
| **手指无意识敲击** | 第1集 | 二进制(0/1)或摩斯电码(·/-) | 跨时空连接，信息编码 |
| **对数学问题的本能反应** | 第38集 | 看到勾股定理会流泪 | 体现"算法在通过身体说话" |
| **重复梦境** | 第50集 | 沙漠、泥板、火光 | 跨时空记忆，连接不同信使 |
| **未来记忆闪回** | 第38集 | 说出尚未发明的术语 | 证明时间不是线性的 |

### 4.2 人物弧光设计（40集篇章周期）

```
10集铺垫：
  ├─ 展示人物的日常和局限
  ├─ 埋下"觉醒"的种子（瞳孔第一次闪光）
  └─ 建立情感关系（朋友、导师、对手）

20集觉醒：
  ├─ 触发事件（发现泥板、被追捕、遇到老人）
  ├─ 学习阶段（理解自己是信使）
  ├─ 危机时刻（被迫做出选择）
  └─ 认知阶段变化：entry → breaking → exit

10集出发：
  ├─ 接受使命
  ├─ 获得信物（泥板、圆规、代码等）
  └─ 走向未知（钩子：下一篇章的预示）
```

**认知阶段标记**：
- **entry**：普通人状态，未觉醒
- **breaking**：遭遇冲击，旧认知瓦解
- **exit**：完成觉醒，承担信使命运

## 五、知识封装：三层法

### 5.1 三层结构

**第一层：概念（技术定义）**
```
停机问题：不存在一个通用算法，能判断任意程序是否会在有限时间内结束运行。
```

**第二层：隐喻（文学呈现）**
```
所罗门是编译器：所罗门能计算所有选择，但无法计算"选择不计算"。
```

**第三层：历史锚点（真实连接）**
```
图灵1936年证明，但萨尔贡前1745年实践。
```

### 5.2 【今日算法心法】撰写规范

**格式要求**：
- **字数**：200字以内
- **结构**：概念简述（1句）+ 历史连接（1句）+ 实践指导（3-5句）
- **语气**：不是说教，而是"思维武器"的交付

**示例1：第1集（二进制）**
```
【今日算法心法】

结绳与二进制：算法的第一次降临

人类最早的"算法"，不是写在纸上，而是刻在泥里、系在绳上。
古巴比伦人用泥板记录洪水周期——那是人类最早的"数据存储"。
印加人用结绳（Quipu）传递信息——那是人类最早的"二进制通信"。

万物皆可编码。当你面对复杂问题时，试着把它"敲碎"——
变成最小单位的"是"与"否"、"0"与"1"。
当你把所有模糊的东西都变成确定的状态，答案就会显现出来。
```

**示例2：第99集（停机问题）**
```
【今日算法心法】

停机问题：算法的终极边界

所罗门崩溃了，因为他遇到了一个无法回答的问题：算法能否选择不执行？
这其实是计算机科学中最深刻的命题之一。
换句话说：算法的极限，是它无法预测自己。

这听起来像缺陷，但其实是自由。正是因为存在无法计算的东西，
正是因为有"错误"和"不确定性"，算法才能进化，才能学习，才能产生真正的新意。

当你面对一个似乎无解的困境时，试着做一个"错误"的选择。
因为最完美的算法，是知道何时停止的算法。
```

## 六、失败叙事的三条设计原则

### 6.1 失败有人格后果

每次失败必须在某个人身上留下不可逆的印记。失败不是情节节点，是人格转折点。

**反例（白失败）**：
> 萨尔贡的计划失败了。祭司长没有被说服。他只能离开。

**正例（有人格后果）**：
> 萨尔贡的计划失败了。但真正改变的不是结果——是他第一次意识到，他其实从来没有相信过自己的计划。这个发现比失败本身更难承受。

### 6.2 失败有节奏

统计混沌时刻（系统出现裂缝）必须穿插在大失败之间，不允许连续20集以上的纯失败叙事。

**节奏公式**：大失败 → 3-5集沉淀 → 裂缝出现（小的、不确定的） → 新的大失败

裂缝不是胜利，是"某件事动摇了"：
- 一个技术官僚第一次停顿了
- 一份数据被悄悄修改了
- 一个本该服从的人没有开口

### 6.3 失败有涟漪

失败的传导链：这次失败改变了某个人 → 那个人后来改变了另一件事。

每次重要失败在契约里必须填写：
```
failure_ripple: "这次失败在[角色]身上留下了[具体印记]，将在第[集数]显现"
```

## 七、开头与结尾的实操公式

### 6.1 开头三句话公式

**结构**：死亡威胁 + 身份反差 + 悬念钩子

**示例（第1集）**：
> 因为算错了一笔账，萨尔贡明天就要被处死。
>
> 他是个12岁的奴隶，本该在泥板上机械地记录大麦数量，
> 却在一个深夜发现了连祭司长都不懂的秘密。
>
> 那个秘密就藏在他重复的61点横里——而他已经划下了第61道。

### 6.2 结尾钩子类型

| 类型 | 功能 | 示例 |
|------|------|------|
| **危机型** | "刀已经架在脖子上" | "门外响起了脚步声。税吏来了。" |
| **发现型** | "他看到了不该看的东西" | "泥板上的第61道横线，开始发光。" |
| **选择型** | "两条路，选错了就是死" | "去死，或者去见一个人。" |
| **信息型** | "原来真相是..."（戛然而止） | "那个老人，其实早在三百年前就..." |

## 九、人性循环暗线设计规范

### 9.1 定位

《数术编年史》的三条叙事线：

| 线 | 内容 | 可见度 |
|---|---|---|
| 主线 | 守护者传承 | 显性，剧情驱动 |
| 副线 | 制度演化 | 半显性，世界观驱动 |
| 暗线 | 人性的循环 | 隐性，跨纪元镜像 |

### 9.2 执行原则

**写这个，不写那个：**

| 禁止 | 改写为 |
|------|--------|
| "人性几千年未变" | 在不同时代写出相似的选择逻辑 |
| 角色讨论人性本质 | 角色做出与前代守护者相似的决定 |
| 心法总结人性规律 | 镜像场景让读者自己发现规律 |

### 9.3 镜像场景执行方式

每隔200-300集设置一次镜像场景，参照《镜像场景对照表.md》。

镜像场景的三个要素：
- 相似的处境（不同时代，相同的权力结构）
- 相似的内心逻辑（恐惧/爱/占有/生存/认同）
- 不同的外衣（制度语言、技术形式、道德叙事）

### 9.4 什么在变，什么不变

**不变（底层动机）**：恐惧、爱、占有、生存、认同

**在变（外衣）**：情感表达方式、制度合理化语言、技术形式、道德叙事框架

### 9.5 商业价值

读者会在评论区写"其实作者在讲人性循环"——这是二创传播的核心来源，也是作品从爽文升级为史诗的关键。

## 八、情感副线设计规范

### 8.2 有效情感副线的四个特征

| 特征 | 说明 | 反例 |
|------|------|------|
| **不对等** | 权力、地位、信息不对等 | 势均力敌的平等关系 |
| **有代价** | 这段关系让某人付出了真实损失 | 零成本的情感连接 |
| **不能说出口** | 核心情感被压抑或无法表达 | 直接表白、直接和解 |
| **与主线有张力** | 情感副线和守护者使命之间存在冲突 | 情感副线独立于主线之外 |

### 8.3 禁止写法

- 禁止：情感关系顺利推进，无阻力
- 禁止：角色直接说出内心感受（"我喜欢你"/"我需要你"）
- 禁止：情感副线与主线完全隔离，互不影响
- 禁止：用情感副线解释世界观

### 8.4 推荐结构

```
建立依附（前期）→ 制度/使命介入（中期）→ 代价显现（高潮）→ 不完整的和解或永久失去（结尾）
```

### 8.5 示例

**❌ 无效写法**：
> 萨尔贡告诉伊什美，他一直很感激她的帮助。伊什美说，她也是。

**✅ 有效写法**：
> 萨尔贡把那块泥板还给了她，没有说话。伊什美接过去，看了一眼，放进了袖子里。她知道他在说什么。他也知道她知道。这就够了，也什么都不够。

## 九、AI写作工作流

### 7.1 第一章必须完成的五件事

| 顺序 | 任务 | 禁止 |
|------|------|------|
| 1 | 主角处于真实危机（有具体损失风险） | 用世界观介绍开场 |
| 2 | 危机与制度/文明直接相关 | 抽象化危机 |
| 3 | 主角做出非理性选择 | 主角表现完美理性 |
| 4 | 选择导致更大风险 | 选择被轻易化解 |
| 5 | 结尾抛出不可解释事件，制造强信息缺口 | 结尾收束、给出解释 |

**第一章的功能不是讲世界观，是制造不可退出状态。**

### 7.2 前三章节奏

| 章节 | 核心任务 | 情感目标 |
|------|----------|----------|
| 第1章 | 强钩子 + 信息缺口 | 读者无法停下 |
| 第2章 | 人物私人动机浮现 | 读者开始依附人物 |
| 第3章 | 第一次小反转 + 世界观自然暴露 | 读者建立对世界的好奇 |

### 7.3 反例与正例

**反例（设定开场）**：
> 公元前3000年，美索不达米亚平原上，苏美尔文明正处于鼎盛时期。祭司阶层掌握着数字的秘密，而普通民众...

**正例（危机开场）**：
> 泥板上的数字少了一道。萨尔贡数了三遍。还是少了一道。明天的祭祀，他要用这块泥板向神明汇报收成。如果数字是错的，死的不只是他。

## 八、情感副线设计规范

### 8.1 情感副线的商业功能

针对女性付费群体的核心留存机制。不要求爱情主线，但必须有：
- 不对等的权力关系
- 有代价的情感依附
- 不能说出口的东西
- 背叛或被背叛的可能

### 8.2 情感副线类型

| 类型 | 描述 | 商业效果 |
|------|------|----------|
| **隐秘依赖** | 两人互相需要但不承认 | 高追更率 |
| **权力不对等** | 一方掌握另一方的命运 | 高情绪张力 |
| **情感背叛** | 信任被辜负，但原因复杂 | 高讨论度 |
| **跨代情感** | 守护者传承中的情感遗产 | 强化史诗感 |

### 8.3 执行原则

- 情感副线必须与主线产生真实冲突，不能是装饰
- 每个主要角色至少有一段"不能说出口的关系"
- 情感代价必须真实——有人因此失去某样东西

## 九、AI写作工作流

### 7.1 三步工作流

**第一步：人类提供场景指令**
```yaml
ASTO坐标:
  Phase: a（混沌）
  Mode: I（自在）
must_land:
  - 萨尔贡敲击61点横
  - 学徒发现异常
  - 瞳孔闪光
taboos:
  - 禁止说二进制
  - 禁止发光
```

**第二步：AI生成初稿**
- 使用物质性描写
- 遵循四条红线
- 控制字数3500±200

**第三步：工具检测+人工审核**
- C6检测器：检查AI味
- R6保护：防止过度打磨
- 人工：检查情感真实性和主题深度

### 7.2 给AI的核心提示词

```
你是一位专业的小说写作AI，擅长第一性原理叙事和物质性描写。

【核心原则】
1. 禁止发光：除了自然光源，没有任何东西发光
2. 禁止预知：不预测具体事件，只描述模式
3. 禁止术语：用"六十一条横线"代替"二进制"
4. 禁止光环：主角会受伤、会恐惧、会失败

【写作风格】
- 感官优先：用触觉、嗅觉、听觉替代视觉奇观
- 延迟解释：展示现象，不解释原理
- 物质性：泥板、河水、芦苇、伤口是主要意象
- 受限视角：只写POV角色感知到的，不用全知视角

【输出要求】
- 场景1（钩子）：1000字，立即抓住读者
- 场景2（发展）：1200字，展开冲突
- 场景3（高潮）：1000字，核心揭示
- 场景4（心法+钩子）：300字，主题升华+悬念
```
