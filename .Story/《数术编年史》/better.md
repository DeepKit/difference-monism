# 《数术编年史》改进建议记录

---

## 问题1：规划与写作的倒置风险

**结论：不成立。**

AI主写、人类为辅的前提下，规划先行完全正确。5000集的线索锚点一旦断裂，后期修复成本是灾难性的。规划文档是AI的约束边界，没有这些文档AI会漂移。

## 问题2：用AI写作却要去AI味——结构性矛盾

**结论：C6检测器是必要的但不充分的。去AI味的根本解法在输入端。**

### 核心诊断

AI文字的根本问题不是句式，是选择逻辑：
- 人类作者选择细节，因为"我看见了那块泥板"
- AI选择细节，因为"这个细节在统计上合理"
- C6能检测对称句式，但检测不出这种根本性差异

AI写作的另一个典型缺陷：情感精确度不足。人类写悲伤会给出奇怪的、不对称的细节（"他没有哭，只是一直在折那张纸"）。AI给出完整的、合理的、没有任何意外的悲伤。

长篇AI写作的读者疲惫感来源：每一集都有情绪，但情绪和情绪之间没有连接，像一串断掉的珠子。

### 解决方向

去AI味的责任从检测后移到输入前。契约改进方向：
1. 每个角色在本集的"场景意图"——他自己都不完全意识到的欲望
2. 至少一个反常规的具体感官细节（sensory_anchor字段，已加入契约模板）
3. 一个不对称的情感表达方式（不是"他感到悲伤"，而是一个奇怪的行为）

C6作为事后补救保留，但不作为主要防线。

## 问题3：哲学框架有成为"显性内容"的风险

**结论：契约taboos加入硬性禁令，must_land必须用可观察行为表达。已更新契约模板和04写作指南C_实操.md（C7节）。**

- 禁止叙述者或角色使用规划文档概念词（数、术、命、不完备意识、ProtectionCoefficient等）
- must_land写法：❌"展示萨尔贡的不完备意识开始丧失" ✅"萨尔贡第一次没有质疑祭司报出的数字，因为他太累了"

## 问题4：守护者传承第25代=读者——元叙事设计

**结论：采用A进化版（持续共振型）——早期植入 + 长期共振 + 最终确认升级。**

- 不藏秘密，从早期就让读者感受到"这个故事在对我说话"
- 第5000集是情感确认，不是揭秘
- "缺失的一代"设计：不公布第25代，让读者自发猜测并传播
- 每100集一个小轮回，守护者感阶段性确认
- 不同纪元用不同情感触点，第5000集语言足够开放，所有情感都能投射进去
- 详见：情感映射.md

## 问题5：封印机制与长篇叙事的现实冲突

**结论：封印分三层，契约在写作和发布前均可优化。**

| 层级 | 内容 | 规则 |
|------|------|------|
| 圣经层（真正封印） | 数术命三元、五纪时间线、守护者传承结构、核心命题 | 发布后不可改，内部也不可改 |
| 叙事层（发布封印） | 已发布集数的具体情节、角色命运 | 发布后对读者不可改，未发布内容可调整 |
| 规划层（可调整） | 未发布集数的情节规划、支线走向、契约内容 | 随时可调整，不需要封印 |

## 问题6：平台定位与文体身份

**结论：B为主、A为辅的双层结构，"守护者残页"替代"今日算法心法"。**

| 层级 | 内容 | 规则 |
|------|------|------|
| 正文（100%） | 纯叙事，知识只通过冲突、选择、失败、遗物、残简传递 | 禁止讲台式解释 |
| 守护者残页（可选） | 未完成的推演、被涂改的公式、失败实验记录、断裂的逻辑图 | 偶尔出现，不强制每集都有 |

- 情绪化而非概念化：❌"贝叶斯更新的本质是概率修正" ✅"我们不是输给了敌人，是输给了自己的旧信念"
- 让读者在评论区自己写出理论，读者完成抽象时故事变成他们自己的
- 知乎盐选强调：守护、选择、代价、误判、更新。算法是骨架，不做显性卖点

## 问题8：历史研究的规模问题

**结论：创建写作素材考古.md，每100集进行物质细节约定。**

- AI只从考古文档取用物质细节，不自由发明
- 不确定的细节用模糊处理（"一个容器"而非"一个陶罐"）
- 契约taboos已加入：禁止使用考古文档未收录的具体历史器物名称
- 详见：写作素材考古.md

## 商业变现模型优化

**结论：按付费小说变现模型（点击→追读→付费→续订→口碑传播）全面优化，落地六项行动。**

### 核心公式
强缺口 + 高悬念 + 情绪依附 + 稳定高潮循环

### 落地项

| 行动项 | 落地位置 |
|--------|----------|
| 契约新增`surprise_variable`字段 | 04写作指南B_框架.md |
| 契约新增三种集数类型（standard/hook/climax） | 04写作指南B_框架.md |
| 镜像场景对照表（人性循环暗线） | 镜像场景对照表.md |
| 金句库（写作中自然收集，不预先制造） | 金句库.md |
| 前三章商业钩子规范 | 04写作指南C_实操.md 第八节 |
| 情感副线设计规范 | 04写作指南C_实操.md 第九节 |

### 关键原则
- 人物驱动体系，不是体系驱动人物
- "人性循环"作为暗线，禁止角色直接讨论
- 每10集一次结构级爆点，用集数类型模板自动化
- 主角必须犯错（surprise_variable强制执行）



**结论：三条设计原则已写入04写作指南C_实操.md 第六节。**

| 设计 | 内容 |
|------|------|
| 失败有人格后果 | 每次失败在某个人身上留下不可逆的印记，不是白失败 |
| 失败有节奏 | 小裂缝（统计混沌时刻）穿插在大失败之间，不允许连续20集纯失败 |
| 失败有涟漪 | 契约新增failure_ripple字段，记录失败的传导链 |
