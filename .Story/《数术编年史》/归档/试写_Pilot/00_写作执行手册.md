# 《数术编年史》写作执行手册

**版本**：v3.6+  
**适用范围**：第1-10集试写（Pilot阶段）  
**目标**：验证世界观、测试节奏、建立标准

---

## 一、写作流程总览

```
阶段1：准备（每集开始前）
├── 确定ASTO坐标（Mode/Phase/Sequence/ProtectionCoefficient）
├── 选择核心素材（1-2个历史案例）
├── 明确线索关联（埋下/回收/活跃）
└── 确定角色视角（六类角色轮换）

阶段2： drafting（写作中）
├── 场景1：钩子（1000字）
├── 场景2：发展（1200字）
├── 场景3：高潮（1000字）
└── 场景4：心法/钩子（300字）

阶段3：验证（写作后）
├── 自检清单（17项）
├── 历史准确性核查
├── 线索一致性检查
└── ProtectionCoefficient状态确认

阶段4：封印（通过后）
├── 发布到版本控制
├── 更新线索追踪表
└── 标记为不可修改
```

## 二、每集3500字详细结构模板

### 标准集结构（基于ASTO）

---

### 【集首信息】（不计算在3500字内）

```yaml
集数: 第X集
标题: XXX
ASTO坐标:
  Mode: M1/M2/M3/M4/M5
  Phase: P1/P2/P3/P4/P5/P6
  Sequence: S1/S2/S3/S4/S5/S6/S7
  ProtectionCoefficient: 高(0.7-1.0)/中(0.4-0.6)/低(0-0.3)
角色类型: 技术官僚/神秘解释者/反模式执行者/逃避者/日常抵抗者/不完备意识守护者
核心素材: ECO-001/WAR-003/etc.
线索:
  埋下: CLUE-XXX
  回收: CLUE-XXX
  活跃: CLUE-XXX, CLUE-XXX
```

---

### 【场景1：钩子】（1000字）

**功能**：建立悬念、引入角色、展示技术

**结构**：
```
[开篇100字] 感官细节，直接进入场景
  └── 展示某种计数技术如何改变感知（S1感知）

[发展400字] 角色行动，引入冲突
  └── 角色面对技术/制度的具体情境
  └── 展示ProtectionCoefficient的初始状态

[转折300字] 意外发现或事件
  └── "错误"的显现 / 不完备性的暴露
  └── 悬念建立

[钩子200字] 引出下一场景的核心问题
  └── 集间钩子（如果非第1集，呼应上一集）
  └── 本集核心悬念
```

**检查点**：
- [ ] 是否展示了某种计数技术？
- [ ] 是否建立了情感锚点？
- [ ] ProtectionCoefficient状态是否清晰？

---

### 【场景2：发展】（1200字）

**功能**：展开冲突、展示权力运作、引入抵抗

**结构**：
```
[分析400字] 权力机制的展示（S2分析）
  └── 技术官僚的视角：制度如何运作
  └── 或被计数者的视角：压迫如何感受

[干预400字] 日常抵抗或逃避（S3干预）
  └── 偷懒、误读、规则套利、隐藏
  └── 展示命的不可计算性

[升级400字] 冲突加剧
  └── 系统的反应：监视、惩罚、同化
  └── 或抵抗的升级：从个体到群体
```

**检查点**：
- [ ] 是否展示了数术张力？
- [ ] 是否展示了六类角色中的一类？
- [ ] 是否有历史质感的细节？

---

### 【场景3：高潮】（1000字）

**功能**：核心揭示、认识论伦理选择、ProtectionCoefficient检查

**结构**：
```
[揭示400字] 核心真相或选择
  └── 不完备性的暴露 / 完备性宣称的冲突
  └── 角色面临认识论伦理选择

[选择300字] 角色的决定
  └── 保持不完备意识（ProtectionCoefficient高）
  └── 或接受完备性宣称（ProtectionCoefficient低）

[后果300字] 决定的直接后果
  └── 成功/失败/代价
  └── 为场景4铺垫
```

**检查点**：
- [ ] 是否有情感高潮？
- [ ] 是否展示了认识论伦理选择？
- [ ] ProtectionCoefficient是否发生变化？

---

### 【场景4：心法/钩子】（300字）

**功能**：主题升华、集间钩子、读者反思

**结构**：
```
[反思150字] 角色的内心独白或对话（S6反思）
  └── 认识论伦理的总结
  └── 不完备意识的保持或丧失

[钩子150字] 引出下一代技术或人物
  └── 技术演进钩子
  └── 人物传承钩子
  └── 或悬念延续钩子
```

**检查点**：
- [ ] 是否有主题升华？
- [ ] 是否有有效的集间钩子？
- [ ] 是否为读者留下思考空间？

## 三、第1-10集详细写作计划

### 写作顺序建议

| 顺序 | 集数 | 难度 | 优先级 | 原因 |
|------|------|------|--------|------|
| 1 | 第1集 | 高 | 最高 | 世界观入口，必须完美 |
| 2 | 第5集 | 中 | 高 | 情感锚点，母亲记忆 |
| 3 | 第9集 | 中 | 高 | 不完备意识守护者登场 |
| 4 | 第10集 | 中 | 高 | 篇章闭环，钩子回收 |
| 5 | 第2-4集 | 低 | 中 | 填充叙事，建立节奏 |
| 6 | 第6-8集 | 中 | 中 | 冲突升级，逃亡开始 |

### 每集核心任务

#### 第1集《河边的孩子》
**核心任务**：
- 建立世界观基础（数术命三元结构）
- 引入主角萨尔贡（日常抵抗者）
- 展示"六十一点横"（不完备性的具象化）
- ProtectionCoefficient：高（天然不完备意识）
- 埋下长期线索：CLUE-L-1, CLUE-L-2

**关键场景**：
- 开篇：萨尔贡在河边制作泥板（感官细节）
- 发展：发现"错误"——六十一点横
- 高潮：选择是否报告错误
- 结尾：河边老人的出现，抛出终极问题

#### 第2集《学徒的疑惑》
**核心任务**：
- 展示技术官僚视角（学徒）
- 建立友谊与背叛的张力
- ProtectionCoefficient：中（开始感知权力）

#### 第3集《巴比伦的税收官》
**核心任务**：
- 临时插播，展示时代背景
- 经济算法ECO-001的应用
- 结构性暴力的展示

#### 第4集《监视的眼睛》
**核心任务**：
- 神秘解释者视角（祭司长）
- 监视机制的建立
- ProtectionCoefficient：低（监视开始）

#### 第5集《母亲的记忆》
**核心任务**：
- 情感核心集
- 萨尔贡回忆母亲（也是不完备意识者）
- ProtectionCoefficient：高（回忆中的自由）
- 揭示传承的源头

#### 第6集《希帕索斯的幽灵》
**核心任务**：
- 历史插播，知识暴力的先例
- COST-001的应用
- 预示萨尔贡的命运

#### 第7集《六十一点横》
**核心任务**：
- 技术揭示集
- 展示六十一点横的规律
- 不完备性的首次理论化

#### 第8集《背叛的边缘》
**核心任务**：
- 学徒的背叛发生
- 友谊与自保的冲突
- 逃亡的触发点

#### 第9集《河边老人》
**核心任务**：
- 不完备意识守护者正式登场
- 教导不完备意识的四个维度
- 传承的开始

#### 第10集《七十二柱》
**核心任务**：
- 篇章闭环
- 揭示知识垄断集团（72柱的现实版）
- 萨尔贡被标记为"异常"
- 回收短期线索，埋下中期线索

## 四、质量控制清单（17层）

### 内容层（5项）
- [ ] **历史准确性**：技术细节是否准确？
- [ ] **世界观一致性**：是否符合数术命v3.6+？
- [ ] **线索一致性**：埋下/回收/活跃是否正确？
- [ ] **角色一致性**：六类角色行为是否符合定义？
- [ ] **ProtectionCoefficient**：状态是否清晰且合理变化？

### 叙事层（6项）
- [ ] **悬念建立**：开篇是否吸引人？
- [ ] **节奏控制**：3500字内是否有起伏？
- [ ] **情感锚点**：是否有共鸣时刻？
- [ ] **主题揭示**：数术命张力是否展示？
- [ ] **钩子有效性**：集间钩子是否有效？
- [ ] **闭环完整性**：本集悬念是否适当解决？

### 文学层（4项）
- [ ] **感官细节**：是否有历史质感？
- [ ] **对话质量**：是否符合角色身份？
- [ ] **场景转换**：是否流畅自然？
- [ ] **语言风格**：是否统一且适合史诗？

### 技术层（2项）
- [ ] **字数控制**：是否在3500±200字范围内？
- [ ] **格式规范**：是否符合集首信息模板？

## 五、写作工具和工作流

### 推荐工具栈

| 功能 | 工具 | 用途 |
|------|------|------|
| 写作 | Markdown编辑器 | 纯文本，版本控制友好 |
| 版本控制 | Git | 封印机制，历史回溯 |
| 线索追踪 | Excel/Notion | 三层伏笔管理 |
| 素材查询 | 本地素材库 | 快速查找历史案例 |
| 质量检查 | 自检清单 | 17层验证 |

### 文件命名规范

```
第XXX集_[标题]_[状态].md

状态：
- draft：草稿
- review：待审核
- sealed：已封印（通过验证）

示例：
第001集_河边的孩子_sealed.md
```

### 每日写作流程

```
09:00-09:30  准备阶段
             ├── 确定ASTO坐标
             ├── 选择素材
             └── 回顾线索状态

09:30-12:00  写作阶段（场景1-2）
             ├── 场景1：钩子（1000字）
             └── 场景2：发展（1200字）

14:00-16:00  写作阶段（场景3-4）
             ├── 场景3：高潮（1000字）
             └── 场景4：心法/钩子（300字）

16:00-17:00  验证阶段
             ├── 自检清单（17项）
             ├── 历史准确性核查
             └── 修改完善

17:00-17:30  封印阶段
             ├── 更新线索追踪表
             ├── Git提交
             └── 标记为sealed
```

## 六、试写阶段目标

### 第1-10集试写成功标准

1. **世界观验证**：读者能理解数术命三元结构
2. **节奏验证**：3500字/集的节奏是否合适
3. **角色验证**：六类角色是否清晰可辨
4. **线索验证**：三层伏笔系统是否有效
5. **主题验证**：不完备意识是否能被感知

### 试写后评估

完成第1-10集后，进行：
- 读者测试（3-5位目标读者）
- 节奏分析（阅读时间、 dropout率）
- 主题理解测试（是否能解释数术命）
- 调整后续490集的规划
