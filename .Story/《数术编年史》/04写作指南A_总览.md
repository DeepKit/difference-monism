## 二、每集标准结构

### 2.1 字数与节奏

```
总字数：3500字（±200）
阅读时长：约12-15分钟

【场景1：钩子】1000字（3分钟）
├─ 功能：立即抓住读者，建立悬念
├─ 技巧：从动作或对话开始，避免冗长描写
└─ 必须：展示技术如何改变感知

【场景2：发展】1200字（4分钟）
├─ 功能：建立情境，铺垫高潮
├─ 技巧：交替进行人物互动和知识植入
└─ 必须：展示权力运作机制

【场景3：高潮】1000字（3分钟）
├─ 功能：情感或智力的顶点
├─ 技巧：压缩时间，增加紧迫感
└─ 必须：核心揭示，伦理选择

【场景4：心法+钩子】300字（1分钟）
├─ 功能：给读者收获感和期待感
├─ 技巧：心法用比喻，钩子用悬念
└─ 结构：【今日算法心法】+【下一集预告】
```

## 三、创作禁令（四条红线）

> **守住这四条，作品就是独一档。**

### 红线一：禁止发光

- **禁止**：除了火、阳光、自然生物发光，没有任何人体或物体会无端发光
- **替代**：用触觉、温度、质感替代视觉奇观
- ❌ "他的眼睛发出金光"
- ✅ "他的瞳孔在黑暗中扩张，像是要吞噬所有光线"

### 红线二：禁止预知

- **禁止**：河边老人不是预言家，不能准确预测具体事件
- **替代**：他是"历史的复读机"，看清循环模式，能预测大方向
- ❌ "三天后神殿会倒塌"
- ✅ "当账目乱到一定程度，饥荒必至——这不是预言，是算术"

### 红线三：禁止术语

- **禁止**：萨尔贡绝不可说出"二进制"、"算法"、"系统"等现代词汇
- **替代**：用时代语言表达
- ❌ "这是二进制编码"
- ✅ "阴阳，双手，刻痕的逻辑——零和一，世界的基本呼吸"

### 红线四：禁止光环

- **禁止**：萨尔贡会受伤、会恐惧、会失败，没有主角不死定律
- **替代**：他的强大在于看穿了吓唬人的"神迹"背后，仅仅是数学
- ❌ 以一敌百，全身而退
- ✅ 利用逻辑漏洞制造混乱，趁乱逃脱，但留下永久性伤疤

## 四、质量检查清单（核心10项）

### 4.1 内容层（4项）

- [ ] **1. 历史准确性**：技术细节是否准确？虚构人物是否符合时代背景？
- [ ] **2. 世界观一致性**：是否符合数术命三元结构？六类角色行为是否符合定义？
- [ ] **3. 线索管理**：埋下的线索是否正确？回收的线索是否完整？
- [ ] **4. 主线推进**：三条主线是否有推进？是否展示了数术命张力？

### 4.2 叙事层（4项）

- [ ] **5. 悬念建立**：开篇是否吸引人？悬念是否与主题相关？
- [ ] **6. 节奏控制**：3500字内是否有起伏？场景转换是否流畅？
- [ ] **7. 情感锚点**：是否有共鸣时刻？情感是否真实而非煽情？
- [ ] **8. 钩子有效性**：集间钩子是否有效？是否激发读者好奇？

### 4.3 文学层（2项）

- [ ] **9. 物质性描写**：是否用五感细节替代抽象概念？是否有历史质感？
- [ ] **10. 红线检查**：是否违反四条创作禁令？

## 五、快速参考卡

### 5.1 ASTO六阶情绪曲线

| 六阶 | 叙事特征 | 写作要点 | 读者感受 |
|------|----------|----------|----------|
| **a 混沌** | 信息碎片、氛围铺垫 | 意识流、感官细节、不解释 | 困惑、好奇 |
| **b 秩序** | 规则建立、关系明确 | 信息交换、人物关系确认 | 清晰、安心 |
| **c 流变** | 渐变积累、张力蓄压 | 节奏加速、细节累积 | 紧张、期待 |
| **d 脉冲** | 突变、高潮、不可逆 | 压缩时间、强烈冲突 | 震撼、兴奋 |
| **e 崩解** | 旧秩序瓦解、失控 | 失控感、情感释放 | 悲伤、失落 |
| **f 归元** | 沉淀、收束、新平衡 | 慢节奏、留白、新起点 | 释然、期待 |

### 5.2 概念物质化转换

| 抽象概念 | 物质化表达 | 示例 |
|----------|-----------|------|
| 二进制 | 泥板上的横线、手指的敲击 | "六十一条横线，像是一道未完成的栅栏" |
| 算法 | 水流的轨迹、风的路径 | "河水知道该往哪里流，即使没有人告诉它" |
| 信息 | 气味、温度、重量 | "那封信带着她的体温，还有某种他说不上来的悲伤" |
| 系统 | 建筑的结构、身体的习惯 | "他的手指自己动了，像是被某种看不见的线牵引着" |
| 错误 | 裂痕、污渍、不对称 | "那个符号歪了，但正是那个歪斜让它看起来...真实" |

### 5.3 【今日算法心法】格式

**字数**：200字以内  
**结构**：概念简述（1句）+ 历史连接（1句）+ 实践指导（3-5句）  
**语气**：不是说教，而是"思维武器"的交付

**示例**：
```
【今日算法心法】

结绳与二进制：算法的第一次降临

人类最早的"算法"，不是写在纸上，而是刻在泥里、系在绳上。
古巴比伦人用泥板记录洪水周期——那是人类最早的"数据存储"。
印加人用结绳（Quipu）传递信息——那是人类最早的"二进制通信"。

万物皆可编码。当你面对复杂问题时，试着把它"敲碎"——
变成最小单位的"是"与"否"、"0"与"1"。
当你把所有模糊的东西都变成确定的状态，答案就会显现出来。
```

## 六、正文文件命名规范

### 6.1 命名格式

```
0{纪}纪0{篇}篇-{集数4位}.{篇章名}.{集标题}.md
```

**示例**：
```
01纪01篇-0001.觉醒.河边的孩子.md
01纪01篇-0002.觉醒.泥板的秘密.md
01纪02篇-0011.圆的分割者.七十二柱.md
02纪01篇-0101.原子与道.XXX.md
```

### 6.2 规则说明

| 字段 | 格式 | 说明 |
|------|------|------|
| 纪编号 | 01-05 | 两位数字，对应五纪 |
| 篇编号 | 01-10 | 两位数字，每纪10篇 |
| 集数 | 0001-5000 | 四位数字，全局唯一，跨纪连续 |
| 篇章名 | 中文 | 该篇章的名称 |
| 集标题 | 中文 | 该集的标题 |

### 6.3 存放位置

所有正文文件统一存放在对应纪目录下，不再建子目录：
```
第1纪_神谕与刻痕/
├── 第1纪_总纲.md
├── 第1纪_详细规划_第1-100集.md
├── 01纪01篇-0001.觉醒.河边的孩子.md
├── 01纪01篇-0002.觉醒.泥板的秘密.md
└── ...
```
