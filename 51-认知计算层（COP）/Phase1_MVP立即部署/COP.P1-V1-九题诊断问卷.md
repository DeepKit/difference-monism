# COP · Phase 1 · V1 — 九题诊断问卷

> **定位**：这是 `COP Phase 1` 当前场景适配包的输入层。
> **边界**：`A-D + 9题` 只代表当前一条可上线采样包，不等于 `COP` 的全部状态本体。

---

## 使用规则（必须对用户展示）

1. 共 `9` 题，全部为客观选择。
2. 每题只选当前最符合现实的一项，不选“理想情况”。
3. 若无法判断或信息不足，应允许退出问卷并进入补充采样，而不是强迫完成。
4. 本问卷只提供结构诊断入口，不直接提供高风险执行许可。

---

## 一、诊断问卷

### Q6. 你能否看到大量失败案例？

### Q7. 当前策略是否可能被“利用规则套利”？
* A. 基本不可能

### Q8. 当前是否存在明确“停止条件”？
* A. 有清晰停止线
* B. 有模糊标准
* C. 基本没有

### Q9. 当前系统是否存在“外部强制约束”？

## 二、当前四类候选类型

| 类型 | 名称 | 当前含义 |
|:---|:---|:---|
| **Type A** | 共识幻觉型 | 表面一致但内部失真严重 |
| **Type B** | 死锁型 | 双方或多方结构依赖卡死 |
| **Type C** | 阈值失效型 | 边界不断扩张，缺少停止机制 |
| **Type D** | 沉没成本型 | 因既有投入而持续加码 |

```text
当前更稳的处理方式：
1. 先补信息或人工复核
2. 暂不把类型结果直接当成执行许可
3. 若进入高风险状态，按上游接口升级
```

## 四、当前动作输出纪律

### 1. `RESOLVE`

允许输出低强度结构建议，
例如：

- 增补责任接口
- 明确冲突点
- 设置停止条件

### 2. `MIXED`

允许输出候选类型差异与补采样建议，
不应强压成单一结论。

### 3. `FREEZE / UNKNOWN / REFER`

只允许输出以下类型动作：

- 补录信息
- 暂停自动闭环
- 升级人工复核
- 回接 `TAT / ODD / ECET`

不允许输出：

- 高强度行为命令
- 直接替代人类裁决
- 绕过责任门槛与工程审计

## 五、各类型的当前候选探针

这些不是“必须动作命令”，

而是仅在 `RESOLVE / MIXED` 下可用的低强度候选探针。

### Type A：共识幻觉型

- 候选探针：公开核对关键前提、私下验证真实意见、补录责任确认点
- 避免动作：继续顺从表面一致、把沉默误判成共识

### Type B：死锁型

- 候选探针：显式画出依赖链、把冲突方放到同一讨论场、记录谁拥有拍板权
- 避免动作：继续单点传话、继续让中间人吸收冲突

### Type C：阈值失效型

- 候选探针：补写停止条件、补写新增需求交换规则、限定试点范围
- 避免动作：继续接受无边界追加、继续拖延阈值决策

### Type D：沉没成本型

- 候选探针：把过去投入与未来选择分开核算、补写退出条件、引入外部复核视角
- 避免动作：只因既有投入继续决策、把未来收益当作过去损失的自动补偿
