## 架构决策：一套引擎，两个产品（品牌隔离，数据暗合）

> 经专家组从第一性原理论证，COP 引擎采用以下架构：

| | **goodmem.cn/009** | **asto.cc/diag** |
|:---|:---|:---|
| **目标用户** | 夹心层（L1） | 出资人/老板（L3） |
| **漏斗位置** | 付费后的个性化导读 | 付费前的免费体检 |
| **入场框架** | 个人项目困境（你卡住的那件事） | 组织财务损耗（你的系统摩擦税） |
| **结果页 CTA** | 推荐优先阅读篇章 → 96篇文章库 | 失血量估算 → TAT首诊申请 |
| **视觉风格** | 温暖·知性 | 极简·冷酷·临床级 |
| **后端引擎** | ✅ 共用同一套 COP 算法 | ✅ 共用同一套 COP 算法 |
| **数据归属** | 匿名汇入同一样本库（供 V4 训练） | 匿名汇入同一样本库（供 V4 训练） |

**品牌上绝对隔离，数据上暗线合并。两者共享 V4 训练数据，加速双倍飞轮。**

## 当前最优先动作（Phase 1 战役目标）

> **把 V1-V3 做成两个独立的网页产品 MVP。**

**第一优先：`asto.cc/diag`（L3 老板线）**
1.  用 V1 的 9 道题（组织视角包装）+ V2 权重矩阵做前端
2.  用 V3 的 `/analyze` API 做后端计算
3.  输出结果页：类型 + 副标签 + 系统摩擦税估算 + 风险等级
4.  结果页底部硬植入《TAT首诊申请表》入口（一号位验资）

**第二优先：`goodmem.cn/009`（L1 夹心层）**
1.  相同后端算法，更换个人视角的题目包装
2.  结果页：类型 + 对应推荐篇章列表（付费用户专属）
3.  与 asto.cc 完全隔离，不共享任何视觉或入口

**Phase 1 MVP 覆盖的核心痛点：**
- ✅ 老板进入泊位后"感到极度恐惧"→ 精准类型判定做到
- ✅ 信任凭空建立→"懂病"胜过任何广告
- ✅ 100%回流 asto.cc→ 没有任何外链跳出
- ✅ 品牌隔离→ 老板不会看到夹心层产品
- ✅ 数据飞轮→ 两条线共同喂养 V4
