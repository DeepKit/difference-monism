# COP · Phase 2 · V4 — 有门槛的动态权重学习机制

> **启动条件**：不再是“有 500 条反馈就自动启动”，而是必须同时满足 `反馈量 + 人工复核 + 版本回滚条件`。
>
> **核心价值**：不是让系统“自己越学越大胆”，而是让它在误判成本可控、复核一致性达标的前提下小步校准。

---

## 核心思想

> 哪种判定在 `已确认执行 + 已过观察窗口 + 已完成人工复核` 的条件下带来更好结果，才允许有限修正对应权重。

换句话说：

1. 不能只看用户一句 `improved / worse`
2. 不能把未执行动作、环境扰动、服从性偏差直接学进模型
3. 不能在高误判成本窗口里继续自动学习

---

## 一、启动前提

Phase 2 当前至少要求同时满足：

- `500+` 条有效反馈
- `100+` 条高风险人工复核样本
- `triage_status agreement >= 0.85`
- 近一窗口未触发回滚条件

并且已经具备：

- `COP.校准样本协议.v1.md`
- `COP.误判样本库规范.v1.md`
- `COP.版本发布纪律.v1.md`

若不满足，`V4` 只保留数据记录，不进入自动学习。

---

## 二、权重结构（新增可训练层）

每个权重值由两层构成：
```json
{
  "base_weight": 3,      // Phase 1 人工定义的初始值（固定）
  "learned_weight": 1.0  // Phase 2 开始随数据动态修正（初始值 1.0）
}
```
实际生效权重：
```
final_weight = base_weight × learned_weight
```

---

## 三、反馈进入学习池的前提

只有同时满足以下条件的反馈，才允许进入学习池：

1. 用户确实执行了动作
2. 已经过最小观察窗口
3. 结果不是明显受外部不可控事件单独主导
4. 样本已通过最小人工复核或抽查

不满足时，应标记为：

- `not_executed`
- `confounded`
- `disputed`
- `log_only`

这些样本可留档，但不应直接改权重。

进入学习池前，样本资格与标注格式应优先遵守 `COP.校准样本协议.v1.md`。

---

## 四、反馈更新规则

| 复核后结果 | 权重修正量 |
|:---|:---|
| `result = "improved"` | `learned_weight += 0.05` |
| `result = "unchanged"` | `learned_weight -= 0.02` |
| `result = "worse"` | `learned_weight -= 0.07` |

**防崩坏限制**：
```python
learned_weight = clamp(learned_weight, 0.5, 1.5)
```

---

> 上表只是当前 `v1` 的占位更新幅度，不得在未满足启动前提时自动生效。

## 五、动作效果学习

每个动作独立维护成功率：
```json
{
  "action": "拉群对撞",
  "success_count": 31,
  "fail_count": 12,
  "success_rate": 0.72
}
```

更新逻辑：
```python
if result == "improved":
    success += 1
else:
    fail += 1
success_rate = success / (success + fail)
```

输出时附带置信度：
```json
"actions": [
  {"name": "拉群对撞", "confidence": 0.72},
  {"name": "引入上级决策", "confidence": 0.58}
]
```

## 六、学习过程中的硬纪律

1. `False Safe 优先级最高`
只要高风险漏判上升，就应先停学习，再谈精度提升。

2. `先看复核一致性，再看总体准确率`
若复核者都无法稳定同意，模型不应继续自改。

3. `必须保留回滚能力`
任何一轮更新都必须能退回上一稳定版本。

4. `不能让学习层吞掉基础权重`
`base_weight` 仍是稳定锚点，`learned_weight` 只是受限修正层。

5. `误判样本必须先入库再谈学习`
若根因标签、复核结论与争议状态未按 `COP.误判样本库规范.v1.md` 留档，则该样本不得参与自动校准。
