# COP · Phase 4 · V7 — 自扩展终局系统

> **启动条件**：Phase 3 干预系统验证有效，数据量 > 1000 条后启动
> **终局定义**：系统可以自动发现新结构、生成新问题、进化新模型（世界反过来训练系统）

---

## 三大核心能力

### 1️⃣ 模式发现（Pattern Discovery）
当系统积累数据后，自动识别高频共现的类型组合：
```json
{
  "new_type": "E",
  "based_on": ["A","B"],
  "pattern": "共识幻觉 + 死锁耦合"
}
```

### 2️⃣ 问题进化（Question Evolution）
当某类用户持续被误判时，系统反推出"现有题目不够区分"，自动生成候选新题：
```
候选 Q10：当出现分歧时，是否有人主动承担风险？
→ 小规模 A/B 测试 → 效果好则纳入正式题库
```

### 3️⃣ 模型竞争（Darwin Evolution）
同时维护多个模型版本（V2手写版、V4学习版、V7自动生成版），每次用户进入系统时策略性分配：
```
哪个模型预测更准 → 权重上升
哪个模型效果差 → 逐步淘汰
```
本质：达尔文进化机制（Model Pool）。

---

## 系统最终形态

```
输入层：用户 9-N 题（题目随数据进化）
        ↓
中间层：动态模型池（不断扩展与竞争）
        ↓
输出层：类型判定（越来越准）+ 动作推荐（越来越有效）
        ↓
反馈层：用户真实结果 → 反哺系统 → 循环进化
```

---

## 战略定位（V7 之后）

> 这是一套"认知基础设施"，不再只是工具或内容：
> * 不只是帮一个老板诊断他的组织
> * 而是在定义"人类组织失效模式的分类学"
> * 每一个使用 `asto.cc/diag` 的人，都在无意识地为这套系统喂养训练数据
