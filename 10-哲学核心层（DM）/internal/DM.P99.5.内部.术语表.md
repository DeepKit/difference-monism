## 一、使用规则

1. `以封版主链为准`
若内部讨论和本文冲突，应先回读 `P00-P11`。

2. `一词一核心义`
不允许同一术语在不同文档里随意漂移。

3. `对外先给近邻概念，再讲 DM 差异点`
不要直接把缩写和新词扔给外部读者。

4. `不追求百科式穷尽`
这里给的是最低稳定义、可对接对象与常见误读，不承担全部长论证。

## 三、最容易混淆的几组词

| 不应混淆 | 这里的区分 |
|----------|----------|
| 差异 / 结构 | 差异是起点条件；结构是差异进入非任意约束后的组织形态 |
| 时间 / 过程词 | DM 优先用成立条件依赖与继承序，避免用未经压缩的过程词顶替论证 |
| 内部性 / 自指 | 内部性早于自指；自指是内部性进一步折回处理的结果 |
| 报告 / 内部进入 | 报告是证据链一环，不等于内部进入本身 |
| 暂稳态 / 刚性稳定 | 前者保留跃迁与改道空间，后者优先压低变化 |
| 结构描述 / 价值选择 | 前者描述何种结构出现，后者声明 DM 站在哪边 |
| 责任 / 责备 | 责任先是可归属性和承接结构，责备是更高阶的规范与制度判断 |
| 发生学解释 / 正当性结论 | 前者说明“如何长出来”，后者说明“为什么应当被接受” |

## 四、对外输出的最低纪律

1. `先给近邻概念，再给 DM 差异点`
比如先说“纠错能力接近 corrective capacity，但 DM 把它放到文明韧性的核心变量位置”，再进入正文。

2. `不要把术语表当论证替身`
术语对接只是拆门槛，不是替代长论证。

3. `不要把新词当创新本身`
术语新增只有在它确实增加解释力、区分力或裁决力时才成立。

4. `对外若只给一份桥接件，优先给 P99.32`
术语表偏词汇层；公共接口包才是最短完整入口。

## 五、内部结论

最短压缩句：

1. `DM 最怕的不是术语多，而是术语既不能稳定使用，也不能公共对接。`
2. `P99.5 的升级，不是要把 DM 稀释成主流词汇，而是要让外部读者先有共同靶面，再决定是否反对。`
