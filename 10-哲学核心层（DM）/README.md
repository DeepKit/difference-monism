## 冠军命题

> **许多哲学难题之所以长期僵住，是因为我们先把世界想成一组现成实体，再追问它们如何桥接；DM 改从差异结构及其组织阈值出发，把意识重写为内部性问题，把规范与制度重写为可归属性与可复归程序问题。**

## 三条边界

- `DM` 不是已完成同行评审与长期外部验证的终局理论，而是一个可送审、可受损、可继续压缩的研究纲领
- `DM` 目前不宣称已经完成意识问题的第三人称闭合，也不宣称已给出制度或文明的终局裁决
- `DM` 负责最小本体论与规范地基，不直接替代下游工程、治理、制度设计与场景执行细则

## 当前定位

DM 当前最稳的公开描述是：

> **一个以差异结构与组织阈值为统一语法、已形成主链、失败线与首批案例档案的送审/征评型理论包。**

这意味着：

- 它已经不再只是内部零散草稿
- 它已经可以公开发布、公开征评与有限比较
- 它仍需接受外部批评、案例累积与长期验证

## DM 风险防火墙

### `DM-min` 最小承诺

若要避免 `DM` 一受攻击就拖垮全链，当前应优先把 `DM` 压成一个最小可保留内核：

1. 世界至少可以被描述为 `差异`
2. 差异会形成 `结构`
3. 结构需要 `边界`
4. 结构的保持、漂移、断裂与升级需要 `继承 / 约束 / 涌现`

这四项构成当前最小的 `DM-min`。

### `DM` 受争议时如何阅读本层

若外界不接受 `DM` 的强本体论、强意识论或强规范推演，
当前最稳的降级阅读方式是：

- 把 `DM` 先视为研究纲领
- 把 `DM` 先视为优选解释框架
- 把 `DM` 先视为统一术语与问题组织器

而不是继续把它包装成“若不接受就全体系不成立”的终局真理包。

### `DM` 对下游的防火墙纪律

从本层起，默认不再鼓励下游写成：

> 因为 `DM` 完整成立，所以后续层必然成立。

更稳的写法应是：

> `DM` 提供最高一致性的解释；下游层与 `DM` 相容，但可在更弱前提下继续成立。

### 本层防火墙句

> **`DM` 若受压，首先失去的是最强解释权，而不是自动宣告下游协议层、治理层与工程层全部失效。**

## DM 立足六条件

若要从“能说”推进到“站得住”，当前默认按以下六条件阅读 `DM`：

1. `最小硬核`
先守住 `DM-min`，不把所有强结论都塞进地基

2. `陈述分层`
分清 `本体承诺 / 规范承诺 / 机制命题 / 工具协议`

3. `真实失败线`
让 `P00 / P02 / P04 / P05 / P09` 真正保留缩限或重写触发

4. `公共接口`
让外部读者能按同一规则读、比、驳

5. `比较力`
稳定回答“比主要对手多解释了什么、最怕什么反例”

6. `实践裁决力`
在责任、危机、制度、文明判断上给出可复查导向

工具总表见：

- DM.P99.34.内部.DM立足六条件.md

## 对外阅读纪律

- 若目标是公开介绍，不要先抛整套正文，先给统一送审包。
- 若目标是学术比较，先给理论定位申明，再给统一送审包。
- 若目标是正式投稿准备，先给 `审稿人摘要`，再核对引用、案例档案与失败条件接口，再导出外发版本。
- 若目标涉及制度域或文明域的跨案例比较，先核对是否已有对应 `P99.27` 正式记录；当前可直接回指的正式记录为 `八组`。
- 若目标涉及制度域或文明域第一层案例，优先使用档案正文与优先序中的 `四项硬字段` 快照，不要只引用抽象原则句。
- 若目标是正式投稿，不建议整包直投；优先拆成 `单焦点论文`，例如 `P04 最小内部性` 或 `P09 纠错能力核心性`。
