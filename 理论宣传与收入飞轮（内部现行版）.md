# 理论宣传与收入飞轮（内部现行版）

> 性质：根级内部策略文件  
> 用途：回答“这套理论接下来该怎么宣传、怎么产生收入、又怎么把收入继续回流为推广、研究与部署能力”。

---

## 一、总判断

这套体系的良性循环，不应写成：

> 先把 `DM` 讲得越大越好，再自然有人来付费。

更稳的写法应是：

> 先用 `A 类` 法源理论立名，  
> 再用 `B 类` 摘要件传播与获客，  
> 再用 `C 类` 执行法收费，  
> 再把收入回流到研究、案例、产品与下一轮传播。

最短句：

> **不要卖“世界观本身”，要卖“基于这套世界观才能稳定交付的判断、诊断、设计与执行法”。**

---

## 二、三条线必须分开

### 1. 立名线

职责：

- 让外界知道这套理论是谁提出的
- 让 `DM -> ASTO -> ECET -> TAT -> ODD` 形成稳定法源地位
- 让 `DM-P00 / P04 / P09` 这类代表作成为外部讨论入口

主要对象：

- `DM`
- `ASTO`
- `ECET`
- `TAT`
- `ODD`
- `LMM` 公共理论面
- `WSM` 核心理论面

核心产物：

- 统一送审包
- 预印本 / 投稿稿
- 一页定位表
- 审稿人摘要
- 八层总图
- 体系简介

### 2. 传播线

职责：

- 让外部快速理解这套体系解决什么问题
- 把陌生读者变成持续关注者
- 把关注者导向可承接的问题场

主要对象：

- `README`
- 八层总图
- 体系简介
- 各层 `README / 白皮书摘要 / 发布说明`
- `60-科学对话层`
- `70-其他工具层` 的安全入口件

核心原则：

- 传播讲 `问题`
- 不先讲整包形而上学
- 不先抛完整执行法

### 3. 收费线

职责：

- 把理论压成可付费的交付
- 把交付变成案例、现金流与组织能力
- 把收入回流到理论与传播

主要对象：

- `COP`
- `LMM 04-08`
- `Harness`
- 根目录跨层执行协议件
- `Progee2`
- `.BetterCiv`

核心原则：

- 收费卖 `执行法`
- 不卖 `终极真理证明`
- 不把 `C 类` 细节白送

## 三、宣传怎么做

### 1. 先问题化，不先体系化

外部传播不要从“我有一个八层理论体系”开始，
而要从下面这类高痛点问题开始：

- 为什么很多组织会系统性不负责任
- 为什么危机治理不能长期停在永久例外
- 为什么一个制度真正的核心能力是纠错能力
- 为什么很多营销其实在制造责任真空
- 为什么 AI / 工程流程需要可审计、可回滚、可封存的方法

也就是说：

> **对外入口应先是问题，再是框架，最后才是母理论。**

### 2. 用三层传播材料

#### 第一层：抓注意力

用这些材料：

- 一页摘要
- 八层总图
- 体系简介
- 单篇问题文章
- 短讲 / 访谈 / 对话稿

目标：

- 让外部先知道“你在处理什么硬问题”

#### 第二层：建权威

用这些材料：

- `DM / ASTO / ECET / TAT / ODD` 各层摘要入口
- `DM-P00 / P04 / P09` 单焦点论文
- 科学对话层桥接件
- `DM` 的失败线与公共接口材料

目标：

- 让外部知道这不是情绪表达，而是可争辩的理论

#### 第三层：导承接

用这些材料：

- `LMM` 公共理论面
- `WSM` 的结构识别摘要版
- 脱敏案例
- 问题场入口页
- 港口接引页

目标：

- 让真正有问题的人进入你的承接系统，而不是只停在点赞层

### 3. 三类受众分三种话术

#### 学术受众

他们关心：

- 你新增了什么
- 你如何与现有理论比较
- 你最怕什么反例

所以给他们：

- `DM` 主链
- `P00 / P04 / P09`
- `P99.21 / P99.32 / P99.34`

#### 公共受众

他们关心：

- 这套理论能解释什么现实困局
- 它和日常判断有什么关系

所以给他们：

- 总图
- 简介
- 问题导向文章
- 限定性案例

#### 客户 / 合作方

他们关心：

- 你能帮我解决什么
- 你怎么做
- 风险怎么控制

所以给他们：

- 脱敏结构说明
- 交付边界
- 责任与审计纪律
- 付费前后的流程说明

## 四、收入怎么来

### 1. 第一层收费：低门槛入口产品

作用：

- 把“围观者”变成“付费进入者”
- 验证问题场是否真实存在
- 累积首批标准化问题样本

适合卖的对象：

- 结构识别自测件
- 问题场初筛报告
- 小型训练营 / 讲解课
- 问题诊断一页报告
- `LMM / WSM` 的入门级工具件

要求：

- 不泄露 `COP` 阈值与完整协议
- 不提前交付高价值执行法

### 2. 第二层收费：中客单诊断与顾问

作用：

- 把理论转成组织判断力
- 形成第一批高质量现金流
- 累积可脱敏案例

适合卖的对象：

- 责任架构审查
- 危机复归评估
- 组织结构诊断
- AI / 工程流程审计
- 制度与治理边界评估

理论映射：

- `TAT`
- `ODD`
- `ASTO`
- `ECET`
- `LMM`

### 3. 第三层收费：高客单部署与授权

作用：

- 形成真正军费
- 把体系变成不可轻易替代的交付能力

适合卖的对象：

- `COP` 诊断部署
- `Harness` 受控执行接入
- `ODD` 工程方法导入
- `TAT-COP-ODD-Harness` 接口落地
- `Progee2` 场景化部署
- 企业内训、顾问、系统授权

要求：

- 默认只讲脱敏原理，不公开完整模板、阈值、参数与样例库

### 4. 第四层收费：版权与品牌授权

适合对象：

- 课程合作
- 出版合作
- 顾问品牌合作
- 认证体系
- 授权使用方法名、图谱与训练材料

注意：

- 授权卖的是 `使用权`
- 不是把母理论和执行法一次性卖断

## 五、良性循环怎么形成

把它压成六步：

1. `法源立名`
先让 `DM -> ASTO -> ECET -> TAT -> ODD` 站住

2. `摘要传播`
用总图、简介、单篇问题文和各层摘要持续获客

3. `前线接引`
用 `LMM / WSM` 的公共面把问题场显影，把真正有需求的人引到港口

4. `付费承接`
用分层产品、顾问、部署与授权把价值兑现成收入

5. `案例回流`
把交付经验脱敏、整理、复核，回流到 `.BetterCiv` 与 `一元论`

6. `权威增强`
把回流后的案例、论文、白皮书、对话件再次推出，降低下一轮获客成本

最短句：

> **传播不是终点，收费也不是终点；真正的终点是“案例与收入一起回流，反过来增强下一轮传播与交付能力”。**

---

## 六、哪些钱最该先赚

如果要优先选择，当前最稳的不是卖纯哲学订阅，
而是卖这三类：

1. `结构诊断钱`
来自 `LMM / WSM / ASTO` 的问题识别与结构判断

2. `责任与危机评估钱`
来自 `TAT / ECET / ODD` 的责任门槛、例外治理、审计闭环

3. `部署与方法导入钱`
来自 `COP / ODD / Harness / Progee2` 的系统接入、训练与实施

原因很简单：

- 这些钱最容易和现实问题直接挂钩
- 这些钱最不依赖外界先接受整套本体论
- 这些钱最容易反哺案例、论文和产品

---

## 七、哪些钱不要先赚

1. `不要先卖“终极世界观会员”`
这会把理论做轻，也难以形成稳收入

2. `不要先卖完整执行法培训包`
这等于把护城河直接送人

3. `不要把咨询现场全部公开成宣传素材`
这会损害信任，也会把案例资产快速贬值

4. `不要让收入压力反向重写 DM`
成交不是法源，营收也不是证据

---

## 八、收入回流纪律

收入回来之后，优先流向四件事：

1. `案例补强`
让理论不只停在说服力，还停在案例厚度

2. `论文与对话`
让权威持续增长，而不是只停在客户层

3. `产品化`
把重复劳动压成工具、模板与系统

4. `传播资产`
持续生成更好的一页稿、总图、白皮书、案例摘要与演讲材料

不能允许的回流错误：

- 只养前台销售，不养研究
- 只接短单，不积累案例
- 只追成交，不建设产品底盘

---

## 九、最小动作顺序

如果从现在开始推进，最稳的顺序是：

1. 固定 `DM` 的双口径
对内是母理论，对外是强研究纲领

2. 固定传播入口
只用 `README / 总图 / 简介 / 单篇问题文 / 各层摘要`

3. 固定首批收费对象
先从诊断、责任评估、危机评估、方法导入切入

4. 固定承接边界
公开面只讲原则，收费面才进入 `COP / Harness / Progee2 / LMM 私有层`

5. 固定回流纪律
每一单都至少回流为 `案例 / 术语 / 问题清单 / 一页摘要 / 论文线索` 之一

---

## 十、最短结论

如果只能记一句话，就记这一句：

> **`DM` 用来立名，`B 类` 摘要件用来传播，`LMM / COP / TAT / ODD / Harness / Progee2 / BetterCiv` 用来收费与部署，收入再回流成案例、论文、产品与下一轮传播。**
